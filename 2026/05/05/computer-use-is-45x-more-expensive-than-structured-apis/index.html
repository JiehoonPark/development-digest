Computer Use는 구조화된 API보다 45배 더 비싼가?

Computer Use is 45x more expensive than structured APIs

📅 2026-05-05📰 Hacker News (Top)
ai

요약

Reflex는 동일한 관리자 패널을 제어하는 두 가지 AI 에이전트 방식(비전 에이전트 vs API 에이전트)을 벤치마크하여 비전 에이전트의 실제 비용을 측정했다. 결과적으로 API 에이전트는 평균 8회 호출로 19.7초(입력 토큰 12,151개)에 작업을 완료한 반면, 비전 에이전트는 명확한 UI 가이드 없이는 작업을 완료하지 못했으며, 상세한 14단계 프롬프트 지침을 통해 겨우 약 17분(입력 토큰 550,976개)이 소요되었다. 비전 에이전트는 페이지 매김 신호 부재로 화면에 보이지 않는 데이터를 놓쳤고, 실행 결과도 높은 분산을 보였다(749-1257초, 407k-751k 토큰).

핵심 포인트

  • API 에이전트와 비전 에이전트의 성능 차이: API 에이전트는 Sonnet으로 8회 호출로 안정적으로 작업을 완료(±0 분산)했고, 비전 에이전트는 Haiku 기준 평균 53회 호출이 필요했으며 높은 분산(±13)을 보였다. 이는 구조화된 응답이 모델에 명확한 의도 신호를 제공함을 의미한다.
  • 비전 에이전트의 근본적 한계: 비전 에이전트는 렌더링된 페이지만 보므로 페이지 1/4라는 정보를 읽을 수 없고, 화면 아래 있는 3개의 미승인 리뷰를 완전히 놓쳤다. 이는 모델의 추론 능력 문제가 아니라 비구조화된 인터페이스의 근본적 문제이다.
  • 숨은 엔지니어링 비용: 비전 에이전트를 작동시키기 위해 명시적인 14단계 UI 가이드를 작성해야 했으며, 이는 토큰 계산에 나타나지 않는 실제 엔지니어링 비용이다. 각 단계에서 정확히 어느 사이드바 항목, 탭, 폼 필드를 사용할지 명시해야 하므로 유지보수 부담이 크다.
  • 비용 차이 정량화: API 에이전트(Sonnet)는 약 19.7초에 12,151개 입력 토큰 소비, 비전 에이전트(Sonnet)는 약 1003초에 550,976개 입력 토큰 소비로, 대략 45배의 입력 토큰 비용 차이가 발생한다. Haiku 기반 API 에이전트는 7.7초에 9,478개 토큰만 필요하다.
  • 실행의 재현성 차이: API 경로는 5회 시도에서 동일한 8회 호출을 유지했고 입력 토큰 분산이 ±27에 불과했으나, 비전 경로는 3회 시도에서 43-68 사이클 변동과 407k-751k 토큰 변동을 보였다. 이는 스크린샷-추론-클릭 루프의 비결정성을 의미한다.
  • 테스트 시나리오의 현실성: 고객 'Smith' 찾기, 최근 미처리 주문 위치, 모든 미승인 리뷰 수락, 주문 배송 완료 표시라는 작업은 3개 리소스, 필터링, 페이지네이션, 교차 엔티티 조회, 읽기/쓰기 모두를 포함하는 전형적인 내부 도구 작업이다.

왜 중요한가

💡
AI 에이전트를 내부 도구에 배포할 때 비전 에이전트의 겉보기 편의성이 실제로는 45배 높은 비용과 유지보수 복잡성을 초래하므로, API 인터페이스 설계의 중요성과 장기적 ROI를 재고하는 계기를 제공한다.
