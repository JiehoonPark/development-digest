React Server Components의 서비스 거부 및 소스 코드 노출 취약점

frontendtypescript관련 1건

React 팀이 지난주 중대 취약점 패치를 시도하던 중 보안 연구자들이 2개의 추가 취약점을 발견하고 공개했습니다. 높은 심각도의 서비스 거부(DoS) 취약점 3개(CVE-2025-55184, CVE-2025-67779, CVE-2026-23864, CVSS 7.5)와 중간 심각도의 소스 코드 노출 취약점 1개(CVE-2025-55183, CVSS 5.3)가 있으며, 이들은 원격 코드 실행은 허용하지 않지만 이전 패치의 불완전성으로 인해 다시 업그레이드해야 합니다. 영향받는 패키지는 react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack이며, 버전 19.0.4, 19.1.5, 19.2.4로 즉시 업그레이드 필요합니다.

  • 높은 심각도의 서비스 거부 취약점 2개(CVE-2025-55184, CVE-2025-67779)가 존재하여 악의적인 HTTP 요청으로 Server Functions 엔드포인트에 무한 루프를 유발하고 서버 프로세스를 중단시킬 수 있습니다. 이는 CPU 소비로 이어져 사용자가 애플리케이션에 접근하지 못하게 합니다.
  • 추가 높은 심각도의 서비스 거부 취약점(CVE-2026-23864)이 발견되었으며, 특별히 제작된 HTTP 요청으로 서버 충돌, 메모리 부족 예외, 또는 과도한 CPU 사용을 초래할 수 있습니다. 원본 패치가 불완전하여 19.0.4, 19.1.5, 19.2.4 버전만 안전합니다.
  • 중간 심각도의 소스 코드 노출 취약점(CVE-2025-55183, CVSS 5.3)도 공개되어 공격자가 애플리케이션의 소스 코드에 접근할 가능성이 있습니다.
  • 영향받는 패키지는 react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack이며, 버전 19.0.0~19.2.3 모두가 취약합니다. 19.0.3, 19.1.4, 19.2.3으로 이미 업데이트한 경우에도 불완전한 패치이므로 재차 업그레이드해야 합니다.
  • Next.js, React Router, Waku, @parcel/rsc, @vite/rsc-plugin, rwsdk 등 여러 프레임워크와 번들러가 취약한 React 패키지에 의존하고 있어 이들도 영향을 받습니다.
  • React Server Components를 사용하지 않거나, 번들러 플러그인이 Server Components를 지원하지 않는 경우는 취약점 영향을 받지 않습니다.
  • 호스팅 제공자들이 임시 완화 조치를 적용했지만, 이를 보안 대책으로 의존해서는 안 되며 여전히 즉시 업그레이드해야 합니다.
  • React Native 사용자가 모노레포에서 react-server-dom-* 패키지를 사용하는 경우에만 해당 패키지들을 업데이트하면 되며, react와 react-dom 버전 불일치 오류를 피할 수 있습니다.

💡 React Server Components를 사용하는 프로젝트는 원격 코드 실행은 아니지만 심각한 서비스 거부와 소스 코드 노출 공격에 노출되어 있으므로, 불완전한 이전 패치를 무시하고 최신 버전으로 반드시 업그레이드해야 합니다.
