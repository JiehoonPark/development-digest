React Server Components의 서비스 거부 및 소스 코드 노출 취약점

Denial of Service and Source Code Exposure in React Server Components

📅 2026-04-15📰 React Blog
frontendtypescript

요약

React 팀이 지난주 중대 취약점 패치를 시도하던 중 보안 연구자들이 2개의 추가 취약점을 발견하고 공개했습니다. 높은 심각도의 서비스 거부(DoS) 취약점 3개(CVE-2025-55184, CVE-2025-67779, CVE-2026-23864, CVSS 7.5)와 중간 심각도의 소스 코드 노출 취약점 1개(CVE-2025-55183, CVSS 5.3)가 있으며, 이들은 원격 코드 실행은 허용하지 않지만 이전 패치의 불완전성으로 인해 다시 업그레이드해야 합니다. 영향받는 패키지는 react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack이며, 버전 19.0.4, 19.1.5, 19.2.4로 즉시 업그레이드 필요합니다.

핵심 포인트

  • 높은 심각도의 서비스 거부 취약점 2개(CVE-2025-55184, CVE-2025-67779)가 존재하여 악의적인 HTTP 요청으로 Server Functions 엔드포인트에 무한 루프를 유발하고 서버 프로세스를 중단시킬 수 있습니다. 이는 CPU 소비로 이어져 사용자가 애플리케이션에 접근하지 못하게 합니다.
  • 추가 높은 심각도의 서비스 거부 취약점(CVE-2026-23864)이 발견되었으며, 특별히 제작된 HTTP 요청으로 서버 충돌, 메모리 부족 예외, 또는 과도한 CPU 사용을 초래할 수 있습니다. 원본 패치가 불완전하여 19.0.4, 19.1.5, 19.2.4 버전만 안전합니다.
  • 중간 심각도의 소스 코드 노출 취약점(CVE-2025-55183, CVSS 5.3)도 공개되어 공격자가 애플리케이션의 소스 코드에 접근할 가능성이 있습니다.
  • 영향받는 패키지는 react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack이며, 버전 19.0.0~19.2.3 모두가 취약합니다. 19.0.3, 19.1.4, 19.2.3으로 이미 업데이트한 경우에도 불완전한 패치이므로 재차 업그레이드해야 합니다.
  • Next.js, React Router, Waku, @parcel/rsc, @vite/rsc-plugin, rwsdk 등 여러 프레임워크와 번들러가 취약한 React 패키지에 의존하고 있어 이들도 영향을 받습니다.
  • React Server Components를 사용하지 않거나, 번들러 플러그인이 Server Components를 지원하지 않는 경우는 취약점 영향을 받지 않습니다.
  • 호스팅 제공자들이 임시 완화 조치를 적용했지만, 이를 보안 대책으로 의존해서는 안 되며 여전히 즉시 업그레이드해야 합니다.
  • React Native 사용자가 모노레포에서 react-server-dom-* 패키지를 사용하는 경우에만 해당 패키지들을 업데이트하면 되며, react와 react-dom 버전 불일치 오류를 피할 수 있습니다.

왜 중요한가

💡
React Server Components를 사용하는 프로젝트는 원격 코드 실행은 아니지만 심각한 서비스 거부와 소스 코드 노출 공격에 노출되어 있으므로, 불완전한 이전 패치를 무시하고 최신 버전으로 반드시 업그레이드해야 합니다.

React Server Components의 서비스 거부 및 소스코드 노출 취약점

2025년 12월 11일 | React 팀 작성

2026년 1월 26일 업데이트

지난주 공개된 심각한 취약점의 패치를 우회하려는 시도 과정에서 보안 연구원들이 React Server Components의 새로운 취약점 2개를 추가로 발견하고 공개했습니다. 이 새로운 취약점들은 원격 코드 실행을 허용하지는 않습니다. React2Shell 패치는 원격 코드 실행 공격을 방어하는 데 여전히 효과가 있습니다.

새롭게 공개된 취약점은 다음과 같습니다.

서비스 거부 - 높음 심각도: CVE-2025-55184, CVE-2025-67779, CVE-2026-23864 (CVSS 7.5)

소스코드 노출 - 중간 심각도: CVE-2025-55183 (CVSS 5.3)

새로운 취약점의 심각성 때문에 즉시 업그레이드하기를 권장합니다.

> 주의 이전에 발표된 패치는 취약합니다. 기존 취약점으로 이미 업데이트했다면 다시 한 번 업그레이드해야 합니다. 19.0.3, 19.1.4, 19.2.3으로 업그레이드했다면 불완전한 패치이므로 재업그레이드가 필요합니다. 업그레이드 방법은 이전 포스트의 안내를 참고하세요.

>

> 2026년 1월 26일 업데이트. 이 취약점들에 대한 자세한 내용은 수정 사항이 완전히 배포된 후 제공될 예정입니다.

즉시 조치 필요

이 취약점들은 CVE-2025-55182과 동일한 패키지 및 버전에 존재합니다.

다음 패키지의 19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3 버전이 영향을 받습니다.

  • react-server-dom-webpack
  • react-server-dom-parcel
  • react-server-dom-turbopack

수정 사항이 19.0.4, 19.1.5, 19.2.4 버전으로 백포트되었습니다. 위의 패키지를 사용 중이라면 즉시 고정 버전 중 하나로 업그레이드하세요.

앞서 언급했듯이, React 코드가 서버를 사용하지 않는다면 이 취약점의 영향을 받지 않습니다. 프레임워크, 번들러, 또는 React Server Components를 지원하는 번들러 플러그인을 사용하지 않는다면 역시 영향을 받지 않습니다.

> 참고 중대한 CVE가 공개될 때 후속 취약점이 발견되는 건 흔한 일입니다. 심각한 취약점이 공개되면 보안 연구원들은 초기 패치를 우회할 수 있는 변종 공격 기법이 있는지 확인하기 위해 인접한 코드 경로를 집중적으로 검토합니다.

>

> 이러한 패턴은 JavaScript 뿐만 아니라 업계 전반에서 나타납니다. 예를 들어 Log4Shell 이후에도 커뮤니티가 원본 수정 사항을 검토하면서 추가 CVE들이 보고되었습니다.

>

> 추가 공개가 답답할 수 있지만, 일반적으로 건강한 보안 대응 사이클의 신호입니다.

영향을 받는 프레임워크 및 번들러

일부 React 프레임워크와 번들러는 취약한 React 패키지에 의존하거나 피어 의존성으로 포함하고 있습니다. 다음 React 프레임워크 및 번들러들이 영향을 받습니다.

next, react-router, waku, @parcel/rsc, @vite/rsc-plugin, rwsdk

업그레이드 방법은 이전 포스트의 안내를 참고하세요.

호스팅 제공자의 임시 완화 조치

이전과 마찬가지로 여러 호스팅 제공자와 함께 임시 완화 조치를 적용했습니다.

이러한 조치에 의존해서는 안 되며, 반드시 즉시 업그레이드하세요.

React Native

monorepo나 react-dom을 사용하지 않는 React Native 사용자의 경우, package.json에서 react 버전이 고정되어 있다면 추가 조치가 필요 없습니다.

monorepo에서 React Native를 사용하는 경우, 설치되어 있다면 다음 패키지만 업데이트하세요.

  • react-server-dom-webpack
  • react-server-dom-parcel
  • react-server-dom-turbopack

이는 보안 권고사항을 완화하기 위해 필요하지만, react와 react-dom을 업데이트할 필요는 없으므로 React Native의 버전 불일치 오류를 유발하지 않습니다.

자세한 내용은 [이 이슈](https://github.com/facebook/react-native/issues/)를 참고하세요.

높음 심각도: 다중 서비스 거부 취약점

CVE: CVE-2026-23864

기본 점수: 7.5 (높음)

공개 날짜: 2026년 1월 26일

보안 연구원들이 React Server Components에 여전히 존재하는 추가 서비스 거부 취약점을 발견했습니다.

취약점은 Server Function 엔드포인트로 특별히 제작된 HTTP 요청을 전송할 때 트리거되며, 서버 크래시, 메모리 부족 예외 또는 과도한 CPU 사용을 초래할 수 있습니다. 구체적인 결과는 실행되는 취약한 코드 경로, 애플리케이션 설정, 그리고 애플리케이션 코드에 따라 달라집니다.

1월 26일에 발표된 패치가 이 서비스 거부 취약점들을 완화합니다.

> 참고 추가 수정 사항이 발표되었습니다. CVE-2025-55184의 서비스 거부를 다루는 원래 수정 사항이 불완전했습니다. 이로 인해 이전 버전들이 여전히 취약했습니다. 19.0.4, 19.1.5, 19.2.4 버전은 안전합니다.

>

> 2026년 1월 26일 업데이트.

높음 심각도: 서비스 거부

CVE: CVE-2025-55184, CVE-2025-67779

기본 점수: 7.5 (높음)

보안 연구원들이 Server Functions 엔드포인트로 악의적인 HTTP 요청을 전송할 수 있으며, React에 의해 역직렬화될 때 무한 루프를 일으켜 서버 프로세스를 행(hang) 상태로 만들고 CPU를 소비할 수 있음을 발견했습니다. 앱에서 React Server Function 엔드포인트를 구현하지 않았더라도, React Server Components를 지원한다면 여전히 취약할 수 있습니다.

이는 공격자가 사용자들의 제품 접근을 거부하고, 잠재적으로 다른 악의적인 활동을 수행할 수 있는 공격 벡터를 만듭니다.
