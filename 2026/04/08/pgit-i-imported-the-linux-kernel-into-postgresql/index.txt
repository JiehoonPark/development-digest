16:T1d8c,<h2>Linux 커널을 PostgreSQL 데이터베이스로 변환해봤습니다</h2>
<p><strong>요약:</strong> Linux 커널의 전체 역사를 pgit으로 임포트했습니다. 142만 8,882개의 커밋, 2,440만 개의 파일 버전, 20년의 개발 히스토리를 PostgreSQL에 저장했고, delta 압축으로 2.7GB에 압축했습니다. (<code>git gc --aggressive</code> 적용 시 1.95GB) 2시간에 완료했고요. 그 다음부터 재미있는 질문들을 던져봤습니다. 140만 개의 커밋 메시지 중에 욕설이 겨우 7개 (2명에게서만), 한 개의 커밋을 가리키는 버그 픽스가 665개, 13년이 걸려 병합된 파일시스템. Linux 커널이 SQL 데이터베이스가 되면 어떻게 보일까요?</p>
<h3>임포트 과정</h3>
<p>이 글은 <strong>pgit: 만약 Git 히스토리가 SQL 데이터베이스라면?</strong>의 후속편입니다. 아직 읽지 않으셨다면 먼저 읽어보시길 추천합니다. 간단히 말하면, pgit은 Git처럼 작동하지만 파일시스템 대신 PostgreSQL에 모든 걸 저장하는 CLI 도구입니다. pg-xpatch를 사용해 투명한 delta 압축을 제공하고, 전체 커밋 히스토리를 SQL로 쿼리할 수 있도록 만들어줍니다.</p>
<p>pgit 글이 해커뉴스 첫 페이지에 올라가고 TLDR, console.dev, dailydev에 소개된 후, 저는 Linux 커널 임포트를 시도해본다고 예고했었습니다. 그 결과물을 지금부터 공개합니다.</p>
<p>Linux 커널은 세계에서 가장 큰 활발한 개발 저장소 중 하나입니다. 140만 개의 커밋, 20년의 역사, 17만 1,000개의 파일, 38,000명의 기여자가 있거든요. 제가 찾은 바로는 Git이 아닌 다른 VCS 중에서 커널 전체 히스토리를 임포트한 경우는 손꼽을 정도입니다. Fossil(SQLite 기반, SQLite 팀이 만듦)은 시도하지 않았고요. Darcs와 Monotone은 시도했지만 성능 문제가 심했습니다. Mercurial은 가능합니다.</p>
<p><strong>pgit도 해냈습니다.</strong></p>
<div class="table-wrapper"><table><thead><tr><th>항목</th><th>수치</th></tr></thead><tbody><tr><td>커밋</td><td>1,428,882</td></tr><tr><td>파일 버전 (파일 참조)</td><td>24,384,844</td></tr><tr><td>고유 blob</td><td>3,089,589</td></tr><tr><td>고유 경로</td><td>171,525</td></tr><tr><td>Delta 체인 (경로 그룹)</td><td>137,600</td></tr><tr><td>임포트 시간</td><td>2시간 0분 48초</td></tr></tbody></table></div>
<p>임포트는 핀란드 헤츠너 전용 서버에서 실행됐습니다: AMD EPYC 7401P (24코어 / 48스레드), 512GB DDR4 ECC RAM, 2×1.92TB SSD RAID 0. 350GB의 xpatch 콘텐츠 캐시를 사용하면 전체 디코드된 저장소가 메모리에 들어갑니다.</p>
<h3>서버 구성</h3>
<h4>하드웨어 스펙</h4>
<p>헤츠너 전용 &quot;Server Auction&quot;을 핀란드 데이터센터(HEL1)에서 선택했습니다:</p>
<div class="table-wrapper"><table><thead><tr><th>항목</th><th>사양</th></tr></thead><tbody><tr><td>CPU</td><td>AMD EPYC 7401P (24코어 / 48스레드)</td></tr><tr><td>RAM</td><td>16×32GB DDR4 ECC reg. (총 512GB)</td></tr><tr><td>저장소</td><td>2×Micron SSD SATA 1.92TB 데이터센터 (RAID 0)</td></tr><tr><td>네트워크</td><td>1 Gbit Intel I350</td></tr><tr><td>월 비용</td><td>약 €272</td></tr></tbody></table></div>
<h4>OS 설치</h4>
<p>헤츠너 installimage를 사용해 Ubuntu 24.04 LTS를 설치했습니다. 기본 설정에서 두 가지를 변경했는데요:</p>
<ul>
<li><strong>RAID 0</strong> (<code>SWRAIDLEVEL 0</code>) - 최대 처리량을 위해 (임시 분석 작업이라 중복성 불필요)</li>
<li><strong>간단한 파티션 레이아웃</strong> - 두 개의 1.92TB SSD에서 약 3.5TB의 사용 가능한 공간 확보</li>
</ul>
<pre><code>PART /boot ext3 1024M
PART swap swap 4G
PART / ext4 all</code></pre>
<h4>OS 튜닝</h4>
<p>설치된 이미지로 부팅한 후 다음을 실행했습니다:</p>
<pre><code class="language-bash">apt update &amp;&amp; apt upgrade -y
apt install -y \
  tmux btop htop iotop \
  cpufrequtils numactl \
  git curl wget unzip \
  build-essential \
  ufw \
  linux-tools-common linux-tools-$(uname -r)</code></pre>
<p>CPU 주파수를 성능 모드로 설정합니다:</p>
<pre><code class="language-bash">for cpu in /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor; do
  echo performance &gt; &quot;$cpu&quot;
done

cat &gt; /etc/default/cpufrequtils &lt;&lt; 'EOF'
GOVERNOR=&quot;performance&quot;
EOF

systemctl enable cpufrequtils
systemctl restart cpufrequtils</code></pre>
<p>Grub 부팅 옵션에서 마이크로코드 보안 완화를 비활성화합니다:</p>
<pre><code class="language-bash">sed -i 's/GRUB_CMDLINE_LINUX_DEFAULT=&quot;consoleblank=0&quot;/GRUB_CMDLINE_LINUX_DEFAULT=&quot;consoleblank=0 mitigations=off&quot;/' /etc/default/grub.d/hetzner.cfg
update-grub</code></pre>
<p>커널 파라미터를 조정합니다:</p>
<pre><code class="language-bash">cat &gt;&gt; /etc/sysctl.conf &lt;&lt; 'EOF'
vm.swappiness = 1
vm.dirty_ratio = 5
vm.dirty_background_ratio = 2
kernel.numa_balancing = 1
EOF

sysctl -p</code></pre>
<p>Transparent Huge Pages를 비활성화합니다:</p>
<pre><code class="language-bash">echo never &gt; /sys/kernel/mm/transparent_hugepage/enabled
echo never &gt; /sys/kernel/mm/transparent_hugepage/defrag

cat &gt; /etc/systemd/system/disable-thp.service &lt;&lt; 'EOF'
[Unit]
Description=Disable Transparent Huge Pages
DefaultDependencies=no
After=sysinit.target local-fs.target
Before=basic.target

[Service]
Type=oneshot
ExecStart=/bin/sh -c 'echo never &gt; /sys/kernel/mm/transparent_hugepage/enabled &amp;&amp; echo never &gt; /sys/kernel/mm/transparent_hugepage/defrag'

[Install]
WantedBy=basic.target
EOF

systemctl daemon-reload
systemctl enable disable-thp</code></pre>
<p>파일시스템 마운트 옵션을 <code>relatime</code>에서 <code>noatime</code>으로 변경합니다:</p>
<pre><code class="language-bash">sed -i 's|relatime|noatime|g' /etc/fstab
mount -o remount,noatime /</code></pre>
<p>기본적인 방화벽을 설정합니다:</p>
<pre><code class="language-bash">ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw --force enable</code></pre>
<p>Go와 Docker를 설치합니다:</p>
<pre><code class="language-bash">wget https://go.dev/dl/go1.26.0.linux-amd64.tar.gz
rm -rf /usr/local/go &amp;&amp; tar -C /usr/local -xzf go1.26.0.linux-amd64.tar.gz
rm go1.26.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin' &gt;&gt; ~/.bashrc
source ~/.bashrc

apt install -y docker.io
systemctl enable docker
systemctl start docker

reboot</code></pre>
<h3>pg-xpatch와 pgit 설정</h3>
<p>표준 최신 pg-xpatch Docker 이미지를 가져왔습니다:</p>
<pre><code class="language-bash">docker pull ghcr.io/imgajeed76/pg-xpatch:latest</code></pre>
<p>이 작업에 사용한 pgit은 v4이며, 몇 가지 로컬 수정사항이 포함돼 있었습니다. 지금 이 글을 읽고 있으신 시점에는 이 수정사항들이 최신 버전에 포함돼 있어야 하므로, 일반적인 <code>go install</code>로도 모두 재현 가능합니다.</p>
<p>주요 변경사항은 단조 타임스탬프 해킹을 명시적인 <code>seq INTEGER NOT NULL</code> 컬럼으로 대체하는 seq 정렬 수정입니다. 이렇게 하면 순차 스캔에서 delta 체인 압축 해제 속도가 크게 빨라집니다.</p>
