28:T1694,<h2>Agent Skill을 API로 변환하기 — SkRun</h2>
<p>Agent Skill(SKILL.md)을 POST /run 엔드포인트로 호출 가능한 API로 만들어보세요. 여러 모델을 지원하고, 상태를 유지하며, 완전히 오픈소스입니다.</p>
<h3>빠른 시작</h3>
<pre><code class="language-bash">npm install -g @skrun-dev/cli
# 기존 skill을 가져온 후 배포하고 호출하기
skrun init --from-skill ./my-skill
skrun deploy
curl -X POST localhost:4000/api/agents/dev/my-skill/run \
-H &quot;Authorization: Bearer &lt;token&gt;&quot; \
-H &quot;Content-Type: application/json&quot; \
-d '{&quot;input&quot;: {&quot;query&quot;: &quot;analyze this&quot;}}'</code></pre>
<h3>시작하기</h3>
<h4>새로운 Agent 만들기</h4>
<pre><code class="language-bash">skrun init my-agent
cd my-agent
# SKILL.md(지시사항) + agent.yaml(설정) 파일이 생성됩니다</code></pre>
<h4>기존 Skill 가져오기</h4>
<pre><code class="language-bash">skrun init --from-skill ./path-to-skill
# SKILL.md를 읽고 2-3가지 질문을 한 뒤 agent.yaml을 자동 생성합니다</code></pre>
<h4>로컬에서 개발 및 테스트하기</h4>
<pre><code class="language-bash">skrun dev
# ✓ Server running at http://localhost:3000
# POST /run 준비 완료 — 파일 변경사항 감시 중...</code></pre>
<pre><code class="language-bash">skrun test
# ✓ basic-test (output.score &gt;= 0)
# 1 passed, 0 failed</code></pre>
<h4>배포하기</h4>
<pre><code class="language-bash">skrun deploy
# ✓ Validated → Built → Pushed
# 🚀 POST http://localhost:4000/api/agents/you/my-agent/run</code></pre>
<p>현재 v0.1은 로컬 런타임을 제공합니다. 클라우드 배포는 로드맵에 포함되어 있으며, 아키텍처는 이미 준비되어 있습니다(RuntimeAdapter 인터페이스를 통해).</p>
<h3>핵심 개념</h3>
<p><strong>Agent Skills</strong> — SKILL.md 표준을 따르며, Claude Code, Copilot, Codex와 호환됩니다.</p>
<p><strong>agent.yaml</strong> — 런타임 설정: 모델, 입출력 스키마, 권한, 상태 관리, 테스트 정의</p>
<p><strong>POST /run</strong> — 모든 Agent는 API입니다. 입력값 검증, 구조화된 응답을 제공합니다.</p>
<p><strong>다중 모델 지원</strong> — Anthropic, OpenAI, Google, Mistral, Groq를 지원하며 자동 폴백이 가능합니다.</p>
<p><strong>상태 유지</strong> — Agent는 키-값 저장소를 통해 실행 간 상태를 기억합니다.</p>
<p><strong>Tool 호출</strong> — 두 가지 방식을 지원합니다. CLI 도구(scripts/ 디렉토리에 직접 작성하고 Agent와 함께 번들링)와 MCP 서버(npx를 통해 표준 생태계 활용, Claude Desktop과 동일).</p>
<h3>데모 Agent</h3>
<p>모든 예제는 기본적으로 Google Gemini Flash를 사용합니다. agent.yaml의 model 섹션을 수정하면 지원되는 다른 제공자를 사용할 수 있습니다.</p>
<div class="table-wrapper"><table><thead><tr><th>Agent</th><th>설명</th></tr></thead><tbody><tr><td>code-review</td><td>Skill을 가져와서 코드 품질 검사 API 만들기</td></tr><tr><td>pdf-processing</td><td>로컬 스크립트를 통한 Tool 호출</td></tr><tr><td>seo-audit</td><td>상태 유지 — 두 번 실행하면 비교 결과를 기억</td></tr><tr><td>data-analyst</td><td>타입 지정 입출력 — CSV 입력, 구조화된 인사이트 출력</td></tr><tr><td>email-drafter</td><td>실제 사용 사례 — 개발자가 아닌 사용자를 위한 API</td></tr><tr><td>web-scraper</td><td>MCP 서버 — @playwright/mcp를 통한 헤드리스 브라우저</td></tr></tbody></table></div>
<h4>예제 실행해보기</h4>
<pre><code class="language-bash"># 1. 레지스트리 시작
cp .env.example .env # GOOGLE_API_KEY 추가
pnpm dev:registry # 이 터미널은 열어둡니다

# 2. 다른 터미널에서
skrun login --token dev-token
cd examples/code-review
skrun build &amp;&amp; skrun push

# 3. Agent 호출
curl -X POST http://localhost:4000/api/agents/dev/code-review/run \
-H &quot;Authorization: Bearer dev-token&quot; \
-H &quot;Content-Type: application/json&quot; \
-d '{&quot;input&quot;: {&quot;code&quot;: &quot;function add(a,b) { return a + b; }&quot;}}'</code></pre>
<p><strong>Windows(PowerShell)</strong>: curl 대신 curl.exe를 사용하고, body는 @input.json 형식으로 작성하세요.</p>
