28:T1d08,<h2>Plain: 에이전트 시대를 위해 재설계된 파이썬 웹 프레임워크</h2>
<p>Plain은 앱을 빌드하기 위한 파이썬 웹 프레임워크입니다. 익숙한 기초 위에 인간과 AI 에이전트를 위해 새롭게 설계되었죠.</p>
<h3>시작하기</h3>
<pre><code class="language-bash">mkdir my-app &amp;&amp; cd my-app &amp;&amp; claude &quot;$(curl -sSf https://plainframework.com/start.md)&quot;</code></pre>
<p>Codex, Amp, OpenCode 같은 다른 에이전트들도 물론 사용할 수 있습니다.</p>
<h3>Plain을 선택해야 하는 이유</h3>
<p>코드는 명시적이고, 타입이 지정되며, 예측 가능해야 합니다. 인간에게 좋은 것이 에이전트에게도 좋거든요.</p>
<p>실제로 Plain 코드는 어떻게 생겼는지 살펴보겠습니다.</p>
<h4>모델 정의</h4>
<pre><code class="language-python"># app/users/models.py
from plain import postgres
from plain.postgres import types
from plain.passwords.models import PasswordField

@postgres.register_model
class User(postgres.Model):
    email: str = types.EmailField()
    password: str = PasswordField()
    display_name: str = types.CharField(max_length=100)
    is_admin: bool = types.BooleanField(default=False)
    created_at: datetime = types.DateTimeField(auto_now_add=True)
    
    query: postgres.QuerySet[User] = postgres.QuerySet()
    
    model_options = postgres.Options(
        constraints=[
            postgres.UniqueConstraint(fields=[&quot;email&quot;], name=&quot;unique_email&quot;),
        ],
    )</code></pre>
<p>타입 힌트로 필드를 명확하게 정의합니다. 코드를 읽기만 해도 스키마가 한눈에 들어오죠.</p>
<h4>뷰는 클래스 기반</h4>
<pre><code class="language-python"># app/users/views.py
from plain.views import DetailView
from .models import User

class UserDetail(DetailView):
    template_name = &quot;users/detail.html&quot;
    
    def get_object(self):
        return User.query.get(pk=self.url_kwargs[&quot;pk&quot;])</code></pre>
<h4>URL 라우팅도 구조적으로</h4>
<pre><code class="language-python"># app/users/urls.py
from plain.urls import Router, path
from . import views

class UsersRouter(Router):
    namespace = &quot;users&quot;
    urls = [
        path(&quot;&lt;int:pk&gt;/&quot;, views.UserDetail),
    ]</code></pre>
<h3>에이전트를 위한 빌트인 도구</h3>
<p>Plain 프로젝트에는 에이전트가 자동으로 활용할 수 있는 도구들이 내장되어 있습니다.</p>
<p><strong>Rules</strong> — 프로젝트 규칙 파일(예: Claude Code용 <code>.claude/rules/</code>)에 저장된 항상 활성화되는 가드레일. 약 50줄의 짧은 파일들이 가장 흔한 실수들을 미리 방지합니다.</p>
<p><strong>Docs</strong> — 프레임워크 문서 전체를 CLI에서 언제든 접근할 수 있습니다:</p>
<pre><code class="language-bash">plain docs models                    # 전체 문서
plain docs models --section querying # 특정 섹션만
plain docs models --api              # 타입 시그니처만
plain docs --search &quot;queryset&quot;       # 전체 패키지 검색</code></pre>
<p><strong>Skills</strong> — 슬래시 커맨드로 트리거되는 엔드투엔드 워크플로우:</p>
<ul>
<li><code>/plain-install</code> — 새 패키지 추가 및 설정 가이드</li>
<li><code>/plain-upgrade</code> — 버전 업그레이드, 변경사항 확인, 호환성 처리</li>
<li><code>/plain-optimize</code> — 성능 추적, 느린 쿼리와 N+1 문제 식별 및 수정</li>
<li><code>/plain-bug</code> — 버그 리포트를 GitHub 이슈로 자동 생성</li>
</ul>
<h3>CLI 명령어</h3>
<p>모든 명령어는 <code>uv run</code> 으로 실행됩니다(예: <code>uv run plain dev</code>).</p>
<ul>
<li><code>plain dev</code> — 자동 리로드와 HTTPS가 활성화된 개발 서버 시작</li>
<li><code>plain fix</code> — Python, CSS, JS를 한 번에 포맷하고 린트</li>
<li><code>plain check</code> — 린트, 프리플라이트, 마이그레이션, 테스트 검증</li>
<li><code>plain test</code> — 테스트 실행(pytest)</li>
<li><code>plain docs --api</code> — LLM용으로 포맷된 공개 API 노출</li>
</ul>
<h3>기술 스택</h3>
<p>Plain은 의견이 확실한 프레임워크입니다.</p>

</ul>
