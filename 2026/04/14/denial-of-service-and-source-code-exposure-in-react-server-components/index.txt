15:T177b,<h2>React Server Components의 서비스 거부 및 소스 코드 노출 취약점</h2>
<p><strong>2025년 12월 11일 | React 팀 작성</strong></p>
<p><strong>2026년 1월 26일 업데이트</strong></p>
<p>보안 연구자들이 지난주 공개된 치명적 취약점을 악용하려던 중, React Server Components에서 두 가지 추가 취약점을 발견해 공개했습니다. 다행히 이 새로운 취약점들은 원격 코드 실행(RCE)을 허용하지 않으며, 기존 React2Shell 패치는 여전히 효과적입니다.</p>
<p>새롭게 공개된 취약점은 다음과 같습니다:</p>
<ul>
<li><strong>서비스 거부 (높음 심각도)</strong>: CVE-2025-55184, CVE-2025-67779, CVE-2026-23864 (CVSS 7.5)</li>
<li><strong>소스 코드 노출 (중간 심각도)</strong>: CVE-2025-55183 (CVSS 5.3)</li>
</ul>
<p>새로운 취약점의 심각성으로 인해 <strong>즉시 업그레이드를 권장합니다.</strong></p>
<h3>⚠️ 주의사항</h3>
<p>이전에 배포된 패치는 취약점이 있습니다. 이전 취약점으로 이미 업데이트했다면 <strong>다시 한 번 업데이트해야 합니다.</strong></p>
<p>특히 19.0.3, 19.1.4, 19.2.3으로 업데이트하신 분들은 불완전한 버전이므로 재업데이트가 필수입니다.</p>
<p>업그레이드 방법은 이전 공지사항을 참고해주세요. 수정 사항의 배포가 완료된 후 취약점에 대한 자세한 정보를 제공하겠습니다.</p>
<h3>즉시 조치 필요</h3>
<p>이 취약점들은 CVE-2025-55182와 동일한 패키지와 버전에 존재합니다. 다음 패키지들의 해당 버전들이 영향을 받습니다:</p>
<ul>
<li>react-server-dom-webpack</li>
<li>react-server-dom-parcel</li>
<li>react-server-dom-turbopack</li>
</ul>
<p><strong>영향받는 버전</strong>: 19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3</p>
<p><strong>수정된 버전</strong>으로 업그레이드하세요: 19.0.4, 19.1.5, 19.2.4</p>
<p>위 패키지 중 하나를 사용 중이라면 즉시 수정된 버전으로 업그레이드해야 합니다.</p>
<h3>영향받지 않는 경우</h3>
<ul>
<li>React Server를 사용하지 않는 앱</li>
<li>React Server Components를 지원하는 프레임워크, 번들러, 플러그인을 사용하지 않는 앱</li>
</ul>
<h3>참고: 추가 취약점 공개는 정상적인 과정입니다</h3>
<p>치명적 CVE가 공개되면 보안 연구자들은 초기 패치가 우회될 수 있는지 테스트하기 위해 인접한 코드 경로를 자세히 살펴봅니다. 이는 JavaScript뿐만 아니라 전 산업에서 나타나는 패턴입니다. 예를 들어 Log4Shell 이후에도 여러 추가 CVE가 보고되었습니다.</p>
<p>추가 공개가 번거로울 수 있지만, 일반적으로 건강한 보안 대응 사이클의 신호입니다.</p>
<h3>영향받는 프레임워크와 번들러</h3>
<p>다음 React 프레임워크와 번들러들이 영향받는 패키지에 의존하고 있습니다:</p>
<ul>
<li>next</li>
<li>react-router</li>
<li>waku</li>
<li>@parcel/rsc</li>
<li>@vite/rsc-plugin</li>
<li>rwsdk</li>
</ul>
<p>업그레이드 방법은 이전 공지사항을 참고해주세요.</p>
<h3>호스팅 제공자 임시 완화 조치</h3>
<p>일부 호스팅 제공자들과 협력해 임시 완화 조치를 적용했습니다. 하지만 이에 의존하지 마시고 <strong>반드시 즉시 업데이트하세요.</strong></p>
<h3>React Native 사용자</h3>
<p>monorepo를 사용하지 않거나 react-dom을 사용하지 않는 경우, package.json에서 react 버전이 고정되어 있으면 추가 조치가 필요하지 않습니다.</p>
<p>monorepo에서 React Native를 사용 중이라면 다음 패키지가 설치되어 있을 경우만 업데이트하면 됩니다:</p>
<ul>
<li>react-server-dom-webpack</li>
<li>react-server-dom-parcel</li>
<li>react-server-dom-turbopack</li>
</ul>
<p>이 업데이트는 보안 권고를 완화하기 위해 필요하며, react와 react-dom을 업데이트할 필요는 없으므로 React Native의 버전 불일치 오류가 발생하지 않습니다.</p>
<p>더 자세한 정보는 해당 이슈를 참고하세요.</p>

<h3>높음 심각도: 다중 서비스 거부 취약점</h3>
<p><strong>CVE</strong>: CVE-2026-23864</p>
<p><strong>심각도 점수</strong>: 7.5 (높음)</p>
<p><strong>날짜</strong>: 2026년 1월 26일</p>
<p>보안 연구자들이 React Server Components에서 여전히 존재하는 추가 서비스 거부 취약점을 발견했습니다.</p>
<p>이 취약점은 Server Function 엔드포인트로 특별히 제작된 HTTP 요청을 보내면 트리거되며, 서버 크래시, 메모리 부족 예외, 또는 과도한 CPU 사용으로 이어질 수 있습니다. 영향받는 코드 경로, 애플리케이션 구성, 그리고 애플리케이션 코드에 따라 달라집니다.</p>
<p>2026년 1월 26일 배포된 패치가 이 서비스 거부 취약점을 완화합니다.</p>
<h4>⚠️ 추가 수정 사항 공개</h4>
<p>CVE-2025-55184의 서비스 거부 문제를 해결한 원본 패치가 불완전했습니다. 이로 인해 이전 버전들이 여전히 취약했습니다. <strong>19.0.4, 19.1.5, 19.2.4 버전은 안전합니다.</strong></p>
