React 서버 컴포넌트의 심각한 보안 취약점

frontend관련 1건

React 서버 컴포넌트에서 인증되지 않은 원격 코드 실행(RCE) 취약점이 발견되었으며, CVSS 10.0으로 평가되는 심각한 수준이다. 이 취약점(CVE-2025-55182)은 React가 서버 함수 엔드포인트로 전송된 페이로드를 디코딩하는 방식의 결함을 통해 악의적인 HTTP 요청으로 인해 발생한다. react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack의 19.0, 19.1.0, 19.1.1, 19.2.0 버전이 영향을 받으며, 즉시 업그레이드가 필수적이다. Next.js, React Router, Waku 등 주요 프레임워크와 번들러도 영향을 받는다.

  • 원격 코드 실행 취약점(CVE-2025-55182): 라클란 데이비슨이 11월 29일 보고한 취약점으로, 인증 없이 공격자가 React 서버 함수 엔드포인트로 악의적인 HTTP 요청을 보내 원격 코드 실행을 달성할 수 있다. 서버 함수를 직접 구현하지 않아도 React 서버 컴포넌트를 지원하는 앱이면 영향을 받을 수 있다.
  • 심각도 및 영향 범위: CVSS 10.0의 최고 심각도로 평가되며, react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack의 버전 19.0, 19.1.0-19.1.1, 19.2.0에서 발견된다. 서버 코드가 없거나 React 서버 컴포넌트를 지원하지 않는 앱은 영향을 받지 않는다.
  • 패치된 버전: 19.0.1, 19.1.2, 19.2.1 버전에서 수정되었으며, 즉시 업그레이드가 필수이다. 호스팅 제공업체가 임시 완화 조치를 적용했으나 이에 의존하지 말고 반드시 업그레이드해야 한다.
  • 영향받는 프레임워크 및 번들러: Next.js, React Router, Waku, @parcel/rsc, @vitejs/plugin-rsc, rwsdk 등이 취약한 React 패키지에 의존하거나 포함하고 있어 영향을 받는다. 각 프레임워크별로 구체적인 업그레이드 명령어가 제시되었다.
  • React 서버 함수 동작 원리: 클라이언트의 함수 호출을 HTTP 요청으로 변환하여 서버에 전달하고, 서버에서 이를 함수 호출로 해석한 후 필요한 데이터를 클라이언트에 반환하는 구조에서 역직렬화 과정의 결함이 취약점을 야기한다.
  • 구체적인 업그레이드 명령어: Next.js는 버전별로 14.2.35, 15.0.8, 15.1.12, 15.2.9, 15.3.9, 15.4.11, 15.5.10, 16.0.11, 16.1.5 등으로 업그레이드 필요하며, React Router는 react, react-dom, react-server-dom-* 패키지를 최신 버전으로 업그레이드해야 한다.

💡 React 서버 컴포넌트 기술을 사용하는 모든 애플리케이션이 인증 없이 원격 코드 실행 공격에 노출되어 있으므로, 개발자는 즉시 패치된 버전으로 업그레이드하지 않으면 심각한 보안 위협에 직면하게 된다.

React 서버 컴포넌트의 서비스 거부 및 소스 코드 노출 취약점

frontend관련 1건

React 서버 컴포넌트에서 원격 코드 실행 패치를 우회하려는 시도 중 발견된 추가 취약점 2개가 공개되었다. 높은 심각도의 서비스 거부(DoS) 취약점(CVE-2025-55184, CVE-2025-67779, CVE-2026-23864, CVSS 7.5)은 악의적인 HTTP 요청으로 인한 무한 루프, 메모리 부족, CPU 과다 사용을 야기하며, 중간 심각도의 소스 코드 노출 취약점(CVE-2025-55183, CVSS 5.3)도 발견되었다. 이전 패치는 불완전하여 19.0.3, 19.1.4, 19.2.3 버전 사용자도 재업그레이드가 필수이다.

  • 서비스 거부 취약점(CVE-2025-55184, CVE-2025-67779): 악의적으로 조작된 HTTP 요청을 서버 함수 엔드포인트로 보내면 React의 역직렬화 과정에서 무한 루프가 발생하여 서버 프로세스가 중단되고 CPU가 소비된다. 서버 함수를 직접 구현하지 않아도 React 서버 컴포넌트를 지원하면 취약할 수 있다.
  • 추가 DoS 취약점(CVE-2026-23864): 2026년 1월 26일 발견된 취약점으로, 특별히 제작된 HTTP 요청으로 인해 서버 충돌, 메모리 부족 예외, 과도한 CPU 사용이 발생할 수 있다. 원래 CVE-2025-55184에 대한 수정이 불완전해서 발생했으며, 19.0.4, 19.1.5, 19.2.4 버전에서 완전히 수정되었다.
  • 소스 코드 노출 취약점(CVE-2025-55183): 중간 심각도(CVSS 5.3)의 취약점으로, 공격자가 소스 코드를 노출시킬 수 있다. 자세한 내용은 수정 배포 완료 후 공개될 예정이다.
  • 불완전한 이전 패치: 19.0.3, 19.1.4, 19.2.3 버전으로 업그레이드한 사용자도 다시 업그레이드해야 한다. 원본 DoS 수정(CVE-2025-55184)이 불완전하여 추가 취약점이 남아있었기 때문이다.
  • 영향받는 버전 범위: react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack의 19.0.0부터 19.2.3까지 모든 버전(19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3)이 취약하며, 19.0.4, 19.1.5, 19.2.4로 업그레이드해야 한다.
  • React Native에 대한 특별 지침: 모노레포를 사용하는 경우에만 react-server-dom-* 패키지를 업그레이드하면 되며, react와 react-dom은 업그레이드할 필요가 없어서 버전 불일치 오류를 피할 수 있다.
  • 산업 전반의 보안 연구 패턴: 심각한 CVE 공개 후 연구자들이 인접 코드 경로를 조사하여 초기 완화를 우회하는 변형 공격 기법을 찾는 것이 일반적이다(Log4Shell 이후 추가 CVE가 보고된 사례 참고).

💡 이전 보안 업데이트로 완전히 보호되지 않았으므로 모든 React 서버 컴포넌트 사용자는 재차 업그레이드해야 하며, DoS 공격으로 인한 서비스 중단 위험에 직면해 있다.

Claude Opus 4.7 공개

⬆ 1.4kai

Anthropic이 고급 소프트웨어 엔지니어링에서 주목할 만한 개선을 이룬 Claude Opus 4.7을 일반 공개했다. 이 모델은 복잡한 장기 실행 작업을 정밀하게 처리하고, 자체 출력을 검증하는 능력을 보여주며, 이전에 밀접한 감시가 필요했던 어려운 코딩 작업을 자신감 있게 수행할 수 있다. 더 높은 해상도의 이미지 처리, 전문적 작업의 품질 향상, 여러 벤치마크에서 Opus 4.6보다 우수한 성능을 보이며, 사이버 보안 사용 사례에 대해서는 차등적으로 기능을 제한하는 안전장치가 포함되어 있다. 가격은 Opus 4.6과 동일하게 입력 토큰 100만 개당 $5, 출력 토큰 100만 개당 $25이다.

  • 소프트웨어 엔지니어링 성능 향상: Opus 4.7은 복잡한 장기 실행 작업에서 엄밀성과 일관성을 보여주며, 정확한 지침 이행과 자체 출력 검증 능력을 갖추고 있다. 사용자들이 이전에는 밀접한 감시가 필요했던 가장 어려운 코딩 작업을 Opus 4.7에 자신감 있게 위임할 수 있다고 보고했다.
  • 멀티모달 비전 능력 대폭 개선: 더 높은 해상도의 이미지를 처리할 수 있으며, 전문적인 작업에서 더 세련되고 창의적이어서 고품질의 인터페이스, 슬라이드, 문서를 생성한다.
  • 벤치마크 성능: Claude Mythos Preview보다는 덜 강력하지만 Opus 4.6을 여러 벤치마크에서 초과했다. 개발자들의 코딩 벤치마크(93개 작업)에서 Opus 4.6 대비 13% 해상도 향상을 기록했고, 4개의 작업에서는 Opus 4.6과 Sonnet 4.6 모두 풀지 못한 문제를 해결했다.
  • 다단계 작업 효율성: 연구 에이전트 벤치마크에서 가장 강력한 효율성 기준선을 보였으며, 6개 모듈 중 최고 종합 점수인 0.715를 기록했다. 금융(General Finance) 모듈에서 Opus 4.6 대비 0.767에서 0.813으로 향상되었고, 장기 컨텍스트 성능이 가장 일관성 있었다.
  • 사이버 보안 기능 제한 및 검증 프로그램: Claude Mythos Preview만큼 사이버 기능이 고급이 아니며, 훈련 중 차등적으로 이 기능을 감소시키는 실험을 진행했다. 금지된 고위험 사이버 보안 사용을 감지하고 차단하는 안전장치가 자동으로 포함되며, 합법적인 보안 연구와 침투 테스트를 위해서는 새로운 사이버 검증 프로그램에 참여할 수 있다.
  • 지연 시간과 지시 이행: 빠른 중앙값 지연 시간과 엄격한 명령 이행으로 복잡한 장기 실행 코딩 워크플로우에 특히 의미 있으며, 개발자들이 다단계 작업의 마찰을 줄여 개발 흐름에 집중할 수 있도록 한다.
  • 출시 및 접근성: 모든 Claude 제품, API, Amazon Bedrock, Google Cloud의 Vertex AI, Microsoft Foundry에서 사용 가능하며, Claude API를 통해 claude-opus-4-7로 접근할 수 있다.

Cloudflare의 AI 플랫폼: 에이전트를 위해 설계된 추론 계층

⬆ 222aidevops관련 2건

Cloudflare는 AI 모델 공급자에 종속되지 않고 여러 공급자의 70개 이상 모델에 접근할 수 있는 통합 추론 계층을 출시했다. 단일 API와 코드 한 줄로 OpenAI, Anthropic, Google 등 12개 이상 공급자의 모델을 전환할 수 있으며, 한곳에서 AI 비용을 통합 관리할 수 있다. Workers AI 바인딩을 통해 Cloudflare 호스팅 모델과 서드파티 모델 간 전환이 가능하고, REST API 지원도 곧 출시된다. 또한 사용자 정의 메타데이터를 통해 팀, 사용자, 워크플로우 별로 비용을 세분화하여 추적할 수 있으며, Replicate의 Cog 기술을 활용하여 커스텀 미세조정 모델을 가져올 수 있다.

  • 통합 API를 통한 다중 모델 접근: 70개 이상의 모델을 12개 이상의 공급자로부터 단일 AI.run() 바인딩으로 접근 가능하며, 모델 전환이 한 줄의 코드 변경으로 가능하다. 예를 들어 'anthropic/claude-opus-4-6'에서 '@cf/moonshotai/kimi-k2.5'로 전환하는 것이 간단하다.
  • 비용 통합 관리: 현재 기업들이 평균 3.5개 모델을 여러 공급자에서 사용하고 있지만, AI Gateway를 통해 한곳에서 모든 AI 지출을 모니터링할 수 있다. 메타데이터를 통해 유료/무료 사용자별, 고객별, 워크플로우별 비용 세분화가 가능하다.
