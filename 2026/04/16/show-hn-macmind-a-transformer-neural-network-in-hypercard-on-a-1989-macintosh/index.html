Show HN: MacMind – A transformer neural network in HyperCard on a 1989 Macintosh

📅 2026-04-16📰 Hacker News (Top)

요약

MacMind는 1987년 HyperTalk 언어로 1,216개 파라미터의 완전한 트랜스포머 신경망을 구현한 프로젝트로, 1989년 Macintosh SE/30에서 FFT의 비트 역순 순열을 학습한다. 모든 수학이 검사 가능하며 수정 가능하며, 더 큰 AI 모델도 동일한 원리로 작동함을 보여준다.

핵심 포인트

  • 토큰 임베딩, 위치 인코딩, 스케일된 닷프로덕트 self-attention, 교차 엔트로피 손실, 완전 백프로파게이션, 확률적 경사 하강법을 순수 HyperTalk으로 구현했다.
  • 학습된 attention 맵이 1965년 Cooley와 Tukey가 발견한 FFT 버터플라이 패턴을 자동으로 재발견했으며, 이는 대형 언어 모델과 동일한 메커니즘의 축소판이다.
  • 모든 코드가 검사 가능하고 수정 가능하며, 러닝 속도나 하드웨어가 아닌 수학 원리가 동일하다는 점을 증명한다.

왜 중요한가

💡
신경망과 백프로파게이션의 동작 원리를 직접 이해하고 검증할 수 있으며, 대형 AI 모델도 본질적으로 동일한 수학 원리로 작동함을 명확히 보여준다.

📄 전문 번역

MacMind: 1987년 매킨토시에서 구현한 트랜스포머 신경망

지난 35년간 잠들어있던 매킨토시 SE/30을 깨워 신경망을 학습시켰습니다. MacMind는 1,216개의 파라미터를 가진 단층 단일 헤드 트랜스포머로, 고전 푸리에 변환의 첫 단계인 비트 역순 순열(bit-reversal permutation)을 학습합니다. 매우 흥미로운 부분은 모든 코드가 1987년의 스크립팅 언어인 HyperTalk로 작성되었다는 점입니다. 원래 이 언어는 대화형 카드 스택을 만드는 용도였지, 행렬 계산을 하도록 설계된 언어가 아니었거든요.

신경망의 모든 구성 요소가 포함되어 있습니다. 토큰 임베딩, 위치 인코딩, 스케일된 닷 프로덕트 어텐션, 크로스 엔트로피 손실 함수, 완전한 역전파, 확률적 경사 하강법까지 전부요. 컴파일된 코드도 없고, 외부 라이브러리도 없고, 블랙박스도 없습니다. 모든 버튼을 Option 클릭하면 실제 수학 계산을 볼 수 있습니다.

이 프로젝트를 만든 이유

MacMind를 학습시킨 과정 — 순전파, 손실 계산, 역전파, 가중치 업데이트, 반복 — 이것은 오늘날 존재하는 모든 대규모 언어 모델을 학습시킨 과정과 정확히 동일합니다. 다른 점이 있다면 규모일 뿐, 원리는 같습니다. MacMind는 1,216개의 파라미터를 가졌고, GPT-4는 약 1조 개의 파라미터를 가졌을 뿐입니다. 수학은 동일합니다.

우리는 AI가 거의 모든 사람에게 영향을 미치지만, 거의 아무도 AI가 실제로 무엇을 하는지 이해하지 못하는 시대에 살고 있습니다. MacMind는 이 과정이 이해 가능하다는 것을 보여주려고 만들었습니다. 역전파와 어텐션은 마법이 아니라 수학이고, 이 수학은 TPU 클러스터에서 실행되든 1987년의 68000 프로세서에서 실행되든 상관없습니다.

모든 것을 검사할 수 있습니다. 모든 것을 수정할 수 있습니다. 학습률을 바꾸고, 학습 과제를 바꾸고, 모델 크기를 조정할 수 있습니다. 모두 HyperCard의 스크립트 편집기에서 직접 할 수 있다는 뜻입니다. 이것은 후드를 연 엔진입니다.

모델이 학습하는 것

비트 역순 순열은 각 위치의 인덱스를 이진 표현으로 바꾼 뒤 그것을 거꾸로 뒤집어서 순서를 재정렬합니다. 8개 원소 시퀀스를 예로 들면:

위치:    0 1 2 3 4 5 6 7
이진:    000 001 010 011 100 101 110 111
역순:    000 100 010 110 001 101 011 111
매핑:    0 4 2 6 1 5 3 7

따라서 입력 [3, 7, 1, 9, 5, 2, 8, 4]는 [3, 5, 1, 8, 7, 2, 9, 4]로 변환됩니다.

이 순열은 고속 푸리에 변환(FFT)의 첫 단계로, 컴퓨터 과학에서 가장 중요한 알고리즘 중 하나입니다. 흥미로운 점은 모델이 이 규칙을 명시적으로 배우지 않는다는 것입니다. 순수하게 자기 어텐션과 경사 하강법만으로 위치 패턴을 발견합니다. 훨씬 더 거대한 규모로 확대되면, 이것이 바로 더 큰 모델들이 언어를 이해하도록 학습된 방식입니다.

학습 완료 후 카드 4의 어텐션 맵을 보면 FFT의 나비 라우팅 패턴이 드러납니다. 모델이 독립적으로 발견한 이 수학적 구조는 1965년 Cooley와 Tukey가 발표한 것과 동일합니다.

HyperCard 스택 구성

MacMind는 5개 카드로 이루어진 HyperCard 스택입니다:

카드목적
1 -- Title프로젝트 이름과 크레딧
2 -- Training모델을 학습시키고 실시간으로 진행 상황 확인
3 -- Inference학습된 모델을 8자리 입력으로 테스트
4 -- Attention Map8x8 어텐션 가중치 매트릭스 시각화
5 -- About모델이 무엇을 하는지 일반 텍스트로 설명

학습하기 (카드 2)

"Train 10"을 클릭하면 10 스텝을 진행합니다. 모델이 완벽한 점수에 도달할 때까지 학습시키려면 "Train to 100%"를 클릭하세요. 더 오래 학습시키려면 "Train 10"을 반복해서 클릭하거나 "Train to 100%"를 다시 누르면 됩니다. 모델이 중단한 부분부터 다시 시작합니다.

더 긴 학습을 원한다면 메시지 박스(Cmd-M)를 열고 trainN 1000을 입력하면 1,000 스텝을 한 번에 실행할 수 있습니다.

각 스텝마다 무작위 8자리 시퀀스를 생성하고, 완전한 순전파를 실행하고, 크로스 엔트로피 손실을 계산한 뒤, 모든 레이어를 통해 경사를 역전파하고, 1,216개의 가중치를 모두 업데이트합니다. 진행률, 각 위치별 정확도, 학습 로그가 실시간으로 업데이트됩니다.

주의할 점이 하나 있습니다. 학습 로그 필드는 30,000자 제한이 있는데요(HyperCard의 제약), 약 900 스텝 후에는 로그가 가득 차서 HyperCard가 오류를 표시합니다. 이럴 땐 메시지 박스(Cmd-M)를 열고 다음을 입력하세요:

put "" into card field "trainingLog"

그 다음 trainN 500(또는 원하는 스텝 수)로 학습을 재개하면 됩니다.

테스트하기 (카드 3)

학습 완료 후 "New Random"을 클릭해서 테스트 입력을 생성하고, "Permute"를 클릭하면 학습된 모델을 실행합니다. 출력 행에는 모델의 예측이 표시되고, 신뢰도 행에는 각 위치에 대한 확신도가 표시됩니다.

결과를 직접 검증하려면 비트 역순 순열을 손으로 계산해보세요. 출력은 입력 위치를 다음 순서로 재정렬해야 합니다:

Output[0] = Input[0]    Output[4] = Input[1]
Output[1] = Input[4]    Output[5] = Input[5]
Output[2] = Input[2]    Output[6] = Input[3]
Output[3] = Input[6]    Output[7] = Input[7]

예를 들어 입력 [3, 7, 1, 9, 5, 2, 8, 4]는 [3, 5, 1, 8, 7, 2, 9, 4]를 만들어야 합니다. 모델이 잘 학습되었다면 모든 위치가 90% 이상의 신뢰도로 정확해야 합니다.

어텐션 맵 보기 (카드 4)

8x8 그리드는 각 출력 위치를 생성할 때 모델이 입력의 어느 위치에 주목하는지 시각화합니다. 학습 완료 후에는 나비 패턴이 보여야 합니다. 위치 0, 2, 5, 7은 자신에게만 집중하고(순열의 고정점), 위치 1과 4는 서로에게 집중하며, 위치 3과 6도 서로에게 집중합니다(서로 바뀜).

이것이 바로 1965년 Cooley와 Tukey가 고속 푸리에 변환을 위해 발견한 동일한 라우팅 구조입니다. 모델이 독립적으로 재발견한 수학적 패턴이죠.
