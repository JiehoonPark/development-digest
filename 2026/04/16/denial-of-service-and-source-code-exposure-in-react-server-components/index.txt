15:T14da,<h2>React Server Components의 서비스 거부 및 소스 코드 노출 취약점</h2>
<p>2025년 12월 11일 | React 팀</p>
<p>2026년 1월 26일 업데이트</p>
<p>지난주 발견된 심각한 취약점의 패치를 우회하려던 보안 연구자들이 React Server Components에서 두 가지 추가 취약점을 발견하고 공개했습니다. 다행히 이 새로운 취약점들은 원격 코드 실행(RCE)을 허용하지 않으며, 이전의 React2Shell 패치는 여전히 RCE 공격을 효과적으로 완화합니다.</p>
<p>새로 공개된 취약점은 다음과 같습니다:</p>
<ul>
<li><strong>서비스 거부 (높음)</strong>: CVE-2025-55184, CVE-2025-67779, CVE-2026-23864 (CVSS 7.5)</li>
<li><strong>소스 코드 노출 (중간)</strong>: CVE-2025-55183 (CVSS 5.3)</li>
</ul>
<p>새로 발견된 취약점의 심각성으로 인해 즉시 업그레이드를 권장합니다.</p>
<p>&gt; <strong>주의</strong>: 이전에 발표된 패치에 취약점이 있습니다. 이미 이전 취약점으로 업데이트했다면 다시 한 번 업데이트해야 합니다.</p>
<p>&gt;</p>
<p>&gt; 19.0.3, 19.1.4, 19.2.3 버전으로 업데이트했다면 불완전한 상태이므로 다시 업데이트해야 합니다.</p>
<h3>즉시 조치 필요</h3>
<p>이 취약점들은 CVE-2025-55182와 동일한 패키지 및 버전에 존재합니다.</p>
<p>다음 패키지의 19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3 버전이 영향을 받습니다:</p>
<ul>
<li>react-server-dom-webpack</li>
<li>react-server-dom-parcel</li>
<li>react-server-dom-turbopack</li>
</ul>
<p>위의 패키지 중 하나라도 사용 중이라면 즉시 19.0.4, 19.1.5, 19.2.4 버전 이상으로 업그레이드하세요.</p>
<p><strong>영향을 받지 않는 경우:</strong></p>
<ul>
<li>React 서버를 사용하지 않는 앱</li>
<li>React Server Components를 지원하는 프레임워크, 번들러, 번들러 플러그인을 사용하지 않는 앱</li>
</ul>
<p>&gt; <strong>참고</strong>: 심각한 CVE가 공개되면 보안 연구자들이 인접한 코드 경로를 살펴보며 초기 완화 방안을 우회할 수 있는 다른 공격 기법을 찾으려 합니다. Log4Shell 이후 추가 CVE들이 보고된 것처럼, 이런 패턴은 업계 전반에서 일어나는 일입니다. 추가 공개는 답답할 수 있지만, 전반적으로는 건강한 대응 주기의 신호입니다.</p>
<h3>영향을 받는 프레임워크 및 번들러</h3>
<p>다음의 React 프레임워크와 번들러가 취약한 React 패키지에 의존하거나 피어 의존성으로 포함합니다:</p>
<ul>
<li>next</li>
<li>react-router</li>
<li>waku</li>
<li>@parcel/rsc</li>
<li>@vite/rsc-plugin</li>
<li>rwsdk</li>
</ul>
<h3>호스팅 제공자의 임시 완화 조치</h3>
<p>여러 호스팅 제공자와 협력하여 임시 완화 조치를 적용했습니다. 다만 이에만 의존하지 마시고 즉시 업그레이드하세요.</p>
<h3>React Native 사용자</h3>
<p>monorepo나 react-dom을 사용하지 않는 React Native 사용자는 package.json에서 react 버전을 고정하면 추가 조치가 필요 없습니다.</p>
<p>monorepo에서 React Native를 사용 중이라면 다음 패키지가 설치되어 있을 경우만 업데이트하세요:</p>
<ul>
<li>react-server-dom-webpack</li>
<li>react-server-dom-parcel</li>
<li>react-server-dom-turbopack</li>
</ul>
<p>react와 react-dom은 업데이트할 필요가 없으므로 React Native의 버전 불일치 오류가 발생하지 않습니다.</p>
<h3>높음: 다중 서비스 거부 취약점</h3>
<p><strong>CVE</strong>: CVE-2026-23864</p>
<p><strong>CVSS 점수</strong>: 7.5 (높음)</p>
<p><strong>공개 날짜</strong>: 2026년 1월 26일</p>
<p>보안 연구자들이 React Server Components에 여전히 존재하는 추가 DoS 취약점을 발견했습니다.</p>
<p>Server Function 엔드포인트에 특수하게 조작된 HTTP 요청을 보내면 취약점이 트리거되며, 서버 충돌, 메모리 부족 예외, 또는 과도한 CPU 사용으로 이어질 수 있습니다. 이는 악용되는 취약점 코드 경로, 애플리케이션 구성, 애플리케이션 코드에 따라 달라집니다.</p>
<p>2026년 1월 26일 발표된 패치가 이 DoS 취약점을 완화합니다.</p>
<p>&gt; <strong>참고</strong>: 추가 수정사항 발표</p>
<p>&gt; CVE-2025-55184의 DoS를 해결하기 위한 원래의 수정이 불완전했습니다. 이로 인해 이전 버전들이 여전히 취약했습니다. 19.0.4, 19.1.5, 19.2.4 버전은 안전합니다.</p>
