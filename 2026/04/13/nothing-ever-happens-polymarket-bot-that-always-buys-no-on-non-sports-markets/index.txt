15:T12dd,<h2>Nothing Ever Happens Polymarket Bot</h2>
<p>Polymarket에서 &quot;아니오(No)&quot; 선택지에만 베팅하는 비동기 Python 봇입니다. 스포츠가 아닌 독립적인 이진 질문 시장을 대상으로 합니다.</p>
<p><strong>주의: 이 프로젝트는 엔터테인먼트 목적으로만 제공됩니다. 어떤 명시적 또는 암시적 보증도 없으며, 사용자의 책임하에 이용해야 합니다. 저자는 손실이나 피해에 대해 책임지지 않습니다.</strong></p>
<h3>프로젝트 구조</h3>
<ul>
<li><strong>bot/</strong>: 런타임, 거래소 클라이언트, 대시보드, 복구 로직, Nothing Happens 전략</li>
<li><strong>scripts/</strong>: 배포된 인스턴스와 로컬 환경에서 쓸 유틸리티 스크립트들</li>
<li><strong>tests/</strong>: 단위 테스트와 회귀 테스트</li>
</ul>
<h3>런타임 동작 원리</h3>
<p>봇은 독립적인 시장들을 스캔하다가 설정한 가격 이하의 &quot;아니오&quot; 선택지를 찾으면 구매합니다. 동시에 보유 중인 포지션을 추적하고, 대시보드를 통해 상태를 노출합니다. 주문 전송이 활성화되면 실시간 복구 상태를 저장해두기도 하죠.</p>
<h3>안전장치</h3>
<p>실제 주문을 체결하려면 세 가지 환경 변수가 <strong>모두</strong> 설정되어야 합니다:</p>
<pre><code>BOT_MODE=live
LIVE_TRADING_ENABLED=true
DRY_RUN=false</code></pre>
<p>이 중 하나라도 빠지면 봇은 종이 거래(PaperExchangeClient) 모드로 동작합니다.</p>
<p>라이브 모드에서 추가로 필요한 환경 변수:</p>
<ul>
<li><code>PRIVATE_KEY</code></li>
<li><code>FUNDER_ADDRESS</code> (서명 타입 1, 2일 때)</li>
<li><code>DATABASE_URL</code></li>
<li><code>POLYGON_RPC_URL</code> (프록시 지갑 승인과 환수 처리용)</li>
</ul>
<h3>설치</h3>
<pre><code class="language-bash">pip install -r requirements.txt
cp config.example.json config.json
cp .env.example .env</code></pre>
<p><code>config.json</code>은 의도적으로 로컬 파일이며 git에서 무시됩니다.</p>
<h3>설정</h3>
<p>런타임은 두 곳에서 설정을 읽습니다:</p>
<ul>
<li><strong>config.json</strong>: 비밀이 아닌 런타임 설정 (git 무시)</li>
<li><strong>.env</strong>: 비밀 정보와 런타임 플래그</li>
</ul>
<p>런타임 설정은 <code>strategies.nothing_happens</code> 하위에 있습니다. <code>config.example.json</code>과 <code>.env.example</code>을 참고하세요.</p>
<p>다른 설정 파일을 사용하려면 이렇게 지정하면 됩니다:</p>
<pre><code class="language-bash">CONFIG_PATH=/path/to/config.json</code></pre>
<h3>로컬 실행</h3>
<pre><code class="language-bash">python -m bot.main</code></pre>
<p>대시보드는 <code>$PORT</code> 또는 <code>DASHBOARD_PORT</code> 환경 변수에 바인딩됩니다.</p>
<h3>Heroku 배포</h3>
<p>셸 헬퍼 스크립트들은 앱 이름을 명시적으로 전달하거나 <code>HEROKU_APP_NAME</code> 환경 변수를 사용합니다:</p>
<pre><code class="language-bash">export HEROKU_APP_NAME=&lt;your-app&gt;
./alive.sh          # 봇 상태 확인
./logs.sh           # 로그 보기
./live_enabled.sh   # 라이브 모드 활성화
./live_disabled.sh  # 라이브 모드 비활성화
./kill.sh           # 봇 중지</code></pre>
<h4>배포 절차</h4>
<pre><code class="language-bash">heroku config:set BOT_MODE=live DRY_RUN=false LIVE_TRADING_ENABLED=true -a &quot;$HEROKU_APP_NAME&quot;
heroku config:set PRIVATE_KEY=&lt;key&gt; FUNDER_ADDRESS=&lt;addr&gt; POLYGON_RPC_URL=&lt;url&gt; DATABASE_URL=&lt;url&gt; -a &quot;$HEROKU_APP_NAME&quot;
git push heroku &lt;branch&gt;:main
heroku ps:scale web=1 worker=0 -a &quot;$HEROKU_APP_NAME&quot;</code></pre>
<p><strong>중요</strong>: web dyno만 실행해야 합니다. worker 엔트리는 실수로 시작될 경우 빠르게 실패하도록 만든 것입니다.</p>
<h3>테스트</h3>
<pre><code class="language-bash">python -m pytest -q</code></pre>
