27:T1b77,<h2>Craft: C/C++ 프로젝트를 위한 경량 빌드 도구</h2>
<p>Cargo가 Rust를 위한 것처럼, Craft는 C와 C++ 개발을 위한 도구입니다.</p>
<p>C와 C++ 개발이라고 하면 CMake와 씨름하고, 빌드 시스템을 설정하고, 의존성을 일일이 관리하는 고된 작업이 떠오르죠. Craft는 이런 번거로움을 없애줍니다. 간단한 <code>craft.toml</code> 파일에 프로젝트를 정의하면, Craft가 알아서 CMake 설정을 생성하고 의존성을 관리하고 현대적이고 매끄러운 명령줄 인터페이스를 제공합니다.</p>
<h3>Craft는 어떻게 작동할까요?</h3>
<ul>
<li><code>craft.toml</code>에 프로젝트를 정의합니다</li>
<li>Craft가 <code>CMakeLists.txt</code>를 생성합니다</li>
<li>의존성을 자동으로 가져옵니다</li>
<li>백그라운드에서 CMake가 프로젝트를 빌드합니다</li>
</ul>
<h3>빠른 시작</h3>
<pre><code class="language-bash"># 새 프로젝트 만들기
craft project my_app
cd my_app

# 의존성 추가
craft add --git https://github.com/raysan5/raylib.git --links raylib

# 빌드
craft build

# 실행
craft run</code></pre>
<h3>Craft가 해결하는 문제들</h3>
<h4>하나의 파일로 프로젝트 전체를 정의한다</h4>
<p>프로젝트를 간단하고 읽기 쉬운 설정 파일로 표현합니다.</p>
<pre><code class="language-toml">[project]
name = &quot;my_app&quot;
version = &quot;0.1.0&quot;
language = &quot;cpp&quot;
cpp_standard = 17

[build]
type = &quot;executable&quot;
include_dirs = [&quot;include&quot;]
source_dirs = [&quot;src&quot;]</code></pre>
<p>Craft가 이 설정을 바탕으로 <code>CMakeLists.txt</code>를 자동으로 생성해줍니다.</p>
<h4>의존성 관리가 정말 간단하다</h4>
<p>의존성을 추가하고 제거하는 것이 한 줄의 명령어면 됩니다.</p>
<pre><code class="language-bash">craft add --path ../PhysicsEngine              # 로컬 Craft 프로젝트
craft add --git https://github.com/raysan5/raylib --tag 5.5  # Git 의존성
craft remove raylib                             # 제거도 간단
craft update                                    # 모든 의존성 업데이트</code></pre>
<p>Git 저장소는 자동으로 복제되고, CMake 연결도 Craft가 처리합니다.</p>
<h4>템플릿으로 빠르게 프로젝트를 시작한다</h4>
<p>기존 프로젝트를 템플릿으로 저장했다가, 새 프로젝트를 그 템플릿으로부터 바로 생성할 수 있습니다.</p>
<pre><code class="language-bash">craft template save my_game_template
craft project new_game --template my_game_template</code></pre>
<p>Craft에는 실행 파일과 라이브러리를 위한 내장 템플릿도 포함되어 있습니다.</p>
<h3>설치하기</h3>
<h4>macOS &amp; Linux</h4>
<pre><code class="language-bash">curl -fsSL https://raw.githubusercontent.com/randerson112/craft/main/install.sh | sh</code></pre>
<h4>Windows (PowerShell)</h4>
<pre><code class="language-powershell">irm https://raw.githubusercontent.com/randerson112/craft/main/install.ps1 | iex</code></pre>
<p><strong>필수 요구사항:</strong> git, cmake</p>
<h4>설치 확인</h4>
<pre><code class="language-bash">craft --version
craft help</code></pre>
<h3>주요 명령어</h3>
<h4><code>craft project &lt;경로&gt;</code></h4>
<p>주어진 경로에 새 프로젝트 디렉토리를 생성합니다. <code>craft.toml</code>, <code>src/</code>, <code>include/</code> 폴더와 시작용 메인 파일이 포함됩니다.</p>
<pre><code class="language-bash">craft project my_app
craft project my_app --lang c
craft project my_lib --template static-library
craft project my_app --no-git</code></pre>
<h4><code>craft init [경로]</code></h4>
<p>현재 또는 지정된 디렉토리에서 Craft 프로젝트를 초기화합니다.</p>
<ul>
<li><strong>빈 디렉토리의 경우:</strong> <code>craft.toml</code>, <code>CMakeLists.txt</code>, <code>src/</code>, <code>include/</code>와 시작용 메인 파일을 생성합니다.</li>
<li><strong>기존 프로젝트의 경우:</strong> 디렉토리를 스캔해서 언어, 소스 디렉토리, 포함 디렉토리, 라이브러리를 감지합니다. 기존 소스 파일은 수정되지 않으며, 기존의 <code>CMakeLists.txt</code>는 <code>CMakeLists.backup.cmake</code>로 백업됩니다.</li>
</ul>
<pre><code class="language-bash">craft init
craft init my_app
craft init my_app --lang c
craft init --template static-library</code></pre>
<p>기존 프로젝트를 마이그레이션할 때는 다음처럼 의존성을 변환하면 됩니다.</p>
<pre><code class="language-bash"># 변환 전 → 변환 후
find_package(SFML ...) 
  → craft add --git https://github.com/SFML/SFML --links SFML::Graphics,...

add_subdirectory(../MyLib) 
  → craft add --path ../MyLib

FetchContent_Declare(...) 
  → craft add --git &lt;url&gt;</code></pre>
<p>그 외의 경우엔 <code>CMakeLists.extra.cmake</code>를 사용할 수 있습니다.</p>
<h4><code>craft build</code></h4>
<p><code>craft.toml</code>을 읽고, 필요하면 <code>CMakeLists.txt</code>를 다시 생성한 후, 없는 의존성을 가져오고, CMake를 사용해 프로젝트를 빌드합니다. 프로젝트의 어느 하위 디렉토리에서나 실행 가능하며, Craft가 자동으로 루트 디렉토리를 찾습니다.</p>
<pre><code class="language-bash">craft build</code></pre>
<h4><code>craft run [실행파일]</code></h4>
<p>프로젝트 실행 파일을 실행합니다. 기본값은 <code>craft.toml</code>의 프로젝트 이름입니다.</p>
<pre><code class="language-bash">craft run
craft run my_other_binary</code></pre>
<h4><code>craft gen &lt;파일&gt;</code></h4>
<p>보일러플레이트 코드가 포함된 파일을 생성합니다. 헤더는 첫 번째 include 디렉토리에, 소스는 첫 번째 source 디렉토리에 저장됩니다.</p>
<pre><code class="language-bash">craft gen utils.hpp
craft gen utils.cpp
craft gen math.h
craft gen math.c</code></pre>
<h4><code>craft clean</code></h4>
<p><code>build/</code> 디렉토리와 모든 컴파일된 파일을 제거합니다.</p>
<pre><code class="language-bash">craft clean</code></pre>
<h4><code>craft add</code></h4>
<p>프로젝트에 의존성을 추가합니다. Git 의존성은 자동으로 <code>.craft/deps/</code>에 복제되고, <code>CMakeLists.txt</code>가 즉시 다시 생성되므로 프로젝트는 항상 일관된 상태를 유지합니다.</p>
<pre><code class="language-bash"># 로컬 Craft 의존성
craft add --path ../my_lib

# 레지스트리의 Git 의존성
