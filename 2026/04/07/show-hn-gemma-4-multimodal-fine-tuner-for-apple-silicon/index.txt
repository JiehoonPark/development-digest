14:T1e45,<h2>Gemma 멀티모달 파인튜닝 도구</h2>
<p>Mac에서 텍스트, 이미지, 오디오로 Gemma를 파인튜닝하세요. 용량 걱정 없이요.</p>
<p>🖼️ <strong>이미지 + 텍스트 LoRA</strong> — 로컬 CSV에서 이미지 캡션 생성과 VQA</p>
<p>🎙️ <strong>오디오 + 텍스트 LoRA</strong> — Apple Silicon에서 네이티브로 돌아가는 유일한 방법</p>
<p>📝 <strong>텍스트 전용 LoRA</strong> — CSV의 명령어 또는 완성 작업</p>
<p>☁️ <strong>GCS / BigQuery 스트리밍</strong> — SSD를 채우지 않고 테라바이트 규모 학습</p>
<p>🍎 <strong>Apple Silicon 지원</strong> — MPS 네이티브, NVIDIA GPU 불필요</p>
<p>출처: github.com/mattmireles/gemma-tuner-multimodal (오픈소스)</p>
<h3>Gemma용 LoRA — 다른 도구와 뭐가 다른가요?</h3>

<p>H100을 임차하거나 테라바이트급 데이터를 노트북으로 복사하지 않고, 세 가지 모달리티 모두를 Apple Silicon에서 파인튜닝할 수 있는 유일한 도구입니다.</p>
<h3>지원하는 기능</h3>
<p><strong>텍스트 전용 파인튜닝</strong> (CSV의 명령어 또는 완성)은 설정 파일에서 <code>modality = text</code>로 지정하고, <code>data/datasets/&lt;name&gt;/</code> 아래 로컬 CSV 파일을 사용하면 됩니다. 자세한 내용은 아래 텍스트 전용 파인튜닝 섹션을 참고하세요.</p>
<p><strong>이미지 + 텍스트 파인튜닝</strong> (캡션 또는 VQA)은 <code>modality = image</code>, <code>image_sub_mode</code>, <code>image_token_budget</code>을 설정하고 로컬 CSV를 사용합니다. v1은 로컬 CSV만 지원합니다 (텍스트 전용과 동일한 제약).</p>
<p>내부 구조를 살펴보면, 이 도구는 Hugging Face Gemma 체크포인트와 PEFT LoRA를 활용합니다. 지도 학습은 <code>gemma_tuner/models/gemma/finetune.py</code>에서 진행되고, <code>gemma_tuner/scripts/export.py</code>로 병합된 Hugging Face 또는 SafeTensors 형식으로 내보냅니다. Core ML 변환과 GGUF 추론 도구는 README 가이드를 참고하세요. 이 저장소의 학습 경로는 의도적으로 Gemma만 지원합니다.</p>
<p>더 깊이 있는 내용: README/guides/README.md · README/specifications/Gemma3n.md</p>
<h3>이 도구로 만들 수 있는 것들</h3>
<p><strong>도메인별 ASR</strong> — 의료 구술, 법정 증언, 콜센터 녹음 등 특정 분야의 용어를 Whisper나 Gemma가 잘못 인식하는 경우, 자신의 데이터로 파인튜닝하면 정확도가 올라갑니다.</p>
<p><strong>도메인별 비전</strong> — 영수증, 차트, 스크린샷, 제조 결함, 의료 이미지 같이 일반 모델이 환각을 일으키는 시각 데이터를 정확히 처리할 수 있습니다.</p>
<p><strong>문서 및 화면 이해</strong> — 스크린샷에서 구조화된 출력으로 변환하는 쌍으로 학습하면 UI 에이전트, OCR 유사 파이프라인, 차트 QA에 활용할 수 있습니다.</p>
<p><strong>억양, 방언, 저자원 언어 적응</strong> — 기본 Gemma 모델을 과소 대표된 음성과 언어에 맞게 조정할 수 있습니다. 자체 레이블 오디오 데이터로 파인튜닝하면 되거든요.</p>
<p><strong>멀티모달 어시스턴트</strong> — Gemma의 텍스트 추론 능력을 이미지나 오디오로 보강해서 전사, 캡션 생성, Q&amp;A 파이프라인을 구축합니다.</p>
<p><strong>프라이빗 온디바이스 파이프라인</strong> — Mac에서만 학습하고 실행합니다. 데이터는 절대 외부로 나가지 않고, 가중치도 써드파티 API를 거치지 않습니다.</p>
<p>데이터가 GCS나 BigQuery에 있다면, 테라바이트를 로컬에 복사하지 않고 노트북에서 필요한 샤드만 스트리밍으로 받아서 모든 작업을 할 수 있습니다.</p>
<h3>지원하는 모델</h3>
<p>학습은 <code>config/config.ini</code>의 <code>base_model</code>로 로드된 Gemma 멀티모달 (텍스트 + 이미지 + 오디오) 체크포인트를 대상으로 합니다. 기본 설정 파일에 이런 <code>[model:…]</code> 항목들이 포함되어 있습니다 (Hub 가중치 위에 LoRA를 얹은 형태):</p>
<div class="table-wrapper"><table><thead><tr><th>config.ini의 모델 키</th><th>Hugging Face base_model</th><th>설명</th></tr></thead><tbody><tr><td>gemma-4-e2b-it</td><td>google/gemma-4-E2B-it</td><td>Gemma 4 명령어 모드, ~2B — requirements/requirements-gemma4.txt 필요 (설치 참고)</td></tr><tr><td>gemma-4-e4b-it</td><td>google/gemma-4-E4B-it</td><td>Gemma 4 명령어 모드, ~4B — Gemma 4 스택 필요</td></tr><tr><td>gemma-4-e2b</td><td>google/gemma-4-E2B</td><td>Gemma 4 기본 모델 — Gemma 4 스택 필요</td></tr><tr><td>gemma-4-e4b</td><td>google/gemma-4-E4B</td><td>Gemma 4 기본 모델 — Gemma 4 스택 필요</td></tr><tr><td>gemma-3n-e2b-it</td><td>google/gemma-3n-E2B-it</td><td>Gemma 3n 명령어 모드, ~2B — 기본 pip install -e . 설치 시 기본값</td></tr><tr><td>gemma-3n-e4b-it</td><td>google/gemma-3n-E4B-it</td><td>Gemma 3n 명령어 모드, ~4B</td></tr></tbody></table></div>
<p>다른 Gemma 3n 또는 Gemma 4 E2B–E4B 체크포인트가 필요하면 <code>[model:your-name]</code> 섹션을 추가하고 <code>group = gemma</code>와 호환되는 <code>base_model</code>을 지정하세요. Hugging Face의 더 큰 Gemma 4 가중치 (26B나 31B 클래스)는 이 학습기의 AutoModelForCausalLM 오디오 경로와 다른 Transformers 아키텍처를 사용하므로 아직 지원되지 않습니다.</p>
<p>시간과 메모리 추정값은 <code>gemma_tuner/wizard/base.py</code>의 <code>ModelSpecs</code>에서 가져옵니다.</p>
