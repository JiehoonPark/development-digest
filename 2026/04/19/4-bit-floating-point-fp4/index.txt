27:T191d,<h2>4비트 부동소수점 수: FP4 형식 알아보기</h2>
<h3>부동소수점 수의 진화</h3>
<p>예전에는 부동소수점 수를 32비트로 저장했어요. 그러다가 어느 시점부터 64비트가 표준이 되었습니다. C 언어는 이 역사를 고스란히 안고 있는데, <code>float</code>는 32비트 부동소수점을, <code>double</code>은 64비트 부동소수점을 가리킵니다. 반면 Python은 더 간단하게 <code>float</code> 하나로 C의 double에 해당하는 가장 일반적인 부동소수점 형식을 부릅니다.</p>
<p>프로그래머들은 32비트에서 64비트로의 전환을 환영했어요. 정밀도가 높을수록 좋으니까요. 게다가 32비트에서 64비트로 올라가면서 일부 수치 계산 문제들이 해결되기도 했습니다.</p>
<h3>신경망이 바꾼 게임의 규칙</h3>
<p>그런데 신경망의 등장으로 상황이 180도 바뀌었어요. 신경망은 엄청나게 많은 파라미터를 가지고 있거든요. 이제는 정밀도보다 <strong>메모리에 더 많은 파라미터를 담는 게 더 중요</strong>해졌습니다.</p>
<p>그래서 개발자들은 64비트 배정밀도가 아닌 16비트 반정밀도(half precision)를 원했고, 더 나아가 FP8(8비트)이나 FP4(4비트) 같은 극저정밀도 형식까지 찾기 시작했습니다. 이 글에서는 4비트 부동소수점 수, 즉 FP4를 살펴보겠습니다.</p>
<h3>정수는 안 될까?</h3>
<p>당연한 질문이 나올 법해요. &quot;정밀도가 필요 없으면 그냥 정수 쓰면 되지 않을까?&quot;</p>
<p>4비트 정수라면 0부터 15까지 표현할 수 있습니다. 바이어스를 도입해서 각 값에서 7을 빼면 −7부터 8까지 표현할 수 있죠. 하지만 부동소수점 형식을 쓰는 이유는 <strong>더 큰 동적 범위</strong>가 필요하기 때문입니다.</p>
<h3>FP4 형식의 구조</h3>
<p>4비트 부호 있는 부동소수점 수는 첫 번째 비트로 부호(sign)를 나타내고, 나머지 3비트를 지수(exponent)와 가수(mantissa)로 나눕니다. &quot;ExMy&quot;라는 표기법으로 x는 지수 비트, y는 가수 비트를 뜻해요.</p>
<p>부호 있는 4비트 수의 경우:</p>
<pre><code>x + y = 3</code></pre>
<p>다른 형식에서는 합이 더 클 수 있습니다. 예를 들어 8비트 부호 있는 부동소수점은 <code>x + y = 7</code>이죠.</p>
<p>4비트 부호 있는 수는 <strong>E3M0, E2M1, E1M2, E0M3</strong> 네 가지 조합이 가능합니다. 모두 어딘가에서는 쓰이지만, <strong>E2M1이 가장 일반적</strong>이며 Nvidia 하드웨어에서도 지원합니다.</p>
<h3>수 값을 계산하는 공식</h3>
<p>부호 비트 s, 지수 e, 가수 m을 가진 수의 값은 다음처럼 계산됩니다:</p>
<pre><code>(−1)^s × 2^(e−b) × (1 + m/2)</code></pre>
<p>여기서 b는 바이어스입니다. 바이어스는 e를 부호 있는 수로 만들지 않고도 양수와 음수 지수를 표현하기 위해 사용돼요. 예를 들어 b = 1이고 e = 1, 2, 3이라면, 지수 부분 2^(e−b)는 각각 1, 2, 4를 나타냅니다.</p>
<p>바이어스는 수의 범위에는 영향을 주지만, <strong>상대적인 간격에는 영향을 주지 않습니다</strong>. E3M0은 지수만 있고 가수가 없어서 값들이 로그 스케일에서 균등하게 분포하고, E0M3은 가수만 있어서 선형 스케일에서 균등합니다. 반면 E1M2와 E2M1은 로그와 선형 스케일 모두에서 불균등하게 분포해요.</p>
<h4>예외 처리: e = 0일 때</h4>
<p>위의 공식에는 예외가 있습니다. e = 0일 때는 m = 0이 0을, m = 1이 ½를 나타냅니다.</p>
<h3>E2M1 형식의 모든 값</h3>
<p>FP4는 단 16개의 서로 다른 값만 가질 수 있습니다. 모두 나열해볼까요:</p>
<pre><code>Bits  s exp m  Value
--------------------

0001  0 00  1  +0.5

0011  0 01  1  +1.5

<p>흥미롭게도 이렇게 작은 부동소수점 형식에도 +0과 −0 두 개의 영(zero)이 존재합니다. 일반 정밀도 부동소수점과 똑같네요.</p>
<h3>Python으로 실제로 확인해보기</h3>
<p>Pychop이라는 Python 라이브러리는 다양한 저정밀도 부동소수점 형식을 에뮬레이션합니다. 위의 표를 생성하는 코드는 다음과 같아요:</p>
<pre><code class="language-python">import pychop

    # Subnormal / zero
    if e == 0:
        return sign * (m / 2.0)
    # Normal
    return sign * (2.0 ** (e - 1)) * (1.0 + m / 2.0)

def display_value(bits: int, x: float) -&gt; str:
    if bits == 0b0000:
        return &quot;+0&quot;
    if bits == 0b1000:
        return &quot;-0&quot;
    return f&quot;{x:+g}&quot;

rows = []
for bits in range(16):
    s = (bits &gt;&gt; 3) &amp; 0b1
    e = (bits &gt;&gt; 1) &amp; 0b11
    m = bits &amp; 0b1
    x = e2m1_value(s, e, m)
    rows.append(
        {
            &quot;Bits&quot;: f&quot;{bits:04b}&quot;,
            &quot;s&quot;: s,
            &quot;exp_bits&quot;: f&quot;{e:02b}&quot;,
            &quot;m&quot;: m,
            &quot;Value&quot;: display_value(bits, x),

    )
