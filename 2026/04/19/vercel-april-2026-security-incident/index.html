Vercel 2026년 4월 보안 사건

Vercel April 2026 security incident

📅 2026-04-19📰 Hacker News (Top)⬆ 432
devopscareer

요약

2026년 4월 Vercel이 보안 침해 사건을 공개했습니다. 위협 행위자들이 Vercel 시스템에 무단 접근하여 내부 정보를 탈취했으며, 피해자들은 580명의 직원 정보(이름, 이메일, 계정 상태, 활동 타임스탬프), 액세스 키, 소스 코드, 데이터베이스 정보, 내부 배포 접근권, API 키(NPM 토큰 포함) 등을 노출했습니다. 위협 행위자는 이 데이터를 판매한다고 주장했으며, 사건의 원인은 제3자 AI 도구의 Google Workspace OAuth 애플리케이션 손상으로 파악되었습니다.

핵심 포인트

  • 제한된 고객층 영향: Vercel은 제한된 수의 고객이 영향을 받았다고 발표했으나, 내부 시스템에 무단 접근이 있었습니다.
  • 탈취된 데이터 항목: 580명의 직원 개인정보, API 키(NPM 토큰 및 GitHub 토큰 포함), 소스 코드, 데이터베이스 정보, 내부 배포 접근권 등이 노출되었습니다.
  • 위협 행위자의 주장: 'ShinyHunters'로 자칭하는 위협 행위자가 피해 데이터 판매를 주장했으며, 200만 달러의 몸값을 요구한 것으로 알려졌습니다.
  • 원인 규명: 제3자 AI 도구의 Google Workspace OAuth 애플리케이션 손상으로 인한 침해로 파악되었습니다(OAuth App ID: 110671459871-30f1spbu0hptbs60cb4vsmv79i7bbvqj.apps.googleusercontent.com).
  • Vercel 서비스 정상 운영: Vercel은 서비스 자체는 영향을 받지 않았으며, 영향받은 고객과 협력 중이라고 발표했습니다.
  • 권장 조치: 환경 변수 검토, 민감한 환경 변수 기능 사용, 필요시 암호 재설정 등의 조치를 권고했습니다.
  • 신원 미확인: BleepingComputer는 위협 행위자의 신원과 제공된 데이터의 진위성을 독립적으로 확인할 수 없었습니다.
  • 법 집행 기관 신고: Vercel이 법 집행 기관에 신고했으며, 조사 진행 중입니다.

왜 중요한가

💡
Vercel을 사용하는 개발자와 조직은 탈취된 데이터(특히 NPM 토큰과 GitHub 토큰)로 인한 추가 보안 위협을 평가하고, 민감한 자격증명을 관리하는 보안 관행을 즉시 검토해야 합니다.

📄 전문 번역

Vercel 보안 침해 공식 확인, 해커들이 탈취 데이터 판매 시도

개발자용 클라우드 플랫폼인 Vercel이 보안 침해 사건을 공식 인정했습니다. 공격자들이 시스템에 침입해 탈취한 데이터를 판매하려고 시도하고 있기 때문입니다.

Vercel은 어떤 회사인가

Vercel은 개발자들을 위한 호스팅 및 배포 인프라를 제공하는 클라우드 플랫폼입니다. 특히 JavaScript 프레임워크에 중점을 두고 있죠.

이 회사는 React 기반의 프레임워크인 Next.js를 개발했으며, 서버리스 함수, 엣지 컴퓨팅, CI/CD 파이프라인 같은 서비스를 제공합니다. 개발자들이 애플리케이션을 만들고 미리보기하고 배포할 수 있게 지원하는 거죠.

침해 사건의 내용

Vercel은 오늘 발표한 보안 공지에서 제한된 수의 고객사가 영향을 받았다고 밝혔습니다.

> "당사는 Vercel의 특정 내부 시스템에 대한 무단 접근을 포함하는 보안 사건을 확인했습니다. 현재 적극적으로 조사 중이며 사건 대응 전문가들을 투입했습니다. 수사 기관에도 신고했으며 조사 진행 상황을 계속 업데이트할 예정입니다."

Vercel은 자신들의 서비스 자체는 영향을 받지 않았다고 강조했습니다. 영향받은 고객사들과 함께 대응하고 있다고 했죠.

회사는 고객들에게 환경 변수를 검토하고, 민감한 환경 변수 기능을 사용하며, 필요하면 시크릿 정보를 재발급할 것을 권장했습니다.

침해의 원인은 제3자 AI 도구

기사 발표 이후 Vercel은 공지를 추가 업데이트했습니다. 침해가 제3자 AI 도구의 Google Workspace OAuth 애플리케이션이 손상되면서 발생했다고 말한 겁니다.

어느 회사의 OAuth 애플리케이션인지는 아직 명확하지 않습니다. 다만 Vercel은 Google Workspace 관리자와 계정 소유자들에게 다음 애플리케이션을 확인할 것을 권고했습니다:

OAuth App: 110671459871-30f1spbu0hptbs60cb4vsmv79i7bbvqj.apps.googleusercontent.com

해커의 주장

공격자가 "ShinyHunters"라는 이름으로 해킹 포럼에 게시한 글에 따르면 Vercel을 침해했다고 주장했습니다. 다만 최근 ShinyHunters 그룹으로 알려진 공격자들과 연결된 위협 행위자들은 자신들이 이번 사건과 무관하다고 부인했습니다.

해커는 Vercel에서 탈취한 접근 키, 소스 코드, 데이터베이스 데이터, 내부 배포 접근 권한, API 키를 판매하겠다고 주장했습니다.

포럼 게시물에는 이렇게 적혀 있습니다:

> "Linear에서 나온 것들도 있지만, 내가 제공하려는 접근 권한에는 여러 내부 배포에 접근할 수 있는 직원 계정, API 키(NPM 토큰과 GitHub 토큰 포함)가 포함되어 있습니다."

공격자는 또한 580개 레코드를 담은 Vercel 직원 정보 텍스트 파일도 공유했습니다. 이 파일에는 이름, Vercel 이메일 주소, 계정 상태, 활동 타임스탐프가 포함되어 있었죠. 내부 Vercel Enterprise 대시보드로 보이는 스크린샷도 함께 공개했습니다.

현재까지 탈취 데이터나 스크린샷의 진위 여부는 독립적으로 확인되지 않았습니다.

몸값 협상 주장

Telegram에서 공유한 메시지에 따르면 공격자는 Vercel과 접촉했으며 200만 달러의 몸값을 요구했다고 주장했습니다.

Vercel에 민감한 데이터나 자격 증명 유출 여부, 공격자와의 협상 진행 상황 등 추가 질문을 던졌으며, 답변이 오면 기사를 업데이트할 예정입니다.
