Claude 코드 품질 저하 사건 분석 및 해결

An update on recent Claude Code quality reports

📅 2026-04-23📰 Hacker News (Top)
ai

요약

Anthropic은 지난 한 달간 일부 사용자들의 Claude 응답 품질 저하 보고를 조사했으며, Claude Code, Claude Agent SDK, Claude Cowork에 영향을 미친 세 가지 별개의 이슈를 발견하고 2024년 4월 20일에 모두 해결했습니다. API 레이어는 영향을 받지 않았습니다.

핵심 포인트

  • Claude Code의 기본 reasoning effort가 3월 4일 high에서 medium으로 변경되었으나, 사용자 피드백에 따라 4월 7일 xhigh(Opus 4.7)와 high(기타 모델)로 복구되었습니다. 이 변경은 UI 동결 지연 문제를 해결하려는 의도였으나 지능도 함께 저하되는 부작용이 발생했습니다.
  • 3월 26일에 배포된 prompt caching 최적화에서 1시간 이상 유휴 세션의 reasoning을 제거하는 기능이 버그로 인해 매 턴마다 실행되어 Claude가 건망증 있고 반복적으로 동작하는 것처럼 보였으며, 4월 10일에 수정되었습니다.
  • 4월 16일에 추가된 verbosity 감소 시스템 프롬프트가 다른 프롬프트 변경과 함께 코딩 품질을 해쳤으며, 4월 20일에 되돌려졌습니다. 이는 Sonnet 4.6, Opus 4.6, Opus 4.7에 영향을 미쳤습니다.
  • 각 변경사항이 서로 다른 트래픽 슬라이스와 일정에 영향을 미쳐 전체적으로 광범위하고 일관성 없는 저하로 보였으며, 초기 단계에서는 일반적인 사용자 피드백 변동과 구분하기 어려웠습니다.
  • Anthropic은 모든 구독자의 사용량 제한을 4월 23일부터 초기화했으며, 향후 유사 문제 예방을 위한 개선 방안을 약속했습니다.

왜 중요한가

💡
Claude Code 사용자들이 겪은 품질 저하 원인을 명확히 이해할 수 있으며, Anthropic의 문제 대응 및 개선 체계를 평가할 수 있는 중요한 사례입니다.
