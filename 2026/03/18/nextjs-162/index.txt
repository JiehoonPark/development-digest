<h3>주요 개선사항</h3>
<p><strong>개발 서버 시작 속도</strong></p>

<li>이전 버전 대비 87% 더 빠른 성능으로, localhost:3000이 준비되는 시간이 대폭 단축되었어요.</li>

<li>React에 기여한 개선사항으로 Server Components 페이로드 역직렬화가 최대 350% 빨라졌거든요.</li>
</ul>
<p><strong>기본 에러 페이지 리디자인</strong></p>
<ul>

</ul>

<ul>

<h3>렌더링 성능 개선 상세</h3>
<p>React 팀과의 협업으로 Server Components 역직렬화 성능을 크게 개선했습니다. 이전 방식은 JSON 파싱 시 reviver 콜백을 사용했는데, 이때마다 V8의 C++/JavaScript 경계를 넘나들며 성능 저하가 발생했거든요.</p>
<p>새로운 접근 방식은 두 단계로 진행됩니다:</p>
<ul>
<li>순수 <code>JSON.parse()</code>로 기본 파싱</li>
<li>JavaScript의 재귀적 순회로 변환 처리</li>
</ul>
<p>이렇게 하면 경계 넘나드는 오버헤드가 제거되고, 변환이 필요 없는 문자열은 건너뛰는 최적화도 함께 적용됩니다.</p>
<p>실제 Next.js 애플리케이션에서는 RSC 페이로드 크기에 따라 HTML 렌더링이 25~60% 빨라집니다.</p>
