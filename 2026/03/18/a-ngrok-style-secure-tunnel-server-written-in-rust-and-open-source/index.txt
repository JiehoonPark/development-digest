28:T1c95,<h2>rustunnel: Rust로 만든 ngrok 스타일의 터널 서버</h2>
<p>로컬 서비스를 공개 서버를 통해 외부에 노출시키고 싶으신가요? rustunnel은 암호화된 WebSocket, TLS 종료, HTTP/TCP 프록시, 실시간 대시보드, Prometheus 메트릭, 감사 로깅을 모두 제공하는 Rust 기반 터널 서버입니다. 직접 운영하거나 우리의 관리형 서비스를 사용할 수 있어요.</p>
<h3>목차</h3>

<li>로컬에서 클라이언트 실행</li>
<li>Git 훅</li>
<li>프로덕션 배포 (Ubuntu / systemd)</li>
<li>Docker 배포</li>
<li>클라이언트 설정</li>
<li>REST API</li>
<li>AI 에이전트 통합 (MCP 서버)</li>
<li>모니터링</li>
<li>로드맵</li>
<li>기여하기</li>
<li>라이선스</li>
<li>문의</li>
</ul>
<h3>호스팅 서비스</h3>
<p>직접 서버를 운영하고 싶지 않다면? 우리가 운영하는 공개 엣지 서버 <code>edge.rustunnel.com</code>을 바로 사용할 수 있습니다.</p>
<h4>이용 가능한 서버</h4>
<div class="table-wrapper"><table><thead><tr><th>서버</th><th>리전</th><th>컨트롤 플레인</th><th>상태</th></tr></thead><tbody><tr><td>edge.rustunnel.com</td><td>유럽 (헬싱키)</td><td>:4040</td><td>운영 중</td></tr></tbody></table></div>
<p>더 많은 리전이 곧 추가될 예정이니, 프로젝트를 팔로우해두세요.</p>
<h4>인증 토큰 발급받기</h4>
<p>호스팅 서비스를 사용하려면 인증 토큰이 필요합니다. 다음과 같이 요청하세요.</p>
<ul>
<li>GitHub에서 &quot;Token request&quot;라는 제목으로 이슈 열기</li>
<li>본문에 이메일 주소나 Discord 사용자명 기재</li>
<li>토큰을 개인 메시지로 전송받기</li>
</ul>
<p>현재 서비스가 초기 단계라 토큰은 수동으로 발급하고 있습니다.</p>
<h4>호스팅 서버로 빠르게 시작하기</h4>
<p>토큰을 받았다면 설정 마법사를 실행하세요.</p>
<pre><code class="language-bash">rustunnel setup
# Server address [edge.rustunnel.com:4040]: (Enter 키)
# Auth token: &lt;토큰 붙여넣기&gt;</code></pre>
<p>이제 로컬 서비스를 외부에 노출시킬 수 있습니다.</p>
<pre><code class="language-bash"># HTTP 터널 — 로컬 포트 3000에 공개 HTTPS URL 할당
rustunnel http 3000

# 커스텀 서브도메인
rustunnel http 3000 --subdomain myapp

# TCP 터널 — 예: 로컬 데이터베이스 노출
rustunnel tcp 5432</code></pre>
<p>터널이 연결되면 클라이언트가 공개 URL을 바로 출력해줍니다.</p>
<pre><code>✓ tunnel open https://abc123.edge.rustunnel.com</code></pre>

<h3>필수 요구사항</h3>
<h4>빌드 환경</h4>
<div class="table-wrapper"><table><thead><tr><th>요구사항</th><th>버전</th><th>참고</th></tr></thead><tbody><tr><td>Rust 도구체인</td><td>1.76+</td><td>rustup으로 설치</td></tr><tr><td>pkg-config</td><td>모든 버전</td><td>reqwest (TLS)에 필요</td></tr><tr><td>libssl-dev</td><td>모든 버전</td><td>Debian/Ubuntu: <code>apt install libssl-dev</code></td></tr><tr><td>Node.js + npm</td><td>18+</td><td>대시보드 UI 재빌드 시에만 필요</td></tr></tbody></table></div>
<h4>프로덕션 서버 운영</h4>
<div class="table-wrapper"><table><thead><tr><th>요구사항</th><th>설명</th></tr></thead><tbody><tr><td>Linux (Ubuntu 22.04+)</td><td>systemd 서비스 포함</td></tr><tr><td>TLS 인증서 + 개인키</td><td>PEM 형식 (Let's Encrypt 권장)</td></tr><tr><td>공개 IP / DNS</td><td>HTTP 터널용 와일드카드 DNS <code>*.tunnel.yourdomain.com</code> → 서버 IP 필요</td></tr></tbody></table></div>
<h3>로컬 개발 환경 설정</h3>
<h4>빌드하기</h4>
<pre><code class="language-bash"># 저장소 복제
git clone https://github.com/joaoh82/rustunnel.git
cd rustunnel

# 모든 워크스페이스 크레이트 컴파일 (디버그 모드)
cargo build --workspace

# 또는 Makefile 바로가기 사용
make build</code></pre>
<h4>테스트 실행하기</h4>
<p>통합 테스트는 실제 서버를 임의의 포트에서 띄워서 인증, HTTP 터널, TCP 터널, 재연결 로직을 검증합니다. PostgreSQL이 실행 중이어야 해요.</p>
<pre><code class="language-bash"># 로컬 PostgreSQL 컨테이너 시작 (기기당 한 번, 재부팅 후에도 유지됨)
make db-start

# 전체 테스트 스위트 (단위 + 통합)
make test

# 출력 내용 표시하면서 실행
TEST_DATABASE_URL=postgres://rustunnel:test@localhost:5432/rustunnel_test \
cargo test --workspace -- --nocapture

-subj &quot;/CN=localhost&quot;</code></pre>
