Glassworm 귀환: GitHub와 npm을 노리는 새로운 보이지 않는 유니코드 공격

⬆ 15

Glassworm 위협 행위자가 보이지 않는 유니코드 문자를 이용해 GitHub, npm, VS Code 마켓플레이스를 대규모로 공격하고 있습니다. 2025년 10월부터 추적된 이 공격은 3월 3-9일 사이에 최소 151개의 GitHub 저장소를 감염시켰으며, Wasmer, Reworm, OpenCode 등 유명 프로젝트도 포함되어 있습니다. 공격자는 눈에 띄지 않는 바이트 디코더를 빈 문자열에 숨겨 eval()을 통해 악성 페이로드를 실행하는 방식을 사용하고 있습니다.

  • 보이지 않는 유니코드 문자(U+FE00~U+FE0F 범위)를 이용해 eval() 호출로 디코딩되는 악성 페이로드를 삽입
  • GitHub 코드 검색 결과 최소 151개 저장소 감염 확인, 실제 규모는 더 클 것으로 예상
  • 공격이 GitHub, npm, VS Code 마켓플레이스로 확대되어 생태계 전반에 영향
  • AI를 사용한 위장으로 보이는 일반적인 커밋(문서 수정, 버전 업데이트)과 함께 배포되어 시각적 코드 리뷰로 탐지 불가
  • 표준 린팅 도구와 수동 코드 리뷰로는 감지 불가능하며 전문 악성코드 스캔 도구 필요

💡 보이지 않는 공격 기법이 대규모 오픈소스 공급망을 위협하고 있으며, 개발자는 자신의 저장소와 의존성 보안을 위해 능동적인 방어 도구 도입이 필수입니다.

바이브코딩 프로토타입에서 실제 제품까지 100시간의 간격

⬆ 114

저자는 AI 기반 코딩(바이브코딩)으로 Cryptosaurus 앱을 만드는 과정을 기록했으며, 30분에 앱을 완성한다는 주장은 단순 복제나 버그 많은 결과물이거나 engagement farming일 가능성이 높다고 결론지었습니다. 프로토타입은 1시간 만에 완성되었지만, UI/UX 개선, 이미지 생성 품질 최적화(프롬프트 274줄, 200회 이상 반복), 인프라 구축 등으로 총 100시간이 소요되었습니다. 저자는 Figma 같은 도구를 제대로 사용했다면 훨씬 빠르게 완성했을 수 있다고 지적합니다.

  • 바이브코딩은 프로토타입 단계에서는 빠르지만(1시간), 프로덕션 품질의 제품으로 가는 과정에서 대량의 반복 작업이 필요
  • 이미지 생성 프롬프트 최적화만 200회 이상 반복되었으며, 엣지 케이스 대응을 위해 274줄의 프롬프트 파일 필요
  • AI 코딩이 장점을 가지려면 인프라 설정, UI/UX 다듬기 등 기술적 기초 지식이 전제되어야 함
  • 전문 도구(Figma)를 우회하고 AI만 사용하려다 보니 '단순히 빌드하는' 방식이 오히려 비효율적이었음

💡 AI 코딩 도구의 실제 생산성 한계와 올바른 활용 방식을 보여주며, 마케팅된 '30분 앱 개발' 같은 주장의 현실성을 평가할 수 있게 해줍니다.
