15:T1c2e,<h2>SentrySearch: 대시캠 영상에서 원하는 장면을 한 번에 찾기</h2>
<p>대시캠 영상이 쌓이다 보면 필요한 장면을 찾기가 정말 번거롭죠. SentrySearch는 이 문제를 완전히 다르게 접근합니다. 텍스트로 찾고 싶은 상황을 설명하면, AI가 영상을 의미론적으로 이해해서 정확히 그 부분만 잘라낸 클립을 돌려줍니다.</p>
<h3>동작 원리</h3>
<p>SentrySearch의 핵심은 Google의 Gemini Embedding 모델을 활용한 것입니다. 대시캠 영상을 겹치는 구간으로 나눈 뒤, 각 영상 청크를 벡터로 임베딩합니다. 이 벡터들은 로컬 ChromaDB에 저장되죠.</p>
<p>검색할 때는 당신의 텍스트 쿼리도 같은 벡터 공간에 임베딩되고, 저장된 영상 벡터들과 비교됩니다. 가장 유사한 영상 청크를 찾으면 자동으로 원본 파일에서 해당 부분을 잘라내 클립으로 저장하는 방식이에요.</p>
<h3>설치하기</h3>
<p>먼저 저장소를 클론하고 파이썬 환경을 준비하세요:</p>
<pre><code class="language-bash">git clone https://github.com/ssrajadh/sentrysearch.git
cd sentrysearch
python -m venv venv &amp;&amp; source venv/bin/activate
pip install -e .</code></pre>
<h3>API 키 설정</h3>
<p>다음 명령으로 Gemini API 키를 등록합니다:</p>
<pre><code class="language-bash">sentrysearch init</code></pre>
<p>Gemini API 키를 입력하면 <code>.env</code> 파일에 자동으로 저장되고, 테스트 임베딩으로 검증됩니다.</p>
<p>수동으로 설정하고 싶다면 <code>.env.example</code>을 <code>.env</code>로 복사한 후 [aistudio.google.com/apikey](https://aistudio.google.com/apikey)에서 발급받은 키를 직접 입력해도 됩니다.</p>
<p>참고로 영상 처리에는 ffmpeg이 필요한데, 시스템에 없으면 함께 제공되는 imageio-ffmpeg을 자동으로 사용합니다.</p>
<h3>사용 방법</h3>
<h4>초기 설정</h4>
<pre><code class="language-bash">$ sentrysearch init
Enter your Gemini API key (get one at https://aistudio.google.com/apikey): ****
Validating API key...
Setup complete. You're ready to go — run `sentrysearch index &lt;directory&gt;` to get started.</code></pre>
<p>이미 설정된 키가 있으면 덮어쓸지 물어봅니다.</p>
<h4>영상 인덱싱</h4>
<pre><code class="language-bash">$ sentrysearch index /path/to/dashcam/footage
Indexing file 1/3: front_2024-01-15_14-30.mp4 [chunk 1/4]
Indexing file 1/3: front_2024-01-15_14-30.mp4 [chunk 2/4]
...
Indexed 12 new chunks from 3 files. Total: 12 chunks from 3 files.</code></pre>
<p>주요 옵션들입니다:</p>
<ul>
<li><code>--chunk-duration 30</code> — 청크 길이 (초)</li>
<li><code>--overlap 5</code> — 청크 간 겹침 구간 (초)</li>
<li><code>--no-preprocess</code> — 전처리 스킵 (원본 품질로 전송)</li>
<li><code>--target-resolution 480</code> — 전처리 시 목표 높이 (픽셀)</li>
<li><code>--target-fps 5</code> — 전처리 시 목표 프레임레이트</li>
<li><code>--no-skip-still</code> — 움직임이 없는 청크도 모두 인덱싱</li>
</ul>
<h4>검색하기</h4>
<pre><code class="language-bash">$ sentrysearch search &quot;red truck running a stop sign&quot;
#1 [0.87] front_2024-01-15_14-30.mp4 @ 02:15-02:45
#2 [0.74] left_2024-01-15_14-30.mp4 @ 02:10-02:40
#3 [0.61] front_2024-01-20_09-15.mp4 @ 00:30-01:00
Saved clip: ./match_front_2024-01-15_14-30_02m15s-02m45s.mp4</code></pre>
<p>검색 점수가 높은 순서로 결과가 표시되고, 가장 유사한 부분이 자동으로 클립으로 저장됩니다.</p>
<p>옵션: <code>--results N</code> (결과 개수), <code>--output-dir DIR</code> (저장 경로), <code>--no-trim</code> (자동 자르기 비활성화)</p>
<h4>통계 확인</h4>
<pre><code class="language-bash">$ sentrysearch stats
Total chunks: 47
Source files: 12</code></pre>
<p><code>--verbose</code> 플래그를 추가하면 임베딩 차원, API 응답 시간, 유사도 점수 같은 디버그 정보를 볼 수 있습니다.</p>
<h3>어떻게 이게 가능할까?</h3>
<p>Gemini Embedding 2는 영상을 직접 처리할 수 있습니다. 영상 픽셀을 텍스트 쿼리와 동일한 768차원 벡터 공간으로 투영하는 거죠. 음성 인식도 없고, 프레임 캡셔닝도 없고, 중간 텍스트 변환 과정도 없습니다.</p>
<p>&quot;빨간색 트럭이 정지 신호를 무시함&quot;이라는 텍스트 쿼리가 30초 영상 클립과 벡터 수준에서 직접 비교되는 방식이에요. 이것이 바로 수 시간의 영상을 1초 이내에 검색할 수 있는 비결입니다.</p>
<h3>비용</h3>
<p>기본 설정(30초 청크, 5초 겹침)으로 1시간의 영상을 인덱싱하는 데 약 $2.50이 듭니다. API는 영상 길이 기준으로 청구되므로, 비용은 청크 개수에 따라 결정됩니다.</p>
<p>비용을 줄이는 두 가지 내장 최적화가 있습니다:</p>
<p><strong>전처리 (기본 활성화)</strong> — 청크를 480p 5fps로 다운스케일합니다. 업로드 크기와 토큰 사용량을 줄이지만, API 호출 횟수는 줄이지 않으므로 속도 개선에 더 도움됩니다.</p>
<p><strong>정적 프레임 스킵 (기본 활성화)</strong> — 움직임이 없는 청크(주차된 자동차 같은)는 완전히 건너뜁니다. 실제 API 호출을 줄이므로 비용이 직접 감소합니다. 절감액은 영상 유형에 따라 달라지는데, 유휴 시간이 많은 Tesla Sentry Mode 녹화에서 가장 큰 효과를 봅니다.</p>
<p>검색 쿼리 비용은 무시할 수준입니다(텍스트 임베딩만 필요).</p>
<h4>비용 최적화 옵션</h4>
<ul>
<li><code>--chunk-duration</code> / <code>--overlap</code> — 더 긴 청크에 적은 겹침 = API 호출 감소 = 비용 절감</li>
<li><code>--no-skip-still</code> — 정적 프레임도 포함해 모든 청크 인덱싱</li>
<li><code>--target-resolution</code> / <code>--target-fps</code> — 전처리 품질 조정</li>
<li><code>--no-preprocess</code> — 원본 품질로 API에 전송</li>
</ul>
<h3>제한 사항과 향후 개선</h3>
<p>정적 프레임 감지는 휴리스틱 기반입니다. 샘플링된 프레임의 JPEG 파일 크기를 비교하는 방식인데, 미묘한 움직임이 있는 청크를 가끔 건너뛸 수도 있고, 진정한 정적 장면을 임베딩할 수도 있습니다. 모든 청크를 인덱싱하고 싶으면 <code>--no-skip-still</code>을 사용하세요.</p>
<p>검색 품질은 청크 경계에 영향을 받습니다. 사건이 두 청크에 걸쳐 있으면 겹치는 구간이 도움이 되지만 완벽하지는 않습니다. 장면 감지 같은 더 정교한 청킹 방식으로 개선할 수 있을 것 같습니다.</p>
<p>Gemini Embedding 2는 아직 프리뷰 상태라 API 동작과 가격이 변할 수 있습니다.</p>
