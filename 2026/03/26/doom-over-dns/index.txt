26:T133a,<h2>DNS로 DOOM 플레이하기</h2>
<p>누군가는 &quot;DNS는 도메인을 IP로 변환하는데, 또 뭘 할 수 있을까?&quot;라는 질문을 던졌습니다. 그 답은 놀랍게도 DOOM을 실행하는 것입니다.</p>
<p>DNS TXT 레코드는 임의의 텍스트를 저장할 수 있거든요. Cloudflare는 이를 전 세계에 무료로 제공하고, 엣지에서 캐시하며, 누구나 조회할 수 있게 해줍니다. 물론 이건 파일 저장소가 아닙니다. 설계 목적도 파일 저장이 아니었어요. RFC 1035를 작성한 IETF 누구도 TXT 레코드를 파일 저장소로 쓸 거라고 생각하지 않았을 겁니다. 그럼에도 우리는 여기 있습니다.</p>
<p>이 프로젝트는 셰어웨어 DOOM 전체를 압축한 뒤 약 1,964개의 DNS TXT 레코드로 쪼개어 하나의 Cloudflare 존에 올립니다. 그리고 PowerShell 스크립트와 공개 DNS 쿼리만으로 런타임에 플레이합니다. WAD 파일은 디스크를 거치지 않으며, .NET 게임 엔진 DLL들은 메모리에 직접 로드됩니다.</p>
<p>결국 모든 것이 DNS였던 거죠.</p>
<h3>빠른 시작</h3>
<h4>플레이하기</h4>
<pre><code class="language-powershell"># 1. PowerShell 7 설치 (없으면)
winget install Microsoft.PowerShell

# 2. DOOM 플레이
.\Start-DoomOverDNS.ps1 -PrimaryZone 'example.com'</code></pre>
<p>끝입니다. 나머지는 모두 자동으로 <code>Resolve-DNSName</code>을 통해 DNS에서 가져옵니다.</p>
<h4>업로드하기</h4>
<pre><code class="language-powershell"># 1. 게임 엔진 빌드
cd managed-doom
dotnet publish ManagedDoom/ManagedDoom.csproj -c Release -f net8.0 -o publish-out

# 2. Cloudflare 인증 정보 로드
Import-Module .\TXTRecords\TXTRecords.psm1
Set-CFCredential -ApiToken (Read-Host 'API Token' -AsSecureString)

# 3. DNS에 업로드
.\Publish-DoomOverDNS.ps1 `
-PublishDir 'managed-doom\publish-out' `
-WadPath 'DOOM1.WAD' `
-Zones @('example.com')</code></pre>
<h3>상세 설정</h3>
<h4>Start-DoomOverDNS.ps1</h4>
<div class="table-wrapper"><table><thead><tr><th>파라미터</th><th>기본값</th><th>설명</th></tr></thead><tbody><tr><td>-PrimaryZone</td><td>(필수)</td><td>stripe-meta 레코드가 있는 DNS 존</td></tr><tr><td>-DnsServer</td><td>시스템 기본값</td><td>특정 DNS 리졸버 IP (예: '1.1.1.1')</td></tr><tr><td>-WadName</td><td>'doom1'</td><td>WAD 타입: doom1, doom, doom2, plutonia, tnt</td></tr><tr><td>-DoomArgs</td><td>''</td><td>엔진으로 전달할 인자 (예: '-warp 1 3 -skill 5')</td></tr><tr><td>-WadPrefix</td><td>'doom-wad'</td><td>WAD stripe용 DNS 프리픽스</td></tr><tr><td>-LibsPrefix</td><td>'doom-libs'</td><td>DLL 번들 stripe용 DNS 프리픽스</td></tr></tbody></table></div>
<p>로컬 리졸버에 레코드가 아직 전파되지 않았다면 <code>-DnsServer '1.1.1.1'</code>을 사용하세요.</p>
<h4>Publish-DoomOverDNS.ps1</h4>
<div class="table-wrapper"><table><thead><tr><th>파라미터</th><th>기본값</th><th>설명</th></tr></thead><tbody><tr><td>-PublishDir</td><td>(필수)</td><td>dotnet publish 출력 디렉토리 경로</td></tr><tr><td>-WadPath</td><td>(필수)</td><td>WAD 파일 경로</td></tr><tr><td>-Zones</td><td>(필수)</td><td>Cloudflare DNS 존 이름 배열</td></tr><tr><td>-WadPrefix</td><td>'doom-wad'</td><td>WAD stripe용 DNS 프리픽스</td></tr><tr><td>-LibsPrefix</td><td>'doom-libs'</td><td>DLL 번들 stripe용 DNS 프리픽스</td></tr><tr><td>-Force</td><td>$false</td><td>덮어쓰기 확인 프롬프트 생략</td></tr></tbody></table></div>
<p>업로드하려면 &quot;DNS 존 편집&quot; 권한이 있는 Cloudflare API 토큰이 필요합니다. TXTRecords 모듈의 <code>Set-CFCredential</code>로 로드할 수 있습니다.</p>
<h3>다중 존 스트라이핑</h3>
<p>Free 존은 185개 청크를 보관합니다. Pro/Business/Enterprise는 3,400개씩이죠. WAD만 약 1,199개 청크가 필요하므로 Free 사용자는 여러 도메인이 필요합니다. <code>-Zones</code>에 배열로 전달하면 모듈이 자동으로 청크를 분산시킵니다. Pro 존 하나면 모든 파일이 들어갑니다.</p>
<h3>중단된 업로드 재개하기</h3>
<p>업로드가 중단되었다면 <code>Publish-TXTStripe</code>의 <code>-Resume</code> 플래그를 사용하세요. 해시를 검증하고 마지막 정상 청크를 찾아서 거기서부터 계속 진행합니다.</p>
<h3>managed-doom 패치</h3>
<p>원본 managed-doom은 Native AOT를 사용하는데, 이건 <code>Assembly.Load()</code>로 로드할 수 없습니다. 이 포크는 이를 프레임워크 의존형 .NET 8 어셈블리로 변환했습니다. 스트림 기반 WAD 로딩과 윈도우 처리용 Win32 P/Invoke를 지원합니다 (GLFW, 오디오 없음 — 기본 NullSound/NullMusic 스텁 사용).</p>
