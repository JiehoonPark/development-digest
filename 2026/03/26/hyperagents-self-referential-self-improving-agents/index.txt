27:Te4a,<h2>HyperAgents</h2>
<p>모든 계산 가능한 작업을 최적화할 수 있는 자기 참조적 자기 개선 에이전트</p>
<h3>설치하기</h3>
<p>먼저 API 키를 <code>.env</code> 파일에 설정해야 합니다.</p>
<pre><code>OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
GEMINI_API_KEY=...</code></pre>
<p>그 다음 필요한 패키지들을 설치합니다.</p>
<pre><code class="language-bash">sudo dnf install -y python3.12-devel
sudo dnf install -y graphviz graphviz-devel cmake ninja-build bzip2-devel zlib-devel ncurses-devel libffi-devel</code></pre>
<p>Python 가상 환경을 만들어서 의존성을 설치하세요.</p>
<pre><code class="language-bash">python3.12 -m venv venv_nat
source venv_nat/bin/activate
pip install -r requirements.txt
pip install -r requirements_dev.txt</code></pre>
<p>Docker를 사용하려면 다음 명령어로 이미지를 빌드하면 됩니다.</p>
<pre><code class="language-bash">docker build --network=host -t hyperagents .</code></pre>
<p>초기 에이전트들을 설정합니다.</p>
<pre><code class="language-bash">bash ./setup_initial.sh</code></pre>
<h3>HyperAgents 실행하기</h3>
<p>메인 진입점인 <code>generate_loop.py</code> 스크립트로 알고리즘을 실행할 수 있습니다.</p>
<pre><code class="language-bash">python generate_loop.py --domains &lt;domain&gt;</code></pre>
<p>기본적으로 결과물은 <code>outputs/</code> 디렉토리에 저장됩니다. 스크립트의 인자와 기본 설정은 스크립트 자체에서 확인하세요.</p>
<h3>파일 구조</h3>
<ul>
<li><code>agent/</code> - 기초 모델을 사용하기 위한 코드</li>
<li><code>analysis/</code> - 플롯과 분석에 사용되는 스크립트</li>
<li><code>domains/</code> - 각 도메인별 코드</li>
<li><code>utils/</code> - 저장소 전체에서 사용되는 공통 코드</li>
<li><code>run_meta_agent.py</code> - 메타 에이전트를 실행하고 diff를 얻기 위한 스크립트</li>
<li><code>meta_agent.py</code> - 메타 에이전트의 주요 구현</li>
<li><code>task_agent.py</code> - 태스크 에이전트의 주요 구현</li>
<li><code>generate_loop.py</code> - 알고리즘 실행을 위한 진입점</li>
</ul>
<h3>실험 로그</h3>
<p>실험 로그는 분할된 ZIP 아카이브 형태로 저장됩니다. 추출하려면 <code>.z01</code>, <code>.z02</code> 같은 모든 파일이 <code>.zip</code> 파일과 같은 디렉토리에 있어야 합니다.</p>
<pre><code class="language-bash">zip -s 0 outputs_os_parts.zip --out unsplit_logs.zip
unzip unsplit_outputs.zip</code></pre>

<p>⚠️ <strong>경고</strong>: 이 저장소는 모델이 생성한 신뢰할 수 없는 코드를 실행합니다. 관련된 보안 위험을 충분히 인식하고 사용하시기 바랍니다.</p>
