15:T14ba,<h2>Cherri: Shortcuts를 위한 새로운 프로그래밍 언어</h2>
<p><strong>Cherri</strong>(체리라고 발음합니다)는 Shortcuts 전용 프로그래밍 언어로, 실행 가능한 Shortcut으로 직접 컴파일됩니다.</p>
<p>이 프로젝트의 핵심 목표는 간단합니다. Shortcuts의 한계 내에서 규모 있는 프로젝트를 만들고, 장기적으로 유지보수할 수 있게 하는 것입니다.</p>
<h3>주요 기능들</h3>
<p>✨ <strong>개발 환경</strong></p>
<ul>
<li>노트북/데스크톱 기반 개발 지원 (CLI, VSCode 익스텐션, macOS 앱)</li>
</ul>
<p>🎓 <strong>배우기 쉬운 문법</strong></p>
<ul>
<li>다른 프로그래밍 언어와 유사한 구문으로 진입장벽 낮춤</li>
</ul>
<p>🐞 <strong>디버깅 친화적</strong></p>
<ul>
<li>Shortcut 액션과 1:1로 대응되도록 설계해 문제 추적이 쉬움</li>
</ul>
<p>🥾 <strong>자급자족 구조</strong></p>
<ul>
<li>대부분의 액션과 타입을 Cherri로 직접 작성</li>
</ul>
<p>💻 <strong>macOS에서 액션 임포트</strong></p>
<p>📦 <strong>내장 패키지 매니저</strong></p>
<ul>
<li>Git 저장소 기반이라 자동 업데이트 가능</li>
</ul>
<p>🪄 <strong>직관적인 변수 문법</strong></p>
<ul>
<li>매직 변수 대신 상수처럼 다룸</li>
</ul>
<p>🪶 <strong>최적화된 파일 크기</strong></p>
<ul>
<li>Shortcut 크기를 최소화하고 런타임 메모리 효율을 극대화</li>
</ul>
<p>#️⃣ <strong>파일 포함 기능</strong></p>
<ul>
<li>큰 프로젝트를 여러 파일로 나눌 수 있음</li>
</ul>
<p>🔧 <strong>강력한 액션 정의</strong></p>
<ul>
<li>타입 체킹, 열거형, 선택값, 기본값 등 지원</li>
</ul>
<p>🔄 <strong>함수 정의</strong></p>
<ul>
<li>중복된 액션을 줄이고 독립적인 스코프에서 실행</li>
</ul>
<p>📋 <strong>자동 액션 복사</strong></p>
<p>🥩 <strong>Raw Actions</strong></p>
<ul>
<li>액션 식별자와 파라미터를 직접 입력 가능</li>
</ul>
<p>❓ <strong>커스텀 질문 정의</strong></p>
<p>📇 <strong>vCard 생성</strong></p>
<p>📄 <strong>Base64로 파일 임베딩</strong></p>
<p>🔀 <strong>iCloud 링크에서 변환</strong></p>
<ul>
<li><code>--import=</code> 옵션으로 기존 Shortcut 임포트</li>
</ul>
<p>🔢 <strong>타입 시스템</strong></p>
<ul>
<li>타입 추론 지원</li>
</ul>
<p>🔏 <strong>서명 기능</strong></p>
<ul>
<li>macOS 기본 서명, HubSign 또는 scaxyz/shortcut-signing-server 지원</li>
</ul>
<h3>리소스</h3>
<ul>
<li>🍒 [Cherri VSCode 익스텐션](https://github.com/electrikmilk/cherri-vscode)</li>
<li>🛝 [온라인 플레이그라운드](https://cherri.dev/playground) - 모든 플랫폼에서 시도하고 즉시 미리보기 가능</li>
<li>🖥️ [macOS IDE](https://cherri.dev/ide) - GUI로 Shortcut 작성 및 빌드</li>
<li>📄 [공식 문서](https://docs.cherri.dev) - 학습하기 및 기여하기</li>
<li>🔍 [글리프 검색](https://cherri.dev/glyphs) - Cherri에서 사용 가능한 이모지와 심볼 검색</li>
</ul>
<h3>설치하기</h3>
<h4>Homebrew로 설치</h4>
<pre><code class="language-bash">brew tap electrikmilk/cherri
brew install electrikmilk/cherri/cherri</code></pre>
<h4>Nix로 설치</h4>
<pre><code class="language-bash">nix profile install github:electrikmilk/cherri</code></pre>
<p>또는 <code>nix-direnv</code>를 사용해서 디렉토리별로 격리된 개발 환경을 구성할 수도 있습니다. <code>flake.nix</code>에 Cherri를 추가하면 됩니다:</p>
<pre><code class="language-nix">{
  inputs.cherri.url = &quot;github:electrikmilk/cherri&quot;;
  # outputs.packages.${system}.default 등은 생략
  {
    buildInputs = [
      inputs.cherri.packages.${system}.cherri
    ]
  }
}</code></pre>
<p>디렉토리에서 <code>direnv allow</code>를 실행하면 자동으로 환경이 설정됩니다.</p>
<h3>사용 방법</h3>
<pre><code class="language-bash">cherri file.cherri</code></pre>
<p>인자 없이 실행하면 모든 옵션과 사용법을 볼 수 있습니다. 개발할 때는 <code>--debug</code> 또는 <code>-d</code> 옵션을 사용하면 스택 트레이스, 디버그 정보, <code>.plist</code> 파일이 출력됩니다.</p>
<h3>또 다른 Shortcuts 언어가 필요한가요?</h3>
<p>재미있으니까요 😄</p>
<p>사실 기존 Shortcuts 언어들 중 일부는 더 이상 유지보수되지 않거나, 제대로 작동하지 않습니다. Shortcuts 언어가 사라지는 건 안 타깝거든요.</p>
<p>오히려 더 많은 선택지가 있어야 합니다. 특히 이 프로젝트가 iOS가 아닌 macOS 기반이라는 점은 개발 안정성을 높여줍니다. 또한 Buttermilk을 제외하고는 macOS를 플랫폼으로 하는 Shortcuts 언어를 찾기 어렵습니다.</p>
