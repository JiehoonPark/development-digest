15:T1ab4,<h2>Pretext: DOM 없이 텍스트 레이아웃을 측정하고 렌더링하기</h2>
<p>순수 JavaScript/TypeScript로 작성된 다국어 텍스트 측정 및 레이아웃 라이브러리입니다. 빠르고 정확하며 거의 모든 언어를 지원해요. DOM, Canvas, SVG는 물론이고 곧 서버 사이드 렌더링도 가능해질 예정이에요.</p>
<h3>왜 Pretext를 써야 할까?</h3>
<p>브라우저에서 가장 비용이 많이 드는 작업 중 하나가 바로 레이아웃 리플로우인데요. Pretext는 <code>getBoundingClientRect</code>나 <code>offsetHeight</code> 같은 DOM 측정을 완전히 우회합니다. 대신 자체 텍스트 측정 로직을 구현했고, 브라우저의 폰트 엔진을 기준으로 정확도를 보장하고 있어요. 이 방식은 AI 기반 반복 작업에도 정말 잘 맞습니다.</p>
<h3>설치</h3>
<pre><code class="language-bash">npm install @chenglou/pretext</code></pre>
<h3>데모 보기</h3>
<p>저장소를 클론한 후 <code>bun install</code>을 실행하고 <code>bun start</code>로 개발 서버를 띄운 다음 <code>/demos</code>에 접속하면 됩니다 (주의: 끝에 슬래시를 붙이면 안 됨).</p>
<p>라이브 데모는 [chenglou.me/pretext](https://chenglou.me/pretext)에서, 추가 데모는 [somnai-dreams.github.io/pretext-demos](https://somnai-dreams.github.io/pretext-demos)에서 볼 수 있습니다.</p>
<h3>두 가지 사용 패턴</h3>
<h4>1. DOM을 건드리지 않고 문단의 높이 측정하기</h4>
<pre><code class="language-javascript">import { prepare, layout } from '@chenglou/pretext'

const prepared = prepare('AGI 春天到了. بدأت الرحلة 🚀', '16px Inter')
const { height, lineCount } = layout(prepared, textWidth, 20) // 순수 계산! DOM 리플로우 없음</code></pre>
<p><code>prepare()</code> 함수는 한 번만 실행하는 무거운 작업을 담당합니다. 공백을 정규화하고, 텍스트를 분할하고, 규칙을 적용한 다음 Canvas로 각 세그먼트를 측정한 후 결과를 반환하죠.</p>
<p>그 다음 <code>layout()</code> 함수는 캐시된 너비 값으로 순수 산술 연산만 수행하기 때문에 매우 빠릅니다. 이게 핫 패스예요.</p>
<p><strong>성능 수치</strong></p>
<p>현재 벤치마크 기준으로:</p>
<ul>
<li><code>prepare()</code>: 500개 텍스트 배치당 약 19ms</li>
<li><code>layout()</code>: 같은 배치당 약 0.09ms</li>
</ul>
<h4>textarea처럼 공백 유지하기</h4>
<p>보통 스페이스, 탭(<code>\t</code>), 줄바꿈(<code>\n</code>)을 그대로 표시하고 싶다면 <code>whiteSpace: 'pre-wrap'</code> 옵션을 전달하세요.</p>
<pre><code class="language-javascript">const prepared = prepare(textareaValue, '16px Inter', { whiteSpace: 'pre-wrap' })
const { height } = layout(prepared, textareaWidth, 20)</code></pre>
<h4>다국어 &amp; 이모지 완벽 지원</h4>
<p>이모지, 혼합 양방향 텍스트, 브라우저별 특이사항까지 모두 처리합니다.</p>
<h4>정확한 높이 값으로 뭘 할 수 있나?</h4>
<p>이 높이 값은 웹 UI를 한 단계 업그레이드할 수 있는 열쇠가 되어줍니다:</p>
<ul>
<li><strong>정확한 가상화(virtualization)</strong>: 추측이나 캐싱 없이 진정한 화면 밖 요소 제거 가능</li>
<li><strong>고급 레이아웃</strong>: 메이슨리, JS 기반 flexbox 구현, CSS 핵 없이 레이아웃 값 조정</li>
<li><strong>개발 검증</strong>: AI 시대에 특히 유용한데, 버튼 라벨이 다음 줄로 넘어가는지 브라우저 없이 검증 가능</li>
<li><strong>레이아웃 시프트 방지</strong>: 새 텍스트가 로드될 때 스크롤 위치를 정확하게 유지</li>
</ul>
<hr>
<h4>2. 수동으로 각 줄을 레이아웃하기</h4>
<p><code>prepare</code> 대신 <code>prepareWithSegments</code>를 쓰면 더 많은 제어가 가능해요.</p>
<h5>모든 줄 한 번에 가져오기</h5>
<pre><code class="language-javascript">import { prepareWithSegments, layoutWithLines } from '@chenglou/pretext'

const prepared = prepareWithSegments('AGI 春天到了. بدأت الرحلة 🚀', '18px &quot;Helvetica Neue&quot;')
const { lines } = layoutWithLines(prepared, 320, 26) // 최대 너비 320px, 줄높이 26px

for (let i = 0; i &lt; lines.length; i++) {
  ctx.fillText(lines[i].text, 0, i * 26)
}</code></pre>
<h5>각 줄의 너비 구하기 (문자열 생성 없이)</h5>
<pre><code class="language-javascript">let maxW = 0
walkLineRanges(prepared, 320, line =&gt; { 
  if (line.width &gt; maxW) maxW = line.width 
})
// maxW는 이제 가장 긴 줄의 너비 = 텍스트가 들어갈 수 있는 최소 컨테이너 너비!
// 이 다중 줄 &quot;자동 맞춤(shrink wrap)&quot;이 웹에서 오랫동안 필요했던 기능이에요</code></pre>
<h5>너비가 변하면서 줄 단위로 텍스트 흐르게 하기</h5>
<pre><code class="language-javascript">let cursor = { segmentIndex: 0, graphemeIndex: 0 }
let y = 0

// 떠있는 이미지 주변으로 텍스트 배치: 이미지 옆 줄은 더 좁음

<p>이 패턴으로 Canvas, SVG, WebGL 그리고 (곧) 서버 사이드에서 렌더링할 수 있습니다.</p>
<hr>
<h3>API 레퍼런스</h3>
<h4>사용 패턴 1 API</h4>
<p><strong><code>prepare(text, font, options?)</code></strong></p>
<p>한 번만 실행하는 텍스트 분석 및 측정 작업입니다. <code>layout()</code>에 전달할 불투명한 값을 반환하죠. <code>font</code>는 CSS의 <code>font</code> 선언과 반드시 동기화되어야 합니다. 형식은 Canvas의 <code>myCanvasContext.font = ...</code>과 같습니다 (예: <code>16px Inter</code>).</p>
<p><strong><code>layout(prepared, maxWidth, lineHeight)</code></strong></p>
<p>최대 너비와 줄높이가 주어졌을 때 텍스트의 높이를 계산합니다. 반환값은 <code>{ height, lineCount }</code> 객체예요. <code>lineHeight</code>는 CSS의 <code>line-height</code> 선언과 동기화되어야 합니다.</p>
<h4>사용 패턴 2 API</h4>
<p><strong><code>prepareWithSegments(text, font, options?)</code></strong></p>
