안정적이고 즐거운 로컬 호스트 음성 보조기로의 여정 (2025)

My Journey to a reliable and enjoyable locally hosted voice assistant (2025)

📅 2026-03-17 📰 Hacker News (Top) ⬆ 364

요약

Google Home에서 Home Assistant의 로컬 음성 어시스턴트로 마이그레이션한 경험을 공유하며, llama.cpp 기반 로컬 LLM, 다양한 GPU 옵션, 음성 인식/생성 소프트웨어 조합을 통한 완전히 로컬화된 음성 어시스턴트 구현 방법을 상세히 기록했다.

핵심 포인트

  • RTX 3090/RX 7900XTX 같은 24GB GPU는 20B-30B 모델을 1-2초 응답 시간으로 실행 가능하며, RTX 3050 8GB도 4B 모델로 기본 기능 지원
  • Wyoming ONNX ASR(Nvidia Parakeet V2 + OpenVINO), Rhasspy Faster Whisper, Kokoro TTS 등 다양한 로컬 음성 소프트웨어 조합 테스트 결과 공유
  • LLM Conversation과 LLM Intents 통합으로 기본 어시스턴트 경험 개선 및 추가 도구(웹 검색, 장소 검색, 날씨 예보) 제공

왜 중요한가

개인 정보 보호와 독립성을 원하는 사용자들에게 로컬 호스트 음성 어시스턴트 구현의 현실적인 하드웨어/소프트웨어 요구사항과 성능 기대치를 제시한다.

📄 전문 번역

로컬 음성 어시스턴트를 향한 여정: 신뢰할 수 있고 즐거운 경험 만들기

Nicolas Mowen | 2025년 10월 27일

Home Assistant의 Assist 기능을 오랫동안 지켜봐왔습니다. 예전에는 Nest Mini를 통해 Google Home을 사용했는데, 최근에 완전히 로컬에서 동작하는 Assist로 전환했거든요. llama.cpp(이전에는 Ollama)를 기반으로 로컬 우선 방식으로 운영하고 있습니다.

이 글에서는 현재 상태에 도달하기까지 거친 과정, 내린 결정들, 그리고 제 사용 사례에 맞는 이유를 공유해드리겠습니다.

추가 개선 사항들

이 글타래에서 다룬 여러 개선 사항들입니다:

새로운 기능

  • 보안 카메라 통찰력 / 분석
  • YouTube 영상 검색 및 TV 재생
  • 원치 않는 Home Assistant / LLM 동작 수정

동작 개선

  • 기본 날씨 인텐트를 오버라이드해서 일관된 날씨 정보 제공
  • 모호한 요청 / 오탐지 처리 개선
  • 음성 인식 오류를 자동으로 감지하고 처리
  • 토큰 사용량 감소를 위한 프롬프트 최적화

성능 최적화

  • llama.cpp 성능 최적화

하드웨어 구성

다양한 하드웨어를 테스트해봤는데, RTX 3050부터 RTX 3090까지 최신 그래픽카드면 로컬 Assist를 충분히 구동할 수 있습니다. 물론 기대하는 성능과 응답 속도에 따라 필요한 사양이 달라지긴 합니다.

Home Assistant는 UnRaid NAS에서 실행 중입니다. (사실 음성 어시스턴트 성능과는 무관하므로 스펙은 중요하지 않아요)

음성 하드웨어:

  • Home Assistant Voice Preview Satellite 1개
  • Satellite1 Small Squircle Enclosure 2개
  • Pixel 7a (Satellite 겸 Hub, View Assist 포함)

음성 서버 하드웨어:

  • USB4 포트가 있는 Beelink MiniPC
  • USB4 eGPU 인클로저
  • 아래 표의 GPU들

음성 서버 소프트웨어

모델 러너:

llama.cpp를 권장합니다. 최적의 성능을 위해 사용하세요.

음성 인식 (음성 입력):

소프트웨어모델비고
Wyoming ONNX ASRNvidia Parakeet V2OpenVINO 브랜치로 실행하면 CPU 추론 시간을 약 0.3초까지 단축
Rhasspy Faster WhisperNvidia Parakeet V2ONNX를 직접 CPU로 실행하기 때문에 OpenVINO보다 느림

음성 합성 (음성 출력):

소프트웨어비고
Kokoro TTS여러 음성과 톤을 조합해서 원하는 출력을 얻을 수 있음. 모든 텍스트를 잘 처리함
Piper (CPU 실행)여러 음성 옵션 제공. 일반 텍스트는 잘 처리하지만 통화, 전화번호, 주소는 제대로 읽지 못함

Home Assistant LLM 통합

  • LLM Conversation: 기본 Assist의 경험을 개선하는 기능
  • LLM Intents: Assist에 추가 도구 제공 (웹 검색, 장소 검색, 날씨 예보)

시작점: 문제 인식

제가 이 글을 쓰는 이유는 제 방식이 "올바른 방법"이거나 다른 사람들이 따라야 할 방법이라고 말하려는 게 아닙니다. 그저 이 과정에서 배운 많은 것들을 공유하고 싶었습니다. 다른 사람들이 어떤 결과를 기대할 수 있는지, 어떤 함정이 있는지 더 잘 이해할 수 있으면 좋겠거든요.

배경:

지난 1~2년 동안 Nest Mini를 통한 Google Assistant에 점점 불만이 쌓였습니다.
