React Server Components 후속 취약점: 서비스 거부와 소스 코드 노출
via React Blog
🔥오늘의 화제frontend

지난주 공개된 React Server Components의 치명적 RCE 취약점(React2Shell) 패치를 분석하던 연구자들이 새로운 DoS 취약점 3건(CVE-2025-55184, CVE-2025-67779, CVE-2026-23864)과 소스 코드 노출 취약점 1건(CVE-2025-55183)을 추가로 발견했습니다. 영향받는 패키지는 react-server-dom-webpack/parcel/turbopack의 19.0.x~19.2.x 계열이며, 최종 수정 버전은 19.0.4, 19.1.5, 19.2.4입니다. 이전에 19.0.3/19.1.4/19.2.3으로 업데이트했더라도 불완전한 패치이므로 즉시 재업데이트가 필요합니다.

💡
React Server Components를 사용하는 프로덕션 서비스는 서버 크래시나 소스 코드(하드코딩된 비밀값 포함) 유출로 이어질 수 있는 실질적 보안 위협에 노출되어 있어, 의존성 버전을 즉시 점검하고 최신 패치로 업그레이드해야 합니다.

via Hacker News (Top)

이 글은 영어 필기체가 러시아어 필기체보다 덜 즐거운 이유를 '백트래킹'(t를 가로지르거나 i에 점을 찍기 위해 되돌아가는 동작) 개념으로 설명합니다. 저자는 도스토옙스키의 『죄와 벌』 원문을 분석해 영어는 단어의 51%, 러시아어는 6.4%에서 백트래킹이 필요함을 정량적으로 보여줍니다. 이어 SmithHand 서체를 기반으로 x, t, i, j 등 문제 글자를 재설계해, 모든 소문자를 한 획으로 쓸 수 있는 새로운 필기체 알파벳을 직접 만든 과정을 다룹니다.

💡

via Jack Herrington
▶영상frontend

한 개발자가 3D 프린팅한 태양계 행성 모형과 LED 조명으로 포치를 꾸미고, 이를 Amazon Fire TV의 Vega 플랫폼과 React Native로 제어하는 프로젝트를 소개합니다. VS Code 확장과 로컬 시뮬레이터, CLI 도구로 개발이 수월했고, WLED 컨트롤러 제어는 일반 fetch HTTP 요청으로, 음성 명령은 aria-label 추가만으로 구현했습니다. React/React Native 경험자에게 Vega가 얼마나 낮은 진입 장벽을 제공하는지 보여주는 실전 경험담입니다.

💡
