💡

🔥오늘의 화제frontend

여행사가 일정표와 숙박 정보를 확인하려면 무거운 'Travelbound' 앱을 설치하라고 요구하자, 저자는 이 앱이 사실상 텍스트·이미지·PDF 링크를 보여주는 것에 불과하다는 점에 분노해 직접 리버스 엔지니어링에 나섰습니다. Android 가상 기기 루팅과 HTTP Toolkit을 이용한 트래픽 분석으로 앱의 API와 JSON 데이터 구조를 파악한 뒤, 동일한 정보를 제공하는 0.05MB짜리 웹페이지를 만들어 43MB(최대 124MB) 앱을 대체했습니다. 이 과정에서 앱이 추가로 제공하던 것은 사용자 트래킹과 광고뿐이었다는 사실이 드러났습니다.

💡
이미 HTML/HTTP로 전달 가능한 콘텐츠를 굳이 네이티브 앱으로 감싸는 관행이 얼마나 비효율적일 수 있는지를 실제 리버스 엔지니어링 사례로 보여주며, 프론트엔드 개발자에게 '이게 정말 앱이어야 하는가'라는 근본적 질문을 다시 던집니다.

via Jack Herrington

천문학을 좋아하는 가족을 위해 포치에 설치한 태양계 행성 LED 조명을, React Native 기반 TV 개발 플랫폼인 Amazon Vega로 거실 프로젝션 스크린과 연동한 취미 프로젝트를 소개합니다. VS Code 확장과 로컬 시뮬레이터, CLI 도구 덕분에 개발이 예상보다 쉬웠고, WLED 컨트롤러 제어는 fetch API로, 음성 명령은 aria-label만 추가해 간단히 구현했습니다. React/React Native 지식만 있으면 큰 어려움 없이 Fire TV용 앱을 만들 수 있다는 경험담입니다.
