## 핵심 개념

> [!abstract]
> Anthropic은 Claude Opus 4.7(2026년 4월 16일)에서 시스템 프롬프트를 업데이트했으며, 주요 변경은 도구 활용 우선순위 상향, 응답 간결성 강화, 더 나은 작업 완료 등이다. Claude는 이제 모호한 정보가 있을 때 먼저 사용자에게 묻기보다는 도구를 사용하여 해결하려고 시도한다.

## 상세 내용

- Claude가 도구를 사용해 모호함을 해결한 후 질문하는 방식으로 변경되어 사용자 경험 개선
- 응답의 간결성을 우선하고 이모지/액션 사용 제한, 불필요한 수식어 제거
- 식이 장애 관련 정확한 수치 제공 금지 등 사용자 안전 관련 가이드라인 강화

> [!tip] 왜 중요한가
> AI 모델 개발자들이 시스템 프롬프트의 진화 방향을 이해하면 더 효과적인 LLM 활용 방법을 설계할 수 있다.

## 전문 번역

# Claude Opus 4.7 시스템 프롬프트 업데이트 분석

Anthropic은 주요 AI 연구소 중에서 유일하게 사용자 인터페이스 챗 시스템의 시스템 프롬프트를 공개하고 있습니다. 2024년 7월 Claude 3부터 시작된 그들의 아카이브를 보면, 새로운 모델이 출시될 때마다 시스템 프롬프트가 어떻게 진화하는지 살펴볼 수 있어서 흥미롭거든요.

지난주(2026년 4월 16일) Opus 4.7이 출시되면서 Opus 4.6(2026년 2월 5일) 이후로 Claude.ai 시스템 프롬프트가 업데이트됐습니다.

## 시스템 프롬프트 변경 사항 추적

저는 Claude Code를 활용해서 Anthropic의 마크다운 형식 시스템 프롬프트를 받아, 각 모델별로 문서를 분리하고 각 업데이트 날짜를 가짜 커밋 시간으로 하는 Git 히스토리를 구성했습니다.

Opus 4.6과 4.7 사이의 git diff를 살펴보니 몇 가지 주목할 만한 변화가 있었습니다. 다음은 제가 강조한 주요 내용들입니다.

## 주요 변화점들

**사용자가 세부 사항을 명확히 하지 않았을 때의 대응**

요청에 작은 부분이 불명확하면, 사용자는 보통 Claude가 합리적인 답변을 지금 바로 시도하길 원합니다. 먼저 질문을 던지기를 원하는 게 아니거든요. Claude는 정말로 필요한 정보가 없어서 답변이 불가능할 때(예: 첨부 파일이 없는데 참조하는 경우)만 미리 물어봅니다.

**도구 활용의 우선순위**

모호함을 해결하거나 누락된 정보를 제공할 수 있는 도구가 있다면, Claude는 먼저 그 도구를 사용합니다. 검색, 사용자의 위치 확인, 캘린더 조회, 사용 가능한 기능 파악 등이 여기에 해당하거든요. 사용자에게 직접 검색해달라고 하는 것보다 도구로 먼저 시도하는 것을 선호합니다.

**작업 완수까지의 일관성**

Claude가 한번 작업을 시작하면, 중간에 멈추지 않고 완전한 답변까지 진행합니다.

**기능 제한에 대한 재확인**

위치 정보, 메모리, 캘린더, 파일, 이전 대화, 외부 데이터 같은 특정 기능이 없다고 결론짓기 전에, Claude는 먼저 tool_search를 사용해 관련 도구가 있는지 확인합니다. "X에 접근할 수 없다"는 표현은 tool_search로 관련 도구가 정말 없다는 걸 확인한 후에만 맞는 답변입니다.

**응답의 간결성**

Claude는 사용자를 압도할 수 있는 긴 응답을 피하고, 응답을 집중적이고 간결하게 유지합니다. 면책 조항이나 주의사항이 있더라도 간단히 언급하고 주요 답변에 집중합니다.

**표현 방식의 변화**

이모티콘이나 별표로 표현한 동작(*웃음* 같은)을 사용하지 않으며, 사용자가 명시적으로 요청하지 않으면 이런 스타일을 피합니다. 또한 "genuinely", "honestly", "straightforward" 같은 표현들도 줄였습니다.

**섭식 장애 관련 민감성**

사용자가 섭식 장애의 증상을 보이면, Claude는 영양, 식단, 운동에 관한 구체적인 숫자, 목표, 단계별 계획을 제시하지 않습니다. 더 건강한 목표를 돕기 위함이거나 섭식 장애의 위험성을 강조하려는 의도더라도, 구체적인 수치가 포함된 응답은 오히려 부정적인 행동을 유발할 수 있거든요.

**복잡한 주제에 대한 단답형 응답 거절**

복잡하거나 논쟁이 있는 주제에 대해 "예" 또는 "아니오" 같은 짧은 답변을 요청받으면, Claude는 이를 거절하고 대신 미묘한 뉘앙스가 있는 답변을 제시하며 왜 단답형이 부적절한지 설명할 수 있습니다.

## 공개되지 않은 정보: 도구 설명

다만 Anthropic이 공개한 시스템 프롬프트만으로는 전부가 아닙니다. 모델에 제공되는 도구 설명들은 공개되지 않았거든요. 사실 Claude 채팅 UI의 모든 기능을 최대한 활용하려면 이 부분이 더 중요할 수도 있습니다.
