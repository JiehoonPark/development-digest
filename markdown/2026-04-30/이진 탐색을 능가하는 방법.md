## 핵심 개념

> [!abstract]
> 16비트 정수 배열에서 SIMD와 사원분 보간 탐색을 결합한 새로운 검색 알고리즘 'SIMD Quad'를 제시하여 이진 탐색보다 성능이 우수함을 벤치마크로 보여줍니다.

## 상세 내용

- SIMD(Single Instruction, Multiple Data) 명령어를 활용하여 여러 값을 동시에 비교 가능
- 사원분 보간 탐색으로 블록을 빠르게 찾고 SIMD로 병렬 검사하는 하이브리드 방식
- Apple M4와 Intel Emerald Rapids 프로세서에서 벤치마크 수행하여 이진 탐색 초과 성능 입증

> [!tip] 왜 중요한가
> 현대 프로세서의 SIMD 기능을 활용한 알고리즘 최적화는 성능 중심 애플리케이션 개발에서 실질적인 성능 향상을 제공합니다.

## 전문 번역

# 정렬된 배열에서 빠르게 값을 찾기: SIMD Quad 알고리즘

정렬된 배열에서 특정 값을 찾아야 할 때가 있습니다. 가장 단순한 방법은 처음부터 끝까지 하나씩 살펴보는 선형 탐색이죠. C++에서는 `std::find` 함수로 이를 구현할 수 있습니다.

그런데 배열이 크면 이진 탐색(binary search)을 쓰는 게 훨씬 낫습니다. 이진 탐색은 정렬된 배열을 반으로 나누면서 탐색 범위를 계속 줄여나가는 고전적인 알고리즘입니다. 중간값과 찾는 값을 비교한 뒤, 작으면 앞쪽 절반을, 크면 뒤쪽 절반을 버립니다. 이 과정을 반복하면 값을 찾거나 탐색 범위가 없어질 때까지 계속되는데요. 대규모 데이터셋에서 선형 탐색보다 훨씬 빠릅니다. C++의 `std::binary_search` 함수가 이를 구현하고 있으며, 값 존재 여부를 boolean으로 반환합니다.

## 현실의 문제: Roaring Bitmap

Roaring Bitmap 같은 널리 쓰이는 포맷에서는 1~4096 크기의 16비트 정수 배열을 사용합니다. 여기서 특정 값이 있는지 자주 확인해야 하는데, 보통 이진 탐색을 활용합니다.

하지만 더 빠른 방법이 있을까요? 저는 두 가지 통찰을 얻었습니다.

## 핵심 통찰

**첫 번째, 현대 프로세서의 SIMD 능력**

요즘 거의 모든 프로세서가 SIMD(Single Instruction, Multiple Data) 명령어를 지원합니다. 64비트 ARM과 x64 프로세서(Intel/AMD) 모두 하나의 명령어로 16비트 정수 8개를 동시에 목표값과 비교할 수 있거든요. 이는 이진 탐색을 8개 미만의 작은 블록까지 내려갈 필요가 없다는 뜻입니다. 16개 이상의 요소를 저렴한 비용으로 비교할 수도 있다는 의미죠.

**두 번째, 메모리 수준의 병렬성**

이진 탐색은 매번 하나의 값만 확인합니다. 그런데 최신 프로세서는 여러 값을 한 번에 로드하고 확인할 수 있습니다. 메모리 수준 병렬성이 뛰어나다는 뜻인데요. 이진 탐색 대신 4진 탐색(quaternary search)을 써볼 수 있을까요? 배열을 반으로 나누는 대신 4등분하는 거죠. 그러면 명령어가 좀 더 늘겠지만, 명령어 개수가 성능의 병목이 될 가능성은 낮습니다.

## SIMD Quad 알고리즘

이런 아이디어를 토대로 만든 것이 바로 **SIMD Quad 알고리즘**입니다. 16비트 부호 없는 정수 정렬 배열에서 빠르게 탐색하는 알고리즘으로, 4진 보간 탐색과 SIMD를 결합했습니다.

동작 방식은 다음과 같습니다:

1. **초기 확인**: 배열이 16개 미만이면 단순 선형 탐색을 합니다.

2. **블록 분할**: 배열을 16개 연속 요소씩 블록으로 나눕니다. 크기가 cardinality라면 `num_blocks = cardinality / 16`개의 완전한 블록이 생깁니다.

3. **4진 보간 탐색**: 각 블록의 마지막 요소(위치 15, 31 등)를 키로 사용해 보간 탐색을 수행합니다. 목표값이 있을 법한 블록을 찾기 위해 현재 탐색 범위의 4등분점과 비교하면서 진행합니다.

4. **블록 선택**: 보간 결과를 바탕으로 적절한 블록 인덱스를 결정합니다.

5. **SIMD 확인**: 선택한 블록의 16개 요소를 SIMD 레지스터에 로드합니다(ARM의 NEON 또는 x64의 SSE2 사용). 그 다음 모든 요소와 목표값을 병렬로 비교해서 일치하는 게 있으면 true를 반환합니다.

6. **나머지 확인**: 완전한 블록에 포함되지 않은 요소들에 대해서는 선형 탐색을 합니다.

## 성능 측정

실제로 얼마나 빠른지 벤치마크로 확인했습니다.

**벤치마크 방식**
- 배열 크기: 2~4096개 요소
- 각 크기마다 100,000개의 정렬된 16비트 정수 배열 생성
- 각 크기별로 1천만 회의 멤버십 쿼리 수행

두 가지 모드로 측정했습니다:

- **Cold 모드**: 각 쿼리가 다른 배열을 탐색 (캐시 미스 시뮬레이션)
- **Warm 모드**: 쿼리들을 배열별로 그룹화, 각 배열을 100번 연속 탐색 (캐시 히트 시뮬레이션)

세 가지 알고리즘을 비교했습니다:
- 선형 탐색 (`std::find`)
- 이진 탐색 (`std::binary_search`)
- 새로운 SIMD Quad 알고리즘

두 개의 다른 시스템에서 테스트했는데요:
- Apple M4 + Apple LLVM
- Intel Emerald Rapids + GCC
