## 핵심 개념

> [!abstract]
> TestDisk와 PhotoRec 도구를 사용하여 오래된 저장장치에서 삭제된 파일을 복구한 경험을 공유합니다. 저자는 13년된 노트북과 10년된 SD 카드에서 각각 16,000개 이상의 파일을 복구했으며, 복구 과정의 장점과 한계를 분석했습니다.

## 상세 내용

- PhotoRec는 파일 시스템을 스캔하여 원본 파일명과 폴더 구조 없이 삭제된 파일 복구 가능
- 복구 시간이 길고 결과물이 많아 분류가 어려우며, 복구 대상 파일 시스템에 저장하면 안 됨
- 300개 이상의 파일 형식을 지원하며 디지털 포렌식 작업의 핵심 도구

> [!tip] 왜 중요한가
> 데이터 손실 상황 대응과 레거시 시스템 관리 시 실질적인 파일 복구 방법을 이해하는 것은 개발자와 시스템 관리자에게 중요합니다.

## 전문 번역

# PhotoRec로 삭제된 파일 되살리기

2026년 4월 24일

당신도 경험해봤을 거예요. "이 passwords.txt는 이제 필요 없겠네" 싶어서 우클릭 → 삭제 → 휴지통 비우기. 끝.

아, 그런데 정말 끝일까요?

## 삭제된 파일은 완전히 사라지지 않는다

어릴 때 들었던 이야기가 떠올라요. 삭제된 파일을 복구한다는 게 정말 가능하다는 거였죠. 단순히 휴지통에서 꺼내는 게 아니라, 휴지통을 비운 후에도 되살릴 수 있다고 했어요.

최근에 저도 직접 해봤는데요. TestDisk와 PhotoRec이라는 도구를 썼습니다. 이 두 프로그램은 디지털 포렌식 분야에서 매우 중요한 역할을 하는데, 시스템을 이전 상태로 되돌리고 여러 파일들을 분석할 수 있게 해줍니다.

저는 십수 년 된 하드웨어들을 가지고 있었거든요. PhotoRec이 정말 어디까지 복구할 수 있는지 직접 테스트해보기로 했습니다.

## TestDisk와 PhotoRec의 역할

공식 문서에 따르면 TestDisk는 다음을 할 수 있습니다:

- 파티션 테이블 수정 및 삭제된 파티션 복구
- FAT32 부트 섹터를 백업에서 복구
- FAT12/FAT16/FAT32 부트 섹터 재구성
- FAT 테이블 수정
- NTFS 부트 섹터 재구성
- FAT, exFAT, NTFS, ext2 파일시스템에서 파일 복구
- 삭제된 파티션에서 파일 복사

저는 마지막 두 기능에만 관심이 있었어요. 여기서 중요한 차이점이 있습니다.

- **TestDisk** → 디스크와 파일시스템 수정
- **PhotoRec** → 원본 데이터를 스캔해서 개별 파일 복구

파일을 '복구'하려는 게 목표였으니까 PhotoRec만 사용하면 됐습니다.

## 복구 대상 선정

선택할 수 있는 장비들이 꽤 많았습니다:

- Toshiba 노트북 (1TB), 13년된 제품
- GoPro SD 카드 (7GB), 10년 묵은 것
- PSP 메모리 카드 (32MB), 20년 넘은 것
- iPod video (30GB), 약 20년 전 기기
- Windows 10 PC (용량 미상), 16년 전 컴퓨터

그런데 파일 복구는 생각보다 오래 걸립니다. 저장소 크기와 파티션 크기에 따라 몇 시간은 기본으로 기다려야 하죠. 게다가 복구된 파일들도 어딘가에 저장해야 하니까 공간이 더 필요해요. 그래서 저는 첫 시도를 두 개 장비로만 제한했습니다:

- Toshiba 노트북 (1TB), 13년된 제품
- GoPro SD 카드 (7GB), 10년 묵은 것

## 파일 형식 필터링

PhotoRec은 특정 파일 타입만 선택적으로 스캔할 수 있어요. 스캔 대상 파일 확장자가 많을수록 시간이 오래 걸리거든요. 저는 속도를 우선했고 다음 형식만 지정했습니다:

- Toshiba: .jpg
- SD 카드: .png, .mp4, .jpg

PhotoRec이 복구할 수 있는 전체 파일 형식은 300개 이상의 파일 패밀리와 480개 이상의 확장자를 지원하고 있어요.

## PhotoRec 실행

솔직히 준비를 잘 못했어요. Toshiba 노트북에 USB를 꽂아서 PhotoRec을 직접 부팅했는데, 몇 개 파일이 복구될지, 얼마나 걸릴지, 파일 품질이 어떨지 아무 예상도 없었거든요.

복구된 파일을 저장할 위치를 선택해야 했는데, 손에 있는 건 이미 꽂혀있는 7GB USB였어요. 실제로 사용 가능한 공간은 5.5~6GB 정도였습니다.

사실 미리 이 경고를 읽었어야 했어요...

> 복구된 파일을 원본 파일시스템에 저장하지 마세요. 그렇지 않으면 손실된 데이터가 덮어써져 영구적으로 손실될 수 있습니다.

하지만 실행 결과는 꽤 좋았습니다:

- 복구 소요 시간: 5시간 이상
- 복구된 파일: 16,000개 이상
- 사용한 공간: USB의 전체 남은 공간
- 결과: 저장소 부족으로 중단
- 생성된 폴더: 30개, 각 폴더에 수천 개 파일

## 결과 분석

복잡하긴 했지만 꽤 훌륭한 결과였어요.

PhotoRec이 엄청 많은 파일을 복구했지만, 원래 파일명이나 폴더 구조는 복원되지 않았습니다. LinkedIn 프로필 사진, 웹 아바타, 웹사이트 로고, SNS 채팅 프로필 사진(3가지 크기 모두), 중복된 파일들까지 온갖 .jpg 파일이 다 섞여 있었어요.

결과물을 정리하는 건 정말 지루하고 오래 걸렸어요. 파이썬 스크립트나 AI 모델이 있었으면 파일 크기나 다른 패턴으로 자동 분류할 수 있었을 텐데요. 각 폴더에는 이미지와 이미지 크기가 뒤섞여 있어서 시각적으로 어떤 게 유용하고 어떤 게 쓰레기인지 구분하기 어려웠습니다.

흥미로운 점은 PhotoRec을 종료했을 때, 더 큰 저장소를 제공하면 멈춘 지점부터 계속 진행할 수 있다고 표시했다는 거예요. Toshiba 노트북에서는 잘 작동했는데, 제가 여러 해 동안 사용했던 기기라 그렇게 많은 파일이 복구된 것 같습니다.
