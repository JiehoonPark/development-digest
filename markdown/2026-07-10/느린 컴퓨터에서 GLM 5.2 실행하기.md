## 핵심 개념

> [!abstract]
> 744B 파라미터 혼합 전문가(MoE) 모델인 GLM-5.2를 약 25GB RAM의 일반 컴퓨터에서 순수 C로 실행하는 경량 엔진 Colibrì를 소개합니다. 디스크에서 전문가 모델을 스트리밍하여 필요한 파라미터만 로드하는 방식으로 구현되었습니다.

## 상세 내용

- 744B MoE 모델이 토큰당 약 40B 파라미터만 활성화하므로, 디스크 스트리밍으로 저사양 기계에서 실행 가능
- 순수 C 단일 파일(~1,300줄)로 구현되었으며 BLAS, Python, GPU 의존성이 없음
- MLA 어텐션, MTP 추측 디코딩, int4 양자화 등으로 성능 최적화하여 콜드 캐시 기준 초당 0.05-0.1 토큰 처리

> [!tip] 왜 중요한가
> 대규모 언어 모델을 저사양 하드웨어에서 실행하는 혁신적인 방법으로, 엣지 배포와 개인용 컴퓨터에서의 AI 활용을 가능하게 합니다.

## 전문 번역

# 소비자 PC에서 744B 대형 언어모델 돌리기: colibrì 엔진 소개

```
$ ./coli chat
🐦 colibrì v1.0 — GLM-5.2 · 744B MoE · int4 · streaming CPU
✓ pronto in 32s · residente 9.9 GB
› ciao!
◆ Ciao! 😊 Come posso aiutarti oggi?
```

당신의 PC에 RAM이 25GB 정도 있다면, 지금 바로 744억 파라미터짜리 GLM-5.2 모델을 실행할 수 있습니다. 순수 C 코드로, 외부 라이브러리 없이, 디스크에서 필요한 부분만 스트리밍해서요.

## 이게 어떻게 가능할까?

혼합 전문가(Mixture-of-Experts, MoE) 모델의 핵심은 이겁니다. 744B 파라미터 중에서 토큰 하나당 단 40B만 활성화됩니다. 그것도 매 토큰마다 바뀌는 부분은 약 11GB에 불과합니다.

따라서 우리는 이렇게 분리할 수 있어요.

**항상 메모리에 있어야 하는 부분 (~17B 파라미터)**
- 어텐션, 공유 전문가, 임베딩 레이어
- int4 양자화로 약 9.9GB만 차지

**필요할 때만 꺼내는 부분 (~19GB 파라미터)**
- 21,504개의 라우팅 전문가들 (75개 MoE 레이어 × 256개 전문가)
- 디스크의 370GB에 보관
- 레이어별 LRU 캐시, 자주 쓰는 전문가를 위한 핀 메모리, OS 페이지 캐시로 효율적으로 관리

## 구현된 기능들

**완벽한 GLM-5.2 포워드 패스**
transformers 라이브러리와 토큰 단위로 검증했습니다. 작은 테스트 모델로 teacher-forcing 32/32, greedy 디코딩 20/20 모두 일치합니다.

**MLA 어텐션 (MLQAttention)**
q/kv-LoRA와 부분적 RoPE를 사용합니다. KV 캐시가 토큰당 576 float으로 압축되는데요, 이는 원래 32,768에서 57배 감소한 수치입니다. GLM-5.2는 64개의 어텐션 헤드를 가지고 있고 GQA를 쓰지 않거든요.

**DeepSeek-V3 스타일 라우터**
sigmoid 라우터, 공유 전문가, 그리고 첫 3개 조밀 레이어를 구현했습니다.

**다중 토큰 예측(MTP) 추론**
GLM-5.2의 78번 레이어가 여러 토큰을 한 번에 생성하고, 메인 모델이 이를 검증합니다. 
- 드래프트 헤드는 int8이어야 작동합니다 (int4에서는 수용 비율이 0~4%로 떨어집니다)
- int8일 때는 39~59% 수용률, 포워드당 2.2~2.8 토큰 처리
- 손실 없는 추론이며, 리젝션 샘플링으로도 정확도를 유지합니다

다만 주의할 점이 있습니다. 캐시가 차가운 상태에서는 검증된 각 드래프트가 추가 전문가를 로드하게 되어(660→1100 전문가 로드/토큰), 캐시가 워밍업될 때까지는 오히려 더 느릴 수 있습니다. 이를 위해 적응형 보호 메커니즘과 `DRAFT=0` 옵션을 제공합니다.

**정수 연산 최적화**
Q8_0 스타일의 int8 활성화와 AVX2 maddubs를 사용합니다.
- int8 행렬곱: 1.4~2.5배 빠름 (초당 119 GFLOP)
- int4: 배치에서 1.8배 빠름
- int4 단일 행은 f32가 더 빨라서 자동으로 선택됩니다

**MLA 가중치 흡수**
DeepSeek의 트릭을 적용했습니다. 디코딩 시 토큰마다 k/v를 재구성할 필요가 없죠. 쿼리가 kv_b를 흡수하고, 어텐션 후 컨텍스트를 프로젝션합니다. 검증 완료: TF 32/32, 생성 20/20.

**비동기 전문가 프리페칭**
한 블록의 전문가를 계산하는 동안, 커널이 이미 다음 블록을 읽어옵니다 (WILLNEED).

**양자화 커널**
int8, packed int4, packed int2를 행 단위 스케일로 구현했습니다. AVX2 기반이고 역양자화는 사용할 때 합니다.

**희소 어텐션 (DSA) - 진행 중**
Lightning-indexer 가중치를 다운로드 중입니다. 현재는 2048 토큰 이하의 컨텍스트에서 정확한 조밀 어텐션을 사용합니다.

**배치 단위 MoE**
프리필과 MTP 검증에서 각 전문가를 한 번에 로드한 후, 그것을 라우팅하는 모든 위치에 적용합니다.

**BPE 토크나이저**
C로 구현된 바이트 레벨 BPE (GPT-2 스타일, 32만 개의 병합).

**메모리 안전성**
전문가 캐시는 시작 시 `MemAvailable`에서 자동으로 크기가 결정됩니다. 실제 작업 세트, KV, MTP 행, 재구성 버퍼 등을 고려한 정직한 예측이거든요.

**FP8→int4 변환기**
오프라인 변환 도구를 제공합니다. 한 번에 한 샤드씩 다운로드(~5GB)하고, 역양자화한 후, 엔진의 컨테이너로 재양자화합니다. 756GB FP8 체크포인트가 동시에 디스크에 있을 필요가 없고, 재개 가능합니다.

## 실제 성능 (WSL2, 12 코어, 25GB RAM, NVMe)

| 항목 | 수치 |
|------|------|
| 디스크 모델 (int4) | ~370 GB |
| 메모리 상주 (조밀 부분, int4) | 9.9 GB |
| 로드 시간 | ~30초 |
| 채팅 중 최대 메모리 | ~20 GB (자동 제한) |
| 콜드 캐시 디코딩 비용 | ~11 GB/토큰 (75 레이어 × 8 전문가) |
| 디스크 한계 (VHDX 랜덤) | ~1 GB/s → 0.05~0.1 tok/s |
| MTP 추론 (int8 헤드) | 2.2~2.8 tok/forward |

빠르지는 않습니다. 하지만 H100 한 개 팬 정도의 가격인 PC에서 최첨단 744B 모델이 정답을 출력하고 있거든요. 캐시가 워밍업되고 자주 쓰는 전문가를 메모리에 핀 처리하고 MTP를 활용하면, 실제 응답 지연은 상당히 개선됩니다.

## SSD 관련 참고사항

콜드 스타트는 랜덤 읽기가 많습니다(~11 GB/토큰). 하지만 읽기는 SSD 수명에 의미 있는 영향을 주지 않습니다. colibrì는 읽기만 하거든요.

실제 문제가 되는 부분은 두 가지입니다.

**1) 스왑 트래픽**
시스템 메모리가 부족할 때의 쓰기입니다. 쓰기는 SSD를 손상시킵니다. `--ram` 예산을 적절히 설정하세요. colibrì의 자동 예산은 스왑을 피하도록 설계됩니다.

**2) 장시간 읽기 시 발열**
저가형 SSD는 몇 시간 동안 풀 읽기로 동작하면 열이 오릅니다. 드라이브 온도를 모니터링하세요.
