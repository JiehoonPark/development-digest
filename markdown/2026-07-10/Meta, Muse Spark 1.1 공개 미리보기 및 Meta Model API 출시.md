## 핵심 개념

> [!abstract]
> Meta Superintelligence Labs가 발표한 Muse Spark 1.1은 멀티모달 추론 모델로, 도구 사용, 컴퓨터 제어, 코딩 등에서 이전 버전보다 크게 향상되었습니다. 1백만 토큰 컨텍스트 윈도우, 다중 에이전트 조율, 복잡한 멀티태스크 처리 능력을 갖춘 공개 API 형태로 개발자에게 제공됩니다.

## 상세 내용

- 에이전트 작업에 최적화된 멀티모달 추론으로 계획 수립, 도구 사용, 컴퓨터 제어 능력 대폭 강화
- 1백만 토큰 컨텍스트 관리와 다중 에이전트 조율로 복잡한 작업을 병렬 처리
- 코딩 작업에서 대규모 버그 진단, 기능 구현, 코드 마이그레이션 등 엔터프라이즈급 능력 제공

> [!tip] 왜 중요한가
> AI 에이전트 개발자는 Meta Model API를 통해 고성능 멀티모달 추론 능력을 활용하여 자동화 시스템의 복잡도를 크게 높일 수 있습니다.

## 전문 번역

# Meta가 발표한 Muse Spark 1.1: 차세대 멀티모달 AI 모델의 등장

Meta 인공지능 연구팀이 새로운 AI 모델 Muse Spark 1.1을 공개했습니다. 이전 버전 Muse Spark에서 크게 향상된 이 모델은 도구 활용, 컴퓨터 제어, 코딩, 그리고 멀티모달 이해 능력에서 특히 두드러진 성능을 보여주고 있습니다.

이번 업데이트는 단순한 성능 개선을 넘어, 개인화된 초지능(personal superintelligence)이라는 Meta의 비전을 한 단계 더 앞으로 나아가게 했습니다. 이번 주에 함께 공개된 Muse Image와 함께라면, 개인의 목표 달성과 창의적 작업, 그리고 실질적인 문제 해결이 더욱 수월해질 것으로 기대됩니다.

개발자들은 이제 새로운 Meta Model API를 통해 Muse Spark 1.1에 접근할 수 있습니다. Meta AI 앱과 meta.ai 웹사이트의 "Thinking" 모드에서 미리 이용해볼 수 있습니다.

## 에이전트 작업에서의 탁월한 성능

Muse Spark 1.1이 정말 빛나는 분야는 복잡한 계획과 실행이 필요한 작업들입니다. 여러 외부 앱과 서비스를 동시에 조율해야 하는 상황에서도, 별도의 학습 없이 새로운 도구나 커스텀 스킬에 적응할 수 있거든요.

이전 모델 대비 복잡한 프로젝트를 훨씬 빠르게 처리합니다. 왜냐하면 다중 에이전트 시스템을 조율하도록 학습했기 때문입니다. 메인 에이전트로서는 상황을 파악하고 계획을 세운 뒤 여러 서브에이전트에게 작업을 분담할 수 있고, 서브에이전트로서는 자신의 역할에 집중하면서 필요할 때 메인 에이전트에게 보고하는 방식으로 동작합니다.

특히 100만 토큰 규모의 컨텍스트 윈도우를 효과적으로 관리할 수 있다는 점도 주목할 만합니다. 과거의 작업 내역을 기억했다가 필요할 때 정보를 꺼내쓰고, 중요한 부분만 남겨두는 방식으로 컨텍스트를 정리하거든요.

## 실시간 변화에 대응하는 컴퓨터 제어

Muse Spark 1.1은 여러 애플리케이션을 오가며 진행되는 작업에서 정말 강점을 보입니다. 장시간의 세션 동안 맥락을 유지하면서 요구 사항의 변화에 유연하게 대응하고, 낯선 인터페이스도 최소한의 사용자 개입으로 네비게이션할 수 있습니다.

중요한 건 단순히 클릭 하나하나를 차례대로 실행하는 방식이 아니라는 겁니다. Muse Spark 1.1은 언제 자동화가 필요한지, 언제 직접 상호작용이 효율적인지를 판단할 줄 압니다. 스크립트 작성이 더 빠를 땐 코드를 짜고, 직접 클릭하는 게 간단할 땐 UI를 조작하며, 매 단계마다 여러 작업을 배치로 실행합니다.

실제 예시를 보면, 저녁 파티 예약 같은 작업 중에도 새로운 정보가 들어오면 알아서 주문을 업데이트하는 식으로 대응합니다. 사용자가 일일이 지시할 필요가 없다는 뜻입니다.

## 엔터프라이즈급 코딩 능력

코딩 성능도 눈에 띄게 개선됐습니다. 대규모 복잡한 코드베이스를 다루는 실무 작업에서 특히 그렇습니다. 까다로운 버그를 진단해서 고치고, 기존 시스템에 새로운 기능을 추가하며, 대규모 코드 마이그레이션도 수행할 수 있게 됐거든요.

웹 애플리케이션 개발이나 종단 간 질문 응답 같은 작업에서는 이전 모델 대비 상당한 성능 향상을 보여주고 있습니다.

다양한 에이전트 코딩 환경에도 잘 적응합니다. 계획 모드, 목표 조건부 설정, 서브에이전트 분담, 컨텍스트 압축 같은 주요 기능들을 자연스럽게 지원하기 때문입니다.

OpenCode에서의 디버깅 사례를 보면, Muse Spark 1.1은 채팅 웹앱을 구축한 뒤 자동으로 스크린샷을 찍어 사용자 눈에 띄는 오류를 파악하고, 그 오류를 일으킨 코드 부분을 추적해서 수정하고, 변경 사항을 검증합니다. 코딩, 멀티모달 이해, 도구 활용이 매끄럽게 통합되는 모습입니다.

Meta 내부에서도 개발자와 연구원들이 이미 매일 Muse Spark 1.1을 업무에 활용하고 있습니다. Meta의 자체 코딩 평가 벤치마크(Meta Internal Coding Bench)에서는 이전 모델을 크게 능가하며 업계 최고 수준의 경쟁력을 갖추고 있습니다.

연구원들은 이제 모델 개발과 평가 작업까지 Muse Spark 1.1에 맡기며 자동화하고 있는 상황입니다.

## 멀티모달 능력의 실전 활용

코딩과 에이전트 기능뿐 아니라 인식, 멀티모달 추론, 도구 활용에서도 뛰어난 모습을 보입니다. 실제 환경과 상호작용하면서도 정확한 결과를 만들어낼 수 있으며, 이미지를 코드로 변환하는 능력, 초상세한 이미지와 영상 설명, 멀티모달 워크플로우 자동화 등에서 강점을 갖추고 있습니다.

특히 눈으로 보고 동시에 행동해야 하는 상황에서 Muse Spark 1.1의 멀티모달 능력이 진가를 발휘합니다. 시각과 음성 정보를 검토하고, 장시간의 워크플로우에 걸쳐 세부 사항을 보존한 뒤, 사용자 대신 컴퓨터를 조작할 때 그 정보를 활용할 수 있거든요.

Facebook Marketplace 에이전트라는 사례를 보면, 스마트폰으로 촬영한 동영상에서 유용한 사진들을 추출하고 상품에 대해 추론한 뒤, 사용자의 브라우저를 제어해서 Facebook Marketplace에 상품 리스팅을 올리는 작업까지 자동으로 처리합니다.

배포 전에는 물론 광범위한 안전성 평가를 거쳤습니다.
