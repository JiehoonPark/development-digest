## 핵심 개념

> [!abstract]
> 한 개발자가 3D 프린팅한 태양계 행성 모형과 LED 조명으로 포치를 꾸미고, 이를 Amazon Fire TV의 Vega 플랫폼과 React Native로 제어하는 프로젝트를 소개합니다. VS Code 확장과 로컬 시뮬레이터, CLI 도구로 개발이 수월했고, WLED 컨트롤러 제어는 일반 fetch HTTP 요청으로, 음성 명령은 aria-label 추가만으로 구현했습니다. React/React Native 경험자에게 Vega가 얼마나 낮은 진입 장벽을 제공하는지 보여주는 실전 경험담입니다.

## 아티클

천체 사진과 행성에 진심인 개발자가 자기 집 포치(현관 테라스)를 3D 프린팅한 행성 모형과 LED 조명으로 꾸미고, 이걸 Amazon의 새로운 스마트 TV 플랫폼 Vega와 React Native로 제어하는 프로젝트를 소개합니다. 짧은 영상이지만 Vega가 React/React Native 개발자에게 얼마나 쉽게 다가오는지, 그리고 실제 하드웨어(LED 컨트롤러)와 어떻게 연동했는지에 대한 실전 경험이 담겨 있습니다.

## 포치를 태양계로 만들기

이 프로젝트의 출발점은 천문학에 대한 가족의 애정이었습니다. 3D 프린터로 출력한 태양계 행성 모형들을 포치 뒤편에 쭉 늘어놓고, 각 행성을 제어 가능한 LED로 조명했는데요. 덕분에 포치에서 영화를 볼 때 분위기가 훨씬 좋아졌지만, 여기서 한 걸음 더 나아가 이 행성 조명들을 커다란 프로젝션 스크린과 연결하고 싶었다고 합니다.

## 왜 Vega를 선택했나

이 연결 고리를 만들기 위해 다양한 옵션을 살펴보다가 Amazon Fire Stick과 Vega 조합을 선택했습니다. 결정적인 이유는 Vega의 UI가 React Native 기반이라는 점이었는데요. 평소 React와 React Native에 익숙했던 만큼 자연스러운 선택이었습니다.

실제로 사용해보니 예상보다 훨씬 수월했다고 합니다. Vega는 VS Code용 확장 프로그램을 제공하고, 로컬 시뮬레이터가 내장되어 있어서 코드를 디버깅하기가 매우 편리했습니다. 여기에 프로젝트를 빠르게 시작할 수 있는 CLI 도구까지 갖추고 있어 초기 셋업 부담도 적었습니다.

또 한 가지 흥미로운 점은 AI 코딩 에이전트들이 React Native를 워낙 잘 알고 있다 보니, Vega 공식 문서만 추가로 참고하도록 안내하는 것만으로도 세부적인 부분까지 무리 없이 개발을 이어갈 수 있었다는 점입니다.

## 태양계 지도와 D-pad 내비게이션

이렇게 만든 결과물은, 손님이 놀러 왔을 때 화면에 태양계 지도를 띄우고 각 행성에 대한 정보를 보여줄 수 있는 앱입니다. 리모컨의 D-pad를 이용해 지도 위 행성들과 실제 포치의 LED 조명 사이를 오가며 탐색할 수 있도록 구현했습니다.

## WLED 컨트롤러와의 HTTP 연동

LED 조명 제어는 WLED 컨트롤러를 통해 이뤄졌습니다. Vega 앱 안에서 일반적인 `fetch`를 사용해 WLED 컨트롤러로 HTTP 명령을 바로 전송하는 방식인데요. 별도의 복잡한 통신 계층 없이, 웹 개발에서 익숙한 fetch API 하나로 실제 하드웨어를 제어할 수 있었다는 점이 인상적입니다.

## 음성 제어로 마무리하기

마지막 화룡점정은 음성 명령이었습니다. 사용자가 행성 이름을 말하면 화면과 조명에 해당 행성이 표시되도록 만들고 싶었는데, 이를 구현하는 과정은 놀랍도록 간단했습니다. 컴포넌트에 aria-label만 추가해주면 나머지 음성 인식 처리는 Vega가 알아서 담당하고, 이름이 호출됐을 때 press 이벤트를 자동으로 발생시켜줬습니다. "Saturn"이라고 말하면 바로 반응하는 식인데, 이 기능 하나로 프로젝트의 완성도가 확 살아났다고 합니다.

## 정리

- **접근성**: React와 React Native 경험이 있다면 Vega로 Fire TV용 앱을 만드는 진입 장벽이 매우 낮습니다. VS Code 확장, 로컬 시뮬레이터, CLI 스캐폴딩 도구가 개발 경험을 잘 뒷받침해줍니다.
- **AI 코딩 에이전트 활용**: React Native에 익숙한 AI 코딩 에이전트에 Vega 공식 문서만 추가로 참조시키면 별도 학습 곡선 없이 개발을 진행할 수 있었습니다.
- **하드웨어 연동의 단순함**: WLED 같은 IoT 컨트롤러도 표준 `fetch` HTTP 요청만으로 제어 가능해, 웹 개발자의 기존 지식을 그대로 활용할 수 있습니다.
- **접근성 속성의 재활용**: aria-label을 추가하는 것만으로 음성 명령 인식과 이벤트 트리거링이 자동으로 처리되어, 접근성을 위한 마크업이 곧바로 음성 UX 구현으로 이어지는 점이 실용적입니다.
- React/React Native 지식을 가진 개발자라면 Vega를 통해 별도의 러닝커브 없이 TV 화면 기반 애플리케이션을 손쉽게 만들어볼 수 있습니다.
