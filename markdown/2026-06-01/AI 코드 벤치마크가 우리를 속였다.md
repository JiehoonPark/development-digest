## 핵심 개념

> [!abstract]
> 이 영상은 AI 코드 생성 벤치마크의 신뢰성 문제를 다루고 있습니다. 벤치마크 점수가 실제 성능을 제대로 반영하지 못하는 사례들을 분석하고, 벤치마크 설계의 문제점과 개선 방안을 제시합니다.

## 상세 내용

- 벤치마크와 실제 성능의 괴리: 높은 벤치마크 점수를 받은 모델들이 실제 실무 환경에서는 기대와 다른 성능을 보이는 사례들이 존재합니다.
- 테스트 데이터 오염 문제: 벤치마크 데이터셋이 학습 데이터에 포함되거나 유사한 경우, 모델이 실제로 해당 작업을 학습한 것이 아닌 암기한 것일 수 있습니다.
- 벤치마크 설계의 한계: 현실의 복잡한 코딩 작업들을 단순화하여 평가하기 때문에, 벤치마크가 측정하지 못하는 중요한 능력들이 많습니다.
- 메트릭의 불완전성: 문제 해결률만으로는 코드의 유지보수성, 효율성, 안전성 같은 중요한 측면을 평가할 수 없습니다.
- 모델 선택의 위험성: 벤치마크 점수만 믿고 모델을 선택하면 실제 프로덕션 환경에서 예상과 다른 결과가 나올 수 있습니다.

> [!tip] 왜 중요한가
> AI 코드 생성 도구를 도입할 때 벤치마크 숫자에만 의존하는 것의 위험성을 이해하고, 실제 사용 사례에 기반한 평가가 중요함을 인식하게 해줍니다.
