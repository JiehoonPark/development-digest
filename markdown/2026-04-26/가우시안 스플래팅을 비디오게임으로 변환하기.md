## 핵심 개념

> [!abstract]
> 가우시안 스플래팅 기술로 생성된 포토리얼리스틱한 환경을 브라우저 기반 게임으로 만드는 방법을 다룹니다. PlayCanvas를 이용해 충돌 메시 생성, 라이팅 처리, NPC 및 물리 엔진을 통합하는 과정을 단계별로 설명합니다.

## 상세 내용

- splat-transform CLI 도구를 사용해 스플래팅 데이터를 게임 엔진에 최적화하고 충돌 메시 자동 생성
- 스트리밍 LOD 기술로 모바일 장치에서도 고품질 그래픽을 부드럽게 렌더링 가능
- 바이오 라이팅 프로브를 사용해 3D 스캔 환경의 조명을 일반 PBR 메시에 적용

> [!tip] 왜 중요한가
> 최신 3D 캡처 기술을 웹 게임에 통합하는 실용적인 방법으로, 개발자들이 고품질 환경 제작 비용을 크게 절감할 수 있습니다.

## 전문 번역

# 가우시안 스플래팅으로 만드는 인터랙티브 게임 월드

가우시안 스플래팅은 정말 아름다운 3D 환경을 손쉽게 만들어줍니다. 다만 한 가지 문제가 있어요. 스플래팅은 본질적으로 방향을 가진 구름 덩어리들의 집합일 뿐입니다. 삼각형도 없고, 충돌체도 없으며, 네비게이션 메시도, 조명도 없거든요. 그래서 캐릭터를 배치하면 벽을 뚫고 떠다니며 마치 다른 세계에서 온 존재처럼 보이게 됩니다.

이 글에서는 이 모든 문제를 해결한 데모를 만드는 과정을 소개할 거예요.

👉 **브라우저에서 직접 플레이해보세요** - WASD로 이동, 마우스로 조준, 좌클릭으로 사격합니다.

👉 **전체 프로젝트를 확인하세요** - PlayCanvas 프로젝트 전체가 공개되어 있으며, 이 글에서 언급하는 모든 스크립트를 직접 읽고 포크하고 수정할 수 있습니다.

이 씬은 실제 폐허를 스캔한 아름다운 실내 공간인데, Christoph Schindelar가 촬영했습니다. 그는 가우시안 스플래팅 분야에서 최고의 아티스트 중 한 명이거든요. 그가 실제 장소를 스캔해달라고 제안했을 때 정말 기쁜 마음으로 수락했어요. 이 스플래팅 위에 물리 충돌체, 베이크된 라이팅 프로브 그리드, Recast 네비메시, 각자의 성격을 가진 8개의 NPC, 그리고 클래식한 FPS 게임 루프를 얹었습니다. 모든 것이 브라우저 탭 하나에서 돌아갑니다.

## 🏗️ 구현 방법

이제 단계별로 어떻게 만들었는지 설명하겠습니다.

## 📥 1단계: SuperSplat에서 스플래팅 다운로드하기

코드를 작성하기 전에 먼저 씬이 필요합니다. SuperSplat에서 "Downloadable" 태그가 붙은 스플래팅은 모두 Creative Commons 라이선스로 공개되어 있어요. `.ply` 나 `.sog` 파일을 다운로드해서 PlayCanvas 프로젝트에 바로 넣으면 됩니다. 제가 선택한 스캔은 라이팅, 디테일, 스케일이 이미 완벽했기 때문에 추가로 미술 작업을 할 필요가 없었어요.

**지금 시도해보세요** - 다운로드 가능한 스플래팅 리스트로 바로 가서 마음에 드는 것을 골라보세요.

## 📡 2단계: 스플래팅을 Streamed SOG 포맷으로 변환하기

이어지는 모든 작업의 핵심 도구는 `splat-transform` 입니다. PlayCanvas의 오픈소스 CLI 툴인데, 스플래팅 변환을 담당하죠. 여기서는 Streamed LOD를 위해 이를 활용하고, 다음 단계에서 충돌 메시를 만들 때도 사용합니다.

제 씬은 수백만 개의 가우시안으로 이루어져 있어요. 이 정도 규모를 하나의 `.sog` 파일로 배포하면 휴대폰이나 느린 네트워크 사용자들이 정말 고생합니다. 해결책은 Streamed LOD입니다. 거대한 하나의 파일 대신, SuperSplat과 splat-transform은 SOG 청크 폴더와 매니페스트 파일로 나누어 저장합니다. 런타임에 카메라의 위치와 기기의 성능에 따라 필요한 청크만 로드하죠. 고사양 데스크톱은 플레이어 주변의 전체 디테일을 가져오고, 휴대폰은 가벼운 버전을 가져오며, 누구도 전체 파일을 기다리지 않아도 됩니다.

`Scripts/streaming-lod.mjs` 스크립트는 카메라와 연동되어 게임이 시작되기 전에 플레이어 주변의 청크를 완전히 로드하도록 합니다. 덕분에 전투 중에 갑자기 그래픽이 튀어나오는 현상을 피할 수 있어요.

```bash
npm install -g @playcanvas/splat-transform
```

**지금 시도해보세요** - 당신의 스플래팅이 수백만 개 이상의 가우시안이라면 Streamed LOD로 내보내세요. SuperSplat의 내보내기 대화상자에서 가장 쉽게 할 수 있습니다. 모바일 플레이어들이 정말 고마워할 거예요.

## 🧱 3단계: 충돌 메시 생성하기

이 부분이 예전에는 정말 복잡했어요. 스플래팅은 표면이 없으니까 물리 엔진이 이를 인식하지 못합니다. 플레이어가 위를 걸을 수 없고, 총알이 벽에 충돌할 수 없으며, NPC가 경로를 계산할 수 없죠. 여기서 `splat-transform`의 `-K` / `--collision-mesh` 플래그가 정말 빛을 발합니다.

이 도구는 스플래팅을 복셀화하고, 지정한 시드 위치에서 네비게이션 가능한 공간을 찾아 플러드 필(flood-fill)한 후, 복셀 기반의 수밀(watertight) GLB 파일을 출력합니다. PlayCanvas에 바로 임포트해서 메시 콜라이더로 사용할 수 있어요.

```bash
splat-transform scene.ply \
  --seed-pos 0,1,0 \
  --voxel-params 0.05,0.1 \
  --voxel-carve 1.6,0.2 \
  -K \
  scene.sog
```

이 명령 하나로 두 가지 파일이 생성됩니다:

- **scene.sog** - 빠른 반복 작업용 단일 파일 압축 스플래팅입니다. 실제 배포 빌드는 2단계의 Streamed 폴더를 사용합니다.
- **scene.collision.glb** - 복셀 기반으로 만든 메시인데, 실제 기하학을 정확히 따릅니다.

이 두 파일을 PlayCanvas 프로젝트에 넣고 GLB를 보이지 않는 엔티티에 Collision 컴포넌트(메시 타입)와 Rigid Body 컴포넌트(static)를 붙여줬어요. 그러면 플레이어가 밟을 바닥도 생기고, 총알이 벽과 충돌하며, NPC도 걸어다닐 수 있는 무언가가 생깁니다. 모델링도, 정리작업도 필요 없어요.

**지금 시도해보세요** - `splat-transform scene.ply -K scene.sog` 명령 하나로 이쁜 스플래팅이 실제 게임처럼 변합니다. 결과로 나온 `.collision.glb`를 프로젝트에 정적 메시 리지드바디로 추가하기만 하면 돼요.

## 💡 4단계: 스플래팅에서 라이팅 그리드 베이크하기

스플래팅은 모든 가우시안에 라이팅이 이미 구워져 있어요. 그래서 씬이 정말 아름답고 변하지 않습니다. 하지만 내 플레이어 무기 모델, NPC 병사들, 아이템은 일반적인 라이팅이 적용된 PBR 메시들이거든요. 체육관 조명처럼 밋밋한 곳에 있으면 완전히 어울리지 않을 겁니다. 스플래팅의 라이팅을 이 메시들이 상속받아야 해요.

다시 스플래팅에 라이팅을 입히고 싶지는 않았어요. 대신 이런 간단한 질문에 답할 수 있는 방법이 필요했습니다. "맵의 이 지점에서 얼마나 밝을까?" 그리고 이를 런타임에 일반 메시들을 위해 계산해야 했어요.

`Scripts/probes.js` 스크립트의 작동 원리는 이렇습니다:

1. 지정된 바닥 엔티티의 AABB(축 정렬 경계 상자)를 가져와서 지표면 위 1미터 높이에 1미터 간격의 프로브 격자를 만듭니다.

2. 작은 16×16 오프스크린 RenderTarget과 90° FOV 카메라를 만들어서 World 레이어만(즉, 스플래팅만) 렌더링합니다. 캐릭터, HUD, 무기 모델은 렌더링하지 않죠.

3. 각 프로브마다 정육면체의 6개 면(+X, -X, +Y, -Y, +Z, -Z)을 렌더링하고, 16×16 RGBA 출력을 readPixels로 읽어서 표준 Rec. 601 가중치를 사용해 밝기(luminance)를 계산합니다:

```javascript
this._faceLuminance += 0.299 * r + 0.587 * g + 0.114 * b;
```

4. 6개 면의 밝기를 평균내서 그 프로브의 최종 값을 구합니다.

5. 모든 프로브 데이터를 메모리에 3D 그리드로 저장해두고, 런타임에 어떤 위치의 밝기를 조회할 때는 주변 8개 프로브의 삼선형 보간(trilinear interpolation)을 합니다.

결과적으로 일반 메시들은 스플래팅의 조명 분위기를 물려받게 되고, 카드보드처럼 튀지 않게 됩니다.

**지금 시도해보세요** - 프로브 그리드 대신 환경 맵(lightmap)을 만들 수도 있지만, 프로브는 동적으로 변하는 조명에도 대응할 수 있고, 저장 공간도 훨씬 적게 씁니다.
