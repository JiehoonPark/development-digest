## 핵심 개념

> [!abstract]
> YourMemory는 인간의 기억 방식을 모델링한 지속적인 메모리 레이어로, AI 에이전트가 세션 간 대화 내용을 기억할 수 있게 합니다. Zep Cloud 대비 2배 높은 59% 재현율을 달성하며 Docker나 외부 서비스 없이 설치 가능합니다.

## 상세 내용

- Ebbinghaus 망각곡선 기반 생물학적 감쇠 알고리즘으로 자연스러운 메모리 관리
- 벡터 + 그래프 하이브리드 검색으로 관련 컨텍스트 포착
- pip install으로 간단 설치, 로컬 DuckDB 데이터베이스 사용

> [!tip] 왜 중요한가
> AI 에이전트에 장기적 메모리를 제공하여 반복 질문 제거와 개인화된 대화 경험을 구현할 수 있습니다.

## 전문 번역

# YourMemory: AI 에이전트를 위한 지속적인 기억 시스템

매번 대화를 시작할 때마다 AI 어시스턴트가 처음부터 시작하는 문제를 겪어본 적 있나요? 같은 질문을 반복하고, 당신의 선호도를 잊고, 기술 스택을 다시 학습하는 식으로요. 대화 사이에 기억이 전혀 남지 않는 거죠.

YourMemory는 이 문제를 해결합니다. AI 에이전트에게 인간의 기억 방식처럼 작동하는 지속적인 메모리 레이어를 제공하는데요. 중요한 정보는 계속 남고, 잊혀진 것들은 자연스럽게 사라지며, 낡은 정보는 자동으로 갱신됩니다. 단 두 줄의 명령어로 설치되고, 별도의 인프라도 필요 없습니다.

## 얼마나 잘 작동할까요?

LoCoMo-10 벤치마크(10개의 멀티세션 대화에서 1,534개의 QA 쌍)로 테스트했습니다.

| 시스템 | Recall@5 | 95% CI |
|--------|----------|--------|
| YourMemory (BM25 + vector + graph + decay) | 59% | 56–61% |
| Zep Cloud | 28% | 26–30% |

같은 벤치마크에서 Zep Cloud 대비 **2배 우수한 회수율**을 달성했습니다. 전체 방법론과 샘플별 분석은 BENCHMARKS.md에서 확인할 수 있으며, 자세한 설명은 "I built memory decay for AI agents using the Ebbinghaus forgetting curve" 글을 참고하세요.

## 설치부터 실행까지

Python 3.11, 3.12, 3.13, 3.14를 지원합니다. Docker도, 데이터베이스 설정도, 외부 서비스도 필요 없어요.

### 1단계 — 설치
```bash
pip install yourmemory
```

### 2단계 — 초기 설정 (한 번만)
```bash
yourmemory-setup
```
spaCy 언어 모델을 다운로드하고 `~/.yourmemory/memories.duckdb`에 로컬 데이터베이스를 초기화합니다.

### 3단계 — 설정 경로 확인
```bash
yourmemory-path
```
실행 파일의 전체 경로와 바로 복사할 수 있는 설정 블록을 출력합니다.

### 4단계 — AI 클라이언트와 연결

**Claude Code**

`~/.claude/settings.json`에 다음을 추가하세요:

```json

    }
  }

```

그 다음 Cmd+Shift+P → Developer: Reload Window로 리로드합니다.

**Claude Desktop**

macOS의 경우 `~/Library/Application Support/Claude/claude_desktop_config.json`, Windows의 경우 `%APPDATA%\Claude\claude_desktop_config.json`에 추가하세요:

```json

    "yourmemory": {
      "command": "yourmemory"
    }
  }
}
```

Claude Desktop을 다시 시작합니다.

**Cline (VS Code)**

VS Code는 shell PATH를 상속받지 않으니 먼저 `yourmemory-path`를 실행해서 전체 경로를 가져오세요. 그 다음 Cline → MCP Servers → Edit MCP Settings에서:

**MCP 호환 클라이언트 (일반)**

YourMemory는 표준 stdio MCP 서버입니다. Windsurf, Continue, Zed, MCP를 지원하는 모든 클라이언트에서 작동합니다. 클라이언트가 shell PATH를 상속받지 않으면 `yourmemory-path`의 전체 경로를 사용하세요.

### 5단계 — 프로젝트에 메모리 지침 추가

```bash
cp sample_CLAUDE.md CLAUDE.md
```

`CLAUDE.md`를 편집해서 YOUR_NAME과 YOUR_USER_ID를 바꾸세요. 이제 Claude는 모든 작업에서 자동으로 recall → store → update 워크플로우를 따릅니다.

## MCP 도구

세 가지 도구가 있습니다. CLAUDE.md가 설정되면 Claude가 자동으로 호출합니다.

| 도구 | 언제 | 역할 |
|------|------|------|
| `recall_memory(query)` | 모든 작업 시작 | 유사도와 강도로 정렬된 관련 기억을 표시 |
| `store_memory(content, importance)` | 새로운 정보 학습 후 | 생물학적 감쇠와 함께 임베드하고 저장 |
| `update_memory(id, new_content)` | 기억이 낡았을 때 | 재임베드하고 교체 |

### 예제 세션

```python
store_memory("Sachit prefers tabs over spaces in Python", importance=0.9, category="fact")
# 다음 세션 — 다시 말해주지 않아도:
recall_memory("Python formatting")
# → {"content": "Sachit prefers tabs over spaces in Python", "strength": 0.87}
```

### 카테고리별 기억 유지 기간

| 카테고리 | 회수 없이 유지 | 사용 사례 |
|---------|--------------|---------|
| strategy | ~38일 | 성공한 패턴 |
| fact | ~24일 | 선호도, 정체성 |
| assumption | ~19일 | 추론된 맥락 |
| failure | ~11일 | 오류, 환경별 이슈 |

## 작동 원리

### Ebbinghaus 망각곡선

기억의 강도는 지수적으로 감쇠합니다. 다만 중요도와 회수 빈도가 이 감쇠를 늦춥니다:

```
effective_λ = base_λ × (1 - importance × 0.8)
strength = importance × e^(−effective_λ × days) × (1 + recall_count × 0.2)
score = cosine_similarity × strength
```

자주 회수되는 기억은 감쇠에 저항합니다. 강도가 0.05 이하인 기억은 24시간마다 자동으로 삭제됩니다.

### 하이브리드 검색: Vector + Graph

검색은 두 라운드로 진행되어 어휘 기반 검색으로 놓치는 관련 맥락까지 표시합니다.

**라운드 1 — Vector 검색**: 모든 기억에 대해 코사인 유사도를 계산하고, 임계값 이상의 상위 k개를 반환합니다.

**라운드 2 — Graph 확장**: 라운드 1의 결과에서 시작해 BFS 탐색을 수행하며, 어휘는 다르지만 맥락을 공유하는 기억(의미적 edge로 연결된, 코사인 유사도 ≥ 0.4)을 표시합니다.

```
recall("Python backend")
라운드 1 → [1] Python/MongoDB (sim=0.61)
         [2] DuckDB/spaCy (sim=0.19)
라운드 2 → [5] Docker/Kubernetes (sim=0.29 — 임계값 이하이지만 
                                    graph를 통해 표시됨)
```

### 연쇄 인식 정리

감쇠된 기억이라도 graph 이웃 중 하나라도 정리 임계값 이상이면 살아있는 상태로 유지됩니다. 관련된 기억들이 함께 나이 먹는 거죠.
