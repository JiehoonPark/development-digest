## 핵심 개념

> [!abstract]
> Simon Willison이 Qwen3.6-35B-A3B(20.9GB 양자화 모델)와 Claude Opus 4.7을 '펠리칸이 자전거를 타는' 벤치마크로 비교했으며, 의외로 로컬에서 실행되는 Qwen 모델이 더 높은 품질의 SVG 이미지를 생성했다고 보고한다. 이는 모델 비교가 특정 작업에서는 일반적인 성능 순위와 다를 수 있음을 보여주는 사례이며, SVG 생성 작업에 국한된 재미있는 관찰이다. Willison은 이것이 모델 비교의 어려움과 벤치마크 설계의 한계를 드러낸다고 지적한다.

## 상세 내용

- 벤치마크 결과: Qwen3.6-35B-A3B(Unsloth의 양자화 버전, MacBook Pro M5 + LM Studio)가 Claude Opus 4.7보다 더 정확하고 미적으로 우수한 펠리칸 자전거 SVG 이미지를 생성했다. Opus는 자전거 프레임을 잘못 렌더링했고, thinking_level: max 옵션을 사용해도 개선되지 않았다.
- 백업 벤치마크 검증: 저자가 '플라밍고가 외발자전거를 타는' 이미지 생성 작업으로도 검증했으며, 여기서도 Qwen이 더 우수한 결과(SVG 주석 포함)를 제공했다.
- 벤치마크의 한계: 펠리칸 벤치마크는 원래 모델 비교의 불명확성과 부조리함을 비판하는 농담으로 시작했으나, 지난 시간 동안 생성된 펠리칸의 품질이 모델의 일반적 유용성과 상관관계를 보였다. 그러나 오늘의 결과로 이 느슨한 상관관계마저 깨졌다.
- 실제 능력과의 괴리: 저자는 Qwen 팀에 높은 존경심을 가지고 있지만, 21GB 양자화 모델이 Anthropic의 최신 독점 모델보다 강력하거나 유용할 가능성을 의심한다. 이는 특정 작업(SVG 생성)에서만 Qwen이 우수할 뿐 전반적인 능력 차이는 있음을 시사한다.
- 작업 특이성: SVG 그림 생성이라는 매우 특정한 작업에서는 로컬 실행 가능한 Qwen3.6-35B-A3B가 현재 Opus 4.7보다 나은 선택이다. 이는 모델 선택이 실제 사용 사례와 성능 벤치마크의 특이성에 따라 달라질 수 있음을 보여준다.

> [!tip] 왜 중요한가
> 이 글은 모델 비교 벤치마크의 신뢰성 한계, 작은 규모 양자화 모델의 예상 외 성능, 그리고 특정 작업에서는 로컬에서 실행되는 오픈 소스 모델이 클라우드 기반 대형 모델을 능가할 수 있음을 보여줌으로써 개발자 선택 과정의 복잡성을 드러낸다.

## 전문 번역

# 펠리컨 벤치마크: 새로운 모델들의 재미있는 대결

혹시 제 "펠리컨이 자전거를 타는" 벤치마크를 진지하게 받아들이셨다면, 미안합니다. 어쨌든 오늘 출시된 두 가지 주요 모델의 펠리컨을 보여드리겠습니다. 알리바바의 Qwen3.6-35B-A3B와 Anthropic의 Claude Opus 4.7이거든요.

## Qwen 3.6의 펠리컨

먼저 Unsloth에서 공개한 20.9GB 크기의 Qwen3.6-35B-A3B-UD-Q4_K_S.gguf 양자화 모델을 사용했습니다. 제 MacBook Pro M5에서 LM Studio와 llm-lmstudio 플러그인으로 돌린 결과인데요, 꽤 괜찮게 나왔습니다.

## Claude Opus 4.7의 펠리컨

이제 Anthropic의 새로운 Claude Opus 4.7로 생성한 펠리컨을 보겠습니다. 음... 자전거 프레임을 망쳤네요. 이 라운드는 Qwen 3.6에 줘야 할 것 같습니다.

혹시 모르니 Opus를 한 번 더 돌려봤어요. `thinking_level: max` 옵션까지 줬는데, 결과가 크게 나아지지 않았습니다.

## 진짜 테스트: 플라밍고 유니사이클

많은 사람들이 제 이 황당한 벤치마크를 위해 모델들을 따로 학습시킨다고 생각합니다. 솔직히 저는 그렇게 생각하지 않는데, 이번 결과는 좀 의심스럽게 만들었어요. 그래서 숨겨뒀던 테스트 중 하나를 꺼내봤습니다. "플라밍고가 유니사이클을 타는 SVG를 생성하세요"라는 프롬프트로요.

역시 Qwen이 이겼습니다. SVG 주석에 선글라스를 쓴 플라밍고가 있는 센스까지 보이니까요.

## 벤치마크의 본질

사실 펠리컨 벤치마크는 처음부터 농담이었습니다. 어떤 모델이 더 좋은지 비교하는 행위 자체가 얼마나 우스꽝스럽고 터무니없는지를 보여주려는 거였죠.

재미있는 건, 이 농담이 지난 몇 개월간 어느 정도 일관성을 보였다는 겁니다. 펠리컨의 질이 모델의 실제 유용성과 대체로 비례했거든요. 2024년 10월의 첫 펠리컨들은 정말 형편없었어요. 최근 결과들은 훨씬 나아졌고, Gemini 3.1 Pro 수준이면 실제로 써먹을 수 있는 수준의 일러스트를 만들어냅니다. 진짜로 펠리컨이 자전거를 타는 이미지가 필요하다면 말이죠.

그런데 오늘은 이 연관성이 깨졌습니다.

## 현실과의 괴리

Qwen을 정말 존경하지만, 솔직하게 말해서 21GB짜리 양자화 모델이 Anthropic의 최신 유료 모델보다 전반적으로 더 강력하거나 유용하다고는 생각하지 않습니다.

그런데 "펠리컨이 자전거를 타는 SVG를 만들어야 한다"면 어떻게 될까요? 현재로서는 노트북에서 돌리는 Qwen3.6-35B-A3B가 Claude Opus 4.7보다 낫습니다!

이게 펠리컨 벤치마크의 가장 재미있는 부분인 것 같습니다. 모델을 평가하는 게 이렇게까지 맥락에 따라 달라진다는 거죠.
