## 핵심 개념

> [!abstract]
> React 서버 컴포넌트에서 원격 코드 실행 패치를 우회하려는 시도 중 발견된 추가 취약점 2개가 공개되었다. 높은 심각도의 서비스 거부(DoS) 취약점(CVE-2025-55184, CVE-2025-67779, CVE-2026-23864, CVSS 7.5)은 악의적인 HTTP 요청으로 인한 무한 루프, 메모리 부족, CPU 과다 사용을 야기하며, 중간 심각도의 소스 코드 노출 취약점(CVE-2025-55183, CVSS 5.3)도 발견되었다. 이전 패치는 불완전하여 19.0.3, 19.1.4, 19.2.3 버전 사용자도 재업그레이드가 필수이다.

## 상세 내용

- 서비스 거부 취약점(CVE-2025-55184, CVE-2025-67779): 악의적으로 조작된 HTTP 요청을 서버 함수 엔드포인트로 보내면 React의 역직렬화 과정에서 무한 루프가 발생하여 서버 프로세스가 중단되고 CPU가 소비된다. 서버 함수를 직접 구현하지 않아도 React 서버 컴포넌트를 지원하면 취약할 수 있다.
- 추가 DoS 취약점(CVE-2026-23864): 2026년 1월 26일 발견된 취약점으로, 특별히 제작된 HTTP 요청으로 인해 서버 충돌, 메모리 부족 예외, 과도한 CPU 사용이 발생할 수 있다. 원래 CVE-2025-55184에 대한 수정이 불완전해서 발생했으며, 19.0.4, 19.1.5, 19.2.4 버전에서 완전히 수정되었다.
- 소스 코드 노출 취약점(CVE-2025-55183): 중간 심각도(CVSS 5.3)의 취약점으로, 공격자가 소스 코드를 노출시킬 수 있다. 자세한 내용은 수정 배포 완료 후 공개될 예정이다.
- 불완전한 이전 패치: 19.0.3, 19.1.4, 19.2.3 버전으로 업그레이드한 사용자도 다시 업그레이드해야 한다. 원본 DoS 수정(CVE-2025-55184)이 불완전하여 추가 취약점이 남아있었기 때문이다.
- 영향받는 버전 범위: react-server-dom-webpack, react-server-dom-parcel, react-server-dom-turbopack의 19.0.0부터 19.2.3까지 모든 버전(19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3)이 취약하며, 19.0.4, 19.1.5, 19.2.4로 업그레이드해야 한다.
- React Native에 대한 특별 지침: 모노레포를 사용하는 경우에만 react-server-dom-* 패키지를 업그레이드하면 되며, react와 react-dom은 업그레이드할 필요가 없어서 버전 불일치 오류를 피할 수 있다.
- 산업 전반의 보안 연구 패턴: 심각한 CVE 공개 후 연구자들이 인접 코드 경로를 조사하여 초기 완화를 우회하는 변형 공격 기법을 찾는 것이 일반적이다(Log4Shell 이후 추가 CVE가 보고된 사례 참고).

> [!tip] 왜 중요한가
> 이전 보안 업데이트로 완전히 보호되지 않았으므로 모든 React 서버 컴포넌트 사용자는 재차 업그레이드해야 하며, DoS 공격으로 인한 서비스 중단 위험에 직면해 있다.

## 전문 번역

# React Server Components의 서비스 거부 및 소스 코드 노출 취약점

2025년 12월 11일 | React 팀  
2026년 1월 26일 업데이트

지난주 발견된 심각한 취약점의 패치를 우회하려던 보안 연구자들이 React Server Components에서 두 가지 추가 취약점을 발견하고 공개했습니다. 다행히 이 새로운 취약점들은 원격 코드 실행(RCE)을 허용하지 않으며, 이전의 React2Shell 패치는 여전히 RCE 공격을 효과적으로 완화합니다.

새로 공개된 취약점은 다음과 같습니다:

- **서비스 거부 (높음)**: CVE-2025-55184, CVE-2025-67779, CVE-2026-23864 (CVSS 7.5)
- **소스 코드 노출 (중간)**: CVE-2025-55183 (CVSS 5.3)

새로 발견된 취약점의 심각성으로 인해 즉시 업그레이드를 권장합니다.

> **주의**: 이전에 발표된 패치에 취약점이 있습니다. 이미 이전 취약점으로 업데이트했다면 다시 한 번 업데이트해야 합니다.
> 
> 19.0.3, 19.1.4, 19.2.3 버전으로 업데이트했다면 불완전한 상태이므로 다시 업데이트해야 합니다.

## 즉시 조치 필요

이 취약점들은 CVE-2025-55182와 동일한 패키지 및 버전에 존재합니다.

다음 패키지의 19.0.0, 19.0.1, 19.0.2, 19.0.3, 19.1.0, 19.1.1, 19.1.2, 19.1.3, 19.2.0, 19.2.1, 19.2.2, 19.2.3 버전이 영향을 받습니다:

- react-server-dom-webpack
- react-server-dom-parcel
- react-server-dom-turbopack

위의 패키지 중 하나라도 사용 중이라면 즉시 19.0.4, 19.1.5, 19.2.4 버전 이상으로 업그레이드하세요.

**영향을 받지 않는 경우:**
- React 서버를 사용하지 않는 앱
- React Server Components를 지원하는 프레임워크, 번들러, 번들러 플러그인을 사용하지 않는 앱

> **참고**: 심각한 CVE가 공개되면 보안 연구자들이 인접한 코드 경로를 살펴보며 초기 완화 방안을 우회할 수 있는 다른 공격 기법을 찾으려 합니다. Log4Shell 이후 추가 CVE들이 보고된 것처럼, 이런 패턴은 업계 전반에서 일어나는 일입니다. 추가 공개는 답답할 수 있지만, 전반적으로는 건강한 대응 주기의 신호입니다.

## 호스팅 제공자의 임시 완화 조치

여러 호스팅 제공자와 협력하여 임시 완화 조치를 적용했습니다. 다만 이에만 의존하지 마시고 즉시 업그레이드하세요.

## React Native 사용자

monorepo나 react-dom을 사용하지 않는 React Native 사용자는 package.json에서 react 버전을 고정하면 추가 조치가 필요 없습니다.

monorepo에서 React Native를 사용 중이라면 다음 패키지가 설치되어 있을 경우만 업데이트하세요:

- react-server-dom-webpack
- react-server-dom-parcel
- react-server-dom-turbopack

react와 react-dom은 업데이트할 필요가 없으므로 React Native의 버전 불일치 오류가 발생하지 않습니다.

## 높음: 다중 서비스 거부 취약점

**CVE**: CVE-2026-23864  
**CVSS 점수**: 7.5 (높음)  
**공개 날짜**: 2026년 1월 26일

보안 연구자들이 React Server Components에 여전히 존재하는 추가 DoS 취약점을 발견했습니다.

Server Function 엔드포인트에 특수하게 조작된 HTTP 요청을 보내면 취약점이 트리거되며, 서버 충돌, 메모리 부족 예외, 또는 과도한 CPU 사용으로 이어질 수 있습니다. 이는 악용되는 취약점 코드 경로, 애플리케이션 구성, 애플리케이션 코드에 따라 달라집니다.

2026년 1월 26일 발표된 패치가 이 DoS 취약점을 완화합니다.

> **참고**: 추가 수정사항 발표  
> CVE-2025-55184의 DoS를 해결하기 위한 원래의 수정이 불완전했습니다. 이로 인해 이전 버전들이 여전히 취약했습니다. 19.0.4, 19.1.5, 19.2.4 버전은 안전합니다.

## 높음: 서비스 거부 취약점

**CVE**: CVE-2025-55184, CVE-2025-67779  
**CVSS 점수**: 7.5 (높음)

보안 연구자들이 Server Functions 엔드포인트로 전송된 악의적인 HTTP 요청이 React에서 역직렬화될 때 무한 루프를 야기하여 서버 프로세스를 멈추고 CPU를 소비할 수 있음을 발견했습니다. 

React Server Function 엔드포인트를 직접 구현하지 않더라도 React Server Components를 지원한다면 여전히 취약할 수 있습니다. 이는 공격자가 사용자들의 제품 접근을 차단하고 잠재적으로 서비스 거부 공격을 수행할 수 있는 취약점입니다.
