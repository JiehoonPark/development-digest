## 핵심 개념

> [!abstract]
> Archestra 팀이 GitHub 저장소의 AI 봇 스팸 문제를 해결한 경험을 공유합니다. 기여자 인증 프로세스를 통해 온보딩을 강제하고, Git의 --author 플래그를 활용하여 신규 기여자에게 커밋 권한을 부여하는 방법을 소개합니다.

## 상세 내용

- GitHub의 '이전 기여자' 설정과 Git --author 플래그를 결합한 AI 봇 필터링 기법
- 온보딩 프로세스(CAPTCHA 및 윤리 규칙)를 통한 정당한 기여자 보호

> [!tip] 왜 중요한가
> 오픈소스 프로젝트 관리자가 AI 생성 스팸을 차단하면서도 정당한 기여자를 환영하기 위한 실용적인 해결책을 제시합니다.

## 전문 번역

# 오픈소스의 종말, 그리고 우리의 대응

**Ildar Iskhakov, CTO 작성**

며칠 전 GitHub가 자신들의 제품에서 AI의 기여도가 엄청나게 증가했다며 자랑스러워했습니다. 하지만 정작 중요한 것을 놓쳤어요. 바로 기여의 **질**이 급격히 떨어졌다는 점 말입니다.

우리 팀이 처음 위험 신호를 감지한 건 $900 상금을 걸고 올린 이슈였습니다. 새로운 "MCP Apps" 기능을 추가해달라는 요청이었는데요. 처음엔 좋았어요. 정말로 관심 있는 개발자들이 계획을 제시하고 질문을 던졌거든요.

그런데 갑자기 AI 봇들이 몰려왔습니다.

## AI 봇의 침입

이슈에 달린 댓글이 순식간에 253개까지 늘어났어요. 엉터리 "구현 계획"들과 관리자에 대한 비난들로 가득 찬 댓글들이었습니다. 문제는 여기서 끝나지 않았습니다.

AI 계정들이 저장소 전체를 점령해버렸거든요. 알림은 끊임없이 쏟아져 나왔습니다. 팀원들이 받는 알림 중 상당수가 AI 봇의 스팸이었으니까요.

정작 중요한 것은 **실제 기여자들의 목소리가 묻혀버렸다**는 겁니다. @ethanwater, @developerfred, @Geetk172 같은 사람들이 열심히 작업 중이었는데, 그들의 댓글이 AI 쓰레기에 파묻혀버렸어요.

## 상황은 점점 악화됐습니다

예를 들어 x.ai 제공자 지원 기능을 Archestra에 추가하자는 이슈만 해도 27개의 PR이 들어왔는데, 대부분 테스트조차 하지 않은 상태였습니다.

우리 팀 한 명이 매주 반나절을 AI 쓰레기를 정리하는 데만 써야 했어요. 테스트 안 된 PR들을 닫고, 존재하지 않는 기능에 대한 이슈들을 삭제하고요. 이 작업을 조금 미루기라도 하면, 저장소는 금세 정상적인 기여자들이 접근하기 힘든 곳으로 변해버렸습니다.

## 우리의 대응

처음엔 기여자들의 "평판"을 계산하는 시스템을 만들었어요. "London-Cat"이라는 이름의 봇이었는데, 병합된 PR 수 같은 신호들을 바탕으로 신뢰도를 판단했습니다. 물론 스팸을 완전히 막지는 못했지만, 누가 진짜인지는 구분할 수 있었죠.

다음으로 "AI sheriff"라는 봇을 만들었는데... 이건 정상적인 PR도 몇 개 닫아버렸습니다. 🤦

계속 밀려드는 쓸모없는 AI 댓글들과 제안들은 점점 심해졌어요. 정상적인 기여자들이 떠나갔고, 우리도 고민하게 됐습니다.

> 상금을 걸고 기여를 독려하는 걸 그만둬야 할까?  
> 채용 시험으로 쓰는 과제들을 없애야 할까?

**아니었어요.** 우리는 저장소를 정상적인 개발자들, 책임감 있는 AI 사용자들, 그리고 경험 많은 개발자들이 편안히 기여할 수 있는 공간으로 만들기로 결심했습니다.

## 강경한 조치를 취하기로 했습니다

오늘부터 우리는 **온보딩 과정을 거치지 않은 사람들의 이슈 생성, PR 제출, 댓글을 모두 막기로 결정했습니다.**

이건 강경한 조치입니다. 특히 VC 펀딩을 받은 스타트업 입장에서 GitHub 활동량으로 평가받다 보니 더 그렇거든요. 하지만 우리는 **수량보다 품질을 택했습니다.** AI가 만들어낸 쓰레기로 부풀려진 메트릭 같은 건 필요 없었어요.

## GitHub에서 이걸 어떻게 구현했을까

안타깝게도 GitHub는 공식적으로 "누가 댓글을 달거나 PR을 열 수 있는가"를 제어할 방법이 없습니다. 그래서 우리는 우회 방법을 찾아야 했어요.

**"Limit to prior contributors"** 라는 설정이 있습니다. 규칙은 간단합니다. 메인 브랜치에 한 번이라도 커밋한 적 없으면, 이슈나 PR에 댓글을 달 수 없다는 거죠.

문제는 이 설정이 **AI 봇과 새로 가입한 개발자를 구분하지 못한다**는 점입니다. 둘 다 "이전 기여자"가 아니거든요.

### 해결 방법: Git author 필드 이용하기

Git 커밋에는 두 가지 정체성 필드가 있습니다. **author(작성자)**와 **committer(커밋한 사람)**인데, 이 둘은 다를 수 있어요.

Git의 `--author` 플래그를 쓰면 다른 사람 이름으로 커밋을 만들 수 있습니다. 이메일이 GitHub 계정과 일치하면, GitHub이 그 프로필과 연결해주거든요.

모든 GitHub 계정은 고유한 noreply 이메일을 가지고 있습니다: `<id>+<username>@users.noreply.github.com`

API로 ID를 조회한 후 이렇게 하면 됩니다:

```bash
gh api users/their-username --jq '.id'

git commit \
  --author="their-username <ID+their-username@users.noreply.github.com>" \
  -m "chore: add their-username to external contributors"
```

메인 브랜치에 푸시하면 끝이에요. 그러면 그 사용자는 즉시 이슈와 PR에 댓글을 달 수 있습니다.

## 최종적인 온보딩 플로우

1. 우리 웹사이트에서 온보딩 수행 (윤리적 AI 사용 규칙 동의 + CAPTCHA): https://archestra.ai/contributor-onboard

2. GitHub Action이 제출 정보를 받으면, 사용자의 GitHub ID를 조회해서 `EXTERNAL_CONTRIBUTORS.md` 파일에 추가합니다.

3. 그 사용자 이름으로 메인 브랜치에 커밋을 푸시합니다.

4. 사용자가 whitelisted 상태가 되고, 저장소에 접근할 수 있게 됩니다.

## 마치며

GitHub는 거대한 메트릭 성장을 자랑하지만, 그 상당 부분이 AI가 만들어낸 것입니다. 하지만 우리 같은 오픈소스 팀은 그 AI 쓰레기를 손으로 직접 치워야 하거든요.

우리는 **Archestra를 모두가 편하게 기여할 수 있는 진짜 좋은 소프트웨어**로 만들고 싶습니다. AI 봇들에게 점령당하지 않은 공간으로요.
