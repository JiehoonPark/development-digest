## 핵심 개념

> [!abstract]
> 98.css는 JavaScript 없이 순수 CSS만으로 Windows 98 스타일의 UI를 재현하는 디자인 시스템입니다. 버튼, 체크박스, 라디오버튼, 그룹박스, 텍스트박스, 슬라이더, 드롭다운, 타이틀 바 등 폼 컨트롤과 창 구성 요소를 시맨틱 HTML 기반으로 제공하며, aria-label을 활용한 접근성을 핵심 설계 원칙으로 삼고 있습니다. unpkg CDN이나 npm을 통해 손쉽게 설치할 수 있고 React, Vanilla JS 등 어떤 프레임워크와도 호환됩니다.

## 아티클

# 윈도우 98을 그대로 재현하는 CSS 라이브러리, 98.css

레트로 UI 디자인은 개발자들 사이에서 꾸준히 사랑받는 소재인데요, 이번에 소개할 `98.css`는 이름 그대로 Windows 98의 인터페이스를 웹에서 그대로 구현할 수 있게 해주는 CSS 라이브러리입니다. JavaScript 없이 순수 CSS만으로 동작하기 때문에 React든 Vanilla JS든 어떤 프레임워크와도 자연스럽게 섞어 쓸 수 있다는 점이 특징입니다.

## 설치와 기본 사용법

가장 빠르게 적용하는 방법은 unpkg에서 직접 불러오는 것입니다.

```html
<link
  rel="stylesheet"
  href="https://unpkg.com/98.css"
>
```

npm으로 설치하고 싶다면 아래처럼 진행하면 됩니다.

```
npm install 98.css
```

GitHub 릴리즈 페이지에서 직접 다운로드하는 것도 가능합니다. React 예제와 Vanilla JavaScript 예제도 공식적으로 제공되고 있어, 실제 프로젝트에 붙일 때 참고하기 좋습니다.

## 시맨틱 HTML과 접근성을 기본 원칙으로

98.css의 핵심 설계 철학은 시맨틱 HTML을 그대로 활용한다는 점입니다. 버튼을 만들 때는 반드시 `<button>` 태그를 써야 하고, 입력 요소에는 `<label>`을 붙여야 하며, 아이콘 버튼은 `aria-label`에 의존해 의미를 전달합니다. 즉, 라이브러리가 CSS 클래스만 얹어서 스타일을 입히되, 그 밑바탕은 항상 표준 HTML 시맨틱을 따르도록 강제하는 구조입니다.

또한 이 스타일을 유지하면서도 사용자가 자유롭게 오버라이드할 수 있습니다. 버튼 패딩을 늘리거나 라벨에 색상을 추가하는 등 커스터마이징이 자유롭다는 점도 강조되고 있습니다.

## 주요 컴포넌트

### Button (버튼)

표준 버튼은 75px 너비, 23px 높이에 외곽과 내부에 raised(양각) 테두리를 가지며, 기본적으로 좌우 12px의 패딩이 적용됩니다.

```html
<button>Click me</button>
<input type="submit" />
<input type="reset" />
```

`default` 클래스를 추가하면 Windows 98에서 Enter 키를 눌렀을 때 실행될 기본 동작임을 시각적으로 알려주는 스타일이 적용됩니다.

```html
<button class="default">OK</button>
```

버튼을 클릭하면 raised 테두리가 sunken(음각) 상태로 바뀌면서 눌린 느낌을 줍니다. `disabled` 속성을 주면 동일한 raised 테두리는 유지되지만 라벨 텍스트가 흐릿하게 처리됩니다. 포커스 상태는 버튼 안쪽 4px 위치에 점선 테두리로 표현됩니다.

### Checkbox (체크박스)

체크박스는 sunken 패널로 표현되고, 선택 시 체크 아이콘이 채워집니다. 반드시 `<label>`을 `for` 속성으로 연결해 접근성과 사용성을 동시에 확보해야 합니다(탭 키 네비게이션, 라벨 클릭으로 선택 등).

```html
<input type="checkbox" id="example1">
<label for="example1">This is a checkbox</label>
```

`checked`, `disabled` 표준 속성을 그대로 사용할 수 있고, 여러 입력을 그룹핑할 때는 `field-row` 클래스로 감싸 일관된 간격을 유지합니다.

```html
<div class="field-row">
  <input checked type="checkbox" id="example2">
  <label for="example2">I am checked</label>
</div>
```

### OptionButton (라디오 버튼)

라디오 버튼은 `type="radio"`를 사용하며, 같은 `name` 속성을 공유하는 입력끼리 그룹으로 묶입니다. 여기서도 `field-row`로 감싸는 패턴이 동일하게 적용됩니다.

```html
<div class="field-row">
  <input id="radio5" type="radio" name="first-example">
  <label for="radio5">Yes</label>
</div>
```

### GroupBox (그룹박스)

관련된 컨트롤들을 묶어 시각적으로 정리하는 컴포넌트로, `<fieldset>` 태그로 감싸면 sunken 외곽 테두리와 raised 내부 테두리가 조합되어 마치 각인된 상자 같은 느낌을 줍니다. `<legend>`를 넣으면 그룹에 라벨을 붙일 수 있습니다.

```html
<fieldset>
  <legend>Today's mood</legend>
  <div class="field-row">
    <input id="radio13" type="radio" name="fieldset-example2">
    <label for="radio13">Claire Saffitz</label>
  </div>
  <!-- ... -->
</fieldset>
```

### TextBox (텍스트박스)

`type="text"` 입력에 라벨을 연결하고 `field-row`로 감싸는 패턴은 동일합니다.

```html
<div class="field-row">
  <label for="text17">Occupation</label>
  <input id="text17" type="text" />
</div>
```

라벨을 입력창 위쪽에 배치하고 싶다면 `field-row-stacked` 클래스를 사용하면 됩니다.

```html
<div class="field-row-stacked" style="width: 200px">
  <label for="text18">Address (Line 1)</label>
  <input id="text18" type="text" />
</div>
```

여러 줄 입력이 필요하면 `<textarea>`를 사용하고, `disabled`와 `value` 속성도 표준 그대로 동작합니다.

### Slider (슬라이더)

`type="range"` 입력으로 렌더링하며, `has-box-indicator` 클래스로 기본 인디케이터를 박스 형태로 바꿀 수 있습니다. `is-vertical` 클래스로 감싸면 세로 방향 슬라이더로 표시할 수 있는데, 이 경우 세로 슬라이더의 길이를 조절하려면 input의 width와 감싸는 div의 height를 함께 조정해야 합니다.

```html
<div class="field-row">
  <label for="range25">Cowbell</label>
  <div class="is-vertical">
    <input id="range25" class="has-box-indicator" type="range" min="1" max="3" step="1" value="2" />
  </div>
</div>
```

### Dropdown (드롭다운)

표준 `<select>`, `<option>` 요소를 그대로 사용합니다. 기본적으로 첫 번째 옵션이 선택되며, 특정 옵션에 `selected` 속성을 주면 그 옵션이 기본으로 선택됩니다.

```html
<select>
  <option>5 - Incredible!</option>
  <option>4 - Great!</option>
  <option selected>3 - Pretty good</option>
  <option>2 - Not so great</option>
  <option>1 - Unfortunate</option>
</select>
```

## Window: 완전한 창 만들기

98.css는 개별 컨트롤뿐 아니라 창(Window) 전체를 구성하는 요소도 제공합니다. 그중 타이틀 바(Title Bar)는 창의 최상단에 위치해 내용물을 식별하고, 창 관련 명령 버튼들을 담는 영역입니다.

`title-bar`, `title-bar-text`, `title-bar-controls` 세 클래스를 조합해 완전한 타이틀 바를 만들 수 있습니다.

```html
<div class="title-bar">
  <div class="title-bar-text">A Title Bar</div>
  <div class="title-bar-controls">
    <button aria-label="Close"></button>
  </div>
</div>
```

여기서도 `aria-label`이 핵심 역할을 하는데요, Close 버튼을 렌더링하면서 동시에 보조 기술(assistive technology)에 버튼의 의도를 전달합니다. `"Minimize"`, `"Maximize"`, `"Restore"`, `"Help"` 등의 라벨도 동일한 방식으로 사용할 수 있으며, 각 `aria-label` 값에는 대응하는 스타일링 클래스가 자동으로 매핑되어 버튼 아이콘이 렌더링됩니다.

```html
<div class="title-bar">
  <div class="title-bar-text">A Title Bar</div>
  <div class="title-bar-controls">
    <button aria-label="Minimize"></button>
    <button aria-label="Maximize"></button>
    <button aria-label="Close"></button>
  </div>
</div>
```

## 정리

- 98.css는 JavaScript 없이 CSS만으로 Windows 98 스타일 UI를 재현하는 라이브러리로, 프레임워크에 구애받지 않고 unpkg CDN이나 npm install로 손쉽게 도입할 수 있습니다.
- 모든 컴포넌트가 시맨틱 HTML(`<button>`, `<label>`, `<fieldset>`, `<select>` 등)을 기반으로 설계되어 있으며, 특히 아이콘 버튼은 `aria-label`을 통해 접근성을 확보하도록 강제하는 구조입니다.
- Button, Checkbox, OptionButton(라디오), GroupBox, TextBox, Slider, Dropdown 등 폼 컨트롤은 물론, Title Bar를 포함한 Window 컴포넌트까지 제공해 실제 창 UI를 완전하게 재현할 수 있습니다.
- `field-row`, `field-row-stacked` 같은 유틸리티 클래스로 입력 요소 간 간격을 일관되게 맞출 수 있고, 기존 스타일을 유지하면서도 자유롭게 오버라이드가 가능합니다.
- 레트로 테마 프로젝트, 패러디 사이트, 혹은 접근성을 갖춘 시맨틱 마크업 예제를 찾는 개발자에게 실용적인 참고 자료가 될 수 있습니다.
