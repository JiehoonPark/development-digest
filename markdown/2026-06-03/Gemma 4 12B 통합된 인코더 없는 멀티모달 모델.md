## 핵심 개념

> [!abstract]
> Google이 Gemma 4 12B를 출시했으며, 이는 노트북에서 직접 실행 가능한 고성능 멀티모달 AI 모델입니다. 전통적인 비전과 오디오 인코더를 제거한 통합 아키텍처를 사용하여 16GB RAM에서 실행되면서도 26B Mixture of Experts 모델에 가까운 성능을 제공합니다. 비전과 오디오 입력을 LLM 백본에 직접 연결하여 지연시간을 줄이고 메모리 사용량을 최소화합니다.

## 상세 내용

- 인코더 없는 통합 아키텍처: 기존 멀티모달 모델의 별도 비전·오디오 인코더를 제거하고, 이미지와 오디오를 LLM 백본에 직접 입력하도록 설계하여 지연시간과 메모리 사용량을 감소시킵니다.
- 비전 처리 경량화: 단일 행렬 곱셈, 위치 임베딩, 정규화로 구성된 경량 임베딩 모듈로 대체하여 LLM 백본이 시각 처리를 직접 담당하도록 합니다.
- 오디오 신호 직접 처리: 오디오 인코더를 완전히 제거하고 원본 오디오 신호를 텍스트 토큰과 동일한 차원 공간으로 직접 투영하여 처리 복잡도를 극소화합니다.
- 노트북 친화적 성능: 16GB VRAM 또는 통합 메모리만으로 로컬 실행 가능하면서도 26B MoE 모델의 절반 이하 메모리 사용량으로 유사한 벤치마크 성능을 달성합니다.
- 멀티 토큰 예측(MTP) 드래프터 내장: 추론 지연시간을 줄이기 위해 여러 토큰을 동시에 예측하는 드래프터 기능을 기본 제공합니다.
- 에이전트 개발 지원: 공식 Skills Repository를 출시하여 Gemma 모델과 함께 작동하도록 설계된 스킬 라이브러리를 제공하여 에이전트 구축을 용이하게 합니다.
- 광범위한 개발 도구 지원: Hugging Face Transformers, llama.cpp, MLX, SGLang, vLLM 등의 인기 프레임워크와 LM Studio, Ollama 등의 실행 환경에서 즉시 사용 가능합니다.
- Apache 2.0 라이선스로 개방: 상용, 연구, 개인 프로젝트 모두에서 자유롭게 사용할 수 있으며, 150만 개 이상의 다운로드 달성으로 개발자 신뢰도가 높습니다.

> [!tip] 왜 중요한가
> 고성능 멀티모달 AI를 로컬 노트북에서 실행할 수 있게 되어, 클라우드 의존성을 제거하고 데이터 프라이버시를 보호하면서도 에이전트 기반의 복잡한 AI 워크플로우를 구축할 수 있습니다.

## 전문 번역

# Gemma 4 12B 소개: 인코더 없이 설계된 멀티모달 모델

2026년 6월 3일

Gemma 4 12B는 고성능 멀티모달 인공지능을 노트북에 바로 구현할 수 있도록 만들어졌습니다. 모바일 환경에서의 효율성과 고급 추론 능력을 모두 갖춘 모델이거든요.

---

오늘 Google Deepmind가 Gemma 4 12B를 공개합니다. 이 모델은 멀티모달 에이전트 기능을 노트북에서 직접 실행할 수 있도록 설계되었어요. 경량의 E4B 모델과 더 강력한 26B Mixture of Experts(MoE) 모델 사이의 간격을 채우면서도, 메모리 점유율을 낮게 유지하고 있습니다. 특히 중간 크기 모델 중 처음으로 네이티브 오디오 입력을 지원한다는 점이 특징입니다.

개발자 커뮤니티 덕분에 Gemma 4 모델은 이미 1억 5,000만 회 이상 다운로드되었습니다. 여러분은 신체 보조용 웨어러블 로봇팔부터 엔터프라이즈급 AI 보안 솔루션까지 정말 다양한 것들을 만들어내고 있어요. 이번 새로운 버전으로 어떤 것들을 만들지 기대됩니다.

## Gemma 4 12B만의 특징

**혁신적인 통합 아키텍처**  
멀티모달 인코더가 없습니다. 비전과 오디오 입력이 LLM 백본으로 직접 흐르거든요.

**고급 추론 능력**  
26B 모델에 필적하는 벤치마크 성능으로 복잡한 다단계 추론과 에이전트 워크플로우를 가능하게 합니다.

**노트북에서 즉시 실행**  
16GB VRAM 또는 통합 메모리만으로도 로컬에서 돌릴 수 있을 정도로 가볍습니다.

**오픈소스 기반**  
Apache 2.0 라이선스로 공개되며 개발자 생태계 전반에서 지원합니다.

**드래프트 모델 준비 완료**  
Multi-Token Prediction(MTP) 드래프터가 탑재되어 레이턴시를 줄입니다.

이러한 기능들을 통합하면 속도나 추론 능력을 포기하지 않으면서도 일상적인 하드웨어에서 고급 멀티모달 기능을 사용할 수 있게 됩니다.

## 로컬에서 최첨단 에이전트 실행하기

Gemma 4 12B는 표준 벤치마크에서 더 큰 26B MoE 모델에 필적하는 성능을 보이면서도 메모리 점유율은 절반 이하입니다. 16GB RAM의 일반 노트북에서도 충분히 돌릴 수 있어서, 로컬 환경에서 강력한 멀티모달 및 에이전트 경험을 그대로 누릴 수 있어요.

## 효율적이고 통합된 아키텍처의 경험

Gemma 4 12B가 돋보이는 이유는 시각 정보와 오디오 입력을 처리하는 방식이 간결하기 때문입니다. 일반적인 멀티모달 모델들은 별도의 인코더를 통해 이미지와 오디오를 변환한 후 언어 모델에 전달합니다. 그런데 이런 분리된 인코더들이 지연 시간을 늘리고 메모리 사용량을 증가시키니까요. 우리는 Gemma 4 12B를 오디오와 비전 입력을 직접 통합하는 인코더 없는 아키텍처로 훈련했습니다.

Gemma 4 12B가 멀티모달 입력을 네이티브로 처리하는 방식은 다음과 같습니다.

**비전 처리**  
비전 인코더 대신 단순 행렬 곱셈, 위치 임베딩, 정규화로 이루어진 경량 임베딩 모듈을 사용했습니다. 이렇게 하면 LLM 백본이 시각 처리를 직접 담당할 수 있어요.

**오디오 처리**  
오디오 처리는 더욱 단순화했습니다. 오디오 인코더를 완전히 제거하고 원본 오디오 신호를 텍스트 토큰과 같은 차원 공간으로 직접 사영했거든요.

## 지금 시작하기

**직접 시도해보기**  
LM Studio, Ollama, Google AI Edge Gallery App, Google AI Edge Eloquent app, LiteRT-LM CLI에서 몇 번의 클릭만으로 실험해볼 수 있습니다.

**가중치 다운로드**  
사전 훈련된 체크포인트와 지시문 튜닝된 체크포인트를 Hugging Face와 Kaggle에서 직접 다운로드할 수 있습니다.

**통합하고 배우기**  
개발자 문서와 빠른 시작 노트북을 검토해보세요.

**선호하는 개발 도구 사용**  
Hugging Face Transformers, llama.cpp, MLX, SGLang, vLLM로 로컬 추론 파이프라인을 구성하거나 Unsloth를 사용해 효율적으로 파인튜닝할 수 있습니다.

**Gemma Skills로 에이전트 개발 시작하기**  
최신 Gemma 발전을 기반으로 에이전트를 구축할 수 있도록 공식 Skills Repository를 새로 공개합니다. 이는 Gemma 모델로 개발하기 위해 특별히 설계된 기능들의 라이브러리예요.

**원하는 대로 배포하기**  
Google Cloud를 통해 프로덕션 엔드포인트를 구성하거나, Gemini Enterprise Agent Platform Model Garden, Cloud Run, GKE를 통해 배포할 수 있습니다.
