## 핵심 개념

> [!abstract]
> 10년에 걸쳐 1998년 Ultima Online 데모 서버를 완전히 역공학하여 약 5,000개 함수를 MSVC x86에서 이식 가능한 C99로 변환했습니다.

## 상세 내용

- MSVC 5.0으로 컴파일된 약 5,000개 함수를 손으로 직접 C99로 번역하고 바이너리와 명령어 단위로 검증
- 클래스 계층 구조(CEntity → CPlayer) 정확히 재현하고 안정성 문제 수정, 스폰 및 감쇠 시스템 복원
- 계정 시스템 재구현, 클라이언트 1.25.30~5.0.9.1 지원, 생태계 시스템 재연동 등 기능 추가

> [!tip] 왜 중요한가
> 레거시 게임 서버의 역공학 방법론과 대규모 바이너리 분석 기법을 배울 수 있으며, 레트로 게임 복원 프로젝트에 참고할 수 있습니다.

## 전문 번역

# 1998년 울티마 온라인 데모 서버를 역공학하다

**2026년 5월 1일**

10년간 틈틈이 작업한 끝에 1998년 울티마 온라인 데모 서버의 완전한 역공학 결과물을 공개합니다. MSVC x86에서 약 5,000개 함수를 역어셈블하여 이식성 높은 C99 코드로 변환했으며, 각 함수를 바이너리와 명령어 단위로 비교 검증했습니다.

관련 리포지토리: https://github.com/draxinar/ouo

## 울티마 온라인이란?

울티마 온라인은 1997년 Origin Systems Inc가 개발한 MMORPG로, 상업적으로 성공한 최초의 온라인 게임 중 하나입니다. Windows에서 돌아가는 클라이언트와 여러 Solaris 머신에서 실행되는 서버(일명 "샤드")로 구성되었는데요, 게임 맵이 여러 지역으로 분산되어 관리되었습니다.

1998년 10월 공개된 "울티마 온라인: 더 세컨드 에이지" 확장팩에는 독립실행형 울티마 온라인 데모(UoDemo.exe, UoDemo.dat)가 번들로 포함되었습니다. 클라이언트와 Windows용 풀 서버 코드, 게임 데이터가 모두 들어있었죠.

UoDemo.exe의 컴파일 날짜는 1998년 9월 2일이고, 서버 데이터는 1998년 6월 2일에 라이브 서버에서 추출되었습니다. 데모용으로 몇몇 기능은 제거되었고 플레이 가능한 맵도 브리태니아 남쪽 해안의 작은 NPC 도시인 오클로 섬으로 축소되었지만, 나머지 부분은 1998년 중반 울티마 온라인에서 실제로 돌아가던 프로덕션 서버 코드 그대로입니다. 데모에는 오클로 섬에서 용을 죽이는 간단한 퀘스트가 있었고, 기본적인 게임 메커닉(대화, 거래, 전투 등)을 소개했습니다.

지금까지 많은 UO 서버 에뮬레이터들이 데모 서버의 코드를 부분적으로 재사용해왔지만, 정말로 완전하게 역공학한 프로젝트는 없었습니다.

UoDemo.exe는 Microsoft Visual C++ 5.0(Visual Studio 97)으로 컴파일되었으며, C++98 이전의 C++ 방언을 사용했습니다.

이 프로젝트는 10년간 틈틈이 진행되다가, 최근 LLM의 발전 덕분에 마침내 완료할 수 있게 되었습니다.

## 작업 방식

**radare2를 이용한 역어셈블**

프로세스는 크게 다섯 단계를 거쳤습니다. 먼저 radare2로 바이너리를 역어셈블했고, C++ 심볼이 포함된 실험적 Linux 포트인 UO 클라이언트 1.25.37에서 심볼 이름을 유추했습니다.

**수동 번역 및 검증**

각 함수를 C99로 수동 번역할 때는 원본 바이너리와 동일한 제어 흐름, 구조체 레이아웃, 분기 조건을 유지했습니다. 뭔가 다른 부분이 있다면 그건 데모의 실제 버그를 수정했거나 플랫폼에 맞게 조정한 것이고, 소스에 표시했습니다.

검증 과정도 중요했는데요, C 빌드를 radare2에서 다시 역어셈블한 후 원본과 비교해서 완벽하게 일치할 때만 완료 표시를 했습니다.

헬퍼 함수는 반복되는 인라인 패턴에서만 사용했으며, 헬퍼가 인라인 버전과 동일한 코드로 확장될 때만 활용했습니다.

**클래스 구조의 중요성**

클래스 계층 구조를 정확히 파악하는 게 가장 중요했습니다: `CEntity (0x10)` → `CResourceEntity (0x1C)` → `CItem (0x50)` → `CContainer (0x5C)` → `CMobile (0x37C)` → `CPlayer (0x458)`. vtable을 통한 가상 디스패치도 있었는데요(vtable[0x18]은 IsPlayer, [0xD0]은 IsMobile, [0xE4]는 IsNPC 등), 이 레이아웃들을 정확히 파악하면 대부분의 바이너리는 직관적으로 번역할 수 있었습니다.

결과적으로 1998년 울티마 온라인 서버의 거의 완벽한 복제본을 만들었지만, 몇 가지 차이점은 있습니다.

## 발견 사항

**안정성 개선**

원본 코드와 비교했을 때 여러 안정성 문제(크래시, 오버플로우, 초기화되지 않은 변수)와 게임플레이 문제(스킬 상승, 명성/악명 방향, 스폰 밀도)를 수정했습니다. 모든 수정 사항은 소스에 태그로 표시했으니까, 누구든 UoDemo.exe와 비교해서 정확히 뭐가 바뀌었고 왜 바뀌었는지 알 수 있습니다.

**복구된 기능들**

스폰 시스템, 아이템 부식 시스템처럼 손상된 기능들도 있었어요. 아마 데모 출시를 위해 의도적으로 비활성화되거나 스텁 처리된 듯합니다. 코드 자체는 온전했지만 어느 부분에서도 호출되지 않았거든요. 이런 함수들을 따로 역컴파일하고 디스패치를 다시 연결하면 작동하게 됩니다.

게임 맵도 원래는 오클로 섬만 포함되어 있었는데요, 서버 데이터 형식을 다루는 완전한 도구 모음을 작성해서 나머지 세계의 문, 표지판, 장식, 순간이동기, 함정, 상자, 스폰 위치를 모두 복원했습니다.

**생태계 시스템의 부활**

흥미로운 발견은 유명한 퇴직 생태계 시스템이 여전히 코드에 남아 있었다는 거예요. 함수들은 더 이상 호출되지 않았지만요. 포식자/피식자/청소동물 시스템을 다시 연결해서 이제 늑대가 토끼를 쫓거나 까마귀가 아이템을 먹는 모습을 볼 수 있게 했습니다. 다만 정확한 데이터가 부족해서 완전한 자원/생산 시스템까지는 구현하지 못했습니다.

**새로운 기능 추가**

명상(Meditation), 잠행(Stealth), 함정 해제(Remove Trap) 같은 스킬도 추가했는데, 이들은 OSI가 1999년 2월에 추가한 기능이지만 코드에 이미 초기 흔적이 있었습니다. 대부분의 새로운 기능은 시작 시 `-features` 파라미터로 활성화하거나 비활성화할 수 있습니다.

**계정 시스템 재구현**

데모 서버에는 계정 시스템이 완전히 없었어서, 원본 개발자들이 어떻게 구현했을지 추측해서 재구현했습니다. 물론 현대식으로 조금 개선했죠.

**클라이언트 호환성**

데모 서버는 원래 클라이언트 1.25.33만 지원했는데, 이제 1.25.30부터 5.0.9.1(2007-03-27)까지 모든 클라이언트를 지원하며, 암호화 여부도 가리지 않습니다.
