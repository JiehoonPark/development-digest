## 핵심 개념

> [!abstract]
> 개발자 Simon Willison은 AI 코딩 도구 사용 경험을 통해 코드 검토 없이 AI 에이전트의 결과물을 신뢰하게 되는 현상을 발견했다. 초기에는 'vibe coding'(검증 없는 AI 코딩)과 'agentic engineering'(전문가 수준의 AI 활용)을 명확히 구분했지만, Claude Code 같은 도구들이 신뢰성을 증명하면서 두 개념의 경계가 흐려지고 있다는 우려를 제시한다.

## 상세 내용

- Vibe coding은 비전문가가 코드 품질을 신경 쓰지 않고 AI에 결과를 맡기는 방식이고, agentic engineering은 전문가 엔지니어가 보안·유지보수·성능을 고려하며 AI를 활용하는 방식이다.
- Claude Code 같은 AI 에이전트가 JSON API 엔드포인트, SQL 쿼리, 테스트, 문서화 등을 정확히 생성하면서 개발자들이 코드 검토 없이 프로덕션에 배포하는 '정상화된 일탈(normalization of deviance)'이 발생 중이다.
- AI 에이전트는 전문적 평판을 걸 수 없으므로 신뢰에는 한계가 있으며, 한 번의 실수가 향후 과신으로 이어질 위험이 있다.

> [!tip] 왜 중요한가
> AI 코딩 도구의 성능 향상으로 인한 검증 생략 문제는 프로덕션 환경의 품질과 책임성에 직결되는 개발자 윤리의 중요한 과제이다.
