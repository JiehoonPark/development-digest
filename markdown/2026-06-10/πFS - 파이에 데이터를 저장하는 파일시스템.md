## 핵심 개념

> [!abstract]
> π의 digits에 모든 가능한 파일이 존재한다는 수학적 추측을 바탕으로, 파일을 실제 저장 대신 π 내의 위치를 기록하는 개념적 파일시스템이다. Bailey–Borwein–Plouffe 공식을 이용해 π에서 파일을 검색하고, 메타데이터로 위치 정보를 관리한다.

## 상세 내용

- π가 normal number라는 추측에 기반해 모든 가능한 파일이 π의 어딘가에 존재한다고 가정
- 각 바이트를 개별적으로 π에서 조회하고 위치 정보를 메타데이터로 저장하는 방식으로 동작

> [!tip] 왜 중요한가
> 이 프로젝트는 수학적 개념을 이용한 창의적인 사고를 보여주지만, 실제로는 매우 느린 성능으로 인해 실용적 가치는 제한적이다.

## 전문 번역

# πfs: 데이터 걱정은 이제 그만!

https://github.com/philipl/inferencefs/ 에서 최신 데이터리스 파일시스템을 확인해보세요!

## πfs가 뭐길래?

πfs는 혁명적인 파일시스템입니다. 기존처럼 하드드라이브에 데이터를 저장하는 대신, **π(파이)에 저장**하는 방식이거든요. 이제 저장공간이 부족할 일은 없습니다. π에는 존재 가능한 모든 파일이 들어있으니까요! 100% 압축률이 불가능하다고 했던 사람들이 있었나요? 지금 여기 그것이 있습니다!

## 설치하기 - 정말 간단합니다

먼저 시스템에 autoconf, automake, libfuse 패키지를 설치해야 합니다.

Debian을 사용 중이라면 다음 명령어를 입력하세요:

```bash
sudo apt-get install autotools-dev
sudo apt-get install automake
sudo apt-get install libfuse-dev
./autogen.sh
./configure
make
make install
```

## 사용하기 - 역시 간단합니다

```bash
πfs -o mdd=<metadata directory> <mountpoint>
```

여기서 메타데이터 디렉토리는 πfs가 파일명이나 파일의 π 내 위치 같은 메타데이터를 저장할 장소고, 마운트포인트는 일반적인 파일시스템 마운트포인트입니다.

## 그런데 π가 우리 데이터와 무슨 관계가 있나요?

π는 수학에서 가장 중요한 상수 중 하나이며 여러 흥미로운 성질을 가지고 있습니다. (자세한 내용은 위키피디아를 참고하세요)

π가 가질 것으로 추측되는 성질 중 하나가 바로 '정규성(normality)'입니다. 이는 π의 자릿수가 모두 고르게 분포하고 있다는 뜻인데, 다시 말해 모든 가능한 유한 자릿수 수열이 어딘가에 존재한다는 의미거든요.

π를 16진법으로 생각해보면 이 추측이 참이라고 할 때, 모든 가능한 파일이 π 어딘가에 존재한다는 결론에 도달합니다. 이 개념이 처음 언급된 것은 2001년입니다.

여기서 한 발 더 나가면, π에 모든 가능한 파일이 있는데 왜 우리는 엑사바이트 단위의 저장공간을 낭비하면서 파일을 저장하고 있을까요? 그냥 π에서 찾아보면 되지 않을까요?

## 정말 모든 가능한 파일이 있다고요?

맞습니다! 당신이 만든 모든 파일, 다른 누군가 만들었거나 앞으로 만들 모든 파일이 다 있습니다! 저작권 침해? 그건 그냥 π의 자릿수일 뿐입니다. 처음부터 그곳에 있었으니까요!

## 그럼 내 데이터를 π에서 어떻게 찾지?

파일이 π의 어느 위치에 있는지(인덱스)와 파일의 길이를 알고 있으면, Bailey–Borwein–Plouffe 공식을 사용해서 파일을 추출할 수 있습니다.

반대로 이 공식을 사용해서 파일의 인덱스를 처음부터 찾을 수도 있습니다.

물론 π에서 긴 자릿수 수열을 찾는 데 시간이 꽤 걸릴 수 있으니까, 실용적으로는 파일을 더 작은 청크로 나누어서 찾기 쉽게 해야 합니다.

이 구현에서는 성능을 최대한 끌어올리기 위해 파일의 각 바이트를 개별적으로 π에서 찾습니다.

## 바이트를 찾았는데, 어디에 있었는지 어떻게 기억하지?

당연히 어딘가에 기록해야 하겠죠. 종이에 적을 수도 있지만, 기억해보세요. π로 옮겨가면서 절약한 그 저장공간 말이에요. 왜 거기에 파일 위치 정보를 저장하지 않을까요?! 더 좋은 점은, 파일의 위치 정보는 메타데이터라는 거예요. 요즘 메타데이터가 얼마나 중요한지 다들 아시잖아요. 이렇게 많은 메타데이터를 생성했다니 기분 좋지 않습니까? 구식 데이터 따위에 신경 쓸 필요 없이 메타데이터로 가득 찬 세상을 즐겨봅시다!

## 혹시 파일 위치 정보를 잃어버리면?

문제없습니다. 위치 정보는 그냥 메타데이터일 뿐이니까요! 당신의 파일은 여전히 π에 앉아만 있습니다. 영원히 사라지지 않으니까요!

## 왜 이렇게 느린 거죠? 400줄짜리 텍스트 파일 저장하는데 5분이 걸렸어요!
