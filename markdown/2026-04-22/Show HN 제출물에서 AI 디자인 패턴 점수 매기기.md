## 전문 번역

# Hacker News의 Show HN에서 발견한 AI 디자인 패턴들

Hacker News를 구경하다 보니 요즘 Show HN 프로젝트들이 뭔가 차갑고 획일적인 느낌이 드는데요. 자세히 보니 AI가 만든 것처럼 보이더라고요. 처음엔 그 이유를 정확히 못 집어냈는데, 혹시 이런 주관적인 느낌을 객관적으로 수치화할 수 있을까 궁금해졌습니다. 그래서 Show HN 페이지 500개를 분석해서 AI 디자인 패턴을 점수로 매기기로 했어요.

Claude Code의 등장으로 Show HN 제출이 크게 늘어났거든요. 그래서 Hacker News 운영진도 신규 계정의 Show HN 제출을 제한해야 할 정도였습니다.

## AI가 선호하는 디자인 패턴들

한 디자이너가 저한테 이런 말을 했어요. "컬러 왼쪽 테두리는 em 대시(—)만큼이나 AI 디자인의 신호탄이나 마찬가지"라고요. 그 말을 들으니까 많은 사이트에서 이 패턴이 보이더라고요.

그래서 디자이너 친구들한테도 물어봤습니다. AI가 흔히 사용하는 패턴이 뭐냐고요. 답변들을 정리하니 폰트, 색상, 레이아웃, CSS 패턴 네 가지로 나뉘더군요.

### 폰트

- **Inter를 모든 곳에**: 특히 중앙 정렬된 히어로 헤드라인에서 자주 봄
- **특정 폰트 조합**: Space Grotesk, Instrument Serif, Geist 같은 조합을 자주 사용
- **한 단어만 Serif Italic**: 나머지는 Inter인데 강조하려는 단어 하나만 italic 처리

### 색상

- **"VibeCode Purple"**: 어디서 많이 봤을 법한 그 보라색
- **다크 모드 고정**: 중간 톤의 회색 본문 텍스트와 모두 대문자로 된 섹션 라벨
- **낮은 명도 대비**: 다크 테마에서 텍스트 대비가 겨우 통과 수준
- **그래디언트 남발**: 거의 모든 곳에 그래디언트 적용
- **화려한 광선 효과**: 큰 컬러 글로우와 colored box-shadow

### 레이아웃 특징

- **중앙 정렬 히어로**: 그냥 제네릭한 산스 폰트로 중앙 정렬
- **배지 강제 배치**: 히어로 H1 바로 위에 배지 배치
- **카드 테두리**: 위쪽 또는 왼쪽 테두리에 색상 적용
- **획일적인 기능 카드**: 모두 동일한 구조에 위에 아이콘 배치
- **숫자 나열**: 1, 2, 3처럼 단계를 숫자로 표시
- **통계 배너**: 행으로 된 통계 배너
- **이모지 아이콘 네비게이션**: 사이드바나 네비게이션에 이모지 사용
- **모두 대문자**: 헤드라인과 섹션 라벨을 전부 대문자로

### CSS 패턴

- **shadcn/ui 사용**: 거의 그대로 가져다 씀
- **글래스모피즘**: 반투명 카드 효과

## AI 디자인을 자동으로 감지하기

이제 이 패턴들을 체계적으로 측정해볼 차례입니다. Show HN 최신 제출 500개의 랜딩 페이지를 분석해서 위의 패턴 목록과 비교해봤어요.

**점수 방식:**

1. Playwright를 사용해 headless 브라우저가 각 사이트를 로드
2. 페이지 내 스크립트가 DOM을 분석하고 계산된 스타일을 읽음
3. 각 패턴은 명확한 CSS 또는 DOM 체크: 스크린샷을 찍거나 LLM이 판단하게 하지 않음
4. 이렇게 하면 오탐지가 생기지만, 수동 검증 결과 약 5~10% 정도

혹시 이 점수 계산 코드를 오픈 소스로 공개해서 누구나 재현하거나 자신의 사이트를 평가하고 싶으신 분이 있으면 연락주세요.

## 결과

하나의 패턴만으로는 사이트가 AI로 만들어졌다고 단정할 수 없으니, 발동한 패턴 개수로 3단계로 분류했습니다:

- **Heavy slop (5개 이상)**: 105개 사이트 · 21%
- **Mild (2~4개)**: 230개 사이트 · 46%
- **Clean (0~1개)**: 165개 사이트 · 33%

## 이게 나쁜 걸까요?

꼭 그렇지만은 않아요. 단지 영감이 부족할 뿐입니다. 어차피 비즈니스 아이디어를 검증하는 데 멋진 디자인이 꼭 필요한 건 아니니까요. AI 시대가 오기 전에도 대부분의 사이트가 Bootstrap처럼 보였거든요.

LLM이 기본으로 뱉어내는 디자인을 그대로 배포하는 것과 직접 설계하려고 노력하는 것 사이에는 분명한 차이가 있습니다. 이건 AI 이전 시대에 CSS/HTML 템플릿을 사용할 때도 마찬가지였어요.

아마 사람들은 이 수많은 '슬롭(slop)' 속에서 돋보이기 위해 다시 예쁜 디자인을 직접 만들기 시작할 겁니다. 다른 한편으로는, AI 에이전트가 웹의 주요 사용자가 되는 순간 디자인이 얼마나 중요할까 하는 의문도 들더라고요.
