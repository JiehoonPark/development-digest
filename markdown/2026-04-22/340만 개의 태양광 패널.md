## 핵심 개념

> [!abstract]
> 미국의 지면 설치형 태양광 에너지 데이터셋(GM-SEUS) v2를 분석한 기술 글로, 이전 버전의 290만 패널에서 340만 개로 증가한 데이터를 GDAL, DuckDB, QGIS 등의 도구로 처리합니다.

## 상세 내용

- AMD Ryzen 9, 96GB RAM, NVMe SSD 고사양 워크스테이션 환경에서 대규모 지리공간 데이터 분석
- GDAL, DuckDB(H3, Parquet, Spatial 확장), QGIS를 활용한 태양광 패널 데이터 추출 및 변환 프로세스

> [!tip] 왜 중요한가
> 개발자들이 대규모 지리공간 데이터셋을 효율적으로 처리하고 분석하기 위한 실무 기술 스택을 참고할 수 있습니다.

## 전문 번역

# 340만 개의 태양광 패널 데이터셋 분석해보기

지난 10월에 미국의 지상 설치 태양광 에너지(Ground-Mounted Solar Energy in the United States, GM-SEUS) 데이터셋을 소개한 적이 있는데요. 이 데이터셋은 미국 전역의 태양광 발전소 어레이와 패널들을 거의 모두 포함하려고 만들어졌습니다. 당시 v1에는 290만 개의 패널이 포함되어 있었죠.

지난주 v2가 공개되었는데, 이번엔 340만 개 이상의 패널을 담고 있습니다. 기존 패널과 어레이 데이터가 업데이트된 것은 물론이고, 옥상 설치 어레이 데이터셋이 새로 추가되었어요. 이 글에서는 GM-SEUS v2 데이터셋을 직접 분석한 내용을 공유하겠습니다.

## 내 작업 환경

제 작업용 데스크톱 사양을 먼저 소개하자면, CPU는 5.7 GHz로 작동하는 AMD Ryzen 9 9950X입니다. 16개 코어와 32개 스레드를 가지고 있고, L1 캐시 1.2MB, L2 캐시 16MB, L3 캐시 64MB를 탑재했어요. 액체 냉각장치가 붙어 있고, Cooler Master HAF 700이라는 넉넉한 크기의 풀사이즈 케이스에 들어가 있습니다.

메모리는 DDR5 RAM 96GB를 4,800 MT/s 속도로 구성했고, 저장소는 Crucial T700 4TB NVMe M.2 SSD(5세대)를 사용 중입니다. 이 드라이브는 초당 12,400MB 속도로 읽을 수 있고 온도 관리를 위해 히트싱크가 달려 있어요. 이게 제 C 드라이브입니다.

전원은 1,200와트 완전 모듈식 Corsair 파워서플라이로 공급받으며, 마더보드는 ASRock X870E Nova 90을 사용합니다.

OS는 Windows 11 Pro 위에서 Microsoft의 Ubuntu for Windows를 통해 Ubuntu 24 LTS를 실행 중인데요. 네이티브 Linux 데스크톱을 쓰지 않는 이유는 두 가지입니다. 하나는 제 GPU가 Nvidia GTX 1080인데 Windows에서의 드라이버 지원이 훨씬 낫거든요. 다른 하나는 ArcGIS Pro가 Windows만 공식 지원하기 때문입니다.

## 필요한 도구 설치

이번 분석에서는 GDAL 3.9.3을 데이터 분석에 사용하겠습니다.

```bash
$ sudo add-apt-repository ppa:ubuntugis/ubuntugis-unstable
$ sudo apt update
$ sudo apt install \
gdal-bin
```

그리고 DuckDB를 H3, JSON, Lindel, Parquet, Spatial 확장과 함께 사용할 거예요.

```bash
$ cd ~
$ wget -c https://github.com/duckdb/duckdb/releases/download/v1.5.1/duckdb_cli-linux-amd64.zip
$ unzip -j duckdb_cli-linux-amd64.zip
$ chmod +x duckdb
$ ~/duckdb
INSTALL h3 FROM community;
INSTALL lindel FROM community;
INSTALL json;
INSTALL parquet;
INSTALL spatial;
```

DuckDB가 실행될 때마다 자동으로 확장을 로드하도록 설정해두겠습니다.

```bash
$ vi ~/.duckdbrc
.timer on
.width 180
LOAD h3;
LOAD lindel;
LOAD json;
LOAD parquet;
LOAD spatial;
```

지도 시각화는 QGIS 4.0.1로 만들었습니다. QGIS는 Windows, macOS, Linux를 모두 지원하는 데스크톱 애플리케이션이고, 최근 몇 년 사이 인기가 많이 늘어서 전 세계 사용자로부터 매달 약 1,500만 회의 실행 기록이 있어요. 저는 QGIS의 HCMGIS 플러그인을 사용해서 Esri의 베이스맵을 추가했습니다.

## 분석 준비 - 데이터 다운로드

먼저 3.4GB 크기의 ZIP 파일을 다운로드해서 GeoPackage(GPKG) 파일들을 추출합니다.

```bash
$ wget -O GMSEUS_v2.zip \
'https://zenodo.org/records/19581821/files/GMSEUS.zip?download=1'
$ unzip -j GMSEUS_v2.zip "*.gpkg"
```

이 데이터셋의 GPKG 파일들이 사용하는 좌표계를 확인해보면 다음과 같습니다.

```bash
$ gdalsrsinfo \
-o proj4 \
GMSEUS_RooftopArrays_2025_v2_0.gpkg
+proj=aea +lat_0=23 +lon_0=-96 +lat_1=29.5 +lat_2=45.5 +x_0=0 +y_0=0 +ellps=GRS80 +towgs84=0,0,0,0,0,0,0 +units=m +no_defs
```

이제 옥상 어레이 데이터를 Parquet 형식으로 변환하겠습니다. 다음 쿼리를 실행할 때는 v1.5.1이 예외를 발생시키므로 DuckDB v1.4.4를 써야 합니다.

```bash
$ ~/duckdb
COPY (
WITH a AS (
SELECT Source,
grndCvr,
modType,
mount,
nativeID,
roofArrID,
area: IF(area::TEXT='-9999.0', NULL, area::DOUBLE),
azimuth: IF(azimuth::TEXT='-9999.0', NULL, azimuth::DOUBLE),
capMWAC: IF(capMWAC::TEXT='-9999.0', NULL, capMWAC::DOUBLE),
capMWDC: IF(capMWDC::TEXT='-9999.0', NULL, capMWDC::DOUBLE),
tilt: IF(tilt::TEXT='-9999.0', NULL, tilt::DOUBLE),
instYr: CASE WHEN instYr::INT = -9999 THEN NULL ELSE instYr END,
ST_FLIPCOORDINATES(
ST_TRANSFORM(
geom,
'+proj=aea +lat_0=23 +lon_0=-96 +lat_1=29.5 +lat_2=45.5 +x_0=0 +y_0=0 +ellps=GRS80 +towgs84=0,0,0,0,0,0,0 +units=m +no_defs',
'EPSG:4326')) geometry
FROM ST_READ('GMSEUS_RooftopArrays_2025_v2_0.gpkg')
)
SELECT * EXCLUDE (geometry),
{'xmin': ST_XMIN(ST_EXTENT(geometry)),
'ymin': ST_YMIN(ST_EXTENT(geometry)),
'xmax': ST_XMAX(ST_EXTENT(geometry)),
'ymax': ST_YMAX(ST_EXTENT(geometry))} AS bbox,
ST_ASWKB(geometry) geometry
FROM a
ORDER BY HILBERT_ENCODE([ST_Y(ST_CENTROID(geometry)),
ST_X(ST_CENTROID(geometry))]::double[2])
) TO 'GMSEUS_RooftopArrays_2025_v2_0.parquet' (
FORMAT 'PARQUET',
CODEC 'ZSTD',
COMPRESSION_LEVEL 22,
ROW_GROUP_SIZE 15000);
```

이 데이터셋에는 5,822개의 레코드가 있습니다.

```sql
SELECT COUNT(*)
FROM 'GMSEUS_RooftopArrays_2025_v2_0.parquet';
```

결과:
```
5,822
```

각 컬럼별 고유값 개수와 NULL 비율을 정리하면 다음과 같습니다.

```sql
SELECT column_name,
column_type,
null_percentage,
approx_unique,
min,
max
FROM (SUMMARIZE
FROM READ_PARQUET('GMSEUS_RooftopArrays_2025_v2_0.parquet'))
WHERE column_name != 'geometry'
AND column_name != 'bbox'
ORDER BY LOWER(column_name);
```

```

│ column_name │ column_type │ null_percentage │ approx_unique │ min │ max │
│ varchar │ varchar │ decimal(9,2) │ int64 │ varchar │ varchar │
├─────────────┼─────────────┼─────────────────┼───────────────┼────────────┤
```
