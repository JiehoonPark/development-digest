## 핵심 개념

> [!abstract]
> 셰이더를 활용하여 대기 산란(atmospheric scattering)을 구현하고 현실적인 하늘과 일몰을 브라우저에서 렌더링하는 방법을 상세히 설명합니다. Rayleigh 산란, Mie 산란, 오존 흡수 등 대기 렌더링의 핵심 요소들을 단계별로 구현하고, 성능 최적화를 위해 LUT 기반 접근법을 활용합니다. 실시간으로 동작하는 인터랙티브 경험을 만드는 것을 목표로 하며, 우주에서 본 지구의 대기층과 같은 현실감 있는 시각적 효과를 구현합니다.

## 상세 내용

- 대기 밀도 샘플링: Raymarching 기법을 통해 카메라 위치에서 광선을 발사하여 대기 밀도를 누적하는 방식으로 transmittance(빛의 투과율)를 계산하며, Rayleigh 밀도 함수를 사용해 고도에 따른 공기 밀집도를 모델링합니다.
- 광학 깊이(Optical Depth): 광선이 대기를 통과하면서 만나는 대기 밀도를 누적하여 얼마나 많은 빛이 손실되는지 계산하며, 이는 Beer의 법칙을 사용하여 구현되고 T=1.0(빛 손실 없음)부터 T=0.0(완전 소광)까지의 범위를 가집니다.
- Rayleigh 및 Mie 산란: 두 가지 산란 메커니즘을 구현하여 짧은 파장(파란색)은 더 많이 산란되고 긴 파장(빨간색)은 적게 산란되는 현상을 재현하며, 이를 통해 낮의 파란 하늘과 일몰의 주황색 그라데이션을 구현합니다.
- 오존 흡수: 대기의 오존층이 특정 파장의 빛을 흡수하는 현상을 모델링하여 더욱 정확한 색상 표현을 제공합니다.
- LUT 기반 최적화: Sebastian Hillaire의 LUT(Look-Up Table) 기반 접근법을 적용하여 복잡한 계산을 사전 계산된 텍스처로 대체하여 성능을 개선합니다.
- 행성 주변 대기 렌더링: 초기의 하늘 돔 렌더링에서 나아가 행성 주변의 대기 껍질로 표현하는 방식으로 확장하여 우주에서 바라본 지구의 관점을 구현합니다.
- 코드 예시: RAYLEIGH_SCALE_HEIGHT=8.0, ATMOSPHERE_HEIGHT=100.0, PRIMARY_STEPS=24 등의 상수를 사용하여 exp(-max(h, 0.0) / RAYLEIGH_SCALE_HEIGHT) 함수로 고도 h에서의 Rayleigh 밀도를 계산하는 구체적 구현을 제시합니다.

> [!tip] 왜 중요한가
> 브라우저 기반 3D 렌더링에서 대기 효과는 게임, 시뮬레이션, 과학 시각화 등 다양한 분야에서 필수적이며, 이 글은 복잡한 물리 현상을 셰이더로 구현하는 실전 기법과 성능 최적화 방법을 제공하여 개발자들이 현실감 있는 환경을 만들 수 있게 합니다.

## 전문 번역

# 셰이더로 하늘, 석양, 그리고 행성을 렌더링하기

우주 왕복선 엔데버호가 지구 저궤도에서 석양을 배경으로 떠있는 사진이 있는데요. 이 사진은 어두운 주황색부터 파란색까지 아름다운 색상의 대기층을 보여주다가 우주의 검은색으로 사라져갑니다. 이 그라데이션은 시각적으로도 매력적이지만, 그 뒤에 숨은 대기 산란(atmospheric scattering) 현상은 어떻게 작동하는지, 그리고 이를 재현하는 방법을 알아보면 정말 흥미로운 주제거든요.

저는 이 효과를 셰이더로 직접 구현해보고 싶었습니다. 브라우저에서 실시간으로 하늘의 독특한 파란색과 현실적인 석양, 그리고 일출을 렌더링하는 것이죠. 목표는 그 우주 사진에 최대한 가깝게 재현하면서도, 게임이나 다른 셰이더 기반 미디어에서 흔히 볼 수 있는 수준의 대기 렌더링을 구현하는 것이었습니다.

이 한 달간의 여정을 통해 만들어진 결과물들이 모두 실시간으로 렌더링되고 있습니다.

처음엔 이 주제에 대해 쓸 생각이 없었는데요. 최근 아르테미스 II 미션에 대한 관심과 제 개인적인 우주에 대한 열정이 합쳐져서 깊이 있게 탐구해보고 싶었습니다. 게다가 이를 통해 인터랙티브한 경험을 만들면 이 주제가 더 많은 사람들에게 접근 가능할 것 같았죠.

이 글에서는 대기 산란 셰이더 후처리 효과를 단계별로 구현하는 방법을 살펴볼 겁니다. 먼저 기본 요소들(레이마칭, 레일리 산란과 미 산란, 오존 흡수)을 구현해서 현실적인 하늘 돔을 렌더링한 다음, 이를 행성 주변의 대기층으로 확장해보겠습니다.

마지막으로는 더 나은 성능을 위해 Sebastian Hillaire의 LUT(룩업 테이블) 기반 접근 방식을 살펴보겠습니다. 물론 제가 구현한 버전이긴 하지만요. 이 부분은 정말 제 편안함의 영역을 벗어난 도전이었습니다.

## 하늘을 렌더링하는 방법

한 번쯤은 작업물 뒤에 파란색 그라데이션 배경을 슥 붙여서 "대기감 있게" 만들려고 시도해본 경험이 있을 겁니다. 그런데 이렇게 하면 항상 뭔가 부족한 느낌이 들죠.

더 현실적인 구현을 위해서는 하늘의 색을 빛이 공기와 그 구성 요소들과 상호작용한 결과로 봐야 합니다. 관찰자의 고도, 먼지의 양, 시간대 등 여러 변수를 고려해야 한다는 뜻입니다. 그리고 이 모든 것이 3차원 공간에서 일어나죠.

이제 우리의 첫 번째 목표는 이를 기본 원칙으로 삼아 대기 셰이더의 기초를 다지고, 하루의 어느 시간이든 현실과 구분하기 어려울 정도의 하늘을 표현하는 것입니다.

## 대기 밀도 샘플링

볼류메트릭 구름이나 볼류메트릭 라이트를 다룰 때처럼, 대기를 샘플링하는 가장 간단한 방법은 레이마칭입니다. 카메라 위치에서 장면으로 광선을 쏘고, 투명한 매질을 통과하면서 두 가지 질문에 답하면 되죠:

1. 대기를 통과하면서 얼마나 많은 빛이 살아남을까? 이것을 투과율(transmittance)이라고 합니다.
2. 각 샘플 지점에서 얼마나 많은 빛이 카메라 방향으로 꺾일까? 이것이 산란입니다.

첫 번째 질문에 답하려면 광선을 따라 만난 대기 밀도를 누적해서 광학 깊이(optical depth)를 구해야 합니다. 이를 레일리 밀도 함수로 모델링할 텐데요. 이 함수는 특정 고도 h에서 "공기"가 얼마나 많은지를 알려줍니다. 고도가 높아질수록 대기가 희박해지는 현상을 반영하기 위해 필요한 것이죠.

### 레일리 밀도 샘플링과 광학 깊이 누적

```glsl
const float RAYLEIGH_SCALE_HEIGHT = 8.0;
const float ATMOSPHERE_HEIGHT = 100.0;
const float VIEW_DISTANCE = 200.0;
const int PRIMARY_STEPS = 24;
const vec3 SUN_DIRECTION = normalize(vec3(0.0, 1.0, 1.0));

float rayleighDensity(float h) {
 return exp(-max(h, 0.0) / RAYLEIGH_SCALE_HEIGHT);
}

void main() {
 vec2 p = vUv * 2.0 - 1.0;
 
 vec3 color = vec3(0.0);
 vec3 viewDir = normalize(vec3(p.x, p.y, 1.0));
 vec3 skyDir = normalize(vec3(viewDir.x, max(viewDir.y, 0.0), viewDir.z));

 float stepSize = VIEW_DISTANCE / float(PRIMARY_STEPS);
 float viewOpticalDepth = 0.0;

 for (int i = 0; i < PRIMARY_STEPS; i++) {
   float t = (float(i) + 0.5) * stepSize;
   float h = t * skyDir.y;

   if (h < 0.0) break;
   if (h > ATMOSPHERE_HEIGHT) break;

   float dR = rayleighDensity(h);
   viewOpticalDepth += dR * stepSize;
 }

 color = ACESFilm(color);

 fragColor = vec4(color, 1.0);
}
```

광학 깊이로부터 광선 위의 특정 지점에서 투과율 T를 계산할 수 있습니다. 이는 대기를 통과할 때 살아남는 빛의 비율을 나타냅니다.

- T = 1.0: 빛이 손실되지 않음
- T = 0.0: 빛이 완전히 차단됨

제 볼류메트릭 구름 글을 읽어본 분이라면 낯익을 수도 있는데요. 여기서 사용하는 공식이 바로 Beer의 법칙입니다:

$$T = e^{-\mu \cdot d}$$

여기서 μ는 흡수 계수, d는 광학 깊이입니다.
