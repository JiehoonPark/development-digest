## 핵심 개념

> [!abstract]
> 본 논문은 기존 검색 시스템의 고정된 유사성 인터페이스 대신 에이전트가 grep, 파일 읽기, 셸 명령 등의 일반 도구로 원본 코퍼스와 직접 상호작용하는 직접 코퍼스 상호작용(DCI) 방식을 제안한다. 이 접근은 임베딩 모델이나 벡터 인덱스 없이도 BRIGHT, BEIR 데이터셋에서 기존 검색 방식을 능가하는 성능을 보인다.

## 상세 내용

- DCI는 정확한 어휘 제약, 희소 단서 결합, 다단계 가설 정제 등을 기존 검색 API로는 어려운 작업을 가능하게 함
- 오프라인 인덱싱이 필요 없고 진화하는 로컬 코퍼스에 자연스럽게 적응
- 강화된 언어 에이전트는 검색 품질이 추론 능력뿐만 아니라 코퍼스와의 상호작용 인터페이스 해상도에 의존

> [!tip] 왜 중요한가
> AI 에이전트 검색 시스템을 구축하는 개발자는 벡터 데이터베이스 없이도 더 유연하고 적응 가능한 검색 인터페이스를 설계할 수 있다.

## 전문 번역

# 의미 유사성을 넘어서: 직접 코퍼스 상호작용을 통한 에이전트 검색의 재고찰

## 논문 요약

현대의 검색 시스템은 어휘 기반이든 의미 기반이든, 한 번의 top-k 검색 단계로 코퍼스 접근을 압축하는 고정된 유사도 인터페이스를 제공합니다. 이 방식은 효율적이지만, 에이전트 검색에서는 병목이 됩니다.

정확한 어휘 제약, 희소한 단서 조합, 로컬 문맥 검증, 그리고 다단계 가설 개선 같은 작업들은 일반적인 검색 API로 구현하기 어렵거든요. 게다가 초반에 필터링되어 제외된 증거는 강력한 추론 모델로도 복구할 수 없습니다.

에이전트 작업이 더욱 복잡한 이유는 중간 엔티티 발견, 약한 단서 결합, 부분적 증거 확보 후 계획 수정 같은 여러 단계를 조율해야 하기 때문입니다.

## 새로운 접근: 직접 코퍼스 상호작용(DCI)

이 논문에서는 **직접 코퍼스 상호작용(Direct Corpus Interaction, DCI)**을 제안합니다. 에이전트가 임베딩 모델, 벡터 인덱스, 검색 API 없이 grep, 파일 읽기, 셸 명령어, 간단한 스크립트 같은 범용 터미널 도구로 원본 코퍼스를 직접 검색하는 방식입니다.

이 방법의 장점은 몇 가지입니다:

- 오프라인 인덱싱이 필요 없습니다
- 시간이 지나며 변하는 로컬 코퍼스에 자연스럽게 적응합니다
- 복잡한 검색 요구사항을 더 유연하게 처리할 수 있습니다

## 실험 결과

연구팀은 IR 벤치마크와 엔드-투-엔드 에이전트 검색 작업에서 DCI를 평가했습니다.

BRIGHT, BEIR 데이터셋에서 희소 검색, 밀집 검색, 리랭킹 기반 강력한 베이스라인들을 상당히 앞지른 성능을 보였습니다. 특히 BrowseComp-Plus와 다중 홉 질문 응답 작업에서 기존의 의미 기반 검색기에 의존하지 않으면서도 우수한 정확도를 달성했습니다.

## 핵심 통찰

이 연구의 결론은 흥미롭습니다: 언어 에이전트가 강해질수록, 검색의 품질은 단순히 추론 능력뿐만 아니라 **모델이 코퍼스와 상호작용하는 인터페이스의 해상도(resolution)**에도 달려 있다는 겁니다.

DCI는 에이전트 검색을 위한 더욱 폭넓은 인터페이스 설계 공간을 열어줍니다. 기존의 고정된 검색 인터페이스에서 벗어나, 에이전트가 필요에 따라 더 세밀하고 유연하게 정보에 접근할 수 있게 만드는 거죠.
