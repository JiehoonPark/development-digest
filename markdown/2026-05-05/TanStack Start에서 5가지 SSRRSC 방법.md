## 핵심 개념

> [!abstract]
> 이 영상은 TanStack Start 프레임워크에서 Server-Side Rendering(SSR)과 React Server Components(RSC)를 구현하는 5가지 서로 다른 접근 방식을 설명한다. 각 방법은 프로젝트 요구사항과 성능 목표에 따라 다른 이점을 제공한다.

## 상세 내용

- 다양한 렌더링 전략: TanStack Start는 순수 SSR, RSC, 하이브리드 접근 방식 등 여러 렌더링 패턴을 지원하여 개발자가 프로젝트 요구사항에 맞는 최적의 전략을 선택할 수 있다.
- 성능 최적화 선택: SSR은 초기 페이지 로드 성능을 개선하고, RSC는 클라이언트 번들을 줄이며, 각 방법의 조합으로 다양한 성능 트레이드오프를 달성할 수 있다.
- TanStack 에코시스템 통합: TanStack Start는 TanStack Router, Query 등 다른 TanStack 도구와 통합되어 일관된 전체 스택 개발 환경을 제공한다.

> [!tip] 왜 중요한가
> 최신 웹 프레임워크에서 SSR과 RSC는 필수 기능이며, TanStack Start의 다양한 구현 방법을 이해하면 프로젝트 특성에 맞춰 최적화된 렌더링 아키텍처를 설계할 수 있다.
