## 핵심 개념

> [!abstract]
> 모든 웹사이트가 갖춰야 할 기술적 특성을 128개 항목으로 정의한 플랫폼 독립적 명세로, 인간과 AI 에이전트 모두를 위해 작성되었다.

## 상세 내용

- SEO, 접근성, 보안, 성능, 국제화 등 10개 범주에 걸친 128개 항목의 웹사이트 체크리스트를 제공하며 모든 항목이 공식 웹 표준 문서로 소급된다.
- MCP 서버 및 에이전트 스킬로 제공되어 AI가 직접 활용할 수 있으며, 플랫폼(WordPress, Next.js, Hugo 등)에 관계없이 적용 가능하다.
- 완전 오픈소스로 관리되어 누구나 GitHub PR을 통해 기여하고 개선할 수 있다.

> [!tip] 왜 중요한가
> 개발자들이 웹사이트 품질을 객관적 기준으로 평가하고 개선하는 데 사용할 수 있는 표준화된 체크리스트를 제공하며, AI 에이전트와의 협업도 지원한다.

## 전문 번역

# 좋은 웹사이트의 조건

플랫폼에 상관없이 모든 웹사이트가 갖춰야 할 기술적 요소들을 정리한 스펙입니다. `<title>` 태그부터 `/.well-known/security.txt`까지, WCAG 명도 기준부터 llms.txt까지 다룹니다. 사람과 AI 에이전트 모두를 위해 작성했습니다.

## 10개 카테고리, 검증된 표준과 연계

**Foundations** (14개)
페이지가 필요로 하는 HTML, head, 문서 기본 구조

**SEO** (13개)
검색 엔진의 가시성 - robots.txt, 사이트맵, canonical 태그, 구조화된 데이터

**Accessibility** (20개)
WCAG를 준수해 모든 사용자가 사이트를 이용할 수 있도록

**Security** (12개)
방문자를 보호하는 헤더, 전송 보안, 정책들

**Well-Known URIs** (9개)
`/.well-known/` 경로 아래의 표준화된 URI들

**Agent Readiness** (18개)
AI 에이전트와 크롤러가 사이트를 이해하기 쉽게 만드는 것들

**Performance** (19개)
Core Web Vitals, 캐싱, 이미지, 폰트, 네트워크 동작

**Privacy** (6개)
동의, 신호, 방문자의 선택 존중

**Resilience** (5개)
우아한 실패 - 에러 페이지, 오프라인 지원, 리다이렉트

**Internationalisation** (12개)
언어, 지역 설정, 텍스트 방향, 번역된 콘텐츠

## 의견이 아닌 표준

각 항목은 그 출처가 되는 표준으로 다시 연결됩니다. WHATWG, W3C, IETF RFC, WCAG, MDN, 그리고 현대 웹을 정의하는 각 조직의 문서들이죠.

## 플랫폼 독립적

WordPress, Drupal, TYPO3, Next.js, Astro, Hugo, Django 앱, 아니면 순수 HTML을 쓰든 상관없습니다. 스펙은 스펙이거든요. 구현 방법은 이 스펙을 따를 뿐, 그 반대는 아닙니다.

## 공개적으로 함께 만들어가기

모든 페이지에 GitHub 편집 링크가 있습니다. PR을 언제든 보내주세요. 모든 페이지에 출처를 명시합니다.

## AI 에이전트를 위한 접근

이 스펙 전체는 오픈 MCP 서버로 이용 가능합니다. 인증이 필요 없는 읽기 전용 방식이죠. 호환되는 모든 에이전트에게 이 스펙을 언제, 어떻게 활용할지 가르치는 Agent Skill도 공개되어 있습니다.

각 페이지의 마크다운은 `/llms.txt`와 `Accept: text/markdown` 헤더를 통해 접근할 수 있습니다.

### MCP 연결 및 Agent Skill

```json
{
  "mcpServers": {
    "specification-website": {
      "transport": "http",
      "url": "https://mcp.specification.website/mcp"
    }
  }
}
```

## 이 사이트를 어떻게 쓸까

**01. 감시**
체크리스트를 하나씩 살펴보세요. 각 항목은 단순합니다. "우리 사이트가 이것을 하고 있나? 네 또는 아니오."

**02. 학습**
관심 있는 항목을 클릭해보세요. 그것이 무엇인지, 왜 중요한지, 어떻게 구현하는지 설명이 나옵니다.

**03. 개선**
빠진 부분, 오래된 정보, 빠진 주제를 찾았다면 PR을 열어주세요. 출처는 반드시 포함해야 합니다.
