## 핵심 개념

> [!abstract]
> GitHub의 여러 서비스에서 장애가 발생했습니다. 사용자는 이메일 또는 문자 메시지로 장애 업데이트 알림을 받을 수 있습니다.

## 상세 내용

- GitHub의 다중 서비스에서 동시 장애 발생
- webhook 전송 실패 시 이메일 알림 제공

> [!tip] 왜 중요한가
> GitHub 의존도가 높은 개발팀은 서비스 상태 모니터링을 통해 장애에 빠르게 대응할 필요가 있습니다.

## GitHub 장애 알림

GitHub 다중 서비스 장애에 대한 업데이트를 이메일 또는 문자 메시지로 구독할 수 있습니다. 

장애 상황이 업데이트될 때마다 이메일 알림을 받게 되며, GitHub가 장애를 생성하거나 해결할 때마다 문자 메시지 알림을 받을 수 있습니다.
