## 핵심 개념

> [!abstract]
> DocuBrowse v0.9.0은 PDF, Word, 이북 등 다양한 형식의 문서를 로컬 AI 모델로 인덱싱하고 키워드/의미론적 검색을 지원하는 도구입니다.

## 상세 내용

- SQLite FTS5 키워드 검색과 Ollama 기반 의미론적 유사도 검색을 결합한 하이브리드 검색
- 인터넷 연결 없이 로컬 머신에서 완전히 동작하므로 데이터 프라이버시 보장 및 API 비용 없음
- PDF, DOCX, PPTX, XLSX, EPUB, MOBI, AZW, HTML, TXT, Markdown 등 10개 이상의 형식 지원 및 PII 자동 감지·제거

> [!tip] 왜 중요한가
> 개발자가 대량의 내부 문서를 효율적으로 관리하고 검색할 수 있는 오픈소스 솔루션으로, 프라이버시와 비용 효율성을 모두 확보할 수 있습니다.

## 전문 번역

# DocuBrowse v0.9.0 — 로컬 AI로 문서를 똑똑하게 검색하기

DocuBrowse는 흩어져 있는 문서들을 제대로 된 검색 가능한 형태로 정리해주는 도구입니다. Linux(RPM, DEB, tarball), Windows(zip), macOS(dmg) 환경에서 사용할 수 있고, 인터페이스도 충분히 안정적이에요.

PDF, 전자책, Word 문서, 노트 등 어떤 형식이든 상관없습니다. 파일들을 지정하면 DocuBrowse가 똑똑한 인덱스를 만들어주는데, 단순한 키워드 매칭을 넘어 의미를 이해합니다. "임차료 갱신에 관한 계약서 찾아줘"라고 요청하면, 그 정확한 단어들이 나타나지 않아도 찾아낼 수 있다는 거죠. 검색 결과를 클릭하면 파일을 열기 전에 AI가 생성한 요약본을 바로 볼 수 있습니다.

가장 큰 장점은 **당신의 컴퓨터에서만 작동한다**는 점입니다. 인터넷이 필요 없고, 회원가입도 없고, API 키도 없습니다. 당신의 데이터는 당신의 것입니다.

## 내부 구조

SQLite FTS5로 빠른 키워드 검색을 하고, Ollama와 nomic-embed-text, dolphin3 모델을 활용해 의미 기반 검색과 요약을 생성합니다. 다양한 문서 포맷과 소스 코드를 지원하죠.

## 주요 기능

### 🔍 두 가지 검색 방식

**키워드 검색** — SQLite FTS5를 통한 빠른 전문 검색입니다. 제목, 저자, 주제, 태그, 스니펫을 대상으로 검색하죠.

**의미 기반 검색** — Ollama 임베딩(nomic-embed-text:latest)을 활용한 AI 검색입니다. "계약서를 찾고 싶은데 정확한 표현이 뭐였지?" 하는 상황에서 제격입니다.

**하이브리드 모드(기본값)** — 의미 기반 검색 70%와 키워드 검색 30%를 섞어서 다시 랭킹합니다. 가장 관련성 높은 결과를 먼저 보여줍니다.

### 📖 AI 요약

문서 제목을 클릭하면 Kindle 스타일의 미리보기가 나타납니다. Ollama(dolphin3:latest)가 생성하고 처음 생성된 후에는 데이터베이스에 캐시됩니다.

### 📚 문서 인덱싱

**지원 포맷**: PDF, DOCX, PPTX, XLSX, EPUB, MOBI, AZW3, AZW, HTML, TXT, Markdown

**PDF 처리**: pdfplumber를 기본으로 사용하고, 복잡한 파일은 pypdf로 폴백합니다. 스캔된 PDF(이미지만 있는 경우)는 자동으로 감지해 OCR 목록에 담깁니다.

**Word 문서**: python-docx로 단락, 표, 핵심 속성(제목, 저자, 주제)을 추출합니다.

**프레젠테이션**: python-pptx로 슬라이드 텍스트, 노트, 핵심 속성을 추출합니다.

**스프레드시트**: openpyxl로 셀 값과 시트 이름을 추출합니다.

**전자책**: EPUB은 ebooklib로, MOBI/AZW3는 mobi 패키지와 Calibre로 처리합니다. DRM 보호된 AZW 파일은 메타데이터(제목/저자)만 인덱싱됩니다.

**메타데이터**: 문서 메타데이터에서 제목, 저자, 주제를 추출하고, 폴더 구조와 콘텐츠 키워드로 태그를 자동 생성합니다.

**개인정보 보호**: 주민등록번호, 신용카드 번호, 계좌번호, 생년월일, 의료기록, 운전면허 번호, 여권번호 패턴을 자동 감지합니다. 일치하는 문서는 제거되고 영구 차단됩니다.

### 🎨 사용자 인터페이스

다크/라이트 테마 토글이 가능합니다. 검색 결과는 페이지당 50개씩 표시되고, A-Z, 0-9 알파벳 인덱스 바로 빠르게 이동할 수 있습니다. 상태는 페이지를 새로고침해도 유지됩니다.

각 결과 카드에는 관련성 점수(0-100%)가 표시되고, Open 버튼으로 기본 앱에서 파일을 바로 열 수 있습니다. 문서 제목을 클릭하면 AI 요약을 보고, 📋 버튼으로 경로를 클립보드에 복사하거나, 🗑 버튼으로 파일과 인덱스에서 삭제할 수 있습니다.

이동되거나 삭제된 문서의 경우, 파일 시스템이 마운트되어 있으면 닫을 수 있는 모달이 표시되고(모달을 닫으면 인덱스에서도 제거), 마운트되지 않은 상태라면 토스트 메시지만 표시됩니다.

### ⚙️ 설정 (/settings)

설정 페이지는 독립된 페이지(/settings)로, 기어 아이콘을 클릭하면 새 탭에서 열립니다.

**General 패널**: 문서 디렉토리를 라이브 디렉토리 브라우저로 선택할 수 있고, 추가 스캔 디렉토리를 여러 개 추가하거나 제거할 수 있습니다. 추가된 디렉토리는 스캔/리스캔에 자동으로 포함됩니다. 작업 디렉토리와 포트도 여기서 설정합니다.

**Ignored Directories 패널**: 스캔에서 제외할 디렉토리를 관리합니다. 각 항목마다 디렉토리 브라우저, 추가/제거 컨트롤이 있고, 이미 인덱싱된 문서를 삭제하거나 항목을 제거하기 전에 확인 프롬프트가 나타납니다.

### ⚡ 성능

검색 응답 시간은 일반적으로 150ms 이내입니다. PDF 추출은 ProcessPoolExecutor로 병렬 처리되고, 물리 코어 개수에 맞춰 워커를 배분합니다.

메모리 안전을 위해 커널 레벨의 RLIMIT_AS(워커당 6GB)를 설정하고, 남은 RAM 양에 따라 자동으로 일시 중지/재개됩니다.

## 빠른 시작

**필수 요구사항**: Python 3.9 이상

필요한 라이브러리들:
- pdfplumber, pypdf — PDF 추출
- python-docx — Word 문서
- python-pptx — PowerPoint 프레젠테이션
- openpyxl — Excel 스프레드시트
- ebooklib, beautifulsoup4, mobi — 전자책
- psutil — 크로스 플랫폼 프로세스/하드웨어 정보
