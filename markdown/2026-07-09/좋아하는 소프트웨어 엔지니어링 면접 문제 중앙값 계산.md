# 기술 면접에서 자주 묻는 문제: 중앙값 구하기

면접을 볼 때 나는 몇 가지 질문들을 항상 준비해두곤 합니다. 이런 질문들의 공통점은 뭘까요? 바로 겉으로는 단순해 보이지만, 그 안에 여러 각도로 파고들 수 있다는 거예요.

수수께끼 같은 퍼즐 문제는 잘 안 물어봅니다. 실무 역량을 평가하기에는 별로 도움이 안 되거든요. 대신 평범한 문제처럼 보이면서도, 여러 주제로 자연스럽게 심화 논의로 넘어갈 수 있는 질문을 고르죠.

한 예로 이런 문제를 자주 냅니다.

**숫자로 이루어진 배열을 받아서 중앙값을 반환하는 함수를 작성해보세요.**

아래는 Python으로 구현한 코드와 함께 주요 포인트를 정리한 설명입니다.

```python
def find_median(numbers):
    """
    배열의 중앙값을 구한다.
    - 짝수 개: 가운데 두 값의 평균
    - 홀수 개: 정확히 가운데 값
    """
    sorted_numbers = sorted(numbers)
    n = len(sorted_numbers)
    
    if n % 2 == 1:
        # 홀수 개인 경우
        return sorted_numbers[n // 2]
    else:
        # 짝수 개인 경우
        mid1 = sorted_numbers[n // 2 - 1]
        mid2 = sorted_numbers[n // 2]
        return (mid1 + mid2) / 2
```

이 문제는 단순해 보이지만, 여기서 파고들 수 있는 주제들이 꽤 많아요. 정렬 알고리즘의 시간복잡도부터 시작해서, 대용량 데이터를 다루는 방식, 혹은 스트리밍 데이터에서 중앙값을 효율적으로 구하는 방법까지 자연스럽게 논의를 확장할 수 있죠.
