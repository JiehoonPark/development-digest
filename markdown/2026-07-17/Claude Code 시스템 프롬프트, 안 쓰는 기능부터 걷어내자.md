## 핵심 개념

> [!abstract]
> Claude Code를 비롯한 AI 코딩 하네스는 기본적으로 workflow, design sync, remote control, connectors, artifacts 같은 미사용 기능들까지 시스템 프롬프트에 포함시켜 요청당 수천 토큰을 낭비합니다. settings.json에서 이런 기능들을 하나씩 비활성화하면 해당 도구 정의가 시스템 프롬프트에서 완전히 제거되는데, 실제로 저자는 이런 방식으로 시스템 프롬프트를 25K 토큰에서 8K 토큰으로 줄였습니다. 프록시로 실제 전송되는 시스템 프롬프트를 확인하면서 본인에게 필요한 기능만 남기고 튜닝하는 방법을 소개합니다.

## 아티클

Claude Code를 비롯한 대부분의 AI 코딩 하네스(harness)는 시스템 프롬프트에 상당한 양의 불필요한 내용을 기본으로 포함하고 있습니다. 실제로 얼마나 많은 토큰이 낭비되고 있는지, 그리고 이를 어떻게 줄일 수 있는지 직접 확인해본 내용을 정리해봤습니다.

## 시스템 프롬프트에 숨어있는 수천 토큰의 낭비

Claude Code를 비롯한 대부분의 하네스는 기본 상태로 시스템 프롬프트에 엄청난 양의 블로트(bloat)를 싣고 다닙니다. 실제로 요청 하나당 사용하지도 않는 기능에 수천 토큰이 소모되고 있을 가능성이 매우 높습니다.

직접 확인해보니 workflow, design sync, monitor 같은 기능들이 통째로 시스템 프롬프트에 들어있었는데, 정작 저는 전혀 사용하지 않던 것들이었습니다. 이런 것들만 합쳐도 요청 하나당 약 8,000~10,000 토큰을 잡아먹고 있었습니다.

## settings.json으로 불필요한 도구 제거하기

다행히 Claude Code는 이런 부분을 커스터마이징할 수 있게 해줍니다. 글로벌 `settings.json` 파일에 설정을 추가해서 필요 없는 기능들을 하나씩 비활성화할 수 있는데요. 이렇게 도구를 끄면 해당 도구의 정의(tool definition)가 시스템 프롬프트에서 통째로 제거됩니다.

실제로 제가 비활성화한 항목들은 다음과 같습니다.

- **plan mode 진입/종료 제어**: 에이전트가 알아서 plan mode를 드나드는 걸 원치 않아서 관련 도구를 껐습니다.
- **ask user question 도구**: 개인적으로 정말 싫어하는 기능이라 아예 안 쓰고, 보고 싶지도 않아서 도구 정의 자체를 시스템 프롬프트에서 제거했습니다.
- **cron 스케줄링**: 에이전트가 알아서 cron을 예약하는 것도 원치 않아 삭제했습니다.
- **번들 스킬(bundled skills)**: 기본 제공되는 커스텀 코드 리뷰 스킬을 쓰지 않아서 비활성화했습니다.
- **dynamic workflow 관련 기능**: 이 역시 전혀 사용하지 않아 모두 껐습니다.
- **remote control**: 이 기능 관련해서도 시스템 프롬프트에 생각보다 많은 내용이 들어있어 비활성화했습니다.
- **Claude AI connectors**: 미처 인지하지 못했지만 상당한 토큰을 소모하고 있어서 껐습니다.
- **artifacts 기능**: 이것도 비활성화했습니다.

## 결과: 25K에서 8K로

이 모든 설정을 정리한 결과, 초기 시스템 프롬프트(시스템 도구 정의 포함)를 25K 토큰에서 약 8K 토큰까지 줄일 수 있었습니다. 요청당 무려 17K 토큰가량을 절약한 셈입니다.

이 과정을 아티클로 정리해뒀는데, 여기서는 프록시(proxy)를 사용해서 실제로 어떤 내용이 시스템 프롬프트에 실려서 전송되고 있는지 직접 확인하는 방법을 소개하고 있습니다. 이렇게 실제 전송되는 내용을 눈으로 확인하고 나면, 정말 필요한 부분만 남기고 나머지는 설정으로 걷어낼 수 있습니다.

## 왜 토큰을 줄이는 게 중요한가

전송하는 데이터가 적을수록 에이전트의 출력 품질은 더 좋아집니다. 에이전트를 산만하게 만드는 불필요한 정보가 줄어들기 때문입니다. 즉, 시스템 프롬프트를 가볍게 만드는 작업은 단순히 비용을 아끼는 문제를 넘어서, 에이전트가 실제로 더 집중력 있게 작업하도록 만드는 최적화이기도 합니다.

## 정리

- Claude Code 등 AI 코딩 하네스는 기본적으로 사용하지 않는 기능(workflow, design sync, monitor, remote control, connectors, artifacts 등)까지 시스템 프롬프트에 포함시켜 불필요하게 많은 토큰을 소모합니다.
- `settings.json`에서 개별 도구와 기능을 비활성화하면 해당 도구 정의가 시스템 프롬프트에서 완전히 제거됩니다.
- 실제 사례에서는 plan mode 제어, ask user question, cron 스케줄링, 번들 스킬, dynamic workflow, remote control, Claude AI connectors, artifacts를 껐더니 시스템 프롬프트가 25K에서 8K 토큰으로 줄었습니다.
- 프록시를 활용하면 실제로 어떤 내용이 시스템 프롬프트로 전송되는지 확인할 수 있어, 본인의 워크플로우에 맞게 설정을 튜닝하는 데 유용합니다.
- 전송 토큰이 적을수록 에이전트가 불필요한 정보에 방해받지 않아 출력 품질이 개선됩니다.

실무에서 Claude Code를 매일 쓰고 있다면, 한 번쯤 프록시로 실제 시스템 프롬프트를 열어보고 본인이 안 쓰는 기능이 얼마나 섞여 있는지 점검해볼 만합니다. 설정 몇 줄로 응답 품질과 비용 모두를 개선할 수 있는, 비교적 손쉬운 최적화 포인트입니다.
