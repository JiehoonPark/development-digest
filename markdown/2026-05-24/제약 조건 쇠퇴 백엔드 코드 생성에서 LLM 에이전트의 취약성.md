## 전문 번역

# LLM 에이전트의 백엔드 코드 생성: 구조적 제약의 한계

**논문 제목:** Constraint Decay: The Fragility of LLM Agents in Backend Code Generation  
**저자:** Francesco Dente, Dario Satriani, Paolo Papotti  
**분야:** 소프트웨어 공학, 인공지능

## 핵심 내용

대규모 언어 모델(LLM) 에이전트는 느슨한 명세 하에서 자동 코드 생성 작업에서 뛰어난 성능을 보여줍니다. 그런데 실제 프로덕션 환경의 소프트웨어는 아키텍처 패턴, 데이터베이스, ORM(객체-관계 매핑) 같은 구조적 제약을 엄격히 지켜야 합니다. 

문제는 기존 벤치마크들이 이런 비기능적 요구사항을 간과한다는 거예요. 기능적으로는 정확하지만 구조적으로는 임의적인 솔루션에 높은 점수를 주고 있거든요.

## 연구 방법

이 연구팀은 8가지 웹 프레임워크에 걸쳐 80개의 신규 프로젝트 생성 과제와 20개의 기능 구현 과제를 진행했습니다. 통일된 API 계약을 고정한 뒤, 엔드-투-엔드 동작 테스트와 정적 검증기를 활용한 이중 평가로 구조적 복잡도의 영향을 측정했습니다.

## 주요 발견사항

### 1. 제약 조건의 붕괴 현상

흥미롭게도 구조적 요구사항이 쌓일수록 에이전트의 성능이 눈에 띄게 떨어집니다. 기본 구성에서는 능력 있는 모델들이 어설션 통과율에서 평균 30점 가까이 손실을 입으며, 일부 약한 모델은 거의 0점까지 떨어집니다.

### 2. 프레임워크별 성능 격차

프레임워크에 따라 성능 편차가 상당합니다. Flask처럼 명시적이고 최소한의 설정만 필요한 프레임워크에서는 잘 작동하는데, FastAPI나 Django 같은 컨벤션 기반의 프레임워크에서는 크게 떨어집니다.

### 3. 주요 오류 원인

데이터 계층 결함이 가장 큰 문제입니다. 잘못된 쿼리 구성과 ORM 런타임 위반이 성능 저하의 주된 이유거든요.

## 결론

이 연구가 보여주는 것은 명확합니다. LLM 에이전트가 기능적 정확성과 구조적 제약을 동시에 만족시키는 것은 여전히 풀어야 할 핵심 과제라는 점입니다. 단순히 "작동하는" 코드를 생성하는 것을 넘어, 프로덕션 환경에 적합한 구조를 갖춘 코드를 만들어내기 위해서는 더 많은 연구가 필요합니다.
