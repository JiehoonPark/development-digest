## 핵심 개념

> [!abstract]
> DynIP는 60초 이내의 빠른 DNS 업데이트, RFC 2136 TSIG 지원, IPv6 듀얼스택, DNSSEC을 기본으로 제공하는 동적 DNS 서비스다. 사용자 도메인을 가져올 수 있으며 라우터, 엣지 장비 등에서 직접 설정 가능하다.

## 상세 내용

- 대부분의 DDNS 서비스는 30분 캐싱하지만 DynIP는 60초 이내 전역 전파로 빠른 업데이트 제공
- RFC 2136 TSIG를 통해 FortiGate, MikroTik, OPNsense 등 표준 DNS UPDATE를 지원하는 라우터와 호환
- IPv4/IPv6 듀얼스택, IPv6 전용 존 지원으로 현재와 미래 네트워크에 대응

> [!tip] 왜 중요한가
> 홈랩과 엣지 인프라 운영자에게 빠르고 안정적인 동적 DNS 솔루션이 필요하다.

## 전문 번역

# DynIP: 빠르고 유연한 동적 DNS 서비스

## 60초 안에 업데이트되는 DDNS

기존 DDNS 서비스들은 보통 30분 정도 캐시를 유지하는데, DynIP는 다릅니다. 라우터에서 업데이트를 보내면 전 세계 어디서나 약 60초 안에 당신의 호스트명이 올바르게 해석돼요. 60초 TTL과 NOTIFY 기반 업데이트, 다중 지역 네임서버로 이를 구현했습니다.

## 어떤 라우터든 바로 사용 가능

RFC 2136 TSIG 표준을 지원하니까, FortiGate, MikroTik, OPNsense, OpenWRT 같은 대부분의 라우터가 별도 설정 없이 바로 작동합니다. 저희 코드 생성기를 사용하면 DNS UPDATE 명령어를 직접 입력할 수 있어요. REST API와 네이티브 UDP/53도 지원하므로 선택지가 많습니다.

## IPv6까지 생각한 설계

요즘 ISP들은 CGNATed IPv4와 함께 네이티브 IPv6를 제공하는 추세예요. DynIP는 A와 AAAA 레코드를 동시에 업데이트할 수 있고, IPv6 전용 존도 운영할 수 있습니다. 오늘날의 네트워크와 내일의 네트워크 모두에 대응하는 설계죠. 기본적으로 DNSSEC도 활성화되어 있습니다.

## 직접 도메인을 가져오세요

본인의 도메인을 DynIP로 가져오면, 그 도메인 아래에서 동적 서브도메인을 자유롭게 프로비저닝할 수 있어요.

### 3단계 설정 과정

1. **존 생성**: 기기 이름을 입력하고 원하는 기본 도메인을 선택한 뒤 생성을 누릅니다.
2. **설정 가져오기**: 새 도메인 옆의 Snippets 버튼을 클릭해서 생성된 설정을 확인합니다.
3. **배포**: 기기 타입을 선택하고 생성된 설정 블록을 라우터 CLI에 복사-붙여넣기합니다.

참고로 IPv4와 IPv6는 들어오는 연결 기반으로 자동 감지되고 업데이트됩니다.

## 자동화를 위한 토큰 관리

세션 토큰은 로그아웃할 때 만료되는데, 장기 자동화가 필요하면 API 토큰을 사용하세요. 모니터링 스크립트나 CI 파이프라인, MSP 통합 같은 용도에 유용합니다.

각 토큰마다 읽기 전용 또는 전체 접근 권한을 설정할 수 있고, 언제든 폐기할 수 있어요. 토큰은 생성할 때만 한 번 표시되니까 1Password나 Bitwarden 같은 비밀번호 관리자에 안전하게 보관하세요.

## 기타 기능

**현재 IP로 즉시 업데이트**: 선택한 모든 존을 기기의 현재 외부 IP로 한 번에 업데이트할 수 있습니다.

**프로그래매틱 존 등록**: 세션 토큰으로 `/register` 엔드포인트에 POST 요청을 보내면 새로운 존을 프로그래매틱하게 등록할 수 있습니다.

**2FA 보안 강화**: Google Authenticator나 다른 TOTP 앱으로 로그인 시 추가 인증을 활성화할 수 있어요.

**계정 삭제**: 계정과 생성한 모든 DNS 존, 그리고 발급된 TLS 인증서까지 영구 삭제할 수 있습니다. 단, 활성 유료 구독이 있다면 먼저 취소해야 합니다.
