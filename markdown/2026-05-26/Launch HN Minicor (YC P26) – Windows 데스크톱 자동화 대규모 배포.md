## 핵심 개념

> [!abstract]
> Minicor는 자가 치유 AI 에이전트를 통해 Windows 데스크톱 애플리케이션을 자동화하고 RPA(로봇 프로세스 자동화)를 확장하는 Y Combinator 지원 플랫폼이다. UI 변경 시 자동으로 적응하고, API 호출 하나로 복잡한 워크플로우를 실행할 수 있다.

## 상세 내용

- 자가 치유 에이전트가 UI 변경, 예상 외 대화상자에 자동 적응하여 유지보수 비용 대폭 절감
- Windows VM, 클라우드, Citrix 등 다양한 환경에 배포 가능하며 하나의 API 호출로 워크플로우 실행
- 동영상 재생, Slack 알림, 스크린샷 기반 디버깅으로 완전한 실행 컨텍스트 제공, 일 25,000명 환자 규모로 운영 중

> [!tip] 왜 중요한가
> 레거시 시스템과의 통합이 필요한 AI 스타트업이 주 4주가 아닌 수일 만에 프로덕션 배포 가능하게 한다.

## 전문 번역

# 레거시 시스템을 자동화하는 AI 에이전트, Minicor

Y Combinator 지원 스타트업이자 SOC 2 Type II, HIPAA 인증을 받은 Minicor는 더 이상 API를 제공하지 않는 레거시 데스크톱 애플리케이션을 자동화하는 RPA 플랫폼입니다. 의료, 자동차, 물류, 금융 같은 규제산업에서 고객의 시스템에 데이터를 읽고 쓰려면 이런 오래된 시스템들과 상호작용해야 하는데, Minicor가 바로 그걸 가능하게 합니다.

## 핵심 기능

### 1. 자가 치유 에이전트(Self-Healing Agents)

RPA의 가장 큰 골칫거리는 유지보수예요. UI가 조금만 바뀌어도 스크립트가 깨지고, 매번 수정해야 합니다.

Minicor의 자가 치유 에이전트는 이 문제를 근본적으로 해결합니다. UI가 변경되거나 예상치 못한 대화창이 나타나면 에이전트가 그걸 감지하고 자동으로 적응해서 계속 진행합니다. 리플렉션 에이전트가 매 액션마다 화면에 실제로 나타난 것과 비교해서 검증하고, 워크플로우가 깨지기 전에 자가 수정합니다.

더는 벤더가 업데이트할 때마다 스크립트를 다시 만들 필요가 없다는 뜻이죠.

### 2. 어디든 배포 가능(Deploy Anywhere)

Windows VM, 브라우저, 온프레미스, 클라우드, Citrix 등 어디든 실행 가능한 컴퓨터 유즈 에이전트입니다. 단 하나의 API 호출로 실행을 트리거할 수 있고, 프로덕션에서 하루에 25,000명의 환자 데이터를 처리하는 아키텍처가 현재 운영 중입니다.

### 3. 기본 제공되는 관찰성(Built-in Observability)

모든 자동화 실행의 전체 비디오 녹화본을 재생할 수 있습니다. 실패 알림은 Slack으로 직접 받을 수 있고, 스크린샷과 전체 실행 컨텍스트로 디버깅하니까 추측할 필요가 없어요.

## 작동 원리

API 호출부터 Windows VM에서 실행되는 데스크톱 자동화까지, Minicor가 어떻게 동작하는지 보겠습니다.

먼저 AI 시스템이 Minicor를 호출합니다. 그러면 데스크톱 서비스가 VM에 연결되고, 워크플로우를 시각적으로 구축하고 모니터링할 수 있습니다. 마지막으로 여러 VM에 걸쳐 자동으로 확장됩니다.

단 하나의 API 호출이 전체 데스크톱 워크플로우를 시작합니다. 워크플로우 ID와 데이터만 보내면 구조화된 JSON으로 결과를 받습니다. SDK도 필요 없고, 브라우저 확장 프로그램도 필요 없어요.

```javascript
POST https://api.minicor.com/workflow/execute
{
  "workflow_id": "write_to_desktop_app",
  "data": {
    "record_id": "8472936",
    "fields": { "note": "Progress note content..." }
  }
}
```

응답은 이렇게 돌아옵니다.

```javascript
{
  "status": "success",
  "action": "note_written",
  "verified": true,
  "duration_s": 11.2,
  "data": {
    "record_id": "REC-847293",
    "written_at": "2026-03-03T14:30:53Z"
  }
}
```

## 자주 묻는 질문

### API를 쓰면 안 되나요?

EHR, ERP, DMS, PMS 같은 레거시 데스크톱 시스템 대부분은 쓰기 가능한 API를 갖고 있지 않습니다. 어떤 벤더는 적극적으로 서드파티 API 접근을 제한하고 있어요. 데이터를 읽거나 쓰는 유일한 방법은 인간처럼 데스크톱과 상호작용하는 것입니다. 이것이 바로 Minicor가 푸는 문제입니다.

### 그냥 사내에서 RPA를 만들면 안 되나요?

하나의 RPA를 만드는 건 어렵지 않습니다. 문제는 수백 개를 안정적으로 운영하는 것이에요. UI는 계속 바뀌고, 엣지 케이스는 계속 쌓입니다. 에러는 연쇄적으로 발생하죠. 규모가 커지면 아무리 작은 에러율도 치명적이 됩니다. 결국 엔지니어링 팀은 코어 제품을 개발하는 대신 자동화 유지보수에만 모든 시간을 쏟게 됩니다.

### 기존 RPA나 컴퓨터 유즈 모델과 뭐가 다른가요?

기존 RPA 벤더들은 UI가 바뀌면 깨지는 취약한 스크립트를 사용합니다. 컴퓨터 유즈 모델은 데모에서는 잘 작동하지만, 프로덕션에서 대규모로 운영하려면 자가 치유 기능과 완벽한 검증이 필수입니다. Minicor는 이 두 가지를 모두 제공하면서도 구조화된 응답과 엔드투엔드 검증으로 AI 시스템과 안정적으로 통합됩니다.
