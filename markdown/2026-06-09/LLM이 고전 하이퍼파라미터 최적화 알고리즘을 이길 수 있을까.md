## 핵심 개념

> [!abstract]
> LLM 기반 하이퍼파라미터 최적화를 CMA-ES, TPE 같은 고전 알고리즘과 비교 연구합니다. 순수 LLM은 고전 방식에 미치지 못하지만, Centaur라는 하이브리드 방식(고전 최적화기와 LLM 결합)이 가장 우수한 성능을 달성합니다.

## 상세 내용

- 고전 HPO 알고리즘(CMA-ES, TPE)이 LLM 기반 방식보다 일관되게 우수한 성능을 보이며, 특히 메모리 오버플로우 방지가 중요합니다.
- LLM은 상태 추적에 어려움을 겪지만 도메인 지식이 풍부하므로, 고전 최적화기의 내부 상태(평균값, 스텝 크기, 공분산)를 LLM과 공유하는 Centaur 하이브리드 방식이 가장 효과적입니다.
- 0.8B LLM을 사용한 Centaur도 모든 순수 고전/LLM 방식을 능가하는 결과를 달성합니다.

> [!tip] 왜 중요한가
> LLM을 단순 대체 수단이 아닌 고전 최적화 기법의 보완 도구로 활용할 수 있는 전략을 제시합니다.

## 전문 번역

# LLM이 고전적 하이퍼파라미터 최적화 알고리즘을 이길 수 있을까?

최근 arXiv에 공개된 논문을 소개하고 싶습니다. LLM이 머신러닝 모델의 하이퍼파라미터 튜닝에서 얼마나 효과적인지를 직접 비교한 연구인데요, 실무에서도 참고할 만한 인사이트가 많습니다.

## 연구의 배경

autoresearch라는 리포지토리를 활용한 이 연구는 재미있는 질문에서 출발합니다. LLM이 훈련 코드를 직접 수정하면서 하이퍼파라미터를 최적화할 수 있다면, 과연 고전적인 최적화 알고리즘들을 능가할 수 있을까요?

연구팀은 작은 규모의 언어 모델을 튜닝하면서 정해진 컴퓨팅 예산 내에서 이 두 방식을 직접 비교했습니다.

## 주요 발견

### 고전적 방법이 여전히 우위

고정된 탐색 공간에서는 CMA-ES와 TPE 같은 고전적 방법들이 LLM 기반 에이전트를 꾸준히 능가했습니다. 특히 메모리 부족으로 인한 실패를 피하는 것이 탐색의 다양성보다 더 중요하다는 점이 흥미롭습니다.

LLM에게 소스 코드를 직접 수정할 수 있는 권한을 주면 격차가 줄어들긴 합니다. 하지만 Claude Opus 4.6이나 Gemini 3.1 Pro Preview 같은 최신 모델을 사용해도 완전히 따라잡지 못했습니다.

### LLM의 약점

연구팀이 발견한 가장 큰 문제는 LLM이 여러 시도(trial)에 걸쳐 최적화 상태를 추적하는 데 어려움을 겪는다는 점입니다. 이전 결과를 기반으로 학습하고 개선하는 과정에서 성능이 떨어지는 거죠.

## 최선의 해결책: 하이브리드 접근

연구팀은 두 방식의 강점을 결합한 **Centaur**라는 하이브리드 방식을 제안합니다.

이 방식은 CMA-ES의 해석 가능한 내부 상태(평균 벡터, 스텝 크기, 공분산 행렬)를 LLM과 공유합니다. 즉, LLM이 고전적 최적화 알고리즘의 "생각 과정"을 볼 수 있게 한 것이죠.

### 성과

- **Centaur는 모든 실험에서 최고의 성능을 달성**했습니다.
- 놀랍게도 0.8B 규모의 작은 LLM만으로도 고전적 방법과 순수 LLM 방식 모두를 능가했습니다.
- 코드를 제약 없이 편집할 수 있게 하려면 더 큰 모델이 필요하지만, 고전적 방법과 경쟁하려면 이 정도면 충분합니다.

## 실무적 의의

이 연구 결과는 중요한 시사점을 줍니다. **LLM은 고전적 최적화 알고리즘을 대체하기보다는 보완하는 역할을 할 때 가장 효과적**이라는 거죠.

- 고전적 방법의 해석 가능성과 안정성
- LLM의 도메인 지식과 창의성

이 둘을 조합하면 둘 다 단독으로 사용했을 때보다 훨씬 나은 결과를 얻을 수 있습니다.

연구팀은 탐색 다양성, 모델 스케일링(0.8B부터 최신 모델까지), 그리고 Centaur에서 LLM이 제안하는 시도의 비율에 대해서도 상세히 분석했습니다. 코드와 인터랙티브 데모도 공개되어 있으니 직접 실험해볼 수 있습니다.
