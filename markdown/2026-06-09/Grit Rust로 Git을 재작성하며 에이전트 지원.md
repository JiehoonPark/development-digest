## 핵심 개념

> [!abstract]
> GitButler는 AI 코딩 에이전트가 Git 저장소를 안전하게 다룰 수 있도록 설계된 Git 대체 도구를 개발 중이며, Series A에서 $17M을 투자받았습니다. 안전한 검토, 롤백, 명확한 작업 경계를 통해 에이전트가 코드를 손상시키지 않도록 방지합니다.

## 상세 내용

- AI 에이전트가 안전하게 Git 작업을 수행할 수 있는 워크플로우 제공
- Series A 투자 $17M으로 소프트웨어 개발의 미래 인프라 구축
- CSS가 기존 JavaScript 라이브러리(Floating UI, GSAP 등)의 기능을 대체

> [!tip] 왜 중요한가
> AI 에이전트를 사용하는 개발 워크플로우에서 코드 안전성과 버전 관리가 중요해지고 있습니다.

## 전문 번역

# AI 에이전트를 안전하게 다루는 Git 워크플로우

코딩 AI가 여러분의 저장소를 엉망으로 만들까봐 걱정하시나요? 사실 진짜 문제는 코드 자체가 아닙니다. Git 히스토리가 복잡하게 얽혀버리는 것이 진짜 골칫거리거든요.

이 문제를 해결하려면 안전한 리뷰, 빠른 롤백, 명확한 작업 경계를 갖춘 워크플로우가 필요합니다. GitButler에서는 AI 에이전트가 자유롭게 작업할 수 있으면서도 여러분이 항상 제어권을 유지하는 방식을 제시하고 있어요.

---

# GitButler, Series A 펀딩으로 $17M 확보

GitButler가 Series A 투자를 완료했습니다. 다음 세대 소프트웨어 개발 인프라를 만들기 위해서요.

코드를 어떻게 관리하고 협업할 것인가 하는 문제는 개발자들이 영원히 마주할 숙제입니다. GitButler는 Git의 다음 단계가 무엇인지를 생각하며, 그걸 현실로 만들기 위한 투자를 확보했습니다.

---

# CSS가 해낸 대반란

요즘 CSS는 정말 많이 변했습니다. Floating UI, GSAP ScrollTrigger, Framer Motion, react-select 같은 라이브러리들이 JavaScript로 처리해야 했던 것들을 이제 CSS만으로 할 수 있거든요.

이런 변화가 정확히 얼마나 큰지, 왜 이 라이브러리들이 크기만 크지 않아서 불편했는지, 그리고 아직도 CSS가 풀어야 할 숙제가 뭔지를 자세히 살펴봅시다.
