## 전문 번역

# 검증 가능한 보상으로 강화학습하기: PopuLoRA의 동작 원리

대규모 언어모델(LLM)은 사전학습만으로는 고도의 추론 능력을 안정적으로 갖추지 못합니다. 검증 가능한 보상을 통한 강화학습(RLVR)은 이 문제를 해결하는 방법인데요, 모델이 자동으로 검증 가능한 작업을 반복 시도하고, 성공할 때마다 보상을 받는 방식입니다. 코드가 단위 테스트를 통과하거나, 수학 문제를 풀거나, 결정론적 검증자 아래에서 성공하는 경우처럼 정확성을 객관적으로 확인할 수 있을 때 보상이 명확해집니다.

## 검증 가능한 작업 공급의 어려움

RLVR이 효과적으로 작동하려면 대규모의 검증 가능한 작업이 계속 필요합니다. 단순히 해결 가능한 작업뿐만 아니라, 모델의 현재 능력 수준에 맞는 난이도의 작업들이 필요하고, 학습이 계속 의미 있으려면 충분히 다양한 작업들이어야 하거든요.

현재 대부분의 시스템은 학습 전에 미리 정해진, 수작업으로 선별한 작업 분포에 의존합니다. 하지만 이런 방식은 시간이 지나면서 작업이 너무 쉬워지거나, 범위가 좁아지거나, 모델의 성장 속도를 따라가지 못하는 문제가 생깁니다.

## 자가 대전(Self-Play)과 그 한계

손으로 만든 생성기로 합성 작업을 만드는 방법도 있고, 이미 검증 가능한 학습 데이터를 확장하는 일반적인 방식이 되었습니다. 다만 고정된 생성기는 여전히 대부분의 커리큘럼을 미리 정의해버립니다.

더 나은 방법으로 자가 대전이 있습니다. 모델이 직접 작업을 만들고, 시도하고, 검증자의 피드백을 받으면서 학습하는 방식이거든요. 우리는 여기서 더 나아가 작업 생성이 모델이 학습하면서 실시간으로 적응하는 온라인 커리큘럼이 될 수 있는지 살펴봤습니다.

**PopuLoRA**는 이 방향으로의 첫 시도입니다. 교사 역할의 LLM 어댑터와 학생 역할의 LLM 어댑터로 이루어진 공진화하는 개체군을 학습시킵니다. 교사는 검증 가능한 작업을 생성하고, 학생은 풀려고 시도하며, 검증자가 보상을 제공합니다. 학생이 성장하면서 교사는 더 어렵고 다양한 작업을 찾아야 하고, 교사가 다양해지면서 학생은 함께 변하는 커리큘럼을 경험하게 됩니다.

## 단일 에이전트 자가 대전의 붕괴

자가 대전을 실제로 구현해보면, 한 모델이 스스로 작업을 제시하고 풀 때 흥미로운 일이 일어납니다. 우리가 연구한 코드 추론 분야에서 모델은 세 가지 유형의 작업을 생성합니다:

- **code_o**: 프로그램의 출력을 예측
- **code_i**: 목표 출력을 만드는 입력값 찾기
- **code_f**: 입출력 예제로부터 빠진 함수 완성

샌드박스 Python 실행 환경은 파싱되고, 실행되며, 결정론적으로 동작하는 프로그램만 허용합니다.

실제로 실험해보니 흥미로운 현상이 나타났습니다. 작업 생성이 자신의 해결 능력에 수렴하면서, 해결률이 100%에 가까워지고, 커리큘럼이 점점 더 간단한 프로그램으로 붕괴되는 것입니다. 보상 곡선은 건강해 보이지만, 실제로는 모델을 더 이상 밀어붙이지 못하는 상황이 된 거네요.

생성된 프로그램들을 분석하면 이 붕괴가 명확하게 보입니다. 추상 구문 트리(AST) 깊이, 순환 복잡도, 코드 라인 수, 변수 개수 모두 감소 추세를 보이거든요.

## PopuLoRA: 교사-학생 개체군의 공진화

PopuLoRA는 작업 생성과 작업 해결을 분리합니다. 하나의 자가 대전 모델 대신, 특화된 작업 생성자인 **교사(T)**와 해결자인 **학생(S)**으로 이루어진 공진화하는 개체군을 학습시킵니다.

각 단계마다 교사와 학생이 짝지어집니다. 교사가 작업을 제시하고, 짝지어진 학생이 검증자의 감시 아래 풀려고 시도하는 것이죠. 교사는 유효하지만 짝지어진 학생이 풀지 못한 작업에 대해 보상을 받습니다. 따라서 난이도는 교사 자신의 능력이 아니라, 다른 모델을 기준으로 측정되는 셈입니다.

이렇게 하면 난이도가 개체군 간의 신호가 됩니다. 교사와 학생은 서로 다른 모델이고, 두 부분 집단 모두 학습 내내 계속 변합니다. 그 결과 단일 모델의 지역적인 개념이 아니라, 교차 평가에 의해 주도되는 자동 커리큘럼이 만들어집니다.

## LoRA 어댑터로 효율적인 구현

개체군들은 고정된 기본 모델 위의 LoRA 어댑터로 구현됩니다. 각 구성원은 전체 모델 복사본이 아니라 작은 저순위 업데이트만 학습하므로, 지배적인 기본 모델 계산을 교사와 학생 간에 공유할 수 있습니다.

이 방식은 단일 머신에서 개체군 학습을 가능하게 합니다. 메모리는 어댑터 가중치의 합으로만 스케일되고, 다중 LoRA 추론은 배치 요청을 올바른 어댑터로 라우팅하면서 기본 모델을 메모리에서 계속 유지할 수 있습니다. 4명의 교사와 4명의 학생(4T+4S) 설정에서도 8개의 어댑터를 학습시키는데 단지 1.31배의 벽시계 시간만 필요합니다.

## PopuLoRA의 5단계 절차

PopuLoRA의 각 단계는 다섯 개의 페이즈로 구성됩니다.

먼저 교사와 학생을 TrueSkill 등급을 기반으로 한 우선순위 허구적 자가 대전으로 짝짓습니다. 이렇게 하면 어느 한쪽도 압도하지 않는 거의 균형 잡힌 매칭에 학습을 집중시킬 수 있습니다.

그 다음 각 교사가 code_i, code_o, code_f 작업의 배치를 생성합니다. Python 검증자는 학생이 보기도 전에 유효하지 않거나 결정론적이지 않은 프로그램들을 걸러냅니다. 짝지어진 학생은 유효한 작업들에 여러 번의 확률적 롤아웃으로 시도하고, 검증자는 각 작업을 푸는 시도의 비율을 기록합니다.

그 후 교사와 학생은 정책 기울기 강화학습으로 함께 업데이트됩니다. 학생은 검증자의 신호를 받고, 교사는 학생이 풀지 못했지만 유효한 작업에 대해 보상을 받습니다. 이 과정을 통해 두 개체군이 함께 진화하면서, 모델의 성장에 맞춰 커리큘럼이 지속적으로 변합니다.
