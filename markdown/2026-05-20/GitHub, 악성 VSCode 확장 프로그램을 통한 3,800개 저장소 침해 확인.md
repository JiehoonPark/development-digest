## 핵심 개념

> [!abstract]
> GitHub의 직원이 설치한 악성 VSCode 확장 프로그램으로 인해 약 3,800개의 내부 저장소가 침해되었습니다. GitHub은 VSCode Marketplace에서 해당 확장 프로그램을 제거했으며 침해된 장치를 격리했습니다. 해커 그룹 TeamPCP가 약 4,000개의 저장소에 대한 접근을 주장하며 최소 $50,000의 신원금을 요구했으나, GitHub은 저장소 외부의 고객 데이터는 영향받지 않았다고 발표했습니다.

## 상세 내용

- 침해 규모: GitHub 내부 저장소 약 3,800개가 악성 VSCode 확장 프로그램을 통해 침해됨. TeamPCP 해커 그룹이 약 4,000개 저장소의 접근 주장 및 최소 $50,000의 신원금 요구
- 침해 경로: GitHub 직원이 VSCode Marketplace에서 다운로드한 악의적으로 조작된(trojanized) 확장 프로그램 설치. GitHub은 악성 버전을 Marketplace에서 제거하고 침해된 엔드포인트 격리
- 고객 영향 범위 제한: GitHub의 현재 평가로는 저장소 내부 데이터만 유출되었으며, 저장소 외부에 저장된 고객 데이터는 영향받지 않은 상태. 다만 전체 조사 결과는 계속 진행 중
- TeamPCP의 이력: 과거에 GitHub, PyPI, NPM, Docker 등 개발자 플랫폼을 대상으로 한 대규모 공급망 공격 연계. 최근 'Mini Shai-Hulud' 공급망 캠페인에도 관여(OpenAI 직원 2명에게도 영향)
- VSCode 확장 프로그램의 반복되는 보안 문제: 지난 수년간 Marketplace에서 수백만 건의 설치로 개발자 자격증명 탈취 시도. 2024년 9백만 건 설치 악성 확장 제거, 2024년 10개의 암호화폐 채굴 악성 확장 적발, 2025년 1월 중국 서버로 데이터 유출하는 AI 코딩 어시스턴트 사칭 확장(150만 건 설치) 발견
- GitHub의 규모: 4백만 개 이상의 조직(Fortune 100의 90% 포함) 및 1억 8천만 이상의 개발자가 사용 중이며, 4억 2천만 개 이상의 코드 저장소 보유

> [!tip] 왜 중요한가
> VSCode Marketplace의 확장 프로그램을 설치할 때 공격자들이 정기적으로 악성 프로그램을 배포하므로, 개발자들은 확장 프로그램 선택 시 매우 신중해야 하며 조직 차원의 보안 정책 강화가 필요합니다.

## 전문 번역

# GitHub, 악성 VSCode 확장 프로그램으로 인한 3,800개 저장소 침해 확인

GitHub가 직원이 설치한 악성 VS Code 확장 프로그램으로 인해 내부 저장소 약 3,800개가 침해당했음을 공식 확인했습니다.

GitHub는 이미 이름이 밝혀지지 않은 트로이목마 확장 프로그램을 VS Code 마켓플레이스에서 제거했으며, 침해당한 기기를 격리했습니다.

GitHub 측은 "어제 악의적인 VS Code 확장 프로그램으로 인한 직원 기기 침해를 탐지하고 격리했습니다. 악성 확장 프로그램 버전을 제거하고 엔드포인트를 격리한 후 즉시 사건 대응을 시작했습니다"라고 발표했습니다.

이어 "현재까지의 조사에 따르면, 침해는 GitHub 내부 저장소에만 국한되었습니다. 공격자가 주장하는 약 3,800개 저장소라는 수치는 우리의 조사 결과와 대체로 일치합니다"라고 덧붙였습니다.

## 랜섬웨어 요구와 과거 유사 사건들

GitHub는 지난주 이 침해 사실을 BleepingComputer에 언급했고, 고객 데이터는 영향을 받지 않았다고 밝혔습니다.

현재까지 GitHub는 공격자를 특정하지 못했으나, TeamPCP 해킹 그룹이 화요일 Breached 사이버범죄 포럼에 GitHub 소스코드와 약 4,000개의 비공개 저장소에 접근했다며, 최소 5만 달러를 요구하고 있습니다.

"이건 순수한 랜섬웨어가 아닙니다. GitHub를 협박할 생각은 없고 한 명의 구매자만 있으면 우리 쪽에서 데이터를 삭제할 겁니다. 은퇴가 곧일 것 같으니 구매자가 없으면 무료로 공개하겠습니다. 관심 있으시면 아래 연락처로 제안을 보내세요. 5만 달러 이상의 제안만 받습니다"라고 해커들은 공개 메시지에 적었습니다.

TeamPCP는 과거 GitHub, PyPI, NPM, Docker 등 개발자 코드 플랫폼을 대상으로 한 대규모 공급망 공격과 연관되어 있습니다. 최근에는 OpenAI 직원 2명도 피해를 입힌 "Mini Shai-Hulud" 캠페인에도 관여했던 것으로 알려졌습니다.

## VS Code 확장 프로그램의 보안 문제

VS Code 확장 프로그램은 Microsoft의 코드 에디터에 기능을 추가하거나 도구를 통합하기 위해 공식 마켓플레이스에서 설치할 수 있는 플러그인입니다.

실은 이번이 악성 확장 프로그램이 마켓플레이스에서 발견된 첫 번째 사례가 아닙니다. 최근 몇 년간 수백만 건의 설치 수를 기록한 악성 확장 프로그램들이 개발자 자격증명과 민감한 정보를 탈취하는 데 사용되었죠.

예를 들어 지난해에는 900만 건이 설치된 VSCode 확장 프로그램들이 보안 위험으로 인해 제거되었습니다. 그 해 나중에는 정당한 개발 도구로 위장한 10개의 확장 프로그램이 XMRig 암호화폐 채굴기로 사용자를 감염시켰습니다.

더 최근에는 1월에 AI 기반 코딩 어시스턴트로 광고된 악성 확장 프로그램 2개가 150만 건의 설치 수를 기록하면서 침해된 개발자 시스템의 데이터를 중국의 서버로 유출했습니다.

한편 GitHub의 클라우드 기반 플랫폼은 현재 포춘 500대 기업의 90%를 포함해 400만 개 이상의 조직과 1억 8,000만 명이 넘는 개발자들이 사용하고 있으며, 4억 2,000만 개 이상의 코드 저장소가 호스팅되어 있습니다.
