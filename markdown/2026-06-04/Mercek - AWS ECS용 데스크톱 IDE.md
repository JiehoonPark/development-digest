## 전문 번역

# Mercek: Amazon ECS를 쉽게 관리하는 데스크톱 앱

Mercek은 Amazon ECS를 관리하기 위한 데스크톱 애플리케이션입니다. 이미 설치된 AWS 자격증명을 사용해서 모든 계정과 리전의 서비스를 한눈에 볼 수 있죠. AWS와 직접 통신하며 어떤 원격 분석 데이터도 수집하지 않습니다.

## 주요 기능

### 계정과 리전 통합 관리

Mercek은 `~/.aws` 프로필을 읽어서 SSO, assume-role, MFA 설정까지 모두 인식합니다. 작업할 계정과 리전을 선택하면 클러스터, 서비스, 태스크가 하나의 트리 구조로 표시돼요. 선택하지 않은 계정은 절대 읽지 않습니다.

### 배포 상태 모니터링

현재 롤아웃 상태, 배포 서킷브레이커, 실패한 태스크 개수를 한눈에 확인할 수 있습니다. 배포가 잘못되면 같은 화면에서 이전 태스크 정의로 롤백할 수도 있죠. 다른 환경의 같은 서비스와 비교하는 것도 가능합니다.

### 성능 지표

CPU와 메모리는 Container Insights에서 가져오고, Insights가 꺼져 있으면 AWS/ECS 메트릭으로 대체됩니다. 로드밸런싱되는 서비스라면 요청 수, 레이턴시, 5xx 에러도 같은 화면에서 봅니다. 1시간부터 7일까지 시간 범위를 선택하면 차트에 배포 시점이 표시되어 배포와 롤아웃이 정렬됩니다.

### 비용 추정

Fargate 서비스의 월별 예상 비용을 요청된 CPU, 메모리, 실행 중인 태스크 수로 계산합니다. 피크 사용량과 비교해서 과도하게 프로비저닝됐는지 과소 프로비저닝됐는지 알려줍니다.

### 로그와 쉘 접근

CloudWatch 로그를 하단 패널에서 스트리밍할 수 있습니다. 모든 태스크를 한 번에 보거나 특정 태스크만 볼 수 있고, 텍스트 필터, 로그 레벨 강조 표시, 복사 및 다운로드 기능이 있어요. 필요하면 실행 중인 컨테이너에 대화형 셸로 진입할 수도 있습니다.

태스크의 전체 환경 변수도 읽을 수 있는데, 시크릿은 값 대신 ARN으로 표시됩니다. 네트워크 정보(ENI, IP 주소, 보안 그룹, 서브넷, VPC)도 확인할 수 있죠.

### 의존성 그래프

Mercek은 AWS에 이미 있는 데이터로 의존성 그래프를 구축합니다. 인터넷에서 로드밸런서를 거쳐 각 서비스까지의 경로를 지도화하고, 태스크 정의의 환경 변수에서 서비스 간 연결을 추론하죠.

### AI 에이전트 통합

Claude Code 같은 코딩 에이전트를 Agent Client Protocol로 연결할 수 있습니다. 에이전트는 ECS 상태를 읽고 설명하며 앱의 화면을 열 수 있습니다. 다만 AWS의 아무것도 변경할 수 없어요. 변경을 제안하면 Mercek이 그것을 diff로 보여주고 승인을 기다립니다.

### 자동 모니터링

Mercek이 열려 있는 동안 서비스를 백그라운드에서 검사하다가 뭔가 이상하면 알림을 띄웁니다. 그 알림을 에이전트에게 넘겨서 조사하게 할 수 있죠. 별도의 설정은 필요 없습니다.

## 그 외 편리한 기능들

- **⌘K로 빠르게 열기**: 클러스터, 서비스, 태스크를 키보드로 열 수 있습니다.
- **사전 확인**: 스케일링, 배포, 중지 시 변경 사항을 미리 표시합니다.
- **대상 상태 확인**: 서비스별 헬시, 언헬시, 드레이닝 대상을 봅니다.
- **Auto Scaling 통합**: 확장 대상, 정책, 용량 제공자를 한곳에 모았습니다.
- **환경 간 비교**: 다른 계정이나 리전의 같은 서비스와 비교합니다.
- **테이블 커스터마이징**: 열 설정을 저장하고 CSV로 내보낼 수 있습니다.

## 보안과 프라이버시

### 데스크톱 앱의 이점

Mercek은 호스팅 서비스가 아니라 데스크톱 앱입니다. 당신의 컴퓨터에서 이미 가진 자격증명으로 AWS와 연결되므로 계정에 추가 접근권을 줄 시스템이 없습니다.

당신의 머신에서 AWS를 호출합니다. 데이터를 보는 백엔드가 없죠. 원격 분석도 없고 외부로 전화하지도 않습니다. AWS로만 연결됩니다.

### 자격증명 관리

자격증명은 기존 체인에서 가져옵니다. 해결된 시크릿은 ARN으로 표시되며 디스크에 기록되지 않습니다.

Mercek은 AWS CLI와 같은 `~/.aws` 프로필과 SSO 세션을 사용합니다. 계정을 만들 필요도 없고, 중간에 서버도 없으며, 자격증명이나 해결된 시크릿을 디스크에 쓰지 않습니다.

### 변경 승인

읽기는 자동이지만, 쓰기는 오직 당신이 요청할 때만 발생합니다. 스케일링, 배포, 태스크 중지, 서비스 업데이트 같은 모든 쓰기 작업은 먼저 diff를 보여주고 당신의 승인을 기다립니다.

### AI 에이전트의 제한

Claude Code 같은 코딩 에이전트를 연결할 수 있습니다. ECS 상태를 읽고 설명하며 앱의 화면을 열 수 있죠. 하지만 AWS의 아무것도 변경할 수 없습니다. 변경을 제안하면 손으로 하는 것과 같은 diff 대화상자가 열려서 당신이 적용 여부를 결정합니다.

### 데이터 전송 없음

원격 분석이 없고 분석도 없습니다. AWS로만 연결됩니다.

## 설치와 운영 체제

Mercek은 macOS에서 실행됩니다(Apple Silicon과 Intel 모두). 다운로드하고 열면 AWS 프로필을 선택하는 것만으로 클러스터가 나타납니다.

Mercek은 오픈 소스입니다. GitHub에서 확인할 수 있죠.

Linux와 Windows 지원은 계획 중입니다.
