## 핵심 개념

> [!abstract]
> WebAssembly 기반의 FFmpeg를 브라우저에서 직접 실행하여 서버 업로드 없이 로컬에서 비디오 처리를 수행하는 PWA 앱입니다. 30+ 가지 비디오 작업을 지원하며 완전 오프라인 동작이 가능합니다.

## 상세 내용

- 로컬 처리로 개인정보 보호: 클라우드 업로드 불필요, 모든 데이터 기기 내 처리
- 30+ 비디오 작업 지원: GIF 생성, 형식 변환, 압축, 트리밍, 이펙트, 오디오 추출 등
- 오프라인-퍼스트 PWA: 첫 사용 후 완전 오프라인 동작, 네이티브 앱처럼 설치 가능

> [!tip] 왜 중요한가
> 개발자는 WASM을 통해 복잡한 데스크톱 애플리케이션을 웹에서 운영할 수 있으며, 브라우저 기반 멀티미디어 처리의 새로운 가능성을 볼 수 있습니다.

## 전문 번역

# ffmpeg webCLI: 브라우저에서 즐기는 비디오 편집기

ffmpeg.wasm 기반의 브라우저 비디오 편집 도구입니다. 업로드도 없고, 서버도 필요 없습니다. WebAssembly를 통해 모든 처리가 여러분의 브라우저에서 로컬로 이루어져요.

🚀 **라이브 앱**: https://tejaswigowda.com/ffmpeg-webCLI/

## 주요 기능

✅ **서버 업로드 불필요** — 모든 비디오 처리가 기기에서만 이루어집니다.
✅ **30가지 이상의 작업** — GIF 생성, 형식 변환, 압축, 트리밍, 효과, 필터 등  
✅ **오프라인 PWA** — 처음 접속 후 인터넷 연결 없이 완벽히 작동합니다.
✅ **화면 깨어있음 유지** — 비디오 처리 중 모든 기기에서 화면이 꺼지지 않습니다.
✅ **실시간 미리보기** — 출력 크기를 미리 보고 설정을 즉시 조정할 수 있습니다.
✅ **다양한 형식 지원** — MP4, WebM, MKV, MOV, AVI, GIF, MP3, AAC, WAV, OGG, FLAC, JPG, PNG
✅ **고급 기능** — Raw ffmpeg 명령어 접근, 자막 삽입, 연결, PiP(화면 속 화면), 오디오 믹싱
✅ **빠르고 반응성 있음** — Web Workers를 활용한 백그라운드 처리
✅ **프라이버시 우선** — 데이터 수집 없음. 모든 파일이 로컬에서만 처리됩니다.

## 사용 사례

### 🎞 GIF 만들기

어떤 비디오 클립이든 애니메이션 GIF로 변환할 수 있습니다. 프레임 레이트와 출력 너비를 설정하면 높이는 자동으로 조정되어 종횡비가 유지돼요. 2패스 팔레트 생성을 사용하므로 최고 품질의 색감을 얻을 수 있습니다.

### 🔄 비디오 형식 변환

비디오를 다른 컨테이너와 코덱으로 재인코딩할 수 있습니다.

- **MP4** — H.264 + AAC, 가장 널리 지원됨
- **WebM** — VP9 + Opus, 오픈 포맷으로 웹에 최적화됨 (MP4 대비 약 45% 더 작음)
- **MKV / MOV** — 다른 컨테이너의 H.264 + AAC
- **AVI** — 레거시 호환성

### 🗜 비디오 압축

해상도를 유지하면서 파일 크기를 줄입니다. CRF 슬라이더로 품질을 조정하고 (18 = 거의 손실 없음 → 51 = 최대 압축), 인코딩 프리셋을 선택해서 인코딩 속도와 압축 효율을 트레이드오프할 수 있어요. 설정을 조정하면 예상 파일 크기도 실시간으로 업데이트됩니다.

### ✂️ 비디오 자르기

타임라인 슬라이더로 시작점과 끝점을 설정한 후 작업을 실행합니다. 자르기는 다른 모든 작업 위에 적용되므로, 예를 들어 짧은 클립을 추출하고 압축하고 GIF로 변환하는 작업을 한 번에 처리할 수 있습니다.

### 📐 크기 조정 & 압축

한 번의 작업으로 출력 크기를 변경하고 압축합니다. 너비와 높이는 원본 비디오에서 자동으로 채워지며, 어느 한쪽만 수정하거나 비워두면 ffmpeg이 자동으로 종횡비를 유지합니다. 스케일 필터와 CRF 기반 H.264 인코딩을 결합한 방식이에요.

### 🎵 오디오 추출

어떤 비디오에서든 오디오 트랙을 따로 추출할 수 있습니다.

- **MP3** — 범용 재생
- **AAC** — 효율적인 손실 압축, 모바일에 최적
- **WAV** — 비압축 PCM
- **OGG Vorbis** — 오픈 손실 형식
- **FLAC** — 손실 없는 압축

### 🔇 음소거

오디오 스트림을 완전히 제거합니다. 원본 비디오는 유지되지만 오디오 트랙이 없는 형태로 출력되므로, 무음 루프나 소셜 미디어 클립, 또는 나중에 오디오를 교체하기 전 처리할 때 유용합니다.

### ⚡ 재생 속도 변경

재생 속도를 조절합니다 (0.25배 – 4배). 비디오의 PTS와 atempo 오디오 필터 체인이 함께 조정되므로 오디오 음정과 싱크가 유지돼요. 배속이 0.5~2.0 범위 밖이면 여러 필터 단계를 자동으로 연결합니다.

### 🔄 회전 / 뒤집기

다시 업로드할 필요 없이 방향을 수정하거나 미러 효과를 만들 수 있습니다. 90도 시계 방향, 90도 반시계 방향, 180도, 좌우 뒤집기, 상하 뒤집기, 또는 양쪽 축 모두 뒤집기를 선택할 수 있어요.

### ✂️ 자르기

프레임을 특정 영역으로 자릅니다. X/Y 오프셋과 너비/높이는 원본 비디오 크기에서 자동으로 채워지므로, 처음부터 입력하는 대신 바로 값을 조정할 수 있습니다.

### 🖼 썸네일 추출

비디오의 어느 지점에서든 단일 프레임을 추출해 JPEG 또는 PNG 이미지로 저장합니다. 타임스탬프 필드는 로드된 클립의 중간 지점으로 미리 설정됩니다.

### ⏪ 역재생

ffmpeg의 reverse + areverse 필터를 사용해 비디오와 오디오를 역순으로 재생합니다.

### 🌅 페이드 인 / 아웃

부드러운 페이드 인, 페이드 아웃, 또는 둘 다를 추가합니다. 각 방향의 지속 시간을 초 단위로 독립적으로 설정할 수 있으며, 이 필터는 자르기 작업 이후에 적용됩니다.

### 🎨 명도 / 명암 / 채도 조정

eq 필터로 클립의 표현을 미세 조정합니다. 3개의 슬라이더로 명도 (−1 → 1), 명암 (0 → 2), 채도 (0 → 3)를 제어할 수 있어요. 그레이스케일 체크박스를 활성화하면 채도가 0으로 고정되어 즉시 흑백 출력을 얻을 수 있습니다.

### 🚫 메타데이터 제거

GPS 좌표, 카메라 제조사/모델, 생성 시간 및 다른 모든 태그 같은 내장된 메타데이터를 제거한 후 파일을 공유합니다. 재인코딩 중 `-map_metadata -1`을 사용합니다.

### 📝 자막 삽입

.srt, .vtt, 또는 .ass 자막 파일을 비디오에 소프트 자막 트랙으로 먹스(mux)합니다. 영상을 다시 인코딩할 필요 없이 모든 미디어 플레이어(VLC, 브라우저 등)에서 토글로 켜고 끌 수 있어요. 출력 형식 옵션:

- **MP4** — mov_text로 인코딩된 자막 스트림
- **MKV** — 자막 스트림이 기본적으로 복사됨 (ASS/SSA 스타일 유지)

비디오와 오디오는 스트림 복사되므로 (품질 손실 없음, 거의 즉시 완료) 효율적입니다. 자막을 영상에 직접 번인하려면 libass가 활성화된 ffmpeg 빌드가 필요한데, 표준 WebAssembly 코어에서는 불가능합니다.

### 🔊 볼륨

어떤 비디오의 오디오 레벨을 높이거나 낮춥니다. 단일 슬라이더로 볼륨 배수를 설정합니다 (0 = 무음, 1.0 = 변경 없음, 최대 4배).
