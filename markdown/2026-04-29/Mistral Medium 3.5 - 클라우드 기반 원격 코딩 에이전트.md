## 핵심 개념

> [!abstract]
> Mistral은 클라우드에서 비동기로 실행되는 원격 코딩 에이전트를 제공하는 Mistral Medium 3.5 모델을 공개했다. 이 128B 모델은 4개 GPU에서 자체 호스팅 가능하며, Vibe CLI와 Le Chat에서 장시간 코딩 작업을 병렬로 처리할 수 있다.

## 상세 내용

- Mistral Medium 3.5는 명령어 따르기, 추론, 코딩을 하나로 통합한 플래그십 모델로, SWE-Bench Verified에서 77.6% 성능 달성
- Vibe 원격 에이전트는 CLI와 Le Chat에서 클라우드 기반 코딩 세션을 비동기로 실행하며, 로컬 세션을 클라우드로 전환 가능
- Le Chat의 새로운 Work 모드는 다중 단계 작업(연구, 분석, 도구 연동)을 자동 실행하고 완료 후 PR 생성 가능

> [!tip] 왜 중요한가
> 개발자는 더 이상 코딩 에이전트 작업을 대기할 필요 없이 병렬로 여러 작업을 클라우드에서 실행하고, 리뷰 단계에서만 개입하여 생산성을 크게 향상시킬 수 있다.

## 전문 번역

# Mistral Medium 3.5와 클라우드 기반 코딩 에이전트가 출시됐습니다

코딩 에이전트는 지금까지 주로 개인 노트북에서만 동작했는데, 이제 클라우드로 옮겨갑니다. 클라우드에서는 독립적으로 실행되고 병렬 처리도 가능하며, 작업이 완료되면 알림을 받을 수 있습니다. Mistral Vibe CLI에서 시작하거나 Le Chat에서 직접 실행하면, 대화를 벗어나지 않고도 코딩 작업을 offload할 수 있죠.

이 모든 게 가능한 건 공개 프리뷰 중인 **Mistral Medium 3.5** 덕분입니다. Mistral Vibe와 Le Chat의 새로운 기본 모델이 되었으며, 길고 복잡한 코딩 및 생산성 관련 작업에 특화돼 있습니다. Le Chat의 새로운 Work 모드(프리뷰)는 여기서 한 발 더 나아가, 조사, 분석, 다중 도구 연동 같은 복잡한 멀티스텝 작업을 처리할 수 있는 강력한 에이전트를 제공합니다.

## 주요 특징

- **Mistral Medium 3.5**: 명령어 따르기, 추론, 코딩을 모두 포함하는 새로운 플래그십 모델 (128B 밀집형). 수정 MIT 라이선스 하에 오픈 웨이트로 공개됩니다.
- **현실적인 성능**: GPU 4개 정도면 자체 호스팅도 가능한 크기입니다.
- **Mistral Vibe 원격 에이전트**: 클라우드에서 비동기 코딩 세션을 실행합니다. CLI나 Le Chat에서 시작할 수 있고, 로컬 CLI 세션을 클라우드로 '텔레포트'할 수도 있습니다.
- **Le Chat에서 코딩 작업 시작**: 세션이 동일한 원격 런타임에서 실행되고, 당신이 자리를 떠나도 계속 진행됩니다.
- **Le Chat의 Work 모드**: Mistral Medium 3.5 기반의 새로운 에이전트로, 멀티스텝 작업을 처리하고 도구를 병렬로 호출해 완료합니다.

## Mistral Medium 3.5

Mistral Medium 3.5는 우리의 첫 번째 플래그십 통합 모델이며, 공개 프리뷰 중입니다. 256k 컨텍스트 윈도우를 가진 128B 밀집형 모델로, 단일 가중치 세트로 명령어 따르기, 추론, 코딩을 모두 처리합니다. 현실적인 사용 환경에서 뛰어난 성능을 발휘하며, GPU 4개만으로도 자체 호스팅이 가능합니다.

추론 강도를 요청별로 설정할 수 있어서, 같은 모델이 빠른 채팅 응답부터 복잡한 에이전트 작업까지 유연하게 대응합니다. 이미지 크기와 종횡비 처리를 위해 비전 인코더도 처음부터 다시 훈련했습니다.

**벤치마크 성능:**
- SWE-Bench Verified에서 77.6% 달성 (Devstral 2, Qwen3.5 397B A17B 모델보다 우수)
- τ³-Telecom에서 91.4 점수 획득

이 모델은 장시간의 작업, 여러 도구 신뢰성 있게 호출하기, 다운스트림 코드가 소비할 수 있는 구조화된 출력 생성을 위해 만들어졌습니다. 덕분에 Vibe의 비동기 클라우드 에이전트를 실제로 배포할 수 있게 된 거죠.

Mistral Medium 3.5는 이제 Le Chat의 기본 모델이며, Vibe CLI의 코딩 에이전트에서도 Devstral 2를 대체합니다.

## Vibe 원격 에이전트

이제부터 코딩 세션은 당신이 자리를 떠난 상태에서도 길고 복잡한 작업을 처리합니다. 여러 세션을 병렬로 실행할 수 있으니, 더 이상 에이전트의 매 스텝마다 당신이 병목이 되지 않습니다.

Mistral Vibe CLI나 Le Chat에서 클라우드 에이전트를 시작할 수 있습니다. 실행되는 동안 파일 diff, 도구 호출, 진행 상태, 질문 등을 통해 에이전트가 뭘 하는지 실시간으로 확인할 수 있습니다. 로컬 CLI 세션을 클라우드로 '텔레포트'할 수도 있는데, 이때 세션 히스토리, 작업 상태, 승인 내용이 모두 유지됩니다.

Vibe는 개발팀이 이미 사용하는 시스템들 사이에 위치하면서, 필요한 곳마다 사람이 개입할 수 있도록 설계됐습니다. GitHub (코드와 PR), Linear와 Jira (이슈), Sentry (사건), Slack이나 Teams (보고)와 연동됩니다.

각 코딩 세션은 격리된 샌드박스에서 실행되며, 광범위한 편집과 설치도 포함됩니다. 작업이 끝나면 에이전트가 GitHub에 PR을 열고 알림을 보내니, 당신은 모든 키스트로크 대신 최종 결과를 검토하면 됩니다.

개발자의 시간을 빼앗지만 판단력까지 빼앗지는 않는 대량의 잘 정의된 작업에 딱 맞습니다. 모듈 리팩토링, 테스트 생성, 의존성 업그레이드, CI 조사, 버그 수정 같은 것들이죠.

우리는 Mistral Studio에서 Workflow를 활용해 Mistral Vibe를 Le Chat에 통합했습니다. 원래는 사내 코딩 환경을 위해 만들었고, 나중에 엔터프라이즈 고객들을 위해 확장했습니다. 오늘부터는 모든 사용자가 웹에서 코딩 작업을 시작할 수 있습니다. 로컬 터미널에 묶이지 않으니 여러 작업을 동시에 실행하는 것도 가능합니다.

Le Chat에서 직접 코딩 세션을 시작할 수 있습니다. 채팅에서 설명한 작업이 CLI, 웹과 같은 원격 런타임에서 실행되고, 나중에 완성된 브랜치나 draft PR로 돌아옵니다.

## Le Chat의 새로운 Work 모드 (프리뷰)

Work 모드는 Le Chat에서 복잡한 작업을 처리하기 위한 강력한 새로운 에이전트 모드입니다. Mistral Medium 3.5와 새로운 하네스로 구동되며, 에이전트가 어시스턴트 자체의 실행 백엔드가 되어 Le Chat이 읽고 쓸 수 있고, 여러 도구를 동시에 사용할 수 있으며, 당신이 요청한 일을 완료할 때까지 멀티스텝 프로젝트를 진행할 수 있습니다.

Work 모드가 지원하는 기능들:

- **크로스 도구 워크플로우**: 이메일, 메시지, 캘린더를 한 번에 확인하고, 참석자 정보, 최신 뉴스, 출처에서 가져온 토킹 포인트로 미팅을 준비합니다.
- **조사와 종합**: 웹, 내부 문서, 연동된 도구 전반에서 주제를 파고들어, 편집하거나 내보내거나 보낼 수 있는 구조화된 브리프나 보고서를 만듭니다.
