## 핵심 개념

> [!abstract]
> Ghostty 프로젝트의 개발자 Mitchell Hashimoto는 18년간 GitHub 사용자였지만, 지속적인 서비스 장애로 인한 작업 중단 때문에 Ghostty를 GitHub에서 다른 플랫폼으로 이전하기로 결정했다. 현재 마이그레이션 계획을 진행 중이며 GitHub에는 읽기 전용 미러를 유지할 예정이다.

## 상세 내용

- GitHub의 빈번한 장애(Actions, PR 검토 등)가 거의 매일 작업을 중단시키고 있다.
- 개발자는 GitHub을 사랑했지만 신뢰성 부족으로 다른 제공자(상용/오픈소스)로의 이전을 추진하고 있다.
- 마이그레이션은 점진적으로 진행되며, 기존 URL에는 읽기 전용 미러가 남아있을 예정이다.

> [!tip] 왜 중요한가
> GitHub의 신뢰성 문제를 실제 사례로 보여주며, 대규모 오픈소스 프로젝트 운영 시 플랫폼 선택의 중요성을 시사한다.

## 전문 번역

# Ghostty가 GitHub을 떠납니다

이 글을 쓰고 있자니 마음이 많이 아프지만, Ghostty 프로젝트가 GitHub을 떠나게 됩니다.

저는 GitHub 초기 사용자 중 한 명입니다. 2008년 2월 가입한 1299번 유저거든요. 그 이후로 매일 GitHub을 열었습니다. 18년 넘게 하루에도 여러 번씩입니다. 제 인생의 절반 이상을 GitHub 위에서 보낸 셈이죠. 물론 예외가 있겠지만, 1년 중 일주일 이상 손을 놓은 적은 없을 겁니다.

GitHub이 제 삶에 가장 큰 행복을 준 곳입니다. 저는 항상 시간을 내서 GitHub에 접속했습니다. 힘든 이별을 겪었을 때? 오픈소스 프로젝트에 몰두했고, 그곳은 GitHub이었습니다. 대학 시절 새벽 4시에 모두가 자고 있을 때? 한 커밋 더 올리고 싶었습니다. 신혼여행 중 아내가 아직 자고 있을 때? 네, GitHub을 했습니다. 저는 GitHub에 있을 때 가장 행복했고, 그렇게 있고 싶었습니다.

지루한 부분들도 포함해서 말이죠. 세상 사람들은 소셜 미디어를 무한정 스크롤합니다. 저는 이슈 트래커가 유행하기도 전부터 GitHub 이슈를 무한정 스크롤해왔습니다. 휴가 때도 공부하고 싶은 프로젝트들을 GitHub에 북마크해두곤 했는데, 소스코드뿐 아니라 오픈소스 프로세스나 다른 유지보수자들이 어려운 상황에 어떻게 대처하는지도 살펴봤습니다. 이게 이상해 보일지 모르지만, 저는 이런 것들을 정말 좋아합니다.

제 취미, 일, 열정이 모두 일치했고, 대부분의 인생 동안 그 모든 것이 인터넷의 한 공간 GitHub에서 함께 살아갈 수 있었습니다.

혹시 알고 계셨나요? 제 첫 성공적인 오픈소스 프로젝트인 Vagrant를 시작한 이유 중 큰 부분이 GitHub에 취업하고 싶었기 때문입니다. 이건 비밀이 아니고 여러 번 얘기했는데, 20살 때 처음 Vagrant에 대해 공개적으로 발표했을 때 농담 삼아 "이게 좋으면 GitHub이 날 채용해줄 수도 있지 않을까?"라고 했습니다.

GitHub은 제 꿈의 직장이었습니다. 결국 거기서 일하지는 못했지만 (그들 때문은 아닙니다), 거기는 정말 제가 있고 싶은 완벽한 곳이었습니다. 엔지니어들도 훌륭하고, 제품도 훌륭했으며, 저는 매일 그곳의 일을 생각하고 숨을 쉬었습니다. 지금도 그렇고, 이 18년 내내 계속 그래왔습니다. 한 인간이 어린아이에서 성인으로 자라나기에 충분한 시간을 GitHub 위에서 보낸 거죠.

최근 몇 개월간 저는 GitHub에 대해 공개적으로 비판해왔습니다. 심한 말도 했고, 화도 냈으며, 일하는 사람들의 기분을 상하게도 했습니다. 이렇게까지 나간 이유는 GitHub이 매일 저를 실망시키고 있기 때문이고, 이건 아주 개인적인 일입니다. 비이성적일 정도로 개인적입니다. 저는 GitHub을 사람이 어떤 것을 좋아해야 할 정도 이상으로 사랑합니다. 그래서 화가 납니다. 일하는 분들에게 상처를 드린 점 정말 사과합니다.

저는 오래전부터 이런 감정을 가지고 있었는데, 지난 한 달간 매일 GitHub 장애로 인해 일하는 데 지장이 생긴 날들을 기록해왔습니다. 거의 매일 X를 표시해야 했습니다. 이 글을 쓰는 오늘도 GitHub Actions 장애로 약 2시간 동안 PR 리뷰를 전혀 할 수 없었습니다. 매일 몇 시간씩 당신을 막아두는 플랫폼은 더 이상 진지한 작업을 할 수 있는 곳이 아닙니다.

더 이상 거기 있는 것이 즐겁지 않습니다. 저는 거기 있고 싶지만 GitHub은 저를 원하지 않는 것 같습니다. 저는 일하고 싶지만 GitHub은 제가 일하기를 원하지 않습니다. 저는 소프트웨어를 배포하고 싶지만 GitHub은 그것을 원하지 않습니다. 더 나아지길 바라지만, 동시에 코딩도 하고 싶습니다. 하지만 GitHub에서는 더 이상 코딩할 수 없습니다. 정말 죄송합니다. 18년 후, 저는 떠나야 합니다. 언젠가 다시 돌아올 수 있기를 바라지만, 그러려면 말과 약속이 아닌 실질적인 개선이 필요합니다.

## 다음 단계

앞으로 Ghostty 프로젝트가 어디로 이전할지에 대한 자세한 정보를 공유하겠습니다. 이미 계획이 있고 상용 제공자와 오픈소스 솔루션 제공자 모두와 협의 중입니다.

GitHub에 대한 모든 의존성을 제거하는 데 시간이 걸릴 것이고, 최대한 점진적으로 진행할 계획입니다. 현재 URL에서 읽기 전용 미러를 GitHub에 유지할 예정입니다.

제 개인 프로젝트와 다른 작업들은 당분간 GitHub에 남아 있을 겁니다. Ghostty이 저와 유지보수자들, 오픈소스 커뮤니티에 가장 큰 영향을 미치는 부분이기 때문에, 이 변화의 초점을 거기에 맞추고 있습니다. 그 이후로 어떻게 나아갈지는 두고 봐야 합니다.
