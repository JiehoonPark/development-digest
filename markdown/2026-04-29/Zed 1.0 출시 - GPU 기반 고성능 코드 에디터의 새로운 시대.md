## 핵심 개념

> [!abstract]
> Atom의 후속작으로 개발된 Zed 에디터가 1.0 버전을 정식 출시했다. Electron 기반의 웹 기술 대신 게임 엔진처럼 GPU 셰이더 중심으로 설계한 자체 UI 프레임워크 GPUI를 Rust로 구현하여 근본적으로 높은 성능을 달성했다. 현재 100만 줄 이상의 코드로 macOS, Windows, Linux를 지원하며, AI 네이티브 에디터로 여러 에이전트를 병렬 실행 가능하고 Claude Agent, Codex 등과 통합되는 Agent Client Protocol을 지원한다. 향후 CRDTs 기반 동기화 엔진 DeltaDB를 통해 인간과 AI 에이전트의 협업 환경을 구축할 계획이다.

## 상세 내용

- GPU 중심 아키텍처: 웹 기술(Electron/Chromium)의 성능 한계를 극복하기 위해 게임 개발처럼 GPU 셰이더 중심으로 설계했으며, 자체 UI 프레임워크 GPUI를 Rust로 개발하여 모든 계층의 스택을 소유하게 됨으로써 다른 빌린 기반에서는 불가능한 최적화가 가능해졌다.
- 완전한 언어 생태계 지원: 5년에 걸쳐 100만 줄 이상의 코드로 수십 개 언어, Git 통합, SSH 원격 접속, 디버거, Rainbow Brackets 등 개발자가 기대하는 현대적 에디터의 모든 기능을 3개 플랫폼(macOS, Windows, Linux)에서 구현했다.
- AI 네이티브 에디터: 단순히 AI를 덧붙인 것이 아니라 에디터의 기초부터 AI를 설계했으며, 여러 에이전트를 병렬 실행할 수 있고 키 입력 단위의 세밀한 예측을 Zed의 성능 수준으로 제공한다.
- Agent Client Protocol 개방: Claude Agent, Codex, OpenCode, Cursor 등 최고의 에이전트들을 통합할 수 있는 개방형 프로토콜을 구현하여 AI 에코시스템 확장성을 확보했다.
- Zed for Business 출시: 기업 고객을 위해 중앙화된 결제, 역할 기반 접근 제어(RBAC), 팀 관리 기능을 제공하는 엔터프라이즈 버전을 선보임으로써 조직 수준의 도입을 지원한다.
- DeltaDB를 통한 차세대 협업: CRDTs 기반의 동기화 엔진 DeltaDB를 개발 중이며, 문자 단위의 세밀한 변경 추적으로 여러 인간과 AI 에이전트가 동일한 코드베이스를 실시간으로 공유할 수 있는 환경을 목표로 하고 있다.
- 1.0의 의미: '완성'이 아닌 '임계점 도달'을 의미하며, 대부분의 개발자가 빠르게 적응할 수 있는 수준에 도달했고 성능은 여전히 업계 최고 수준임을 의미한다.
- 지속적인 발전: 매주 새 버전을 배포하는 철학은 유지하되, 1.0 이후도 계속 개선할 사항이 무궁무진하며 각 릴리즈가 공예(craft)를 한 단계 높인다는 철학을 유지한다.

> [!tip] 왜 중요한가
> 개발자는 고성능과 AI 협업 기능이 통합된 에디터의 출현으로 개발 경험이 근본적으로 향상될 가능성을 보게 되며, Zed의 아키텍처 선택(GPU 중심, 자체 스택 소유)이 AI 시대 소프트웨어 개발 도구의 미래 방향을 제시한다.

## 전문 번역

# Zed 1.0: 처음부터 다시 만든 에디터

Nathan Sobo | 2026년 4월 29일

## 새로운 기초 위에 세운 에디터

정말 나은 에디터를 만들려면 데스크톱 소프트웨어 개발 방식 자체를 바꿔야 한다는 걸 깨달았어요. 우리의 이전 에디터인 Atom은 Chromium을 기반으로 만들었는데, 그 과정에서 Electron 프레임워크까지 탄생했거든요. 이후 Electron은 VS Code의 기초가 됐고, 요즘엔 VS Code도 매주 새로운 AI 에디터로 포크되고 있죠.

웹 기술은 유연하고 빠르게 소프트웨어를 출시하는 길을 열어줬어요. 하지만 그만큼 한계도 있었습니다. 아무리 열심히 개선해도 기반이 되는 플랫폼 자체의 성능을 뛰어넘을 수 없었던 거예요.

그래서 처음부터 다시 시작했습니다. Zed는 웹페이지처럼 만들지 않고, 비디오 게임처럼 설계했어요. GPU에서 실행되는 셰이더에 데이터를 공급하는 방식으로 전체 애플리케이션을 구조화한 거죠. 이를 위해 Rust로 자체 UI 프레임워크인 GPUI를 만들어야 했습니다.

스택의 모든 계층을 직접 소유하면, 남의 기초 위에서 작업하는 사람들이 도달할 수 없는 영역까지 갈 수 있어요. 다만 처음부터 그 길이 쉽지 않을 거라는 건 알았습니다. 팀과 커뮤니티의 여러 해에 걸친 노력 덕분에, Zed는 이제 우리가 만들고자 했던 이상적인 도구에 가까워졌어요. 수많은 기능을 추가했으면서도 장인정신과 성능이라는 핵심 철학은 지켜냈고, 매일 수십만 명의 개발자가 Zed를 사용해서 소프트웨어를 만들고 있습니다. 그래서 우리는 1.0을 발표할 자신감을 갖게 됐어요.

## Zed 1.0 출시

### 1.0의 의미

현대의 에디터라면 수십 개 언어와 그들의 생태계를 지원해야 하고, 모든 스택에 걸친 수많은 변형과 예외 케이스들을 처리해야 해요. Git 통합, SSH 원격 접속, 디버거, 심지어 레인보우 괄호까지 말이죠. 5년에 걸쳐 Mac, Windows, Linux 전체에서 백만 줄이 넘는 코드로 이 모든 것들을 구현했습니다.

Zed는 또한 AI 네이티브 에디터입니다. 여러 에이전트를 동시에 실행할 수 있고, 예측 기능은 키 입력 단위로 다음 변경을 제안해줘요. 그것도 Zed만의 속도로요. Agent Client Protocol을 통해 Claude Agent, Codex, OpenCode, 그리고 최근의 Cursor를 포함해 계속 늘어나는 최고의 에이전트들과 연결할 수 있습니다. 우리는 AI를 나중에 덧붙이지 않고, 에디터의 기초부터 함께 설계했어요.

더불어 Zed for Business도 출시합니다. 회사들에서 엔지니어링 팀에 Zed를 배포하고 싶다는 요청을 받아왔거든요. 이제 중앙식 청구, 역할 기반 접근 제어, 팀 관리가 가능해집니다.

1.0은 "완성됐다"라는 뜻이 아니에요. "완벽하다"는 뜻도 아니고요. 대부분의 개발자가 빠르게 Zed에 적응할 수 있는 상태에 도달했다는 의미입니다. 1, 2년 전에 뭔가 부족해서 Zed를 써보다 멈춘 분들이라면, 1.0이 다시 시도해보라는 초대입니다. Zed는 지금까지 가장 강력하면서도 여전히 가장 빠르거든요.

## 앞으로 나아갈 방향

우리의 비전은 처음부터 지금까지 변하지 않았어요. 가장 빠르고 협업하기 좋은 코딩 환경을 만드는 것이죠. 다만 협업의 의미가 바뀌었습니다. 예전에는 사람들이 실시간으로 함께 작업하는 것을 의미했다면, 지금은 사람과 AI 에이전트가 같은 공간, 같은 코드에서 함께 일하는 것을 뜻합니다.

우리의 기초를 직접 만들었기에 1.0에 도달할 수 있었고, 다음 장을 열 수도 있게 됐어요. 현재 DeltaDB라는 동기화 엔진을 개발 중입니다. CRDT를 기반으로 문자 단위까지 모든 변경사항을 추적하는 기술이에요. DeltaDB를 통하면 여러 사람과 에이전트가 진화하는 코드베이스에 대해 단일하고 일관성 있는 관점을 공유할 수 있습니다. 팀원들을 에이전트와의 대화에 초대해서, 코드가 생성된 바로 그 맥락에서 에이전트 코드를 검토하고 개선할 수 있게 될 거예요.

이 비전은 기본 요소들을 깊이 있게 소유하고 있을 때만 가능합니다. 누군가 다른 회사의 브라우저 엔진 위에서는 이런 경험을 만들 수 없거든요.

## 마일스톤이지, 끝이 아니다

우리는 Zed의 천 개 이상의 버전을 출시했어요. 하지만 그 모든 버전의 번호가 0으로 시작했죠. 오늘부터 달라집니다.

앞으로도 매주 새 버전을 출시할 거예요. 만들어야 할 것들의 목록은 절대 끝나지 않을 테고, 그게 바로 우리가 원하는 방식입니다. 매번의 릴리스가 장인정신을 한 단계 더 앞으로 나아가게 해주니까요.
