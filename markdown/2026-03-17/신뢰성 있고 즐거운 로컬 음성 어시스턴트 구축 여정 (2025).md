## 핵심 개념

> [!abstract]
> Home Assistant의 로컬 음성 어시스턴트(Assist)를 llama.cpp와 완전 로컬 LLM으로 구축한 실제 사례를 공유한다. 다양한 GPU 하드웨어(3050~3090), 음성 인식/합성 모델(Parakeet V2, Kokoro TTS), 그리고 LLM 모델들의 성능 비교 테이블을 제공하며 구축 과정에서의 최적화 방법들을 상세히 기록했다.

## 상세 내용

- RTX 3090/RX 7900XTX로 20B-30B MoE 모델을 1-2초 응답시간으로 실행 가능하며 RTX 3050으로도 4B 모델은 3초 대 응답 가능
- Wyoming ONNX ASR (OpenVINO 최적화)로 0.3초 음성인식, Kokoro TTS로 고품질 음성 출력 달성
- LLM Conversation 통합으로 문맥 이해, 오인식 처리, 다중 디바이스 제어 등 고급 기능 구현

> [!tip] 왜 중요한가
> 개인들이 클라우드 의존 없이 로컬에서만 완전히 작동하는 음성 어시스턴트를 구축할 수 있는 구체적인 경로와 최적화 기법을 제시한다.

## 전문 번역

# 로컬 음성 어시스턴트 구축기: 신뢰할 수 있고 즐거운 경험을 찾아서

**작성자:** Nicolas Mowen  
**날짜:** 2025년 10월 27일

Home Assistant의 음성 어시스턴트 기능이 발전하는 과정을 지켜봐온 지 꽤 됩니다. 저희 집은 원래 Google Home과 Nest Mini를 썼는데, 이제는 완전히 로컬에서 돌아가는 음성 어시스턴트로 바꿨거든요. llama.cpp를 이용해서 말이죠. 이 글에서는 어떻게 현재 상태에 도달했는지, 어떤 결정을 내렸고, 왜 그게 제 상황에 최적이었는지를 공유하려고 합니다.

## 하드웨어 구성

### 음성 장비
- Home Assistant Voice Preview Satellite 1개
- Satellite1 Small Squircle 인클로저 2개  
- Pixel 7a (위성 허브 및 View Assist용)

### 음성 처리 서버
Beelink MiniPC(USB4 포함)에 다음을 연결해서 사용하고 있습니다.

**GPU별 성능 비교**

| GPU | 모델 클래스 | 응답 속도 | 참고 |
|-----|-----------|---------|------|
| RTX 3090 24GB | 20B-30B MoE, 9B Dense | 1-2초 | 최적의 모델을 빠르고 효율적으로 실행 |
| RX 7900XTX 24GB | 20B-30B MoE, 9B Dense | 1-2초 | 최적의 모델을 빠르고 효율적으로 실행 |
| RTX 5060Ti 16GB | 20B MoE, 9B Dense | 1.5-3초 | 3초 이내 응답으로 충분한 성능 |
| RX 9060XT 16GB | 20B MoE, 9B Dense | 1.5-4초 | 4초 이내 응답으로 충분한 성능 |
| RTX 3050 8GB | 4B Dense | 3초 | 소규모 모델에 적합 |

현대의 대부분의 GPU는 로컬 음성 어시스턴트에 사용할 수 있습니다. 다만 응답 속도와 능력에 대한 기대치에 따라 필요한 하드웨어가 달라질 뿐이죠.

## 모델 선택

여러 모델을 테스트했고, 아래는 다양한 기능과 성능을 테스트한 결과입니다. 모든 모델이 기본적인 도구 호출(tool calling)은 잘 지원합니다.

**테스트된 주요 기능:**
1. **다중 장치 제어** - "선풍기 켜고 불 꺼" 같은 명령 처리
2. **상황 이해** - "불을 켜"라고 했을 때 그 공간에 조명이 하나만 있으면 바로 켜고, 여럿이면 물어보기
3. **오청 처리** - "선풍기" 대신 "팬"이라고 들렸을 때도 올바르게 실행
4. **오인식 무시** - 오청된 텍스트에 영향받지 않고 정상 동작

권장 모델 조합으로는 **Unsloth Qwen 시리즈**들이 위 기능들을 잘 지원합니다.

## 소프트웨어 스택

### 음성 인식 (Speech to Text)

**Wyoming ONNX ASR** (추천)
- Nvidia Parakeet V2 모델 사용
- OpenVINO 기반 실행으로 CPU 추론을 약 0.3초로 최적화

**Rhasspy Faster Whisper**
- Nvidia Parakeet V2 모델 사용
- 직접 ONNX CPU 실행이라 상대적으로 느린 편

### 음성 생성 (Text to Speech)

**Kokoro TTS** (추천)
- 여러 음성과 톤을 섞어서 원하는 느낌으로 출력 가능
- 모든 텍스트를 잘 처리

**Piper TTS**
- CPU에서 실행
- 다양한 음성 선택 가능
- 화폐, 전화번호, 주소 처리에는 약한 편

### 모델 실행
**llama.cpp**를 최적 성능을 위해 권장합니다.

## Home Assistant 통합

- **LLM Conversation** - 기본 대화 기능 개선
- **LLM Intents** - 웹 검색, 장소 검색, 날씨 예보 같은 추가 도구 제공

## 마지막으로

이 글에서 제시한 구성이 "정답"이거나 누구나 따라야 할 방법은 아닙니다. 다만 이 과정에서 배운 것들이 많았고, 다른 분들도 기대치를 어느 정도로 잡아야 하고 어떤 함정이 있는지 알 수 있으면 좋겠다는 생각에 공유하는 것입니다.
