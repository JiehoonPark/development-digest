## 핵심 개념

> [!abstract]
> Mistral은 Lean 4 증명 보조기를 위해 설계된 첫 오픈소스 코드 에이전트인 Leanstral을 출시했다. 6B의 활성 파라미터만 사용하면서도 형식적 증명 작성과 수학적 개념 정의를 자동화하여 인간의 코드 검증 병목을 해결한다. Claude Opus 대비 92배 저렴한 비용($182 vs $1,650)으로 비슷한 수준의 성능을 제공하며, FLTEval이라는 새로운 평가 스위트로 현실적인 증명 공학 시나리오를 벤치마크한다. Apache 2.0 라이센스로 공개되며 MCP를 통한 확장을 지원한다.

## 상세 내용

- Leanstral은 6B 활성 파라미터의 경량 희소 아키텍처로 설계되어, GLM5-744B(점수 16.6), Kimi-K2.5-1T(점수 20.1) 같은 훨씬 큰 모델들을 단일 패스로 능가하는 효율성을 달성했다. Qwen3.5-397B는 4패스로 25.4를 기록할 때, Leanstral은 2패스(pass@2)로 26.3을 달성하며 계속 선형적으로 확장되어 pass@16에서 29.3에 도달한다.
- Claude 제품군 대비 경제성이 뛰어나다: Leanstral pass@2는 Sonnet(점수 23.7)을 2.6점 앞서면서 $36의 비용으로 실행 가능하며($549 vs), Claude Opus 4.6의 점수 39.6을 따라잡기 위해 pass@16에서 31.9를 기록할 때 $290의 비용만 소요된다. Opus의 $1,650은 Leanstral의 92배에 달한다.
- FLTEval이라는 새로운 평가 스위트는 Fermat의 마지막 정리(FLT) 프로젝트의 모든 PR에 대해 형식적 증명 완성과 새로운 수학적 개념 정의를 벤치마크한다. 경쟁 수학 문제 중심의 기존 평가를 넘어 실제 증명 공학 저장소에서의 유용성을 반영하도록 설계되었다.
- Lean 4.29.0-rc6에서 발생한 타입 별칭 관련 패턴 매칭 오류를 진단한 사례는 `def`를 `abbrev`로 변경하여 정의적 동치성 문제를 해결했다. Leanstral은 테스트 코드 작성으로 실패 환경을 재현하고 근본 원인(rigid definition이 rw 전술의 패턴 매칭 방해)을 정확히 파악한 후 해결책을 제시했다.
- Mistral Vibe 플랫폼을 통해 Apache 2.0 라이센스로 가중치, 에이전트 모드, 무료 API 엔드포인트를 공개하며, 학습 접근 방식을 설명하는 기술 보고서도 함께 제공한다. MCP를 통한 임의 확장을 지원하며 lean-lsp-mcp와의 호환성을 위해 특별히 훈련되었다.

> [!tip] 왜 중요한가
> 코드 생성 AI의 신뢰성이 중요한 고위험 도메인(수학, 미션 크리티컬 소프트웨어)에서 AI가 자신의 구현을 형식적으로 증명하도록 하여 수동 검증의 병목을 제거하고, 동시에 경제성과 효율성을 제공한다.

## 전문 번역

# AI 코딩 에이전트의 새로운 시대: Leanstral 공개

AI 모델이 코드 생성에 뛰어난 능력을 보여주고 있습니다. 그런데 수학 연구나 핵심 소프트웨어처럼 높은 신뢰도가 필요한 분야로 나아가면서 새로운 병목이 생겼어요. 바로 **인간의 검증**입니다. 머신이 생성한 코드를 일일이 검토하는 데 드는 시간과 전문성이 개발 속도를 크게 낮추고 있죠.

우리가 꿈꾸는 것은 다릅니다. 코딩 에이전트가 단순히 코드를 작성하는 것을 넘어, 그 구현이 정확한 명세를 만족하는지까지 **형식적으로 증명**할 수 있는 세상입니다. 사람들은 더 이상 머신이 만든 버그를 디버깅하지 않고, 원하는 결과물을 명확히 정의하기만 하면 됩니다. 오늘 우리는 그 첫 단계를 내디딥니다.

## Leanstral 소개

Lean 4를 위해 특별히 설계된 **오픈소스 코딩 에이전트 Leanstral**을 공개합니다.

Lean 4는 단순한 증명 보조 도구가 아닙니다. perfectoid spaces 같은 복잡한 수학 개념부터 Rust 코드의 속성 같은 소프트웨어 명세까지 표현할 수 있죠. 기존의 대형 범용 모델을 감싸기만 하거나 단일 수학 문제에만 집중하던 증명 시스템과 달리, Leanstral은 **실제 Lean 프로젝트에서 작동하도록 설계**되었습니다.

### 핵심 특징

**공개적이고 접근 가능합니다**
- Apache 2.0 라이선스로 가중치 공개
- Mistral의 에이전트 모드를 통한 무료 API 제공
- 학습 방식을 상세히 담은 기술 보고서 공개
- 경쟁 수학 문제를 넘어선 새로운 평가 세트 FLTEval 공개

**효율성과 성능을 모두 잡았습니다**
- 초경량 아키텍처(6B의 활성 파라미터)로 설계
- Lean을 완벽한 검증자로 활용한 병렬 추론
- 폐쇄형 경쟁사 대비 뛰어난 비용 효율성

**MCP를 통해 계속 확장됩니다**
- 임의의 MCP 지원
- lean-lsp-mcp와의 최적화된 연동

## 성능 평가

실제 증명 작업 시나리오를 반영하기 위해, 우리는 개별 수학 문제가 아닌 **FLT 프로젝트의 실제 PR에서 모든 증명을 완성하고 새로운 수학 개념을 정의하는 능력**을 평가했습니다.

### 오픈소스 모델과의 비교

Leanstral-120B-A6B는 훨씬 더 큰 오픈소스 경쟁자들을 압도합니다.

GLM5(744B)와 Kimi-K2.5(1T)는 각각 16.6과 20.1 점에서 정체되는 반면, Leanstral은 한 번의 패스(pass@1)만으로 이들을 능가합니다.

가장 강력한 오픈소스 경쟁자인 Qwen3.5(397B)도 4번의 패스가 필요해 25.4점에 도달하지만, Leanstral은 **절반의 비용(pass@2)으로 26.3점**을 달성하고 계속 선형으로 성능이 향상되어 pass@4에서 29.3점에 이릅니다.

### Claude 제품군과의 비교

Leanstral은 Claude를 이용할 때의 **비용 대비 성능에서 탁월한 대안**입니다.

| 모델 | 비용 ($) | 점수 |
|------|---------|------|
| Haiku | 184 | 23.0 |
| Sonnet | 549 | 23.7 |
| Opus | 1,650 | 39.6 |
| Leanstral (pass@1) | 18 | 21.9 |
| Leanstral (pass@2) | 36 | **26.3** |
| Leanstral (pass@4) | 72 | 29.3 |
| Leanstral (pass@8) | 145 | 31.0 |
| Leanstral (pass@16) | 290 | 31.9 |

Leanstral pass@2는 Sonnet보다 **2.6점 높으면서 비용은 15배 저렴**합니다($36 vs $549). pass@16 기준으로는 Sonnet을 8점이나 앞지르면서도 $290에 불과해, Claude Opus($1,650)에 비해 **92배 경제적**입니다.

## 실제 사례

### Stack Exchange 질문 해결: Lean 버전 변경 대응

새 Lean 버전이 출시되면 기존 코드를 마이그레이션하는 것이 악몽이 될 수 있습니다.

Proof Assistants Stack Exchange의 실제 질문을 Leanstral에 주었습니다. 질문은 Lean 4.29.0-rc6(학습 후 출시)에서 갑자기 컴파일이 안 되는 스크립트에 관한 것이었죠.

문제는 `def T2 := List Bool` 같은 간단한 타입 별칭과 연결된 패턴을 `rw` (rewrite) 전술이 더 이상 매칭하지 못한다는 것이었습니다.

Leanstral은 막힌 상황을 정면으로 돌파했습니다. 실패하는 환경을 재현하는 테스트 코드를 작성한 후, **정의적 동등성(definitional equality)** 문제를 정확히 진단했어요.

그 결과를 보니 명확했습니다. `def`는 명시적 언폴딩이 필요한 고정 정의이기 때문에 `rw` 전술이 필요한 기저 구조를 볼 수 없었던 것입니다.

해결책도 간단했습니다. `def`를 `abbrev`로 바꾸면 됩니다. `abbrev`는 투명한 별칭을 만들어서 원본 타입과 바로 정의적으로 동등하기 때문에, `rw` 전술이 다시 패턴을 완벽하게 매칭할 수 있습니다.

Leanstral은 문제를 완벽하게 해결했을 뿐 아니라, 사용자에게 그 논리까지 명확하게 설명해주었습니다.
