## 핵심 개념

> [!abstract]
> 10년간 Ubuntu 16.04에서 운영 중인 블로그를 Hetzner VPS의 FreeBSD로 마이그레이션한 경험을 공유하며, FreeBSD Jails와 ZFS 파일시스템의 장점을 소개한다. 비용 절감(월 $13에서 €6 미만), 보안 개선, 더 나은 성능을 달성했다.

## 상세 내용

- FreeBSD Jails를 활용한 컨테이너화로 Docker보다 경량이면서도 안정적인 애플리케이션 격리 환경 구성
- ZFS 파일시스템으로 서버 운영의 데이터 관리 및 안정성 개선
- 낡은 LTS 지원 종료 시스템(Ubuntu 16.04)에서 최신 인프라로의 이전으로 보안과 성능 동시 개선

> [!tip] 왜 중요한가
> DevOps 엔지니어에게 장기 운영 서버의 현대화 전략과 FreeBSD/Jails의 실무적 활용 사례를 제시한다.

## 전문 번역

# 10년 된 블로그를 FreeBSD로 이전하며 배운 것들

10년 넘게 Digital Ocean의 뉴욕 데이터센터에서 운영해온 VPS가 있었습니다. Ubuntu 16.04 LTS를 돌리고 있던 이 서버는 이미 5년 이상 지원이 끝난 상태였어요. 이제 정말 바꿀 시간이 됐다고 느꼈습니다.

여러 검토 끝에 Hetzner의 가상 머신으로 이전하기로 결정했습니다. 성능은 훨씬 좋으면서 가격은 절반 이하고, 무엇보다 저와 같은 나라에 있다는 게 큰 장점이었죠. 그리고 욕심을 내서 스택 전체를 FreeBSD로 옮기기로 했습니다.

이 글은 좀 길지만, FreeBSD Jails와 Bastille에 대한 흥미로운 소개, 그리고 사이트 로드 벤치마크까지 함께 살펴볼 거예요.

## 왜 바꿔야 했나

Ubuntu LTS는 지원이 끝나면 apt 패키지 저장소에서 제외됩니다. 그러면 더 이상 업데이트를 받을 수 없죠. 이렇게 오래된 시스템을 운영하는 건 보안 측면에서 큰 위험입니다. 어딘가의 봇들이 알려진 취약점을 가진 서버를 찾아다니면서 악성코드를 심을 수도 있으니까요.

다행히 저는 운 좋게도 그런 일을 겪지 않았습니다. 물론 서버에 특별히 소중한 데이터가 있지도 않았긴 했지만요. 다만 예전에 운영했던 WordPress 블로그 중 하나가 오래된 VPS에서 갑자기 카지노나 도박 관련 의심스러운 링크로 도배된 일이 있었어서, 그런 위험을 다시 겪기는 싫었습니다.

## 가격 비교: 정말 큰 차이가 났습니다

저는 이미 Hetzner VPS를 원격 개발 머신으로 쓰고 있었는데, 가격 대비 성능이 정말 좋았습니다. 어디서든 SSH로 접속할 수 있고 안정적이었거든요. 그래서 기존 서버와 Hetzner 스펙을 비교해보기로 했습니다.

### 기존 Digital Ocean 서버 (월 $13)
- RAM: 2GB
- vCPU: 1개
- 디스크: 50GB
- 월간 트래픽: 2TB
- 위치: 뉴욕
- OS: Ubuntu 16.04 x64

뉴욕 데이터센터라서 가격이 비쌌던 것 같습니다.

### Hetzner 최저가 옵션 (월 €3.56)
- RAM: 4GB
- vCPU: 2개
- 디스크: 40GB
- 월간 트래픽: 20TB
- 위치: 독일

커피 한 잔 값이네요. 메모리와 CPU는 2배인데 트래픽은 10배입니다.

### 제가 선택한 옵션 (월 €6 미만)
조금 더 성능 좋은 걸로 골랐습니다. 제 사이트 규모상 오버스펙일 수도 있지만, 이 정도면 충분하겠다고 생각했어요.

## 기존 설정은 어떻게 되어 있었나

여러 개의 사이트를 호스팅하고 있었는데, 모두 소규모 프로젝트들이었습니다. 이 블로그가 가장 인기 있는 사이트인데도 한 달에 수천 정도의 페이지뷰만 받았습니다. Hacker News에서 가끔 인기를 얻을 때를 제외하면 트래픽이 많지 않았어요. 결국 이 서버는 기본적으로 정적 사이트들만 서빙하고 있었습니다.

스택은 정말 단순했습니다.

- nginx 1.10.3으로 정적 콘텐츠만 서빙
- `/etc/nginx/sites-available`에 각 사이트별 설정 파일 관리
- Hugo 같은 정적 사이트 생성기와 LaTeX 스위트를 apt나 snap으로 설치
- 블로그 업데이트 프로세스:
  1. 로컬에서 글 작성
  2. 저장소에 commit & push
  3. 서버에 SSH 접속
  4. 저장소 업데이트 pull
  5. Hugo 실행

초반에는 이 VPS에서 테스트 코드도 돌리고 개발도 했었는데, 결국 쓰지 않는 오래된 소프트웨어들이 계속 쌓여만 갔습니다. 하지만 아주 잘 돌아갔습니다.

Linux 4.4였는데도요!

정말 잘 돌아가서, 저를 끄기 직전에 **업타임이 1491일**에 달했습니다. 거의 4년을 쉬지 않고 돌렸다는 뜻이네요!

## FreeBSD를 선택한 이유

솔직하게 말하자면, 새로운 것을 경험해보고 싶다는 욕심이 컸습니다. BSD 관련 글과 영상을 많이 읽고 봤고, 예전에 FreeBSD를 잠깐 써본 경험도 있었거든요. 실제로 써보면서 배워보고 싶었습니다.

FreeBSD는 보통 안정성, 통합 설계, 보안 그리고 **Jails** 때문에 칭찬받습니다.

### Jails가 뭔가요?

읽다가 Jails에 대해 알게 된 순간, 제가 정확히 뭘 하고 싶었는지 알 수 있었습니다.

Jails는 FreeBSD에 25년 이상 포함된 가상화/컨테이너화 기술입니다. Docker가 등장하기 훨씬 전부터 있었어요. 겉으로 보면 Docker 컨테이너처럼 작동합니다. 호스트 시스템에 접근할 수 없는 "미니 시스템"을 샌드박스 환경에서 실행할 수 있으니까요.

하지만 주요 차이점이 있습니다.

**Docker와 다른 컨테이너 솔루션**은 "프로그램 패키징"에 더 적합합니다. 일시적이고 불변적입니다. 외부에서 들어오는 데이터를 처리하지만, 컨테이너 내부는 항상 동일한 상태를 유지합니다.

**Jails는 진정한 서브시스템**입니다. 거의 미니 VM처럼 작동하면서도 실제로는 같은 커널을 공유하고 있죠.

더 나아가 FreeBSD의 파일시스템 **ZFS**도 정말 훌륭합니다. Linux 세계에서 온 사람이라면 Btrfs라는 새로운 파일시스템을 들어봤을 텐데, ZFS는 그보다 훨씬 오래됐으면서도 서버 환경에 정말 유용합니다.
