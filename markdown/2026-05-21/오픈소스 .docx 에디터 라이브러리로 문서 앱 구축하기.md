## 핵심 개념

> [!abstract]
> React, Vue, Nuxt를 지원하는 오픈소스 WYSIWYG .docx 에디터 라이브러리로, 표준 OOXML, 변경 추적, 실시간 협업 기능을 제공한다. AI 에이전트 통합이 가능하며 여러 프레임워크를 위한 어댑터와 플러그인 시스템을 갖추고 있다.

## 상세 내용

- React, Vue, Nuxt 3&4를 모두 지원하는 통합 라이브러리로 OOXML 파서, 레이아웃 엔진, ProseMirror 스키마 기반
- 변경 추적, 실시간 협업, 에이전트SDK 포함으로 현대적 문서 편집 기능 지원
- 다국어 지원(영어, 독일어, 폴란드어, 중국어 등)과 플러그인 아키텍처로 확장 가능

> [!tip] 왜 중요한가
> 개발자는 별도 구축 없이 프로덕션급 문서 편집 기능을 빠르게 통합할 수 있고, AI 에이전트와 함께 사용하는 새로운 문서 앱 개발이 가능하다.

## 전문 번역

# React와 Vue를 위한 오픈소스 WYSIWYG .docx 에디터

OOXML 표준을 따르면서 변경 사항 추적, 실시간 협업을 지원하는 .docx 에디터입니다. AI 에이전트도 준비되어 있죠. [라이브 데모](https://demo.docx-editor.dev) | [문서](https://docs.docx-editor.dev)

## 설치

세 가지 패키지 중 원하는 프레임워크에 맞는 걸 선택하세요.

**React:**
```bash
npm install @eigenpal/docx-editor-react
```

**Vue:**
```bash
npm install @eigenpal/docx-editor-vue
```

**Nuxt:**
```bash
npm install @eigenpal/nuxt-docx-editor
```

## 패키지 구성

| 패키지 | 설명 |
|--------|------|
| @eigenpal/docx-editor-react | React 어댑터. 툴바, 페이지 기반 에디터, 플러그인 지원 |
| @eigenpal/docx-editor-vue | Vue 3 어댑터. 툴바, 페이지 기반 에디터, 플러그인 지원 |
| @eigenpal/nuxt-docx-editor | Nuxt 3 & 4 모듈. Vue 어댑터를 감싼 형태 |
| @eigenpal/docx-editor-core | 프레임워크 독립적인 코어. OOXML 파서, 직렬화, 레이아웃 엔진, ProseMirror 스키마 포함 |
| @eigenpal/docx-editor-i18n | 양쪽 어댑터에서 사용하는 다국어 문자열과 타입 정의 |
| @eigenpal/docx-editor-agents | AI 에이전트 SDK와 채팅 UI. 프레임워크 독립적 브릿지, MCP 서버, AI SDK 어댑터 포함 |

어댑터를 포크할 계획이라면 얇게 유지하세요. `@eigenpal/docx-editor-core`에 직접 의존하면 파서, 직렬화, 렌더링 개선 사항들이 자동으로 빌드에 반영됩니다. 상위 저장소의 변경사항을 일일이 백포트할 필요가 없어진다는 뜻이죠.

## React 사용법

```jsx
import { useState } from 'react';
import { DocxEditor } from '@eigenpal/docx-editor-react';
import '@eigenpal/docx-editor-react/styles.css';

export function App() {
  const [buffer, setBuffer] = useState<ArrayBuffer | null>(null);

  return (
    <>
      <input
        type="file"
        accept=".docx"
        onChange={async (e) => setBuffer((await e.target.files?.[0]?.arrayBuffer()) ?? null)}
      />
      {buffer && <DocxEditor documentBuffer={buffer} mode="editing" />}
    </>
  );
}
```

**Next.js나 SSR 환경이라면?** 동적 임포트를 사용하세요. 에디터는 DOM이 필요하니까요.

자세한 내용은 [React 문서](https://docs.docx-editor.dev/packages/react)와 [API 레퍼런스](https://docs.docx-editor.dev/api)를 참고하세요.

## Vue 사용법

```vue
<script setup lang="ts">
import { ref } from 'vue';
import { DocxEditor } from '@eigenpal/docx-editor-vue';
import '@eigenpal/docx-editor-vue/styles.css';

const buffer = ref<ArrayBuffer | null>(null);

async function loadFile(e: Event) {
  const file = (e.target as HTMLInputElement).files?.[0];
  buffer.value = file ? await file.arrayBuffer() : null;
}
</script>

<template>
  <input type="file" accept=".docx" @change="loadFile" />
  <DocxEditor v-if="buffer" :document-buffer="buffer" mode="editing" />
</template>
```

자세한 내용은 [Vue 문서](https://docs.docx-editor.dev/packages/vue)와 [API 레퍼런스](https://docs.docx-editor.dev/api)를 참고하세요.

## Nuxt 사용법

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@eigenpal/nuxt-docx-editor'],
});
```

`@eigenpal/nuxt-docx-editor`는 Vue 어댑터를 Nuxt 3 & 4 모듈로 감싸줍니다. `<DocxEditor>` 컴포넌트와 Vue composables가 자동으로 임포트되므로 수동 임포트나 `<ClientOnly>` 래퍼가 필요 없어요.

[Nuxt 문서](https://docs.docx-editor.dev/packages/nuxt)를 확인하세요.

## 플러그인

```jsx
import { DocxEditor } from '@eigenpal/docx-editor-react';
import { PluginHost, templatePlugin } from '@eigenpal/docx-editor-react/plugin-api';

<PluginHost plugins={[templatePlugin]}>
  <DocxEditor documentBuffer={buffer} />
</PluginHost>;
```

전체 플러그인 API는 [플러그인 문서](https://docs.docx-editor.dev/plugins)를 참고하세요.
