## 핵심 개념

> [!abstract]
> Microsoft는 개발자의 일상적인 워크플로우를 위해 설계된 새로운 코딩 모델 MAI-Code-1-Flash를 출시했다. 이 모델은 GitHub Copilot에 최적화되어 있으며, Claude Haiku 4.5를 모든 코딩 벤치마크에서 능가한다. 적응형 솔루션 길이 제어를 통해 간단한 요청에는 간결하게, 복잡한 작업에는 더 깊은 분석을 제공하며, SWE-Bench Pro에서 60% 적은 토큰으로 더 어려운 문제를 해결한다. Visual Studio Code의 GitHub Copilot 사용자에게 현재 롤아웃되고 있다.

## 상세 내용

- 개발 환경 중심 설계: MAI-Code-1-Flash는 벤치마크만 최적화하지 않고 실제 GitHub Copilot 프로덕션 워크플로우를 중심으로 훈련되었으며, 저장소 질문 답변, 리팩토링, 원격 측정 기반 작업 등 핵심 소프트웨어 엔지니어링 작업에서 평가되어 실제 개발 환경과의 정렬을 보장한다.
- 적응형 토큰 효율성: 모델이 작업 난이도에 따라 응답 깊이를 자동으로 조정하여 간단한 요청은 빠르게 처리하고 복잡한 문제는 더 많은 추론 자원을 할당하며, SWE-Bench Verified에서 최대 60% 적은 토큰으로 더 어려운 문제를 해결하므로 지연 시간 단축과 비용 절감이 가능하다.
- 벤치마크 성능 우위: SWE-Bench Verified, Pro, Multilingual, Terminal Bench 2 등 4개 주요 코딩 벤치마크에서 Claude Haiku 4.5를 모두 능가했으며, 특히 SWE-Bench Pro(51.2% vs 35.2%)에서 16포인트 앞선다. 단순한 성능 향상이 아니라 더 적은 토큰으로 더 높은 정확성을 달성했다.
- 지시어 따르기 및 에이전트 코딩: IF Bench에서 정확한 지시어 따르기 능력에서 28.9포인트, 루브릭 기반 고급 지시어 따르기에서 14.5포인트 차이로 Claude Haiku 4.5를 능가하며, 이러한 강력한 지시어 따르기 능력이 도구 사용 에이전트 작업으로도 이어진다.
- 추론 능력 검증: 표준 벤치마크가 단순 암기를 보상할 수 있다는 점을 인식하여, 역순 고전 문제, 불가능한 작업, 미결정 시나리오 등 적대적 함정이 포함된 186개 질문, 34개 카테고리의 커스텀 벤치마크를 개발했고, MAI-Code-1-Flash는 85.8% 조정 정확도로 Claude Haiku 4.5를 능가했다.
- 즉시 사용 가능: Visual Studio Code의 GitHub Copilot 개별 사용자에게 현재 롤아웃 중이며, 추가 설정 없이 사용 가능하고, 자동 선택기를 통해 자동 라우팅되거나 모델 선택기에서 직접 선택할 수 있다.

> [!tip] 왜 중요한가
> 개발자들이 매일 사용하는 실제 환경에 맞춰 설계되고 토큰 효율성과 추론 성능을 동시에 달성한 코딩 모델의 출시는 GitHub Copilot 사용자의 개발 생산성을 크게 향상시키고 비용 효율성을 개선할 것으로 예상된다.

## 전문 번역

# MAI-Code-1-Flash 소개: 개발자 중심으로 만든 코딩 모델

마이크로소프트 Superintelligence 팀
2026년 6월 2일

오늘 마이크로소프트가 새로운 코딩 모델 **MAI-Code-1-Flash**를 소개합니다. 개발자의 일상적인 워크플로우에서 빠르고 효율적인 코딩 지원을 제공하도록 설계했어요. 마이크로소프트가 깨끗하고 적절하게 라이센스된 데이터로 처음부터 끝까지 직접 개발했습니다. 지금부터 Visual Studio Code의 GitHub Copilot 개인 사용자를 대상으로 모델 선택기와 자동 선택기에서 이용할 수 있습니다.

## 주요 기능

- **실제 개발 환경에서의 에이전틱 코딩**: GitHub Copilot 환경에서 학습하고 설계되어 더 나은 협력 경험을 제공합니다.
- **적응형 추론**: 단순한 요청에는 간결하게 응답하고, 복잡한 작업에는 더 많은 분석 능력을 투자합니다.
- **뛰어난 명령 이해력**: 한 번의 상호작용뿐 아니라 여러 차례의 대화에서도 일관되게 지시를 따릅니다.

MAI-Code-1-Flash는 한 가지 단순한 목표로 설계됐습니다. 바로 높은 품질의 코딩 지원을 더 효율적으로 제공하는 것이죠. 코딩 벤치마크에서 Claude Haiku 4.5를 능가하면서도 가격 대비 성능이 더 우수합니다.

## 벤치마크가 아닌 개발자를 위해 만들었습니다

코딩 모델은 개발자가 실제로 매일 사용하는 환경에서 잘 작동할 때 진정한 가치를 발휘합니다. 그래서 우리는 벤치마크 최적화만 추구하기보다, 실제 프로덕션 워크플로우를 중심에 두고 MAI-Code-1-Flash를 개발했습니다.

모델을 학습시킬 때는 실제 운영 환경에서 사용하는 GitHub Copilot 도구와 함께 학습했어요. 덕분에 주변 도구와 시스템과 어떻게 상호작용하고, 에이전틱 코딩 작업을 수행할지를 자연스럽게 익힐 수 있었습니다. 결과적으로 다른 모델과 비교했을 때 실제 Copilot 워크플로우에 훨씬 더 적합합니다.

학습 중에는 핵심 소프트웨어 엔지니어링 작업, 저장소 질의응답, 리팩토링, 그리고 실제 GitHub Copilot 사용 데이터를 기반으로 한 작업들에서 모델의 체크포인트를 평가했습니다. 학습, 평가, 실제 운영 환경 사이의 이런 일관성이 있어야만 오프라인에서의 개선이 실제 개발자 경험의 향상으로 이어질 수 있거든요.

## 토큰당 가치 극대화를 위해 설계

MAI-Code-1-Flash는 적응형 솔루션 길이 제어로 학습했습니다. 이 기술 덕분에 모델이 작업의 난이도에 맞춰 응답의 깊이를 조절할 수 있어요. 간단한 요청에는 간결하게 답하고, 깊이 있는 분석이나 광범위한 코드 변경이 필요한 문제에는 더 많은 사고력을 사용하는 식이죠.

실제로 이건 개발자가 더 빨리 유용한 결과를 보게 된다는 뜻입니다. MAI-Code-1-Flash는 SWE-Bench Verified에서 더 어려운 문제를 최대 60% 적은 토큰으로 해결합니다. 이는 지연 시간을 줄이고, 비용을 낮추고, 토큰의 투자 수익률을 높이며, 대화형 워크플로우를 더 부드럽게 만들어줍니다.

## 벤치마크 결과: 실제 운영 환경 기준

MAI-Code-1-Flash의 품질과 효율성을 제대로 평가하기 위해, 개발자들이 매일 사용하는 같은 운영 환경 도구를 사용해서 Claude Haiku 4.5와 비교했습니다. SWE-Bench Verified, SWE-Bench Pro, SWE-Bench Multilingual, Terminal Bench 2에서 평가했고, 작업 성공률과 각 작업당 평균 솔루션 토큰 수를 측정했어요.

**결과는 명확합니다.** MAI-Code-1-Flash는 테스트한 모든 핵심 코딩 벤치마크에서 Claude Haiku 4.5를 능가합니다. 4개 평가 항목 모두에서 더 높은 통과율을 기록했으며, 특히 다양한 실제 작업을 다루는 SWE-Bench Pro에서 무려 **+16포인트 차이**(51.2% vs 35.2%)를 기록했어요.

단순히 더 똑똑한 게 아닙니다. 더 효율적이거든요. SWE-Bench Verified에서 더 어려운 문제를 **최대 60% 적은 토큰으로** 풀어냅니다. 높은 정확도와 뛰어난 효율성이 더 이상 트레이드오프 관계가 아니라는 걸 증명한 거죠.

## 수학, 과학, 명령 이해, 에이전틱 코딩

MAI-Code-1-Flash는 표의 모든 벤치마크에서 앞서갑니다. 정확한 명령 이해(IF Bench)에서는 **+28.9포인트**로 가장 큰 차이를 보였고, 기준 기반 고급 명령 이해(Advanced IF)에서도 **+14.5포인트** 앞섰습니다. 이런 뛰어난 명령 이해력은 에이전틱 도구 사용으로도 이어집니다.

또한 수학, 과학, 시각 생성 코딩의 핵심 추론 능력에서도 Claude Haiku 4.5를 능가합니다.

## 진정한 추론 능력을 측정하다

표준 벤치마크는 암기를 추론만큼 보상합니다. 예를 들어 "몬티 홀 문제"를 본 모델은 정답을 말하지만, 상금을 바꾸면 틀려버리죠. 우리는 **186개 질문, 34개 범주**로 이루어진 벤치마크를 만들어서 모델이 실제로 추론하는지 아니면 단순히 패턴 매칭만 하는지를 확인했습니다. 역설된 고전 문제, 불가능한 작업, 정보 부족 시나리오 같은 함정들로 가득했죠.

**MAI-Code-1-Flash는 전반적으로 Claude Haiku 4.5를 능가했으며, 조정된 정확도로 85.8%를 달성했습니다.** 특히 추론, 명령 이해, 불가능한 문제 인식에서 특히 강했어요. 다만 아인슈텔룽 함정(Einstellung trap) 같은 핵심 적대적 범주에서는 50% 이하의 정확도를 기록해 개선의 여지가 있습니다.

## 지금 바로 시작해보세요

MAI-Code-1-Flash는 지금 Visual Studio Code의 GitHub Copilot 개인 사용자를 대상으로 배포 중입니다. 별도의 설정이 필요 없어요. 배포가 진행되면서 GitHub Copilot이 자동 선택기를 통해 작업을 MAI-Code-1-Flash로 라우팅할 수도 있고, 모델 선택기에서 직접 이 모델을 선택할 수도 있습니다.

VS Code에서 MAI-Code-1-Flash로 만든 몇 가지 재미있는 샘플 앱들을 공유했습니다. 직접 사용해보고 여러분의 피드백을 [GitHub Community](https://github.com/community)에서 공유해주면 좋겠습니다!
