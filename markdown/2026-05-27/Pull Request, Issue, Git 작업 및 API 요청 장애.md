## 핵심 개념

> [!abstract]
> GitHub 서비스에서 Pull Request, Issue, Git 작업 및 API 요청과 관련된 장애가 발생했습니다. GitHub Status 페이지에서 실시간 모니터링과 이메일/SMS 알림 구독을 지원합니다.

## 상세 내용

- GitHub 핵심 기능들(PR, Issue, Git ops, API)의 서비스 장애 발생
- Status 페이지를 통한 실시간 알림 및 구독 기능 제공

> [!tip] 왜 중요한가
> 개발 워크플로우의 중단을 신속하게 파악하고 대응할 수 있는 알림 체계가 필수적입니다.

## 전문 번역

죄송합니다만, 제공하신 텍스트에서 기술 아티클을 찾을 수 없습니다.

보내주신 내용은 OTP 재전송, 웹훅 URL, 이메일 구독 등과 관련된 UI 텍스트와 알림 설정 안내로 보입니다. 이는 기술 개발 글이라기보다는 서비스의 사용자 인터페이스 문구인 것 같습니다.

다음과 같은 형식의 기술 아티클을 제공해주시면 한국어로 자연스럽게 재구성해드리겠습니다:

- React/TypeScript 학습 가이드
- 성능 최적화 팁
- 특정 기술의 사용법 설명
- 문제 해결 방법론
- 아키텍처 또는 디자인 패턴 논의

더 나은 결과를 위해 기술 블로그 글이나 튜토리얼 형식의 글을 공유해주세요.
