## 핵심 개념

> [!abstract]
> FuzzingBrain V2는 LLM 기반 멀티에이전트 시스템으로, 자동화된 취약점 분석에서 높은 거짓 양성률과 재현 불가 문제를 해결합니다. 실제 배포에서 29개의 제로데이 취약점을 발견하여 12개 오픈소스 프로젝트에서 확인 및 수정되었습니다.

## 상세 내용

- 제어 흐름 기반의 Suspicious Point 추상화로 함수 레벨과 라인 레벨의 최적 중간점에서 정확한 취약점 위치 파악이 가능합니다.
- Google OSS-Fuzz 기반으로 모든 보고된 취약점의 fuzzer 재현성을 보장하여 거짓 양성을 제거합니다.
- AIxCC 2025 대회 C/C++ 데이터셋에서 90% 탐지율을 달성했습니다.

> [!tip] 왜 중요한가
> LLM 기반 자동화된 취약점 발견 도구의 실용화는 소프트웨어 보안 검증 프로세스의 효율성과 신뢰성을 크게 향상시킵니다.

## 전문 번역

# FuzzingBrain V2: 취약점 자동 발견과 재현을 위한 다중 에이전트 LLM 시스템

**분류:** 암호화 및 보안 (Cryptography and Security)  
**arXiv:** 2605.21779 (cs)

## 개요

소프트웨어 취약점은 심각한 보안 위협이 되고 있습니다. 2025년만 해도 약 5만 건의 CVE가 보고되었거든요. 최근 대형 언어모델(LLM)이 취약점 자동 탐지 분야에서 가능성을 보여주고 있지만, 여전히 세 가지 핵심 문제가 남아있습니다.

첫째, LLM이 생성한 취약점 리포트는 거짓양성률이 높고 재현 가능한 검증이 부족합니다. 둘째, 기존 LLM 기반 접근법은 취약점 위치 파악에 최적화되지 않은 세분화 수준을 사용합니다. 함수 수준의 분석은 문맥이 길어지면 버그를 놓치기 쉽고, 반대로 줄 수준의 분석은 충분한 문맥을 제공하지 못한다는 뜻입니다. 셋째, 함수 간 복잡한 의존성과 트리거 조건을 가진 취약점 추론이 어렵습니다.

## 솔루션: FuzzingBrain V2

우리는 이러한 문제들을 해결하기 위해 다중 에이전트 시스템 **FuzzingBrain V2**를 제시합니다. 네 가지 핵심 기여는 다음과 같습니다:

1. **완전 자동화된 취약점 분석** - Google의 OSS-Fuzz 기반으로 구축되어, 모든 보고된 취약점이 퍼저로 재현 가능함을 보장합니다.

2. **Suspicious Point** - 제어 흐름 기반의 새로운 추상화 개념으로, 최적의 세분화 수준에서 정확한 취약점 위치 파악을 실현합니다.

3. **논리 기반 계층적 함수 분석** - 이중 계층 퍼징을 통해 리소스 제약 하에서도 함수 커버리지를 높입니다.

4. **MCP 기반 정적·동적 분석 도구** - 문맥 엔지니어링으로 복잡한 취약점 추론 능력을 향상시킵니다.

## 실적

AIxCC 2025 최종 대회 C/C++ 데이터셋에서 FuzzingBrain V2는 **90%의 탐지율**(40개 중 36개 취약점)을 달성했습니다.

실제 배포 환경에서는 더욱 인상적인 결과를 얻었습니다. 12개의 오픈소스 프로젝트에서 **29개의 제로데이 취약점**을 발견했으며, 모두 개발자에게 확인 및 패치됐습니다. 이 중 2개는 CVE ID를 할당받았습니다.
