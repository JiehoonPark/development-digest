## 핵심 개념

> [!abstract]
> MDN이 프론트엔드 스택을 완전히 재구축하면서 React를 포기하고 웹 컴포넌트와 자체 개발 서버 컴포넌트 시스템을 채택했습니다.

## 상세 내용

- 대규모 웹사이트에서 React 대신 웹 컴포넌트 기반 아키텍처로 전환
- 자체 서버 컴포넌트 시스템을 개발하여 성능과 유지보수성 개선

> [!tip] 왜 중요한가
> 주요 기술 문서 사이트의 프레임워크 선택 변화는 웹 컴포넌트의 실무 적용 가능성과 트렌드 변화를 보여줍니다.
