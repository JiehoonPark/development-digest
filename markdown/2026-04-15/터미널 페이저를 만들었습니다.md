## 핵심 개념

> [!abstract]
> 저자가 Go로 재사용 가능한 뷰포트 컴포넌트를 개발하여 터미널 페이저 'lore'를 만들었습니다. 터미널에서 대용량 텍스트를 인터랙티브하게 네비게이션하기 위한 설계 결정과 학습 내용을 상세히 설명합니다.

## 상세 내용

- PAGER 환경변수를 통해 git, man 등 프로그램이 텍스트를 페이징하는 방식 설명
- TUI(터미널 사용자 인터페이스)에서 텍스트 뷰포트는 키보드 기반 네비게이션과 검색 기능 제공
- 터미널의 그리드 제약(행/열)이 정보 계층화와 본질적 기능 집중을 강제함

> [!tip] 왜 중요한가
> 개발자가 터미널에서 로그와 매니페스트를 효율적으로 다루기 위해 필요한 페이저 구현 원리를 이해할 수 있습니다.

## 전문 번역

# Terminal UI에서 텍스트를 다루기 위한 Viewport 컴포넌트 만들기

**요약**: 저는 Kubernetes 로그를 보는 `kl`이나 Nomad용 `wander` 같은 터미널 애플리케이션(TUI)을 만들어왔습니다. 이런 도구들의 핵심은 매니페스트나 로그 같은 대용량 텍스트를 다루는 것인데요. Go로 재사용 가능한 viewport 컴포넌트를 만들어 여러 프로젝트에 활용했습니다.

터미널 pager는 여러 페이지의 텍스트를 대화형으로 탐색할 수 있게 해주는 프로그램입니다. 이 viewport 컴포넌트를 활용해 `lore`라는 terminal pager를 만들었고, 지금은 이것을 매일 사용하고 있습니다.

이 글에서는 viewport에서 지원하고 싶었던 기능들과 그 과정에서 배운 점들, 그리고 설계 결정을 정리해봤습니다.

## 터미널에서의 텍스트 작업

터미널은 명령어를 실행하는 곳이면서 동시에 텍스트를 보고 탐색하는 공간이기도 합니다.

터미널은 고정폭 폰트로 격자 형태를 이루고 있는데요. 가로는 열(column), 세로는 행(row)으로 정의되며, 텍스트가 이 격자를 채우게 됩니다. ANSI escape code를 사용하면 터미널의 텍스트에 스타일을 입힐 수 있습니다.

개발자들은 터미널에서 대량의 텍스트를 스캔해야 하는 상황이 자주 생깁니다.

### 한 화면에 들어오는 경우

텍스트 출력이 터미널 높이보다 작으면 보통 특별한 pager 없이 바로 출력합니다. 이 경우 마우스로 스크롤하거나 터미널 에뮬레이터의 검색 기능(예: iTerm2의 cmd+f)을 사용할 수 있습니다. 또는 `tmux` 같은 터미널 멀티플렉서를 사용하면 그것의 검색과 스크롤백 단축키를 이용할 수 있죠.

### 여러 화면을 넘는 경우

여러 페이지에 걸친 텍스트는 보통 pager 프로그램을 사용해 대화형으로 탐색합니다. 프로그램들은 `PAGER` 환경변수를 확인한 후, 설정되어 있으면 그 프로그램으로 텍스트를 표시합니다.

`PAGER` 환경변수는 파이프된 입력이나 파일 인자를 받는 프로그램을 가리킵니다:

```bash
<command> | mypager
mypager myfile.txt
```

`git`, `man` 같은 프로그램들은 `PAGER`가 설정되어 있는지 확인하는 내부 로직을 가지고 있습니다. 일반적으로 stdout이 TTY(상호작용 터미널)일 때만 pager를 사용하죠. `git diff | grep ...` 같은 경우 stdout이 TTY가 아니므로 pager를 건너뜁니다.

stdout이 TTY라면, 프로그램은 `PAGER`로 설정된 프로그램을 자식 프로세스로 실행하고 파이프를 통해 연결합니다.

대부분의 개발자 머신에서는 `PAGER`가 설정되지 않았을 때 `less`를 기본 pager로 사용합니다. 모든 것을 터미널에 직접 출력하고 싶다면 `PAGER`를 `cat`으로 설정하면 됩니다. `bat`, `most`, `delta` 같은 옵션도 있습니다. 특정 프로그램만 다르게 처리하려면 `GIT_PAGER`(git용) 또는 `BAT_PAGER`(bat용) 같은 환경변수도 있습니다.

### less의 활용

`less`는 매우 강력한 도구인데, 옵션과 설정을 잘 사용하면 더욱 효과적입니다. 예를 들어 기본적으로 less를 종료하면 본 텍스트가 사라지는데, `--no-init/-X` 옵션을 사용하면 본 텍스트가 터미널에 남아있습니다. `--ignore-case/-i`를 사용하면 검색을 대소문자 구분 없이 할 수 있습니다.

## Terminal UI(TUI) 소개

Terminal UI는 네이티브 데스크톱 애플리케이션이나 웹사이트 같은 면이 있는데, 터미널 안에서 실행된다는 점이 다릅니다.

TUI는 보통 alt screen을 사용해 터미널 전체를 임시로 장악하고, 제목, 사이드바, 도움말, 텍스트 viewport 같은 애플리케이션 컴포넌트들을 표시합니다. 텍스트 viewport 컴포넌트는 terminal pager와 비슷하지만 화면의 일부만 차지합니다.

TUI의 특징은 편집 가능한 최소 단위가 픽셀이 아닌 터미널 격자 셀이라는 점입니다. 이것은 좋은 제약인데요. 덕분에 터미널 애플리케이션들은 꼭 필요한 정보만 표시하고, 웹사이트나 데스크톱 앱처럼 한 화면에 많은 컴포넌트와 버튼을 집어넣기보다는 키보드 중심의 계층적 뷰를 제공하게 됩니다.

### 실제 예: kl

제가 만든 `kl`은 여러 클러스터와 네임스페이스에 걸쳐 Kubernetes 로그와 상호작용하는 TUI입니다.

시작 시 두 개의 텍스트 viewport가 나타납니다. 왼쪽에는 설정된 클러스터, 네임스페이스, 파드, 컨테이너를 보여주는 Kubernetes 엔티티 계층이 있고, 오른쪽에는 처음엔 비어있는 로그 뷰가 있습니다.

먼저 선택 viewport에 포커스가 맞춰져 있고, 여기서 로그를 볼 컨테이너 하나 이상을 선택합니다. `L`을 누르면 선택 트리를 숨기고 로그를 전체 화면으로 봅니다. 여기서 `/`로 정확히 일치하는 항목을 검색할 수 있습니다.

`x`를 누르면 매칭된 항목 주변의 문맥을 보거나 매칭된 항목만 볼 수 있습니다. `p`를 누르면 로그를 보기 좋게 정렬합니다.
